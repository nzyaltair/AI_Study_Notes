# 信息检索与文本挖掘

## 1. 概述

信息检索（Information Retrieval, IR）研究如何从大规模文本集合中高效地找到与用户信息需求相关的文档。它是搜索引擎的核心技术，也是 RAG（检索增强生成）系统的基础组件。文本挖掘（Text Mining）则关注从非结构化文本中提取有价值的结构化信息和知识。

信息检索经历了从"词袋匹配"（TF-IDF/BM25）到"语义检索"（稠密向量检索）的演进。大模型时代，混合检索（BM25 + 稠密检索）结合重排序（Re-ranking）成为主流方案。

## 3. 核心概念

### 3.1 检索的基本流程

```
查询（Query）→ 查询处理 → 检索 → 候选文档 → 重排序 → 最终结果
```

| 阶段 | 方法 | 目标 |
|:---|:---|:---|
| 查询处理 | 分词、扩展、纠错 | 规范化查询 |
| 一阶段检索 | BM25 / 稠密检索 | 高效召回相关文档（Top-K） |
| 重排序 | 交叉编码器 / LLM | 精确排序 |
| 结果呈现 | 摘要、高亮 | 用户体验 |

### 3.2 稀疏检索 vs 稠密检索

| 维度 | 稀疏检索（BM25） | 稠密检索（DPR） |
|:---|:---|:---|
| 表示 | 词袋/倒排索引 | 密集向量 |
| 匹配 | 精确词项匹配 | 语义相似度 |
| 优势 | 精确匹配、无需训练 | 语义泛化、同义词匹配 |
| 劣势 | 词汇鸿沟、无语义理解 | 需要训练、精确词匹配弱 |
| 索引 | 倒排索引 | 向量索引（FAISS/HNSW） |

### 3.3 文本挖掘任务

| 任务 | 说明 |
|:---|:---|
| 文本聚类 | 将相似文档分组（K-Means、层次聚类） |
| 关键词提取 | 提取文档的核心关键词（TextRank、TF-IDF） |
| 主题模型 | 发现文档集合中的潜在主题（LDA、NMF） |
| 文本分类 | 将文档分配到预定义类别 |
| 趋势分析 | 分析文本数据的时间变化趋势 |

## 4. 技术原理

### 4.1 TF-IDF

词频-逆文档频率，衡量词对文档的重要性：

$$\text{TF-IDF}(t, d) = \text{TF}(t, d) \times \text{IDF}(t)$$

$$\text{IDF}(t) = \log\frac{N}{\text{df}(t)}$$

- $\text{TF}(t, d)$：词 $t$ 在文档 $d$ 中的频率
- $\text{df}(t)$：包含词 $t$ 的文档数
- $N$：总文档数

直觉：在少数文档中高频出现的词比在所有文档中出现的词更有区分度。

### 4.2 BM25

BM25（Okapi BM25）是 TF-IDF 的概率改进版本，对 TF 做饱和处理并加入文档长度归一化：

$$\text{BM25}(t, d) = \text{IDF}(t) \cdot \frac{\text{TF}(t, d) \cdot (k_1 + 1)}{\text{TF}(t, d) + k_1 \cdot (1 - b + b \cdot |d|/\text{avgdl})}$$

- $k_1 \approx 1.2$：控制 TF 饱和速度（$k_1 \to \infty$ 退化为线性 TF）
- $b \approx 0.75$：文档长度归一化强度（$b=0$ 不归一化，$b=1$ 完全归一化）
- $\text{IDF}(t) = \log\frac{N - \text{df}(t) + 0.5}{\text{df}(t) + 0.5}$

文档 $d$ 对查询 $q$ 的 BM25 分数为所有查询词的 BM25 之和：

$$\text{BM25}(q, d) = \sum_{t \in q} \text{BM25}(t, d)$$

**为什么 BM25 至今仍是基线？** 精确词匹配在关键词查询（产品名、代码名、专有名词）场景下不可替代，且无需训练、效率极高。

### 4.3 稠密检索（Dense Retrieval）

将查询和文档编码为稠密向量，通过向量相似度（余弦或内积）检索：

$$\text{score}(q, d) = \text{sim}(E_q(q), E_d(d))$$

**DPR（Dense Passage Retrieval）**：

- 使用两个独立的 BERT 编码器（共享或不共享权重）
- 训练目标：对比学习

$$L = -\log \frac{e^{\text{sim}(q, d^+)/\tau}}{e^{\text{sim}(q, d^+)/\tau} + \sum_{d^-} e^{\text{sim}(q, d^-)/\tau}}$$

- 正例 $d^+$：与查询相关的文档
- 负例 $d^-$：不相关文档（随机采样或难负例挖掘）

**向量索引**：使用 FAISS、HNSW 等近似最近邻（ANN）索引实现大规模高效检索。

### 4.4 ColBERT：延迟交互

ColBERT 介于稀疏检索和稠密检索之间——保留词级别的交互但使用神经编码：

$$\text{score}(q, d) = \sum_{i} \max_{j} \text{sim}(E_q(q_i), E_d(d_j))$$

查询的每个 token 向量与文档所有 token 向量做最大相似度匹配后求和。精度高于 DPR，效率高于交叉编码器。

### 4.5 重排序（Re-ranking）

一阶段检索召回的候选文档用更精确的模型重新排序：

**交叉编码器（Cross-Encoder）**：将查询和文档拼接后输入 BERT：

$$\text{score}(q, d) = \text{BERT}([\text{CLS}] \; q \; [\text{SEP}] \; d \; [\text{SEP}])$$

精度最高但计算开销大（每对查询-文档需独立计算），仅用于小规模候选集重排序。

### 4.6 主题模型：LDA

潜在狄利克雷分配（Latent Dirichlet Allocation）发现文档集合中的潜在主题：

- 每篇文档是多个主题的混合分布：$\theta_d \sim \text{Dir}(\alpha)$
- 每个主题是词的分布：$\phi_k \sim \text{Dir}(\beta)$
- 每个词由主题生成：$z \sim \theta_d$, $w \sim \phi_z$

使用吉布斯采样或变分推断训练。

## 5. 关键方法/模型

| 方法 | 类型 | 精度 | 效率 | 适用场景 |
|:---|:---|:---|:---|:---|
| BM25 | 稀疏 | 中 | 极高 | 关键词查询、精确匹配 |
| DPR | 稠密 | 高 | 高 | 语义检索 |
| ColBERT | 延迟交互 | 高 | 中 | 高精度检索 |
| Cross-Encoder | 交叉编码 | 极高 | 低 | 重排序 |
| SPLADE | 稀疏神经 | 高 | 高 | 稀疏+语义结合 |
| 混合检索 | BM25+DPR | 高 | 高 | 通用场景 |

## 6. 优势与局限

### 优势

1. **效率高**：倒排索引和 ANN 索引支持毫秒级检索
2. **可扩展**：支持亿级文档的检索系统
3. **互补性强**：BM25 和稠密检索各有优势，混合检索兼顾精确匹配和语义泛化

### 局限

1. **查询理解**：用户查询通常简短且有歧义，意图理解困难
2. **多跳检索**：需要多次推理才能找到答案的复杂查询仍具挑战
3. **时效性**：文档更新后索引需重建
4. **多语言**：跨语言检索需要统一语义空间

## 7. 应用场景

- **搜索引擎**：Google/Bing/百度等通用搜索
- **企业搜索**：企业内部文档检索
- **RAG 系统**：为 LLM 提供外部知识检索
- **推荐系统**：基于内容的推荐
- **问答系统**：从知识库中检索答案段落
- **法律/医疗检索**：专业领域的精确信息查找
