# 参数高效微调

## 背景与发展

参数高效微调（Parameter-Efficient Fine-Tuning, PEFT）是一类仅训练模型少量参数即可实现高效微调的技术。随着模型规模从数十亿增长到数千亿参数，全参数微调的算力和存储成本变得不可接受。PEFT 通过冻结大部分参数、仅训练少量可训练参数，在保持接近全量微调性能的同时，将可训练参数降至 0.1%-3%。

## 核心思想

在预训练模型参数固定的基础上，引入少量可训练参数（低秩矩阵、适配器模块、前缀向量等）来捕获任务特定知识。推理时可将可训练参数与基础模型合并，不增加推理开销。

## 技术原理

### 主流方法对比

| 方法 | 原理 | 可训练参数占比 | 推理开销 | 性能 |
|------|------|-------------|---------|------|
| [[LoRA]] | 冻结原始权重，训练低秩矩阵 $W + BA$ | ~1% | 无（可合并） | 接近全量 |
| [[QLoRA]] | 4-bit 量化基模 + LoRA | ~0.1% | 无（可合并） | 接近全量 |
| [[Adapter]] | 在层间插入小型瓶颈模块 | ~3% | 有（额外层） | 略差于 LoRA |
| Prefix Tuning | 学习虚拟前缀嵌入 | ~0.1% | 有 | 略差于 LoRA |
| Prompt Tuning | 学习软提示嵌入 | ~0.01% | 有 | 适合大模型 |

### LoRA 核心公式

$$W' = W + \Delta W = W + BA$$

其中 $B \in \mathbb{R}^{d \times r}$，$A \in \mathbb{R}^{r \times d}$，$r \ll d$。仅训练 $A$ 和 $B$，参数量为 $2dr$（远小于 $d^2$）。

### 关键参数选择

| 参数 | 推荐范围 | 说明 |
|------|---------|------|
| LoRA 秩 $r$ | 8-64 | 越高表达能力越强，计算量越大 |
| 缩放因子 $\alpha$ | $2r$ | 调整 LoRA 贡献，通常设为秩的 2 倍 |
| 学习率 | 1e-5 ~ 5e-4 | 高于全参数微调 2-10 倍 |
| 目标模块 | Q, V 投影 | 注意力层最常用，可扩展到全部线性层 |
| Dropout | 0.05-0.1 | 防止过拟合 |

### 方法选择指南

| 场景 | 推荐方法 |
|------|---------|
| 资源受限（单卡） | [[QLoRA]] |
| 多任务学习 | [[LoRA]] / [[Adapter]] |
| 生成任务 | Prefix Tuning |
| 特定任务微调 | Prompt Tuning |
| 追求最佳性能 | [[LoRA]]（大秩 + 全线性层） |

## 发展演进

Adapter (2019, Houlsby) → Prefix Tuning (2021) → LoRA (2021, Microsoft) → QLoRA (2023, 单卡 70B) → AdaLoRA / DoRA / PiSSA (2024)

## 关键算法·模型

- [[LoRA]]：低秩适配，最广泛使用的 PEFT 方法
- [[QLoRA]]：4-bit 量化 + LoRA，单卡微调 70B 模型
- [[Adapter]]：瓶颈结构适配器模块
- **AdaLoRA**：动态调整不同层的 LoRA 秩
- **DoRA**：动态缩放因子的 LoRA 变体
- **PiSSA**：用 SVD 初始化 LoRA 矩阵

## 应用场景

- **大模型微调**：在有限 GPU 资源下微调数十亿参数模型
- **多任务部署**：多个 LoRA 适配器共享同一基座模型
- **领域适配**：为不同领域训练专用适配器
- **持续学习**：通过添加新适配器学习新任务，避免灾难性遗忘

## 与其他技术关系

- PEFT 是 [[监督微调]] 的主流实现方式
- [[QLoRA]] 依赖模型量化技术
- PEFT 可与 [[RLHF]] / [[DPO]] 结合
- 推理时可与模型量化、推理加速技术配合
