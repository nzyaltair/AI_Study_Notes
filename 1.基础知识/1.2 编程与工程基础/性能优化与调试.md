# 性能优化与调试

## 背景

AI模型规模的不断扩大使性能优化成为关键挑战。从训练阶段的GPU利用率优化到推理阶段的延迟降低，性能优化直接影响AI系统的成本和用户体验。

## 核心思想

性能优化遵循"测量优先"原则——先定位瓶颈，再针对性优化。优化涉及多个层次：算法、代码、系统、硬件。

## 技术原理

### 性能分析工具

- **Python profiler**：`cProfile`、`line_profiler`定位慢代码
- **GPU监控**：`nvidia-smi`、`torch.cuda.memory_summary()`
- **PyTorch profiler**：`torch.profiler`分析训练瓶颈
- **关键指标**：
  - 训练：吞吐量（samples/sec）、GPU利用率、显存占用
  - 推理：延迟（ms）、吞吐量（tokens/sec）、首token延迟

### GPU优化

- **混合精度训练**：FP16计算+FP32主权重，减少显存、加速计算
  ```python
  from torch.cuda.amp import autocast, GradScaler
  scaler = GradScaler()
  with autocast():
      output = model(input)
      loss = criterion(output, target)
  scaler.scale(loss).backward()
  scaler.step(optimizer)
  ```
- **梯度累积**：小batch模拟大batch效果
- **梯度检查点**：用计算换显存，适用于大模型
- **数据预加载**：`DataLoader(num_workers=4, pin_memory=True)`

### 内存优化

- **显存管理**：及时释放不需要的tensor，`del tensor; torch.cuda.empty_cache()`
- **模型并行**：将大模型拆分到多个GPU
- **ZeRO优化**：优化器状态、梯度、参数分片（DeepSpeed）
- **CPU offload**：将不活跃参数卸载到CPU

### 推理优化

- **批处理**：合并多个请求提高GPU利用率
- **量化**：FP16/INT8/INT4降低计算精度，减少显存
- **KV-Cache**：缓存注意力键值对，避免重复计算（详见[[KV-Cache机制]]）
- ** speculative decoding**：小模型预测+大模型验证，加速生成
- **Flash Attention**：IO-aware的注意力计算优化

### 调试技术

- **梯度检查**：`torch.autograd.gradcheck`验证梯度正确性
- **形状检查**：在关键位置添加`assert tensor.shape == expected`
- **NaN/Inf检测**：`torch.isnan(tensor).any()`检查数值异常
- **可视化**：TensorBoard、Weights & Biases监控训练曲线
- **过度拟合测试**：在小数据集上验证模型能否记住所有样本

### 分布式训练

- **数据并行（DDP）**：每个GPU完整模型副本，不同数据
- **张量并行**：将单个层拆分到多个GPU
- **流水线并行**：将不同层分配到不同GPU
- **混合并行**：结合数据并行和模型并行

## 应用领域

- **[[深度学习]]训练**：GPU利用率优化、显存管理
- **[[大语言模型]]推理**：延迟优化、吞吐量提升
- **[[系统工程化与运维]]**：模型部署性能调优
- **[[量化技术]]**：降低模型精度以加速推理
