# 视觉语言模型

## 背景

视觉语言模型（VLM）连接视觉与语言两个模态，使模型能"看图说话"和"按文绘图"。CLIP（2021）开创了对比学习对齐范式，LLaVA（2023）将视觉编码器接入LLM实现了多模态对话，GPT-4V/Gemini代表了商业多模态大模型的前沿。

## 核心思想

通过跨模态对齐，将视觉信息映射到语言模型的语义空间，使LLM能够理解和推理视觉内容。

## 技术原理

### 架构范式

| 范式 | 架构 | 代表模型 |
|------|------|---------|
| **对比学习** | 双塔编码器+对比损失 | CLIP |
| **桥接式** | 视觉编码器+投影层+LLM | LLaVA, Qwen-VL |
| **原生多模态** | 从头训练的统一模型 | GPT-4V, Gemini |
| **Flamingo式** | Perceiver Resampler+交叉注意力 | Flamingo |

### CLIP：对比学习对齐

- 图像编码器（ViT）+ 文本编码器（Transformer）
- 对比损失：拉近匹配的图文对，推远不匹配的对
- $L = -\log\frac{\exp(\text{sim}(I,T^+)/\tau)}{\sum_j \exp(\text{sim}(I,T_j)/\tau)}$
- 零样本分类：用文本描述类别，选择相似度最高的

### LLaVA架构

```
图像 → ViT视觉编码器 → 视觉Token → MLP投影层 → 接入LLM的Token序列
                                                        ↓
文本 → Tokenization → 文本Token ──────────────→ LLM → 输出
```

- 简单有效：视觉编码器（CLIP ViT）+ 投影层 + 开源LLM
- 两阶段训练：图像描述对齐 → 指令微调

### 多模态对齐方法

| 方法 | 原理 | 代表 |
|------|------|------|
| **线性投影** | 简单MLP映射 | LLaVA-1 |
| **Q-Former** | 可学习查询提取视觉特征 | BLIP-2 |
| **交叉注意力** | LLM层间插入交叉注意力 | Flamingo |
| **原生融合** | 视觉Token直接作为序列一部分 | GPT-4V |

### 关键模型

| 模型 | 架构特点 | 能力 |
|------|---------|------|
| **CLIP** | 双塔对比学习 | 零样本分类、图文检索 |
| **BLIP-2** | Q-Former桥接 | 图文理解+生成 |
| **LLaVA** | ViT+投影+Vicuna | 多模态对话 |
| **Qwen-VL** | ViT+Adapter+Qwen | 中文多模态、OCR |
| **GPT-4V** | 原生多模态 | 通用视觉理解 |
| **Gemini** | 原生多模态 | 图文音视频统一 |

### 训练策略

1. **预训练对齐**：用大规模图文对训练视觉-语言对齐
2. **指令微调**：用多模态指令数据微调（GPT-4生成指令）
3. **偏好对齐**：RLHF/DPO适配多模态偏好

## 发展演进

CLIP（2021）→ BLIP-2（2023）→ LLaVA（2023）→ GPT-4V（2023）→ Gemini（2023）→ 原生多模态统一模型（2024+）

趋势：从桥接式架构走向原生多模态，最终实现"Any-to-Any"统一模型。

## 应用领域

- **多模态对话**：带图片的ChatGPT
- **视觉问答（VQA）**：图像内容问答
- **OCR与文档理解**：表格、公式、图表识别
- **[[Agent智能体|多模态Agent]]**：UI操作、视觉导航
- **图像检索**：文本到图像搜索
