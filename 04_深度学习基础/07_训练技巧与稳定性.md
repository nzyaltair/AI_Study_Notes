# 训练技巧与稳定性

## 1. 概述

深度学习模型训练是一个复杂的系统工程，面临梯度消失/爆炸、数值不稳定、显存瓶颈、训练发散等挑战。训练技巧将优化器、归一化、正则化等基础组件与工程实践（混合精度、分布式训练、梯度管理）结合，确保从单卡小模型到千卡大模型的训练过程稳定高效。

- **目标**：在给定的硬件和时间预算内，训练出收敛稳定、泛化良好的模型
- **难度递增**：单卡小模型训练（技巧简单）→ 多卡大模型训练（分布式+混合精度）→ 千卡LLM训练（全链路工程优化）

## 2. 发展历史

- **2012前**：单卡训练，梯度裁剪手动实现
- **2012-2015**：多GPU数据并行（DataParallel）普及
- **2015**：Batch Normalization极大降低深层训练的初始化敏感度
- **2017**：混合精度训练概念提出（Micikevicius et al., ICLR 2018）
- **2019**：ZeRO（Rajbhandari et al., 2019）将千亿参数模型训练的成本降至学术可接受范围
- **2020+**：Megatron-LM的张量并行+流水线并行+数据并行的3D并行成为大模型训练标准范式
- **2023**：FlashAttention（Dao et al.）彻底改写注意力计算的GPU访存模式

## 3. 核心概念

### 3.1 训练稳定的三个层次

| 层次 | 问题 | 典型解决方案 |
|:---|:---|:---|
| **数值稳定** | 梯度下溢/溢出、NaN | 混合精度训练、Loss Scaling |
| **优化稳定** | 训练发散、Loss震荡 | 梯度裁剪、Warmup、合适的初始化 |
| **工程稳定** | 硬件故障、通信超时 | 弹性训练、Checkpoint恢复 |

### 3.2 显存预算的构成

训练时GPU显存主要被以下组件占用：

$$\text{Total Memory} = \underbrace{\text{参数}}_{\text{Model Params}} + \underbrace{\text{梯度}}_{\text{Gradients}} + \underbrace{\text{优化器状态}}_{\text{Optimizer States}} + \underbrace{\text{激活值}}_{\text{Activations}}$$

- **参数**：FP32时 $4P$ 字节，FP16/BF16时 $2P$ 字节
- **梯度**：与参数同阶
- **优化器状态**（Adam为例）：一阶矩 $\mathbf{m}$ + 二阶矩 $\mathbf{v}$ = $2 \times 4P = 8P$ 字节（FP32）
- **激活值**：随batch size和序列长度线性增长

总计：FP32+Adam ≈ $16P$ 字节，即 1B参数模型需要约 16GB 仅用于存储

## 4. 技术原理

### 4.1 梯度管理

#### 梯度裁剪（Gradient Clipping）

限制梯度的全局范数，防止梯度爆炸导致训练崩溃：

$$\tilde{\mathbf{g}} = \begin{cases} \mathbf{g} & \|\mathbf{g}\|_2 \leq v \\ v \cdot \frac{\mathbf{g}}{\|\mathbf{g}\|_2} & \|\mathbf{g}\|_2 > v \end{cases}$$

- **按范数裁剪（Norm Clipping）**：保持梯度方向，限制其长度
- **按值裁剪（Value Clipping）**：$g_i = \text{clamp}(g_i, -v, v)$，逐个分量裁切
- **LLM训练标配**：$\text{max\_norm}=1.0$ 是LLaMA等模型的常用值

#### 梯度累积（Gradient Accumulation）

在显存不足以支持大batch时，通过多次前向/反向累积梯度模拟大batch训练：

$$\text{有效 batch size} = \text{micro\_batch\_size} \times \text{accumulation\_steps}$$

```
for step in range(accumulation_steps):
    loss = model(micro_batch) / accumulation_steps
    loss.backward()  # 梯度累加，不更新参数

optimizer.step()     # 累积完成后统一更新
optimizer.zero_grad()
```

**注意事项**：如果模型中使用了BatchNorm，梯度累积会改变BN的统计量（mini-batch ≠ 单步micro-batch），需改用SyncBN或GN/LN。

#### 梯度检查点（Gradient Checkpointing / Activation Recomputation）

以时间换空间的经典权衡：不保存全部中间激活值，反向传播时从最近的检查点重新计算。

- **显存节省**：可将激活值存储从 $\mathcal{O}(L)$ 降至 $\mathcal{O}(\sqrt{L})$ 或 $\mathcal{O}(\log L)$
- **计算代价**：约增加33%的前向计算量（每层约重新计算一次）
- **应用**：训练长序列Transformer或大batch时的必备技巧

### 4.2 混合精度训练（Mixed Precision Training）

使用FP16/BF16进行前向和反向计算，同时在关键位置保留FP32精度：

```
参数主副本（FP32）
    ↓ 转换为低精度
前向传播（FP16/BF16）
    ↓
损失计算（FP32）
    ↓
反向传播（FP16/BF16）
    ↓ 梯度转换为FP32
参数更新（FP32）
```

#### FP16 vs BF16

| 特性 | FP16 | BF16 |
|:---|:---|:---|
| 指数位 | 5 bits | 8 bits（与FP32相同） |
| 尾数位 | 10 bits | 7 bits |
| 动态范围 | 小（~65504 max） | **大**（与FP32相同） |
| Loss Scaling | **必需**（防止梯度下溢） | **不需要** |
| 硬件支持 | V100+ | A100+ |
| LLM训练 | 早期使用 | **现代标配** |

- **BF16的优势**：动态范围与FP32一致，不需要Loss Scaling这一容易出错的手动技巧
- **现代实践**：A100/H100上LLM训练几乎全面使用BF16

#### Loss Scaling（FP16专用）

FP16的最小正数约为 $6 \times 10^{-8}$，而某些梯度值可能小于此阈值（下溢为0）。Loss Scaling通过放大损失值来保护小梯度：

$$\text{scaled\_loss} = \text{loss} \times \text{scale\_factor}$$

反向传播后，再缩小梯度恢复原始尺度。实践中通常使用动态Loss Scaling，根据梯度溢出情况自动调整scale factor。

### 4.3 分布式训练策略

#### 数据并行（Data Parallelism, DP）

每张GPU持有完整模型副本，处理不同的数据子集，梯度通过AllReduce同步：

```
GPU 0: data[0:B] → forward → backward → ∇L₀  ┐
GPU 1: data[B:2B] → forward → backward → ∇L₁  ├→ AllReduce(∇L₀+∇L₁+∇L₂+∇L₃)/4
GPU 2: data[2B:3B] → forward → backward → ∇L₂  │
GPU 3: data[3B:4B] → forward → backward → ∇L₃  ┘
```

- **DDP（DistributedDataParallel）**：PyTorch的标准实现，梯度同步与反向传播重叠（通信与计算overlap）
- **局限**：模型必须能放入单卡显存

#### 模型并行（Model Parallelism）

**张量并行（Tensor Parallelism）**：将单个矩阵运算切分到多卡并行计算

以 $Y = XW$ 为例：
- 按列切分 $W = [W_1 | W_2]$，GPU 0/1 分别计算 $XW_1$ 和 $XW_2$，拼接得 $Y$
- Megatron-LM (Shoeybi et al., 2019) 将此法系统化应用于Transformer的多头注意力和FFN层

**流水线并行（Pipeline Parallelism）**：按层切分模型，各GPU负责不同层，形成加工流水线

- GPipe (Huang et al., 2019) 引入微批次（micro-batch）实现流水线填充
- 1F1B schedule：交替执行前向和反向，减少显存峰值

#### ZeRO（Zero Redundancy Optimizer）

微软DeepSpeed提出的三阶段优化策略，消除数据并行中的冗余存储：

| 阶段 | 分片内容 | 显存节省 | 通信开销 |
|:---|:---|:---|:---|
| ZeRO-1 | 优化器状态 | 4× | 无损（与DP相等） |
| ZeRO-2 | 优化器状态 + 梯度 | 8× | 无损 |
| ZeRO-3 | 优化器状态 + 梯度 + 参数 | $N_{\text{GPU}}$ × | 增加50%参数通信 |

- **ZeRO-3**使单卡训练13B模型、多卡训练175B+模型成为可能
- 与Megatron-LM的张量/流水线并行组合使用，形成3D并行（数据×张量×流水线）

#### 3D并行（Data + Tensor + Pipeline）

现代千亿级LLM训练的标准配置：

```
┌──────────────────────────────────────────┐
│  3D Parallelism 拓扑                       │
│                                            │
│  ┌──── 流水线并行（跨节点）───────────┐    │
│  │ GPU0 → GPU1 → GPU2 → GPU3          │    │
│  │  Layer  ──── Layer ──── Layer       │    │
│  └────────────────────────────────────┘    │
│  ┌──── 张量并行（节点内）─────────────┐    │
│  │ GPU0 ←→ GPU1 (切分Attention/FFN)    │    │
│  └────────────────────────────────────┘    │
│  ┌──── 数据并行（纵向扩展）───────────┐    │
│  │ 副本0 ──── 副本1 ──── 副本2        │    │
│  └────────────────────────────────────┘    │
└──────────────────────────────────────────┘
```

## 6. 优势与局限

### 关键权衡

| 技术 | 收益 | 代价 |
|:---|:---|:---|
| 混合精度 | 2×加速 + 50%显存节省 | FP16需Loss Scaling |
| 梯度检查点 | 大幅节省激活存储 | ~33%额外计算 |
| 梯度累积 | 模拟大batch效果 | 不改变BN统计量 |
| 数据并行 | 线性加速（理想） | 模型须放入单卡 |
| ZeRO-3 | 可将优化器状态分布在所有GPU | 额外通信开销 |
| 流水线并行 | 突破单卡模型大小限制 | 流水线气泡（bubble） |

### 大模型训练的"不可能三角"

在大模型训练中，以下三个目标无法同时最优达到：
1. **低通信开销**：数据并行通信简单但显存压力大
2. **低显存占用**：ZeRO-3显存友好但通信开销增加
3. **高计算效率**：流水线并行通信轻量但存在bubble

实际工程需在3D并行空间中寻找最优解。

## 7. 应用场景

- **小规模实验（<1B参数）**：单卡/多卡DDP + BF16 + Gradient Checkpointing
- **中等规模训练（1B-10B）**：DDP + ZeRO-2 + BF16 + FlashAttention
- **大规模LLM训练（10B-100B+）**：3D并行（ZeRO-3 + 张量并行 + 流水线并行）+ BF16 + FlashAttention + 弹性训练
- **微调（Fine-tuning）**：LoRA/QLoRA大幅降低显存需求，单卡可微调70B模型

## 9. 前沿发展

1. **FlashAttention演进**
   - FlashAttention-2（2023）：进一步优化SM间并行，2-3×加速
   - FlashAttention-3（2024）：针对H100的Hopper架构深度优化
   - 核心思想：避免从HBM读写完整注意力矩阵，在SRAM中完成分块计算

2. **通信优化**
   - **Ring Attention**：将序列维度的注意力计算跨多GPU分布式执行，无需全局通信
   - **通信-计算重叠**：在反向传播计算期间完成AllReduce梯度同步

3. **弹性训练（Elastic Training）**
   - 支持训练过程中动态增减GPU数量，适应云环境的抢占式实例
   - PyTorch TorchElastic：worker故障后自动恢复，从checkpoint继续训练

4. **混合精度2.0**
   - FP8训练（H100支持）：进一步2×加速与显存节省
   - 微观混合精度：不同层使用不同精度（对精度敏感的LayerNorm/RMSNorm保留FP32）
