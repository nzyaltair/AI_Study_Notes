# 数据处理与工程化

## 背景与发展

AI 系统的性能上限由数据质量决定。原始数据通常存在缺失、噪声、格式不一致等问题，需经过采集、清洗、变换、特征工程等步骤才能成为有效的模型输入。数据工程化将这一过程标准化、自动化、可复现，是机器学习工程实践的核心环节。

## 核心思想

数据工程化的核心是构建可复现的数据管道（Pipeline），将数据从原始状态到模型输入的变换过程封装为一系列确定性的、可测试的、可监控的步骤，确保训练数据与线上特征一致。

## 技术原理

### 数据清洗

**缺失值处理**：
- 删除法：缺失率低时直接丢弃样本或特征
- 填充法：均值/中位数/众数填充，或插值估计
- 预测法：用其他特征训练模型预测缺失值

**异常值检测**：
- 统计方法：3$\sigma$ 准则、IQR（四分位距）方法
- 距离方法：KNN 距离、局部异常因子（LOF）
- 孤立森林：基于随机划分的异常检测

### 特征工程

**数值特征**：
- 标准化：$x' = \frac{x - \mu}{\sigma}$
- 归一化：$x' = \frac{x - x_{\min}}{x_{\max} - x_{\min}}$
- 对数变换：$x' = \log(x + 1)$，压缩长尾分布

**类别特征**：
- One-Hot 编码：无序类别，维度等于类别数
- 目标编码：用目标变量条件均值替换类别
- 嵌入编码：学习低维稠密向量表示

**特征选择**：
- 过滤法：相关系数、互信息、卡方检验
- 包装法：递归特征消除（RFE）
- 嵌入法：L1 正则化（Lasso）、树模型特征重要性

### 数据管道

Scikit-learn Pipeline 与 ColumnTransformer 实现数值特征与类别特征的差异化预处理并串联模型。

```python
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import StandardScaler, OneHotEncoder

preprocessor = ColumnTransformer([
    ('num', StandardScaler(), numeric_features),
    ('cat', OneHotEncoder(handle_unknown='ignore'), categorical_features)
])
pipeline = Pipeline([('preprocessor', preprocessor), ('model', model)])
```

Pipeline 保证训练与推理时特征变换一致，避免数据泄露（data leakage）。

### 数据存储

- **关系型数据库**：PostgreSQL、MySQL，结构化事务数据
- **数据仓库**：ClickHouse、Snowflake，OLAP 分析查询
- **对象存储**：S3、HDFS，大规模非结构化文件
- **向量数据库**：Milvus、Pinecone，嵌入向量近似检索

### 数据版本管理

- **DVC**：基于 Git 的数据版本控制，追踪数据与代码的关联
- **Delta Lake / Apache Iceberg**：数据湖上的 ACID 事务支持
- **数据血缘**：记录数据来源、变换链路和依赖关系

## 发展演进

手工数据脚本（2010s 初）→ Scikit-learn Pipeline 标准化 → 数据版本管理工具（DVC, 2017）→ 湖仓一体架构（Delta Lake, 2019）→ 特征存储平台（Feast, Tecton）→ 大语言模型时代的数据清洗与对齐流水线。

## 关键算法·模型

- **标准化/归一化**：基于统计量的线性变换
- **孤立森林**：随机划分的异常检测，时间复杂度 $O(n \log n)$
- **互信息**：$I(X; Y) = \sum_{x,y} p(x,y) \log \frac{p(x,y)}{p(x)p(y)}$，衡量特征与目标的非线性依赖
- **L1 正则化**：$\min_w \|y - Xw\|_2^2 + \lambda \|w\|_1$，产生稀疏解实现特征选择

## 应用场景

- 经典机器学习：特征工程直接影响模型性能
- 深度学习：数据增强、归一化、数据加载优化
- RAG 系统：文档分块、嵌入构建、向量索引
- 大模型训练：预训练数据去重、质量过滤、指令对齐数据构建
