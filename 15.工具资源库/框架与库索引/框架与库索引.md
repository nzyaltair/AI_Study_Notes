# 框架与库索引

> 本文件为 AI 工程生态中主流和新兴框架、库与工具链的结构化导航目录，帮助开发者快速选型、理解能力边界并高效接入项目。

## 概述

本索引收录了 AI 工程实践中最常用的框架和库，按功能类别组织，覆盖从基础深度学习框架到专业领域工具链的完整生态。每个条目包含核心功能、适用场景、典型使用模式和对比优势，旨在为团队提供技术选型的参考依据。

## 深度学习框架

### 基础框架

- [ ] [[PyTorch]] - 动态计算图深度学习框架，以灵活性和易用性著称
  - **核心功能**：动态计算图、自动微分、丰富的神经网络模块
  - **适用场景**：研究实验、快速原型开发、生产部署
  - **典型使用模式**：
    ```python
    import torch
    import torch.nn as nn
    model = nn.Linear(10, 1)
    ```
  - **关键能力**：支持分布式训练、量化、LoRA 微调、长上下文
  - **对比优势**：社区活跃、文档丰富、易于调试
  - **常见坑点**：内存使用较高、Windows 支持相对较弱

- [ ] [[TensorFlow]] - 静态计算图深度学习框架，由 Google 开发
  - **核心功能**：静态计算图、TensorBoard、TPU 支持
  - **适用场景**：生产级部署、大规模分布式训练
  - **典型使用模式**：
    ```python
    import tensorflow as tf
    model = tf.keras.Sequential([tf.keras.layers.Dense(1)])
    ```
  - **关键能力**：支持分布式训练、量化、TPU 优化
  - **对比优势**：企业级稳定性、部署工具链完善
  - **常见坑点**：API 变更频繁、学习曲线较陡

- [ ] [[JAX]] - Google 开发的基于函数式编程的机器学习框架
  - **核心功能**：自动微分、XLA 编译、函数变换
  - **适用场景**：研究实验、高性能计算
  - **典型使用模式**：
    ```python
    import jax
    import jax.numpy as jnp
    ```
  - **关键能力**：支持 JIT 编译、GPU/TPU 优化
  - **对比优势**：性能优异、函数式编程风格
  - **常见坑点**：生态相对较小、学习曲线较陡

## 大模型训练/推理框架

- [ ] [[Hugging Face Transformers]] - 最流行的 NLP 模型库，提供丰富的预训练模型
  - **核心功能**：模型加载、微调、推理、评估
  - **适用场景**：快速模型应用、研究实验、生产部署
  - **典型使用模式**：
    ```python
    from transformers import AutoModelForCausalLM, AutoTokenizer
    model = AutoModelForCausalLM.from_pretrained("gpt2")
    ```
  - **关键能力**：支持分布式训练、量化、LoRA 微调、长上下文
  - **对比优势**：模型种类丰富、API 友好、社区活跃
  - **常见坑点**：内存使用较高、部分模型依赖复杂

- [ ] [[vLLM]] - 高性能大语言模型推理引擎
  - **核心功能**：连续批处理、KV Cache 优化、PagedAttention 技术
  - **适用场景**：大模型服务部署、高并发 API 调用、低延迟推理
  - **典型使用模式**：
    ```python
    from vllm import LLM, SamplingParams
    llm = LLM(model="gpt2")
    ```
  - **关键能力**：支持量化、长上下文、流式输出
  - **对比优势**：单卡高吞吐、易于集成
  - **常见坑点**：依赖较新的 CUDA 版本

- [ ] [[DeepSpeed]] - Microsoft 开发的深度学习训练优化库
  - **核心功能**：ZeRO 优化、混合精度训练、流水线并行
  - **适用场景**：超大模型训练、分布式训练加速
  - **典型使用模式**：
    ```python
    from deepspeed import initialize
    model, optimizer, _, _ = initialize(args=args, model=model, optimizer=optimizer)
    ```
  - **关键能力**：支持分布式训练、内存优化
  - **对比优势**：显著降低显存使用、提高训练速度
  - **常见坑点**：配置复杂、调试难度较大

- [ ] [[Megatron-LM]] - NVIDIA 开发的大规模语言模型训练框架
  - **核心功能**：模型并行、流水线并行、分布式训练
  - **适用场景**：超大规模模型训练
  - **关键能力**：支持分布式训练、模型并行
  - **对比优势**：专为大规模训练优化
  - **常见坑点**：资源要求高、部署复杂

- [ ] [[Axolotl]] - 大语言模型训练框架，专注于简化训练流程
  - **核心功能**：配置驱动训练、多种微调方法支持
  - **适用场景**：快速模型微调、实验管理
  - **关键能力**：支持 LoRA 微调、量化
  - **对比优势**：配置简单、易于使用
  - **常见坑点**：功能相对有限

## 多模态库

- [ ] [[CLIP]] - OpenAI 开发的多模态预训练模型
  - **核心功能**：图像-文本对齐、零样本分类
  - **适用场景**：多模态检索、图像分类、跨模态任务
  - **典型使用模式**：
    ```python
    from transformers import CLIPModel, CLIPProcessor
    model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32")
    ```
  - **关键能力**：支持零样本学习、跨模态对齐
  - **对比优势**：性能优异、广泛应用
  - **常见坑点**：模型较大、推理速度较慢

- [ ] [[OpenCLIP]] - CLIP 的开源实现和扩展
  - **核心功能**：CLIP 模型实现、扩展训练
  - **适用场景**：自定义 CLIP 训练、模型研究
  - **关键能力**：支持自定义训练、模型扩展
  - **对比优势**：完全开源、灵活扩展
  - **常见坑点**：文档相对较少

- [ ] [[LAVIS]] - 多模态视觉-语言模型库
  - **核心功能**：多模态模型集合、训练工具
  - **适用场景**：多模态研究、模型开发
  - **关键能力**：支持多种多模态任务
  - **对比优势**：模型种类丰富、工具链完整
  - **常见坑点**：依赖较多

- [ ] [[MMEngine]] - OpenMMLab 开发的多模态训练引擎
  - **核心功能**：统一训练接口、模型 zoo
  - **适用场景**：多模态模型训练、研究
  - **关键能力**：支持分布式训练、模型集成
  - **对比优势**：模块化设计、易于扩展
  - **常见坑点**：学习曲线较陡

## 生成式 AI 工具链

- [ ] [[Diffusers]] - Hugging Face 开发的扩散模型库
  - **核心功能**：稳定扩散、ControlNet、Text-to-Image
  - **适用场景**：图像生成、编辑、风格转换
  - **典型使用模式**：
    ```python
    from diffusers import StableDiffusionPipeline
    pipe = StableDiffusionPipeline.from_pretrained("runwayml/stable-diffusion-v1-5")
    ```
  - **关键能力**：支持多种生成模型、优化推理
  - **对比优势**：易于使用、模型丰富
  - **常见坑点**：内存使用较高、生成速度较慢

- [ ] [[LangChain]] - 大语言模型应用开发框架
  - **核心功能**：链式调用、工具集成、记忆管理
  - **适用场景**：构建复杂 AI 应用、RAG 系统、Agent 系统
  - **典型使用模式**：
    ```python
    from langchain import LLMChain
    from langchain.prompts import PromptTemplate
    ```
  - **关键能力**：支持工具集成、记忆管理、多模型适配
  - **对比优势**：模块化设计、丰富的集成
  - **常见坑点**：API 变更频繁、性能优化复杂

- [ ] [[LlamaIndex]] - 数据感知型 LLM 应用框架
  - **核心功能**：数据索引、查询转换、路由选择
  - **适用场景**：知识库问答、数据分析、个性化推荐
  - **典型使用模式**：
    ```python
    from llama_index import VectorStoreIndex, SimpleDirectoryReader
    documents = SimpleDirectoryReader("data").load_data()
    index = VectorStoreIndex.from_documents(documents)
    ```
  - **关键能力**：支持数据索引、查询优化
  - **对比优势**：数据处理能力强、查询优化
  - **常见坑点**：内存使用较高

- [ ] [[Ollama]] - 本地大语言模型运行工具
  - **核心功能**：本地模型运行、API 服务
  - **适用场景**：本地开发、离线部署
  - **典型使用模式**：
    ```bash
    ollama run llama2
    ```
  - **关键能力**：支持多种模型、本地运行
  - **对比优势**：易于部署、使用简单
  - **常见坑点**：性能依赖硬件

## 优化与部署工具

- [ ] [[ONNX]] - 开放神经网络交换格式
  - **核心功能**：模型格式转换、跨平台部署
  - **适用场景**：模型部署、跨框架使用
  - **典型使用模式**：
    ```python
    import onnx
    model = onnx.load("model.onnx")
    ```
  - **关键能力**：支持跨平台部署、模型优化
  - **对比优势**：广泛支持、性能优异
  - **常见坑点**：部分操作不支持

- [ ] [[TensorRT-LLM]] - NVIDIA 开发的大语言模型推理优化库
  - **核心功能**：模型编译优化、INT8/FP16 量化
  - **适用场景**：GPU 推理加速、生产部署
  - **关键能力**：支持量化、低延迟推理
  - **对比优势**：GPU 性能优异、企业级支持
  - **常见坑点**：依赖 NVIDIA 硬件、配置复杂

- [ ] [[GGML/GGUF]] - 高效推理格式
  - **核心功能**：模型量化、CPU 推理优化
  - **适用场景**：CPU 推理、边缘设备部署
  - **关键能力**：支持多种量化格式、CPU 优化
  - **对比优势**：低资源需求、跨平台支持
  - **常见坑点**：性能低于 GPU 优化方案

- [ ] [[bitsandbytes]] - 8 位量化库
  - **核心功能**：模型量化、内存优化
  - **适用场景**：低显存训练、推理
  - **典型使用模式**：
    ```python
    import bitsandbytes as bnb
    model = bnb.nn.Linear8bitLt(...)
    ```
  - **关键能力**：支持 8 位量化、内存优化
  - **对比优势**：易于集成、显著降低显存使用
  - **常见坑点**：部分模型兼容性问题

## 数据处理与评估库

- [ ] [[Datasets]] - Hugging Face 开发的数据集库
  - **核心功能**：数据集加载、预处理、流式处理
  - **适用场景**：模型训练数据准备、数据探索
  - **典型使用模式**：
    ```python
    from datasets import load_dataset
    dataset = load_dataset("imdb")
    ```
  - **关键能力**：支持流式处理、内存优化
  - **对比优势**：数据集丰富、易于使用
  - **常见坑点**：部分数据集下载较慢

- [ ] [[evaluate]] - Hugging Face 开发的评估库
  - **核心功能**：统一评估接口、多种评估指标
  - **适用场景**：模型评估、性能对比
  - **典型使用模式**：
    ```python
    import evaluate
    metric = evaluate.load("accuracy")
    ```
  - **关键能力**：支持多种评估指标、易于扩展
  - **对比优势**：接口统一、指标丰富
  - **常见坑点**：部分指标计算较慢

- [ ] [[TRL]] - 大语言模型训练库
  - **核心功能**：RLHF 训练、PEFT 方法
  - **适用场景**：模型对齐、指令微调
  - **典型使用模式**：
    ```python
    from trl import SFTTrainer
    trainer = SFTTrainer(model=model, tokenizer=tokenizer, ...)
    ```
  - **关键能力**：支持 RLHF、PEFT 方法
  - **对比优势**：易于使用、功能完整
  - **常见坑点**：依赖较复杂

- [ ] [[OpenAI Evals]] - OpenAI 开发的评估框架
  - **核心功能**：模型评估、基准测试
  - **适用场景**：模型能力评估、基准测试
  - **关键能力**：支持多种评估任务、结果分析
  - **对比优势**：评估维度全面、权威参考
  - **常见坑点**：主要针对 OpenAI 模型
