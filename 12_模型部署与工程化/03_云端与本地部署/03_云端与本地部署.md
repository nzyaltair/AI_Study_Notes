# 云端与本地部署

## 背景与发展

[[模型部署]]的运行环境选择直接影响成本、延迟、隐私和运维复杂度。云厂商从 2020 年起陆续推出托管 LLM 部署服务（AWS SageMaker、Azure OpenAI、Google Vertex AI），提供弹性 GPU 实例和自动扩缩容能力。与此同时，llama.cpp、Ollama 等工具使本地和边缘部署成为可能，满足数据不出域和低延迟需求。混合部署模式将敏感数据留在本地、弹性流量转发到云端，成为企业级部署的主流方案。

## 核心思想

云端部署利用弹性资源应对流量波动，按需付费降低初期投入；本地部署通过固定硬件投资换取数据主权和可控延迟。选择取决于数据敏感度、流量模式、性能要求和预算约束。生产环境通常采用混合策略：核心模型本地部署保障隐私和延迟，弹性容量由云端承接。

## 技术原理

### 云端平台

| 平台 | 特点 | 适用场景 |
|------|------|----------|
| **AWS SageMaker** | 托管部署，集成 S3/CloudWatch | 全托管 LLM 服务 |
| **Azure ML / OpenAI** | Azure OpenAI 托管 GPT 系列 | 企业级 GPT 部署 |
| **Google Vertex AI** | TPU 加速，AutoML 集成 | TPU 推理场景 |
| **阿里云 PAI** | 国内合规，预训练模型丰富 | 国内企业部署 |

### 硬件选型

| 硬件 | 典型产品 | 支持模型规模 | 特点 |
|------|----------|-------------|------|
| **GPU** | A100/H100/L4 | 7B-405B | 通用并行计算，Tensor Core 加速 |
| **TPU** | TPU v4/v5 | 大规模分布式推理 | 专为矩阵运算优化 |
| **NPU** | 华为昇腾 910/310 | 7B-70B | 国产硬件，能效比高 |
| **CPU** | AMD EPYC / Intel Xeon | 1B-7B（量化后） | 通用性强，成本最低 |

### 本地推理引擎

| 引擎 | 核心特点 | 硬件支持 |
|------|----------|----------|
| **llama.cpp** | 轻量 C++ 实现，GGUF 量化格式 | CPU/GPU/Metal |
| **Ollama** | 一键部署，CLI 友好 | CPU/GPU/Metal |
| **ONNX Runtime** | 跨平台，硬件后端丰富 | CPU/GPU/NPU |
| **mlx** | Apple Silicon 优化，统一内存 | M 系列芯片 |
| **TensorRT-LLM** | NVIDIA GPU 内核级优化 | NVIDIA GPU |

### 并行部署策略

- **张量并行（TP）**：将模型单层内权重切分到多 GPU，需高带宽互联（NVLink）
- **流水线并行（PP）**：将模型不同层分配到不同 GPU，通信量小但存在气泡
- **数据并行（DP）**：多副本独立处理不同请求，通过负载均衡分发

### 弹性伸缩

基于 Kubernetes HPA 实现：监控 GPU 利用率或请求队列长度，触发 Pod 副本增减。冷启动优化通过模型预热和镜像缓存减少扩容延迟。

## 发展演进

云托管服务 → 专用推理引擎 → 边缘 LLM（llama.cpp/Ollama） → 混合云部署 → Serverless LLM → WebGPU 浏览器端推理

## 关键算法·模型

- **GGUF 量化格式**：llama.cpp 定义的模型存储格式，支持 INT2-INT8 量化，通过 k-quants 方法减少精度损失
- **模型分片加载**：大模型按层切分存储，按需加载到显存，支持超过单卡显存的模型推理
- **MIG（Multi-Instance GPU）**：A100/H100 支持将单 GPU 切分为多个隔离实例，提高资源利用率

## 应用场景

- **云端生产服务**：面向公众的 LLM API 平台，需弹性扩缩容
- **企业私有化**：金融、医疗等数据敏感行业的本地部署
- **边缘推理**：移动端、IoT 设备上的离线模型推理
- **混合部署**：核心推理本地化，峰值流量溢出到云端
