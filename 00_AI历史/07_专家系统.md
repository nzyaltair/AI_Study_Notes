# 专家系统

## 背景与发展

[[专家系统]]是[[符号主义]]在 1970s-1980s 的商业化成果，也是 AI 第一次在工业界取得广泛应用的尝试。其核心思想是将人类专家的知识编码为规则，让计算机模拟专家的推理过程进行决策。[[知识工程]]为构建专家系统提供了系统化方法论。

## 核心思想

专家系统由知识库（Knowledge Base）和推理引擎（Inference Engine）组成。知识库存储领域专家的知识（通常以 IF-THEN 规则形式），推理引擎根据输入事实和知识库中的规则进行推理，得出结论或建议。系统将知识与推理分离，使知识库可独立更新而不影响推理逻辑。

## 技术原理

### 系统架构

- **知识库**：存储领域知识，通常以产生式规则表示
  - 规则形式：`IF 条件 THEN 结论 (CF)`，其中 CF 为确定性因子
  - 例：`IF 发烧 AND 咳嗽 THEN 感冒 (0.8)`
- **推理引擎**：
  - **前向链**（Forward Chaining）：从已知事实出发，匹配规则前提，推导新事实（数据驱动）
  - **后向链**（Backward Chaining）：从目标假设出发，逆向寻找支持目标的规则（目标驱动）
- **工作内存**：存储当前已知事实和推导中间结果
- **解释器**：向用户展示推理链路，说明结论的得出过程

### 不确定性处理

现实世界知识常带有不确定性，需专门机制处理：

- **确定性因子**（Certainty Factor, MYCIN）：为规则和事实附加 $[-1, 1]$ 的置信度，组合规则时按特定公式传播不确定性
- **贝叶斯推理**（PROSPECTOR）：用概率理论处理不确定性，$P(H|E) = \frac{P(E|H)P(H)}{P(E)}$
- **模糊逻辑**（Fuzzy Logic）：处理模糊和渐变概念，用隶属函数表示程度的连续性

### 知识获取

知识获取是构建专家系统的核心环节，也是最困难的环节。[[知识工程]]研究从领域专家处提取知识并形式化的方法：

- 结构化访谈与协议分析
- 概念组织与领域建模
- 机器辅助规则提取

### 推理流程

前向链示例：

```
已知: 发烧=True, 咳嗽=True
规则1: IF 发烧 AND 咳嗽 THEN 感冒 (0.8)
规则2: IF 感冒 THEN 建议休息 (0.9)
推导: 感冒(0.8) → 建议休息(0.8×0.9=0.72)
```

## 关键算法·模型

| 系统 | 年代 | 领域 | 技术特点 |
|------|------|------|----------|
| DENDRAL | 1965 | 化学质谱分析 | 首个专家系统，启发式规则推断分子结构 |
| MYCIN | 1976 | 医学诊断 | 确定性因子处理不确定性，解释能力强 |
| XCON | 1980 | 计算机配置 | 首个商业成功案例，为 DEC 节省数千万美元 |
| PROSPECTOR | 1979 | 地质勘探 | 贝叶斯推理，成功预测钼矿位置 |

## 应用场景

- 医学诊断：MYCIN 诊断细菌感染并推荐抗生素
- 工业配置：XCON 配置 VAX 计算机系统
- 地质勘探：PROSPECTOR 评估矿产前景
- 化学分析：DENDRAL 从质谱数据推断分子结构

## 与其他技术关系

- 专家系统是[[符号主义]]的商业化应用
- [[知识工程]]为其提供系统化的构建方法论
- 知识表示采用[[知识表示与推理]]中的产生式规则和框架方法
- 知识获取瓶颈的暴露直接推动了[[机器学习]]的兴起——让机器从数据自动学习
- 专家系统的规则思想在现代规则引擎和大语言模型 Agent 系统中以新形式延续
