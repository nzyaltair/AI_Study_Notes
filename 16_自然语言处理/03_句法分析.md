# 句法分析

## 1. 概述

句法分析（Syntactic Parsing）研究如何将线性排列的词序列解析为层次化的语法结构。它是从"词"到"句"的关键一步——理解句子中词与词之间的语法关系（谁是主语、谁是谓语、哪个词修饰哪个词），从而构建句子的结构化表示。

句法分析主要有两大方向：**成分句法分析**（Constituency Parsing）和**依存句法分析**（Dependency Parsing）。成分分析关注短语的层次嵌套结构（NP、VP 等），依存分析关注词与词之间的直接修饰关系。在大模型时代，显式的句法分析在端到端模型中逐渐隐化，但句法结构仍是理解语言深层机制的重要理论工具。

## 3. 核心概念

### 3.1 成分句法分析（Constituency Parsing）

将句子解析为嵌套的短语结构树。使用**上下文无关文法（CFG）**定义：

$$S \rightarrow NP \; VP$$
$$NP \rightarrow Det \; N \mid Det \; Adj \; N \mid ...$$
$$VP \rightarrow V \; NP \mid V \; NP \; PP \mid ...$$

**成分句法树示例**：

```
                S
               / \
             NP    VP
            /  \   / \
          Det   N  V   NP
          |    |   |   / \
         The  cat ate Det N
                         |  |
                        the fish
```

**概率上下文无关文法（PCFG）**：为每条规则赋予概率，选择概率最大的解析树。

### 3.2 依存句法分析（Dependency Parsing）

分析词与词之间的直接修饰关系，构建依存树：

```
        ate
       /    \
     cat    fish
     |       |
    The     the
```

- **核心词（Head）**：被修饰的词
- **依存词（Dependent）**：修饰核心词的词
- **依存关系类型**：nsubj（主语）、dobj（宾语）、det（限定词）、amod（形容词修饰）等

**Universal Dependencies（UD）**：跨语言统一的依存关系标注体系，覆盖 200+ 语言。

### 3.3 成分 vs 依存

| 维度 | 成分句法 | 依存句法 |
|:---|:---|:---|
| 结构 | 层次嵌套短语树 | 词与词的直接关系图 |
| 信息 | 短语类型（NP/VP） | 语法功能（主语/宾语） |
| 表示 | 树（含非终结节点） | 树（仅终结节点） |
| 应用 | 句法理论、语法检查 | 信息抽取、机器翻译 |
| 标注集 | Penn Treebank | Universal Dependencies |

两者可以相互转换——成分树可通过"中心词投影"转换为依存树。

## 4. 技术原理

### 4.1 PCFG 解析

PCFG 为每条重写规则 $A \rightarrow \beta$ 赋予概率 $P(\beta | A)$，解析树的概率为所有规则概率的乘积：

$$P(T) = \prod_{(A \rightarrow \beta) \in T} P(\beta | A)$$

**CYK 算法**（$O(n^3 |G|)$）：动态规划求解最优解析树，$n$ 为句子长度，$|G|$ 为文法规则数。

**局限性**：PCFG 的独立性假设过强——规则的概率不依赖上下文，导致对结构歧义处理不佳。

### 4.2 Transition-Based 依存分析

将依存分析建模为移进-归约（Shift-Reduce）过程，使用栈和缓冲区：

| 动作 | 操作 | 说明 |
|:---|:---|:---|
| Shift | 将缓冲区首词压入栈 | 读入新词 |
| Left-Arc | 栈顶词 → 次顶词（左依存） | 次顶词依存于栈顶 |
| Right-Arc | 次顶词 → 栈顶词（右依存） | 栈顶依存于次顶词 |
| Reduce | 弹出栈顶 | 完成处理 |

使用分类器预测每步动作。Chen & Manning (2014) 用神经网络分类器实现了首个神经依存分析器。

### 4.3 Graph-Based 依存分析

将依存分析建模为在完全图上寻找最大生成树（MST）：

- 为每条可能的边 $(h, d)$（核心词 $h$ 到依存词 $d$）计算分数 $\text{score}(h, d)$
- 寻找总分数最大的依存树：$\hat{T} = \arg\max_T \sum_{(h,d) \in T} \text{score}(h, d)$
- 使用 Chu-Liu/Edmonds 算法求解 MST

McDonald et al. (2005) 的 MSTParser 是经典方法。

### 4.4 基于 BERT 的句法分析

BERT 的中间层表示被发现隐式编码了句法结构。Hewitt & Manning (2019) 发现通过对 BERT 表示做线性变换可以恢复句法依存树：

$$\text{distance}(w_i, w_j) \approx (B h_i)^T (B h_j)$$

其中 $B$ 是可学习的投影矩阵，$h_i$ 是 BERT 的隐藏表示。这表明预训练语言模型内部学习到了句法结构。

## 5. 关键方法/模型

### 5.1 Stanford Parser

斯坦福大学开发的句法分析器，支持成分分析和依存分析，基于 PCFG + 判别式重排序。广泛用于学术研究和工业应用。

### 5.2 spaCy Parser

工业级 NLP 库 spaCy 内置的依存分析器，基于 Transition-based + CNN 特征提取器，速度极快，适合生产环境。

### 5.3 Biaffine Parser

Dozat & Manning (2017) 提出的双仿射注意力依存分析器，使用 BiLSTM + Biaffine 打分函数，精度高且速度快，是预训练模型之前的 SOTA。

## 6. 优势与局限

### 优势

1. **结构化理解**：句法树提供句子的结构化表示，有助于深层语义理解
2. **跨任务复用**：句法信息可用于信息抽取、机器翻译、语法纠错等
3. **理论价值**：句法分析是语言学理论在 NLP 中的直接体现

### 局限

1. **歧义性**：一个句子可能有多种合法的句法分析，消歧困难
2. **领域适应**：在非标准文本（社交媒体、口语）上性能下降明显
3. **语言覆盖**：高质量句法标注仅覆盖少数语言
4. **LLM 时代的角色**：端到端 LLM 隐式处理句法，显式句法分析的独立价值在部分场景减弱

## 7. 应用场景

- **语法纠错**：句法分析帮助检测语法错误
- **信息抽取**：依存路径用于关系抽取的特征
- **机器翻译**：句法树指导源语言到目标语言的转换
- **搜索引擎**：句法分析改善查询理解
- **代码分析**：编程语言的语法分析与自然语言句法分析共享理论基础

## 9. 前沿发展

- **LLM 隐式句法**：研究 LLM 内部如何编码句法结构，探针（Probing）方法分析中间层表示
- **多语言句法**：Universal Dependencies 项目持续扩展语言覆盖
- **语义与句法统一**：将句法分析和语义分析统一到端到端框架
- **句法增强 LLM**：将显式句法结构注入 LLM 以提升结构化理解能力
- **构式语法**：超越传统 CFG，研究构式（Construction）层面的语言结构
