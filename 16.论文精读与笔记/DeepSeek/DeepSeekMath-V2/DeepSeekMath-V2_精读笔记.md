# 🎯 研究背景与问题
- 研究背景：
  - 大型语言模型在数学推理方面取得了显著进展，这是AI的重要测试平台，进一步发展可能会影响科学研究
  - 通过奖励正确最终答案的强化学习来扩展推理能力，LLMs在一年内从表现不佳发展到在AIME和HMMT等定量推理竞赛中达到饱和
  - 然而，这种方法面临根本性限制：正确答案并不保证正确的推理过程
  - 许多数学任务如定理证明需要严格的逐步推导，而不是数值答案，使得最终答案奖励不适用

- 现有方法的核心瓶颈：
  - 追求更高的最终答案准确率并未解决关键问题：正确答案并不保证正确的推理
  - 最终答案奖励机制不适用于不需要数值最终答案的定理证明任务
  - 缺乏自我验证能力，无法在没有参考解决方案的情况下评估证明的正确性
  - 生成-验证差距不足，阻碍了进一步改进

- 作者明确要解决的问题（Problem Statement）：
  - 如何训练一个准确且忠实的基于LLM的验证器，用于定理证明
  - 如何使用验证器作为奖励模型来训练证明生成器，激励生成器在最终确定证明之前识别和解决尽可能多的问题
  - 如何通过扩展验证计算来自动标记难以验证的新证明，创建训练数据以进一步改进验证器

# 🚀 核心贡献（Contributions）
- Contribution 1：
  - 提出了一种训练准确且忠实的基于LLM的验证器的方法，用于定理证明
  - 新在于：验证器能够在没有参考解决方案的情况下识别证明中的问题

- Contribution 2：
  - 设计了一种使用验证器作为奖励模型的证明生成器训练方法，激励生成器在最终确定证明之前识别和解决尽可能多的问题
  - 新在于：使模型明确了解其奖励函数，并通过 deliberate推理而非盲目试错来最大化奖励

- Contribution 3：
  - 提出了一种扩展验证计算的方法，自动标记难以验证的新证明，创建训练数据以进一步改进验证器
  - 新在于：通过这种方式维持生成-验证差距，即使生成器变得更强

# 🧠 方法详解（Method）
## 整体框架
- 模型/算法整体流程说明：
  - 基于DeepSeek-V3.2-Exp-Build构建
  - 训练过程包括两个主要组件：验证器和证明生成器
  - 验证器训练：通过扩展验证计算自动标记难以验证的新证明，创建训练数据
  - 证明生成器训练：使用验证器作为奖励模型，激励生成器识别和解决证明中的问题
  - 迭代改进循环：验证反馈优化证明生成 → 扩展验证计算创建训练数据 → 改进验证器 → 进一步优化证明生成

- 输入 / 输出定义：
  - 输入：数学问题/定理
  - 输出：完整的数学证明

## 关键模块拆解
- 模块 1：验证器（Verifier）
  - 功能：识别证明中的问题，评估证明的正确性
  - 设计：基于LLM，能够在没有参考解决方案的情况下工作
  - 训练：通过扩展验证计算自动标记难以验证的新证明，创建训练数据

- 模块 2：证明生成器（Proof Generator）
  - 功能：生成数学证明
  - 设计：使用验证器作为奖励模型，激励生成器识别和解决证明中的问题
  - 训练：通过验证反馈优化生成策略，使生成器能够迭代改进其证明

- 模块 3：迭代改进循环
  - 功能：维持生成-验证差距，持续改进模型性能
  - 设计：验证反馈优化证明生成 → 扩展验证计算创建训练数据 → 改进验证器 → 进一步优化证明生成

## 关键公式与直觉解释
- 公式：
  - 论文中未明确给出具体公式

- 含义：
  - 验证器通过识别证明中的问题来评估证明质量
  - 证明生成器通过验证器的反馈来优化其生成策略
  - 扩展验证计算用于创建训练数据，改进验证器

- 直觉理解：
  - 类似于人类数学家的工作方式：生成证明，检查错误，修正问题，重复直到证明无懈可击
  - 验证器充当严格的审阅者，帮助生成器发现和解决问题
  - 扩展验证计算相当于让多个专家审阅同一个证明，提高发现问题的概率

- 与传统方法对比：
  - 传统方法：基于最终答案的奖励，不关注推理过程的正确性
  - 新方法：基于验证器的反馈，关注推理过程的严密性和正确性

## 实验设置

  - Putnam 2024

- Baseline：

- 评价指标：

  - 证明正确性：验证器评估的证明质量

- 对照实验设计思路：

  - 在Putnam 2024上取得了近乎完美的118/120分
  - 这些结果是在扩展测试时间计算下取得的

- 是否支撑作者主张：

  - 结果表明，通过验证器反馈和扩展验证计算，可以显著提高模型的定理证明能力

# 🔍 讨论与局限性
- 方法优势：
  - 关注推理过程的正确性，而不仅仅是最终答案
  - 能够处理不需要数值最终答案的定理证明任务
  - 通过迭代改进循环持续提高模型性能
  - 验证器能够在没有参考解决方案的情况下工作，适用于开放问题

- 潜在局限：
  - 扩展验证计算可能需要大量的计算资源
  - 验证器的准确性和忠实性仍需进一步提高
  - 模型可能在处理极其复杂的数学问题时遇到困难
  - 训练过程可能需要大量的高质量证明数据

- 隐含假设：
  - 验证器能够准确识别证明中的问题
  - 扩展验证计算能够有效地创建训练数据
  - 生成-验证循环能够持续提高模型性能
  - 模型能够通过验证器的反馈学会改进其证明

- 实际应用风险：
  - 验证器可能会产生误报或漏报，影响证明评估的准确性
  - 模型可能会过度适应验证器的偏好，而不是真正的数学正确性
  - 计算资源需求可能限制在资源受限环境中的部署
