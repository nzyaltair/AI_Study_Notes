# RLHF（Reinforcement Learning from Human Feedback）

## 1. 简介

RLHF（从人类反馈中进行强化学习）是一种[[偏好对齐方法]]，通过人类反馈来训练和优化大型语言模型，使其输出与人类偏好保持一致。它是目前最成熟、应用最广泛的偏好对齐技术之一。

## 2. 核心原理

### 2.1 RLHF 基本框架

RLHF 包含三个主要阶段：

1. **收集人类偏好数据**：让人类标注者对模型生成的多个输出进行排序或评分
2. **训练奖励模型（RM）**：使用收集到的偏好数据训练一个奖励模型，用于预测人类偏好
3. **强化学习优化**：使用强化学习算法（如 PPO），以奖励模型的输出为指导，优化语言模型

### 2.2 经典 RLHF 流程

```
模型生成候选输出 → 人类排序/评分 → 训练奖励模型 → PPO 优化模型 → 生成更好的输出
```

### 2.3 数学形式

#### 2.3.1 奖励模型训练

对于输入 `x` 和输出 `y`，奖励模型 `r_θ(x, y)` 预测人类对 `y` 的偏好得分。训练目标是最小化预测得分与人类偏好的差异：

```
L(θ) = -E_{(x, y_w, y_l)~D} [log(σ(r_θ(x, y_w) - r_θ(x, y_l)))]
```

其中，`y_w` 是人类偏好的输出，`y_l` 是人类不偏好的输出，`σ` 是 sigmoid 函数。

#### 2.3.2 PPO 优化

PPO 优化的目标函数包含三个部分：

```
L(φ) = E_{(x, y)~D} [r_θ(x, y) - β D_{KL}(p_φ(y|x) || p_{φ0}(y|x))]
```

其中：
- `r_θ(x, y)` 是奖励模型的输出
- `β` 是 KL 散度惩罚系数
- `D_{KL}(p_φ || p_{φ0})` 是当前模型与初始模型的 KL 散度
- `p_{φ0}` 是监督微调后的初始模型

## 3. 工程实现

### 3.1 数据收集

#### 3.1.1 数据格式

- **比较数据**：(输入, 偏好输出, 非偏好输出) 三元组
- **评分数据**：(输入, 输出, 评分) 三元组
- **排序数据**：(输入, [输出1, 输出2, 输出3]) 以及对应的排序

#### 3.1.2 标注指南

- 明确的标注标准，包括相关性、正确性、安全性、有用性等维度
- 定期校准标注者，确保标注一致性
- 使用多人标注，减少个体偏见

### 3.2 奖励模型训练

#### 3.2.1 模型架构

- 通常基于监督微调后的语言模型架构
- 移除生成头，添加回归头输出标量奖励
- 可以冻结部分底层参数，只训练顶层参数

#### 3.2.2 训练技巧

- 使用对比损失，优化模型区分偏好和非偏好输出
- 采用梯度裁剪，防止梯度爆炸
- 使用混合精度训练，提高训练效率

### 3.3 PPO 训练

#### 3.3.1 核心组件

- **策略网络**：需要优化的语言模型
- **价值网络**：预测状态值，减少方差
- **奖励模型**：提供外部奖励

#### 3.3.2 关键参数

| 参数 | 推荐范围 | 说明 |
|------|----------|------|
| `learning_rate` | 1e-6 - 5e-5 | PPO 学习率，通常比监督微调低 |
| `batch_size` | 32 - 256 | 基于 GPU 内存调整 |
| `kl_coef` | 0.1 - 1.0 | KL 散度惩罚系数 |
| `clip_range` | 0.1 - 0.3 | PPO 裁剪范围 |
| `num_epochs` | 3 - 10 | 每个 batch 的训练轮次 |

## 4. 常见问题与解决方案

### 4.1 奖励模型过拟合

- **症状**：奖励模型在训练数据上表现良好，但在新数据上表现差
- **解决方案**：
  - 增加训练数据多样性
  - 添加正则化（如 Dropout）
  - 使用更大的模型作为奖励模型
  - 采用早期停止策略

### 4.2 PPO 训练不稳定

- **症状**：训练过程中奖励波动大，甚至出现模式崩溃
- **解决方案**：
  - 调整 KL 散度惩罚系数
  - 降低学习率
  - 增加 batch 大小
  - 使用更保守的裁剪范围

### 4.3 人类偏好不一致

- **症状**：不同标注者的偏好存在差异
- **解决方案**：
  - 完善标注指南
  - 增加标注者校准
  - 使用多人标注并取平均值
  - 训练鲁棒的奖励模型，能够处理偏好不一致

## 5. 变体与扩展

### 5.1 Iterative RLHF

- **核心思想**：迭代执行 RLHF 流程，不断提高模型性能
- **流程**：
  1. 初始模型 → RLHF 优化 → 更好的模型
  2. 更好的模型 → 生成更多样本 → 人类标注
  3. 重复步骤 1-2

### 5.2 Reward Modeling with Comparison Data

- **核心思想**：使用比较数据而非绝对评分训练奖励模型
- **优势**：
  - 减少标注难度
  - 提高标注一致性
  - 更好地捕获相对偏好

### 5.3 Constitutional AI

- **核心思想**：结合规则和 RLHF，使模型遵循预定义的宪法
- **流程**：
  1. 使用宪法生成自我批评
  2. 基于自我批评优化模型
  3. 结合 RLHF 进一步优化

## 6. 应用场景

- **对话系统**：训练安全、友好、有用的对话模型
- **内容生成**：生成符合人类偏好的文本、代码等
- **搜索引擎**：优化搜索结果排序
- **推荐系统**：生成个性化推荐

## 7. 前沿研究

- **2024 ICLR**：《Improving RLHF Stability with Adaptive KL Coefficients》提出了自适应 KL 系数，提高了 PPO 训练的稳定性
- **2025 NeurIPS**：《Efficient RLHF with Synthetic Feedback》结合合成反馈和人类反馈，降低了对人类标注的依赖
- **2025 ACL**：《Multimodal RLHF for Vision-Language Models》将 RLHF 扩展到多模态领域

## 8. 最佳实践

### 8.1 数据准备

- **多样化数据集**：覆盖不同领域、不同难度的任务
- **高质量标注**：确保标注一致性和准确性
- **定期更新数据**：随着模型改进，不断收集新的标注数据

### 8.2 模型选择

- **奖励模型**：使用与目标模型同架构但稍小的模型
- **策略模型**：基于监督微调后的模型，确保初始性能良好

### 8.3 训练策略

- **逐步增加难度**：先在简单任务上训练，再扩展到复杂任务
- **混合训练**：结合监督微调损失和 PPO 损失，提高稳定性
- **定期评估**：使用自动评估和人工评估相结合的方式，监控模型性能
