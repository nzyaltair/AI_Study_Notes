# 专家系统

## 一句话理解

利用预先编码的领域知识（规则）和推理引擎，模拟人类专家进行特定领域诊断和决策的符号AI系统。

## 背景与发展

专家系统是符号AI黄金时代（1970s-1980s）的标志性产物。其核心思想是将领域专家的知识显式编码为规则，通过推理引擎自动推理。DENDRAL（1965）开创了专家系统先河，MYCIN（1977）展示了医疗诊断的可行性，XCON（1980）证明了商业价值。

| 年代 | 里程碑 | 作者/机构 | 意义 |
|:---|:---|:---|:---|
| 1965 | DENDRAL | Feigenbaum, Lederberg | 首个知识密集型专家系统 |
| 1977 | MYCIN | Shortliffe | 医疗诊断专家系统，引入确定性因子 |
| 1978 | PROSPECTOR | SRI | 矿物勘探，贝叶斯推理 |
| 1980 | XCON (R1) | DEC/Carnegie Mellon | 商业成功——每年节省数千万美元 |
| 1981 | 第五代计算机 | 日本MITI | 国家级专家系统计划 |
| 1986 | 神经网络复兴 | Rumelhart等 | 连接主义挑战符号主义 |
| 1990s | 衰落 | — | 知识获取瓶颈和维护成本导致专家系统衰退 |
| 2010s | 复兴 | — | 规则引擎在推荐、风控领域持续应用 |

## 核心概念

### 基本架构

专家系统由三个核心组件构成：

```
    用户接口
       │
  ┌────┴────┐
  │ 推理引擎 │ ←─── 推理机制（前向/反向链）
  └────┬────┘
       │
  ┌────┴────┐
  │  知识库  │ ←─── 领域知识（规则+事实）
  └────┬────┘
       │
  ┌────┴────┐
  │ 工作内存 │ ←─── 当前事实/中间结论
  └─────────┘
```

- **知识库**（Knowledge Base）：存储领域专家的知识，通常以规则形式
- **推理引擎**（Inference Engine）：根据知识库中的规则和工作内存中的事实进行推理
- **工作内存**（Working Memory）：存储当前问题的事实和中间推理结果

### 知识获取

知识获取是专家系统的**最大瓶颈**——将领域专家的隐性知识转化为显式规则。方法包括：

- **人工访谈**：知识工程师与专家对话，提取规则
- **机器学习**：从数据中自动学习规则（决策树等）
- **类比与案例**：基于案例的推理（CBR）

### 不确定性处理

真实世界知识常带不确定性，专家系统需要处理：

| 方法 | 机制 | 代表系统 |
|:---|:---|:---|
| 确定性因子（CF） | 规则附加置信度 $[-1, 1]$ | MYCIN |
| 贝叶斯概率 | 用先验和似然更新后验 | PROSPECTOR |
| 模糊逻辑 | 真值在 $[0, 1]$ 连续 | 模糊控制 |
| Dempster-Shafer理论 | 证据理论处理无知 | 传感器融合 |

MYCIN的确定性因子传播规则：

$$CF_{combined} = CF_1 + CF_2 \times (1 - CF_1) \quad \text{（两个证据支持同一结论）}$$

## 工作原理

### 前向链推理

从已知事实出发，匹配规则前件，执行规则产生新事实，循环直至目标得证或无规则可触发。

```
已知: 动物有羽毛
规则: IF 有羽毛 THEN 是鸟
→ 推出: 动物是鸟
规则: IF 是鸟 AND 不会飞 AND 体型大 THEN 是鸵鸟
...
```

适合数据驱动的场景（如监控、诊断）。

### 反向链推理

从目标假设出发，查找能推出目标的规则，检查规则前件是否满足，递归直到所有前件被验证或否定。

```
目标: 动物是鸵鸟?
  ← 需要: 是鸟 AND 不会飞 AND 体型大
    ← 需要: 有羽毛? (查规则: IF 有羽毛 THEN 是鸟)
      ← 事实: 有羽毛 ✓
    ← 事实: 不会飞 ✓
    ← 事实: 体型大 ✓
  → 结论: 是鸵鸟 ✓
```

适合目标驱动的场景（如诊断、验证）。

### 冲突消解

当多条规则同时可触发时，需要冲突消解策略：

| 策略 | 说明 |
|:---|:---|
| 优先级 | 按预先设定的优先级选择 |
| 最近匹配 | 选择最近加入工作内存的事实所触发的规则 |
| 最具体匹配 | 选择条件最多的规则（最具体） |
| 首次匹配 | 选择第一个匹配的规则 |

## 应用场景

| 领域 | 系统 | 功能 |
|:---|:---|:---|
| 医疗诊断 | MYCIN, CADUCEUS | 感染诊断、内科诊断 |
| 化学 | DENDRAL | 质谱分析推断分子结构 |
| 地质勘探 | PROSPECTOR | 矿藏评估 |
| 计算机配置 | XCON | VAX计算机配置 |
| 金融风控 | 规则引擎 | 信用评估、反欺诈 |
| 工业控制 | 模糊控制 | 温度控制、过程控制 |

## 优势与不足

### 优势
- 知识显式编码，推理过程可解释
- 在窄领域可达到甚至超越人类专家水平
- 知识与推理分离，便于独立更新

### 不足
- **知识获取瓶颈**：构建和维护规则库成本极高
- **脆弱性**：规则未覆盖的情况会导致系统失效
- **无学习能力**：不能从经验中自动改进（传统专家系统）
- **维护困难**：规则数量增长后冲突和冗余难以管理
