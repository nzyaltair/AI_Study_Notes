# 从零构建GPT

## 背景与发展

本笔记整理自 Andrej Karpathy 的系列教程，从零构建理解GPT的核心机制。通过从最简单的二元语言模型逐步构建到完整的Transformer架构，深入理解[[Transformer]]的内部工作原理。

Transformer的本质是**通信（自注意力）与计算（前馈网络）的交替堆叠**。通过逐步添加各组件，从验证损失2.45的二元模型构建到验证损失1.48的完整GPT。

## 核心思想

通过从最简单的语言模型逐步进化到完整GPT，揭示Transformer各组件的作用：自注意力让序列中每个位置直接交互，因果掩码确保自回归生成，多头注意力捕获不同子空间的模式，残差连接与层归一化稳定深层训练。

## 技术原理

### 语言模型演进路径

```
二元语法模型（Bigram）
  → 多层感知机（MLP + 字符嵌入）
  → 激活函数与梯度（初始化、BatchNorm）
  → 手动反向传播
  → WaveNet（分层网络）
  → Transformer（GPT）
```

### GPT核心组件

1. **自注意力机制**：$\text{softmax}(QK^T/\sqrt{d_k})V$
2. **[[位置编码]]**：注入序列顺序信息
3. **因果掩码**：下三角矩阵，确保只关注左侧token
4. **多头注意力**：多子空间并行注意力
5. **残差连接 + LayerNorm**：稳定深层训练
6. **前馈网络（FFN）**：逐位置非线性变换

### 因果掩码实现

```python
# 下三角掩码，禁止位置i关注位置j>i
mask = torch.tril(torch.ones(seq_len, seq_len))
scores = scores.masked_fill(mask == 0, float('-inf'))
```

### 训练实践

- **数据**：tiny shakespeare（~1MB，65个字符）
- **模型**：~1000万参数
- **结果**：验证损失1.48，生成莎士比亚风格文本

### 分词器

BPE（Byte Pair Encoding）分词原理：从字符级到子词级的过渡，通过迭代合并最高频字节对扩展词表。

## 发展演进

自动微分引擎（Micrograd）→ 字符级语言模型（makemore）→ Transformer核心组件 → 完整GPT → 分词器（BPE）→ 复现GPT-2（1.24亿参数）

nanoGPT项目（Karpathy）是最简洁的GPT实现：`model.py`（~300行）定义模型，`train.py`（~300行）训练逻辑，支持从字符级到GPT-2规模的训练。

## 关键算法·模型

| 阶段 | 核心内容 | 验证损失 |
|------|---------|---------|
| Bigram | 二元统计模型 | 2.45 |
| MLP | 字符嵌入 + 隐藏层 | 2.08 |
| GPT (nano) | 完整Transformer | 1.48 |
