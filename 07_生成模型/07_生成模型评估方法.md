# 生成模型评估方法

## 1. 概述

生成模型评估是一个开放性难题——没有单一指标能完美衡量生成质量。好的评估体系需要同时考虑**样本保真度（Fidelity）**和**分布覆盖度（Diversity）**，以及条件生成场景下的**条件一致性（Alignment）**。

本文档系统梳理生成模型的主流评估指标，包括基于分布的指标（FID、IS）、基于样本的指标、以及覆盖度-保真度框架。

## 2. 发展历史

| 年代 | 里程碑 | 意义 |
|------|--------|------|
| 2016 | Inception Score (IS) | GAN 评估的早期标准 |
| 2017 | FID (Fréchet Inception Distance) | 更稳健的分布距离度量 |
| 2018 | Precision & Recall | 分离保真度与覆盖度 |
| 2019 | KID (Kernel Inception Distance) | 无偏的核距离度量 |
| 2020 | Improved Precision & Recall (k-NN) | 更实用的覆盖度评估 |
| 2022 | CLIPScore | 文本-图像对齐评估 |
| 2023 | CMMD (CLIP-MMD) | 基于 CLIP 嵌入的分布距离 |
| 2023+ | 人工评估 + LLM 评判 | 补充自动指标的不足 |

## 3. 核心概念

### 3.1 评估维度

| 维度 | 定义 | 直觉 |
|------|------|------|
| **保真度 (Fidelity)** | 生成样本是否逼真 | "看起来真实吗？" |
| **多样性 (Diversity)** | 生成样本是否覆盖数据分布 | "够丰富吗？" |
| **一致性 (Alignment)** | 条件生成是否匹配条件 | "符合要求吗？" |
| **新颖性 (Novelty)** | 是否生成训练集中没有的样本 | "有创造力吗？" |

### 3.2 核心矛盾

保真度与多样性之间存在权衡：
- GAN：高保真、低多样性（模式崩溃）
- VAE：低保真、高多样性
- 扩散模型：高保真、高多样性（但采样慢）

好的评估指标应能**分离**这两个维度。

### 3.3 特征提取器

大多数指标依赖于预训练的特征提取器：
- **Inception-V3**（在 ImageNet 上训练）：FID、IS 的标准提取器
- **CLIP**（在图文对上训练）：CMMD、CLIPScore 的提取器
- **DINO/DINOv2**（自监督训练）：更鲁棒的特征表示

特征提取器的选择影响指标的有效性，也是其局限来源。

## 4. 技术原理

### 4.1 Inception Score (IS)

**核心思想**：好的生成模型应产生（1）分类器确信的样本（高保真）和（2）类别分布均匀的样本（高多样性）。

$$\text{IS} = \exp\left(\mathbb{E}_{x \sim p_g}\left[D_{KL}(p(y|x) \| p(y))\right]\right)$$

其中：
- $p(y|x)$：Inception-V3 对生成样本 $x$ 的类别预测分布（应尖锐 = 高保真）
- $p(y)$：所有生成样本的边缘类别分布（应均匀 = 高多样性）

**局限**：
- 不比较真实数据分布（只用生成样本）
- 依赖 Inception-V3（仅 ImageNet 1000 类）
- 对模式崩溃不敏感
- 可被对抗样本欺骗

### 4.2 Fréchet Inception Distance (FID)

**核心思想**：将真实数据和生成数据通过 Inception-V3 提取特征，假设特征服从多元高斯分布，计算两个高斯之间的 Fréchet 距离（即 Wasserstein-2 距离）。

$$\text{FID} = \|\mu_r - \mu_g\|^2 + \text{Tr}(\Sigma_r + \Sigma_g - 2(\Sigma_r \Sigma_g)^{1/2})$$

其中 $(\mu_r, \Sigma_r)$ 和 $(\mu_g, \Sigma_g)$ 分别为真实数据和生成数据特征的均值和协方差。

**优势**：
- 直接比较真实数据与生成数据的分布
- 对模式崩溃更敏感
- 广泛接受的标准指标

**局限**：
- 依赖 Inception-V3 特征
- 高斯假设可能不成立
- 需要足够大的样本量（≥10K）才稳定
- 不能分离保真度和多样性
- 可能与人感知不一致

### 4.3 KID (Kernel Inception Distance)

使用最大均值差异（MMD）替代 Fréchet 距离：

$$\text{KID} = \text{MMD}^2(X_{real}, X_{gen})$$

使用多项式核 $k(x, y) = \left(\frac{1}{d} x^T y + 1\right)^3$。

**优势**：无偏估计（在样本量有限时更可靠），无需高斯假设。

### 4.4 Precision & Recall（改进版）

使用 k-NN 在特征空间中分离保真度和覆盖度：

- **Precision（保真度）**：生成样本落入真实数据流形中的比例
- **Recall（覆盖度）**：真实数据落入生成样本流形中的比例

具体实现：对每组成样本，检查其 $k$ 近邻中是否有足够的真实样本（反之亦然）。

**优势**：明确分离两个维度，能诊断是质量问题还是覆盖问题。

### 4.5 CLIPScore

评估文本-图像对齐程度：

$$\text{CLIPScore} = 2 \cdot \max(\cos(E_{img}(x), E_{text}(c)), 0)$$

其中 $E_{img}$ 和 $E_{text}$ 为 CLIP 图像和文本编码器。

### 4.6 CMMD (CLIP Maximum Mean Discrepancy)

使用 CLIP 嵌入 + MMD 计算分布距离：

$$\text{CMMD} = \text{MMD}^2(\{CLIP(x_i^{real})\}, \{CLIP(x_j^{gen})\})$$

**优势**：不受 Inception 限制，对非 ImageNet 领域更鲁棒。

### 4.7 清晰度与多样性度量

**清晰度**：
- Sharpness（图像梯度方差）
- Inception Score 的保真度分量

**多样性**：
- LPIPS（ Learned Perceptual Image Patch Similarity）距离的均值
- 特征空间中的方差

## 5. 关键方法与模型

### 5.1 指标选择指南

| 场景 | 推荐指标 | 说明 |
|------|---------|------|
| 无条件图像生成 | FID + Precision/Recall | 标准组合 |
| 文本条件生成 | FID + CLIPScore | 质量 + 对齐 |
| 小样本量 | KID | 无偏估计 |
| 跨领域生成 | CMMD | 不依赖 Inception |
| 快速评估 | IS | 简单但不推荐单独使用 |
| 最终评估 | 人工评估 | 金标准 |

### 5.2 评估最佳实践

1. **报告多个指标**：不依赖单一指标
2. **足够的样本量**：FID 至少 10K-50K 样本
3. **固定随机种子**：确保可复现
4. **比较同一特征提取器**：不同提取器不可直接比较
5. **包含人工评估**：自动指标与人感知可能不一致
6. **报告方差**：多次运行的统计显著性

## 6. 优势与局限

### 6.1 共性优势

- 提供定量比较标准
- 自动化、可规模化
- 促进方法间的公平比较

### 6.2 共性局限

- **依赖特征提取器**：Inception-V3 对非 ImageNet 数据不鲁棒
- **高斯假设**：FID 假设特征为高斯分布
- **与人感知不一致**：FID 低的模型不一定人感更好
- **可被优化**：模型可针对性优化指标（goodhart's law）
- **缺乏语义评估**：难以评估语义层面的正确性
- **条件评估不足**：大多数指标针对无条件生成

## 7. 应用场景

- **模型选择**：训练过程中用 FID 选择最佳检查点
- **论文比较**：标准化的定量比较
- **消融实验**：验证设计选择的有效性
- **产品评估**：上线前的质量把关
- **挑战赛**：标准化竞赛评估

## 8. 与其他技术关系

- 评估指标依赖预训练模型（Inception-V3、CLIP），与多模态 AI 的跨模态对齐相关
- FID 的 Wasserstein 距离与 WGAN 的理论基础一致
- Precision/Recall 的概念来自机器学习分类评估
- CLIPScore 依赖 CLIP 模型，与多模态 AI 章节关联

## 9. 前沿发展

- **基于扩散的评估**：利用扩散模型本身的概率评估生成质量
- **多模态评估**：同时评估图像质量、文本对齐、空间一致性
- **LLM 评判**：使用 GPT-4V 等多模态 LLM 评估生成质量
- **人类偏好模型**：训练偏好预测模型模拟人类判断（如 HPS、ImageReward）
- **语义一致性**：超越像素层面，评估语义正确性
- **公平性评估**：评估生成模型中的偏见和公平性
