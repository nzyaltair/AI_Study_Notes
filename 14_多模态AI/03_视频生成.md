# 视频生成

## 1. 概述

视频生成是图像生成在时间维度的扩展，需要同时保证**空间质量**（每帧画面逼真）和**时序一致性**（帧间运动连贯）。Sora（2024）的发布标志着视频生成进入实用化阶段，统一了文本到视频、图像到视频、视频扩展等多种能力。

视频生成解决的核心问题：
- **时空联合建模**：在空间帧质量基础上，建模时间维度的运动和变化
- **长视频一致性**：在数秒到分钟级视频中保持物体身份、场景一致性
- **物理合理性**：生成的运动应符合物理规律（重力、碰撞、形变）
- **可控视频生成**：通过文本、图像、视频等条件精确控制生成内容
- **世界模拟**：通过视频生成模拟物理世界的动态过程

## 3. 核心概念

### 3.1 视频表示

视频是三维数据（空间 H×W + 时间 T），表示方式直接影响生成方法：

| 表示方法 | 原理 | 特点 |
|:---|:---|:---|
| 像素序列 | 原始 RGB 像素 (T, H, W, C) | 数据量大，直接处理 |
| 潜在表示 | VAE 压缩到潜在空间 (T, h, w, c) | 计算量降低 |
| 时空 Patch | 分割为时空 token 块 | 支持可变时长/分辨率 |
| 帧序列 | 逐帧表示，帧间条件 | 自回归生成 |

### 3.2 时空 Patch 化

Sora 的核心创新之一，将视频分割为时空 token：

```
视频 (T × H × W × C)
    ↓
时空 Patch (每个 Patch: t × p × p × C)
    ↓
线性投影 → Token 序列 → Transformer 处理
```

优势：
- **统一表示**：不同分辨率、时长、宽高比的视频统一为 token 序列
- **灵活生成**：支持任意分辨率和时长
- **继承 ViT**：与图像 Patch 化思想一致

### 3.3 时序一致性

视频生成区别于图像生成的核心挑战：

- **帧间一致性**：物体在帧间应保持身份、颜色、形状一致
- **运动合理性**：运动轨迹符合物理规律
- **场景连续性**：背景、光照在帧间平滑变化
- **长程一致性**：随着视频变长，保持一致性更难

解决方案：
- **时间注意力**：帧间 Self-Attention 捕捉长程时序依赖
- **3D 卷积**：时空联合卷积
- **循环生成**：分块生成，条件于前一段
- **运动先验**：显式建模运动场或光流

## 4. 技术原理

### 4.1 视频生成范式

| 范式 | 原理 | 代表模型 | 优势/局限 |
|:---|:---|:---|:---|
| 自回归 | 逐帧/逐 token 生成，条件于前序帧 | VideoGPT, Phenaki | 可变长度，但慢且累积误差 |
| 扩散模型 | 在时空潜在空间去噪 | Sora, SVD | 质量高，但计算量大 |
| Mask 建模 | 随机遮盖视频 Patch 并恢复 | Gaia-1 | 可控，适合补全 |
| GAN | 生成器 vs 判别器 | DVD-GAN | 快速，但质量受限 |
| 自回归+扩散 | LLM 生成语义 token + 扩散生成帧 | 视频大模型 | 结合两者优势 |

### 4.2 Sora 架构

Sora 的技术架构（基于公开信息推断）：

```
输入：文本提示 + 噪声
  ↓
文本编码（DALL·E 3 的重写+T5 编码）
  ↓
视频 → 时空 Patch 化 → 潜在空间压缩（VAE）
  ↓
扩散 Transformer (DiT) 去噪
  - 文本条件注入（交叉注意力）
  - 时空 Self-Attention
  ↓
VAE 解码器 → 生成视频帧
  ↓
输出：高保真视频（最长 60 秒）
```

关键技术创新：
- **时空 Patch 化**：统一不同分辨率和时长的视频表示
- **DiT 架构**：用 Transformer 替代 U-Net，更好的规模定律
- **文本理解**：利用 DALL·E 3 的 GPT-4 文本重写能力增强提示理解
- **可变输出**：支持横屏、竖屏、不同分辨率和时长

### 4.3 Stable Video Diffusion (SVD)

SVD 的图生视频流程：

```
输入图像 → VAE 编码 → 初始帧潜在表示
  ↓
时间嵌入 + 噪声 → 视频潜在表示 (T, h, w, c)
  ↓
U-Net 去噪（3D 卷积 + 时间注意力 + 空间注意力）
  - 条件：初始帧 + 文本
  ↓
VAE 解码 → 视频帧序列
```

训练策略：
1. 图像预训练 → 视频微调
2. 分辨率渐进提升
3. 数据质量筛选

### 4.4 AnimateDiff：动画化扩散模型

将预训练的 SD 模型扩展为视频生成：

- 在 SD U-Net 的 Self-Attention 后插入**运动模块**（Temporal Transformer）
- 运动模块学习帧间时序关系
- 保持原始 SD 的空间生成能力
- 可适配任意 SD 基础模型和 LoRA

### 4.5 视频扩散的数学框架

视频扩散在 4D 时空空间 $(T, H, W, C)$ 上进行：

前向过程：
$$q(\mathbf{x}_t | \mathbf{x}_0) = \mathcal{N}(\mathbf{x}_t; \sqrt{\bar{\alpha}_t}\mathbf{x}_0, (1 - \bar{\alpha}_t)\mathbf{I})$$

其中 $\mathbf{x} \in \mathbb{R}^{T \times H \times W \times C}$

训练目标：
$$\mathcal{L} = \mathbb{E}_{t, \mathbf{x}_0, \epsilon}\left[\|\epsilon - \epsilon_\theta(\mathbf{x}_t, t, c)\|^2\right]$$

条件 $c$ 可包括文本、首帧图像、运动控制等。

## 6. 优势与局限

### 6.1 优势

- **创意内容生产**：大幅降低视频创作门槛和成本
- **物理模拟**：高质量视频生成隐含学习了一定的物理规律
- **数据合成**：为自动驾驶、机器人等生成训练数据
- **世界理解**：视频生成被视为通向"世界模型"的路径

### 6.2 局限

- **计算成本极高**：视频生成的计算量是图像生成的数十到数百倍
- **长视频一致性**：超过 10 秒后物体身份和场景一致性显著下降
- **物理精确性**：复杂物理交互（流体、破碎、多物体碰撞）仍有困难
- **精确控制**：对物体数量、空间布局、运动轨迹的精确控制不足
- **因果关系**：模型可能不理解"因"和"果"的时间顺序
- **推理速度**：生成一段视频需要数十秒到数分钟

## 9. 前沿发展

- **更长视频**：从 10 秒到分钟级长视频生成，关键技术是分段生成 + 全局一致性
- **4K 高清**：Veo 2 等模型实现 4K 分辨率视频生成
- **物理模拟增强**：结合物理引擎或物理先验提升运动合理性
- **交互式生成**：用户可实时干预和引导视频生成方向
- **世界模型**：视频生成作为物理世界模拟器，支持预测和规划
- **统一视频理解与生成**：一个模型同时支持视频分析和生成
- **音频同步**：视频生成同步生成匹配的音效和音乐
- **高效推理**：蒸馏、一致性模型等技术减少采样步数
- **3D 视频生成**：从 2D 视频生成扩展到 3D 场景生成（4D 生成）
- **可控性增强**：更精细的运动、相机、光照控制
