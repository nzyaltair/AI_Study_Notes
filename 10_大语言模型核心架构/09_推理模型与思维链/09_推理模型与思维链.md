# 推理模型与思维链

## 背景与发展

传统LLM通过"快思考"（System 1）直接输出答案，在复杂数学、编程、逻辑推理任务上表现不佳。Chain-of-Thought（CoT）推理和o1-style推理模型通过"慢思考"（System 2）显著提升了推理能力，是LLM领域最重要的突破方向之一。

## 核心思想

通过引导模型生成中间推理步骤（思维链），将复杂问题分解为可管理的子步骤，以更多"测试时计算"（test-time compute）换取更高的推理准确率。

## 技术原理

### 思维链（Chain-of-Thought）

**零样本CoT**：在提示中添加"Let's think step by step"

**少样本CoT**：在示例中展示推理过程

```
Q: 餐厅里有23个苹果，用掉20个做午餐，又买了6个，还有多少？
A: 23 - 20 = 3，3 + 6 = 9。答案是9。
```

### 自洽性（Self-Consistency）

对同一问题多次采样CoT推理路径，取多数投票结果作为最终答案，以计算量换取准确率提升。

### 思维树（Tree of Thoughts, ToT）

将推理过程组织为树结构，在每个节点评估多个分支，支持回溯和搜索，适合规划类任务。

### 推理模型（o1-style）

| 特征 | 传统LLM | 推理模型（o1） |
|------|---------|---------------|
| 思考方式 | System 1（快思考） | System 2（慢思考） |
| 训练方法 | SFT + RLHF | SFT + RL（过程奖励） |
| 输出 | 直接答案 | 隐藏思维链→答案 |
| 测试时计算 | 固定 | 可扩展 |
| 适用场景 | 通用对话 | 数学/编程/逻辑推理 |

**关键技术**：
- **过程奖励模型（PRM）**：对推理的每一步打分，而非只看最终结果
- **强化学习训练**：通过RL优化推理策略
- **测试时计算扩展**：更长的推理链→更好的答案

### ReAct（Reasoning + Acting）

交替进行推理和行动：

```
Thought: 我需要查询今天的天气
Action: search_weather("北京")
Observation: 晴天，25°C
Thought: 天气不错，可以出门
Action: respond("北京今天晴天25度，适合出行")
```

### 其他推理增强技术

| 技术 | 原理 | 应用 |
|------|------|------|
| Least-to-Most | 从简单到复杂逐步解决 | 多步推理 |
| Self-Refine | 自我审查和修正 | 代码生成 |
| Scratchpad | 用草稿纸记录中间结果 | 长计算 |
| Program-Aided | 生成代码辅助计算 | 数学推理 |

## 发展演进

CoT（2022）→ Self-Consistency → ToT（2023）→ ReAct → **o1推理模型**（2024）→ 测试时计算扩展

推理模型代表了LLM从"模式匹配"到"深度推理"的范式转变。

## 关键算法·模型

- **CoT**：通过中间推理步骤提升复杂推理能力
- **Self-Consistency**：多路径采样+多数投票
- **Tree of Thoughts**：树结构搜索+回溯
- **o1**：过程奖励模型+强化学习训练推理策略
- **ReAct**：推理-行动交替的Agent框架

## 应用场景

- **数学推理**：GSM8K、MATH竞赛题
- **代码生成**：复杂算法设计、调试
- **科学推理**：物理、化学问题求解
- **Agent规划**：多步任务分解与执行
- **逻辑分析**：法律推理、医疗诊断

## 与其他技术关系

- CoT推理依赖SFT和RL训练
- ReAct是Agent的核心推理框架
- 推理模型通过推理优化降低长思维链的成本
- 测试时计算扩展是Scaling Laws的新维度
- [[大模型设计模式]]中的应用模式以CoT为核心
