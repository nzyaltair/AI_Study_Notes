# 自监督视觉学习

## 1. 概述

自监督视觉学习（Self-Supervised Visual Learning）是利用未标注图像数据学习视觉表示的方法。它通过设计**前置任务（Pretext Task）**，从数据自身构造监督信号，让模型在不依赖人工标注的情况下学习有用的视觉特征。

自监督学习的动机源于：标注数据获取成本高昂且规模有限，而互联网上有海量未标注图像。如果在无标注数据上预训练出强大的视觉表示，再在少量标注数据上微调，即可达到甚至超越全监督预训练的性能。

自监督视觉学习经历了从早期前置任务（拼图/旋转预测）到对比学习（SimCLR/MoCo/BYOL）再到掩码图像建模（MAE/BEiT）的发展，现已成为视觉预训练的主流范式。

## 3. 核心概念

### 3.1 前置任务（Pretext Task）

从数据中自动构造的代理任务，用于学习特征表示：

| 前置任务 | 描述 | 监督信号 |
|:---|:---|:---|
| 图块位置预测 | 预测两个图块的相对位置 | 相对位置 |
| 拼图还原 | 还原打乱的图块顺序 | 排列索引 |
| 旋转预测 | 预测图像旋转角度（0/90/180/270） | 旋转角度 |
| 着色 | 给灰度图上色 | 原始颜色 |
| 图像拼图 | 预测两张图是否来自同一图像 | 二分类 |

### 3.2 对比学习（Contrastive Learning）

对比学习的核心思想：**拉近正样本对，推开负样本对**。

- **正样本对**：同一图像的两个不同增强视图
- **负样本对**：不同图像的增强视图
- **目标**：学习一个嵌入空间，正样本距离小，负样本距离大

### 3.3 掩码图像建模（Masked Image Modeling, MIM）

受 NLP 中 BERT/MAE 启发：

- 随机遮挡图像部分图块
- 模型预测被遮挡的内容
- 从重建中学习视觉表示

### 3.4 InfoNCE 损失

对比学习的核心损失函数：

$$\mathcal{L}_{InfoNCE} = -\log \frac{\exp(\text{sim}(z_i, z_j^+) / \tau)}{\sum_{k=1}^{K} \exp(\text{sim}(z_i, z_k) / \tau)}$$

- $z_i$：查询（query）的表示
- $z_j^+$：正样本的表示
- $z_k$：包括正样本和 $K-1$ 个负样本
- $\tau$：温度参数
- $\text{sim}(\cdot, \cdot)$：余弦相似度

## 4. 技术原理

### 4.1 SimCLR

SimCLR 的对比学习框架：

```
图像 x → 增强变换 T₁ → x̃₁ → Encoder f → h₁ → Projection g → z₁
      → 增强变换 T₂ → x̃₂ → Encoder f → h₂ → Projection g → z₂
```

**关键设计**：
- **强数据增强**：随机裁剪 + 颜色变换
- **Projection head**：MLP 投影到对比空间，微调时丢弃
- **大 batch size**：batch 内所有其他样本作为负样本
- **NT-Xent 损失**（InfoNCE 的变体）：

$$\mathcal{L}_{SimCLR} = -\frac{1}{2N} \sum_{i=1}^{N} \left[ \ell_{i,j} + \ell_{j,i} \right]$$

$$\ell_{i,j} = -\log \frac{\exp(\text{sim}(z_i, z_j) / \tau)}{\sum_{k \neq i} \exp(\text{sim}(z_i, z_k) / \tau)}$$

### 4.2 MoCo

MoCo 通过**动量编码器**和**队列**解耦负样本数量与 batch size：

```
查询 x_q → Encoder f_q → q
                        ↓
              Queue（大容量负样本队列）← 动量更新 ← Encoder f_k
                        ↓
                    InfoNCE(q, k⁺, k⁻)
```

- **队列**：维护一个大规模负样本队列（如 65536）
- **动量编码器**：$f_k \leftarrow m \cdot f_k + (1-m) \cdot f_q$，保证队列中特征一致性
- **优势**：不需要大 batch size 即可拥有大量负样本

### 4.3 BYOL

BYOL 的创新：**不需要负样本**，通过自蒸馏学习：

```
在线网络：x → f_θ → g_θ → q_θ → 预测 → z_θ
            ↑ Stop-Gradient
目标网络：x' → f_ξ → g_ξ → z_ξ（动量更新）
```

**损失**：在线网络预测与目标网络表示之间的 MSE：

$$\mathcal{L}_{BYOL} = 2 - 2 \cdot \frac{\langle q_\theta(z_\theta), z_\xi \rangle}{\|q_\theta(z_\theta)\| \cdot \|z_\xi\|}$$

**为何不需负样本**：预测器 + 动量更新 + BatchNorm 的隐式对比效应防止表示坍塌。

### 4.4 DINO

DINO（Self-Distillation with No Labels）将自蒸馏应用于 ViT：

```
学生网络：x → ViT_s → softmax → P_s
教师网络：x' → ViT_t → softmax → P_t（动量更新 + 中心化）
         ↓
     Cross-Entropy(P_t, P_s)
```

**关键创新**：
- **中心化**：$c \leftarrow m \cdot c + (1-m) \cdot \mathbb{E}[g_t(x)]$，防止教师输出坍塌
- **锐化**：教师 softmax 使用低温度 $\tau_t$
- **多裁剪**：学生接收多个局部裁剪，教师接收全局裁剪

**关键发现**：DINO 训练的 ViT 注意力图自发地学习了语义分割能力，无需任何标注。

### 4.5 MAE

MAE（Masked Autoencoder）的非对称掩码自编码：

```
图像 → 随机掩码 75% → 可见 25% 图块
                         ↓
                   ViT Encoder（大，仅处理可见图块）
                         ↓
                   + [Mask Tokens] → ViT Decoder（小）
                         ↓
                   重建被遮挡像素 → MSE 损失
```

**设计要点**：
- **高掩码率**（75%）：使任务有挑战性，同时编码器只处理 25% 图块，计算量大幅减少
- **非对称架构**：编码器大（学习表示），解码器小（仅重建）
- **像素级重建**：直接预测 RGB 像素值，无需 Tokenizer
- **微调**：预训练后丢弃解码器，用编码器微调下游任务

### 4.6 BEiT

BEiT 的掩码图像建模使用离散 Token 作为目标：

```
图像 → VQ-VAE Tokenizer → 离散 Token（监督信号）
     → 随机掩码 → ViT Encoder → 预测被掩码 Token
```

与 MAE 的区别：BEiT 预测离散 Token（类似 BERT），MAE 预测连续像素。

### 4.7 DINOv2

DINOv2 = DINO + iBOT 的统一自监督训练：

- **图像级蒸馏**（DINO）：全局视图之间的自蒸馏
- **图块级预测**（iBOT）：被掩码图块的预测
- **大规模数据**：LVD-142M 精选高质量图像
- **高效训练**：Flash Attention、FSDP 并行

DINOv2 在 ImageNet 分类、检测、分割等下游任务上达到或超越有监督预训练。

## 5. 关键方法/模型

### 5.1 方法分类对比

| 范式 | 代表方法 | 核心机制 | 优势 | 劣势 |
|:---|:---|:---|:---|:---|
| 前置任务 | 旋转预测 | 预测旋转角度 | 简单 | 表示质量有限 |
| 对比学习 | SimCLR/MoCo | InfoNCE 损失 | 表示质量高 | 需大量负样本 |
| 非对比学习 | BYOL/SimSiam | 自蒸馏/停梯度 | 不需负样本 | 超参敏感 |
| 掩码建模 | MAE/BEiT | 重建被掩码内容 | 高效、可扩展 | CNN 适配难 |
| 统一方法 | DINOv2 | DINO+iBOT | 综合最优 | 训练复杂 |

### 5.2 性能对比（ImageNet 线性探测）

| 方法 | 预训练数据 | ImageNet Linear | 架构 |
|:---|:---|:---|:---|
| Supervised | ImageNet-1K | 81.0% | ResNet-50 |
| SimCLR | ImageNet-1K | 69.3% | ResNet-50 |
| MoCo v3 | ImageNet-1K | 76.7% | ViT-B |
| BYOL | ImageNet-1K | 74.3% | ResNet-50 |
| MAE | ImageNet-1K | 68.0% (线性) / 83.6% (微调) | ViT-B |

## 6. 优势与局限

### 优势

1. **无需标注数据**：利用海量无标注图像学习表示
2. **预训练质量高**：DINOv2/MAE 等已达到或超越有监督预训练
3. **迁移性强**：预训练表示在多种下游任务上泛化好
4. **可扩展性好**：数据量和模型规模增大时性能持续提升
5. **与 LLM 范式对齐**：类似 NLP 中 BERT/GPT 的预训练范式

### 局限

1. **计算成本高**：预训练需要大量 GPU 时间
2. **评估困难**：如何评估预训练表示质量缺乏统一标准
3. **CNN 适配难**：掩码建模对 CNN 不如 ViT 自然
4. **超参敏感**：对比学习对温度、batch size 等敏感
5. **语义对齐有限**：纯视觉自监督缺乏语义概念对齐（CLIP 通过图文对齐解决）

## 9. 前沿发展

- **统一预训练**：将对比学习和掩码建模统一到单一框架（DINOv2/iBOT）
- **多模态自监督**：图像-文本-视频统一自监督预训练
- **视频自监督**：VideoMAE 等将掩码建模扩展到视频时序
- **世界模型自监督**：从视频中学习物理世界的自监督表示
- **大模型 scaling**：更大数据集和模型规模的自监督训练（ViT-22B 级别）
- **高效自监督**：减少预训练的计算成本，使学术实验室也能训练
- **开放世界表示**：学习开放词汇的视觉表示，支持任意类别识别
