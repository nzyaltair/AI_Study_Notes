# 词法分析

## 1. 概述

词法分析（Lexical Analysis）是 NLP 处理流程中紧接分词之后的层次，主要任务是分析词的语法和语义属性。它包含三个核心子任务：**词性标注**（Part-of-Speech Tagging, POS Tagging）、**命名实体识别**（Named Entity Recognition, NER）和**形态学分析**（Morphological Analysis）。

词法分析是理解句子结构的基础——"book" 在 "I read a book" 中是名词，在 "I book a ticket" 中是动词。词性标注消解此类歧义；NER 识别文本中的关键实体（人名、地名、机构名），是信息抽取和知识图谱构建的前置步骤。

## 3. 核心概念

### 3.1 词性标注（POS Tagging）

为每个词标注其语法类别。常用的标注体系：

| 标注集 | 语言 | 标签数 | 示例 |
|:---|:---|:---|:---|
| Penn Treebank | 英文 | 36 | NN(名词)、VB(动词)、JJ(形容词) |
| 人民日报标注集 | 中文 | ~40 | n(名词)、v(动词)、a(形容词) |
| Universal POS | 多语言 | 17 | NOUN、VERB、ADJ（跨语言统一） |

**核心难点**：一词多义（词性歧义）。如 "fast" 可为形容词（快的）、副词（快地）、动词（禁食）、名词（斋戒）。

### 3.2 命名实体识别（NER）

识别文本中具有特定意义的实体并分类：

| 实体类型 | 标签 | 示例 |
|:---|:---|:---|
| 人名 | PER | 张三、Barack Obama |
| 地名 | LOC | 北京、New York |
| 机构名 | ORG | 微软、联合国 |
| 时间 | DATE/TIME | 2024年1月、3pm |
| 货币/数值 | MONEY/QUANTITY | 100万美元 |

**标注方案**：

- **BIO**：B-X（实体X的开始）、I-X（实体X的内部）、O（非实体）
- **BIOES**：增加 E-X（实体结束）、S-X（单字实体），更精确

```
文本：  张  三  在  北  京  工  作
BIO：   B-PER I-PER O  B-LOC I-LOC O  O
BIOES： S-PER       O  S-LOC       O  O
```

### 3.3 形态学分析

分析词的内部结构，包括：

- **词形还原（Lemmatization）**：将词还原为词典形式。如 "running" → "run"、"better" → "good"
- **词干提取（Stemming）**：截断词缀获取词干。如 "running" → "run"、"happiness" → "happi"
- **形态拆分**：将词分解为语素。如 "unhappiness" → "un" + "happy" + "ness"

| 方法 | "better" | "running" | "studies" |
|:---|:---|:---|:---|
| 词干提取（Porter） | "better" | "run" | "studi" |
| 词形还原 | "good" | "run" | "study" |

词形还原更准确但需要词典支持；词干提取更快但可能产生非词。

## 4. 技术原理

### 4.1 HMM 词性标注

将词性标注建模为隐状态序列（词性标签）产生观测序列（词）：

$$P(\mathbf{x}, \mathbf{y}) = \prod_{t=1}^{T} P(y_t | y_{t-1}) \cdot P(x_t | y_t)$$

- **转移概率** $P(y_t | y_{t-1})$：前一个标签是 $y_{t-1}$ 时当前标签为 $y_t$ 的概率
- **发射概率** $P(x_t | y_t)$：标签 $y_t$ 产生词 $x_t$ 的概率

使用 Viterbi 算法解码最优标签序列。参见 [[05_序列标注模型]]。

### 4.2 CRF 词性标注/NER

CRF 是判别式模型，直接建模 $P(\mathbf{y}|\mathbf{x})$：

$$P(\mathbf{y}|\mathbf{x}) = \frac{1}{Z(\mathbf{x})} \exp\left(\sum_t \left(\sum_k \lambda_k f_k(y_t, \mathbf{x}, t) + \sum_l \mu_l g_l(y_{t-1}, y_t, \mathbf{x}, t)\right)\right)$$

特征函数 $f_k$ 和 $g_l$ 可以利用丰富的上下文特征（词形、前缀、后缀、大小写、词典匹配等）。

### 4.3 BiLSTM-CRF

BiLSTM-CRF 是深度学习时代 NER 的经典架构（Huang et al., 2015）：

```
输入词序列 → 词嵌入 → BiLSTM（提取上下文特征）→ CRF（约束标签转移）→ 输出标签序列
```

- **BiLSTM**：双向 LSTM 提取每个位置的上下文表示
- **CRF 层**：学习标签之间的转移约束（如 B-PER 后不能直接接 I-LOC）

BiLSTM-CRF 结合了深度学习的特征自动提取和 CRF 的序列级解码能力。

### 4.4 BERT-CRF

用 BERT 替换 BiLSTM 作为特征提取器：

```
输入子词序列 → BERT → 隐藏层 → CRF → 输出标签序列
```

BERT 的预训练表示提供了强大的上下文理解能力，显著优于从零训练的 BiLSTM。

### 4.5 LLM 少样本 NER

大模型时代可通过提示工程进行零样本或少样本 NER：

```
提示：从以下文本中识别人名、地名、机构名，以 JSON 格式输出：
文本：张三在北京微软工作
输出：{"人名": ["张三"], "地名": ["北京"], "机构名": ["微软"]}
```

## 5. 关键方法对比

| 方法 | 词性标注 F1 | NER F1 | 优势 | 局限 |
|:---|:---|:---|:---|:---|
| 规则 | ~90% | ~70% | 可解释、无需训练 | 覆盖率低、维护成本高 |
| HMM | ~96% | ~85% | 简单高效 | 独立假设过强 |
| CRF | ~97% | ~88% | 特征灵活、全局最优 | 特征工程依赖人工 |
| BiLSTM-CRF | ~97.5% | ~90% | 自动特征、端到端 | 需要标注数据 |
| BERT-CRF | ~98% | ~92% | 预训练迁移、SOTA | 计算开销大 |
| LLM Few-shot | ~95% | ~85% | 无需训练、零样本 | 精度不如微调 |

## 6. 优势与局限

### 优势

1. **基础性**：词法分析是句法分析、语义理解等高层任务的基础
2. **信息丰富**：词性标签和实体类型为下游任务提供关键结构信息
3. **可解释**：标注结果直观，便于人工审核

### 局限

1. **领域依赖**：训练于新闻语料的模型在医疗/法律文本上性能骤降
2. **标注成本**：高质量标注数据获取成本高
3. **嵌套实体**：如"北京大学信息科学技术学院"中"北京大学"是机构名，整体也是机构名——扁平标注方案无法处理
4. **LLM 时代角色变化**：大模型端到端处理使得独立的词法分析模块在部分场景被弱化

## 7. 应用场景

- **信息抽取**：NER 是关系抽取和知识图谱构建的前置步骤
- **搜索引擎**：实体识别改善查询理解和结果排序
- **问答系统**：实体链接帮助定位答案
- **舆情监控**：人名/机构名识别用于情感对象提取
- **金融风控**：公司名、事件抽取用于风险预警

## 9. 前沿发展

- **统一信息抽取**：一个模型同时完成 NER、关系抽取、事件抽取（如 UIE、GoLLIE）
- **生成式 NER**：将 NER 转化为生成任务，直接输出实体列表（如 LLM-based NER）
- **跨语言 NER**：利用多语言预训练模型实现零样本跨语言迁移
- **嵌套 NER**：使用超图、Span-based 方法处理嵌套实体结构
- **实体链接增强**：NER 与实体链接（Entity Linking）联合优化，将实体映射到知识库
