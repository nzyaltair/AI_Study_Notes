# 序列模型工程实践

## 1. 概述

序列模型的工程实践涵盖了从代码实现、训练优化到生产部署的完整流程。尽管 RNN 系列在主流语言建模任务中已被 Transformer 取代，但其在语音识别、时间序列预测等领域仍广泛使用。掌握这些工程技巧有助于理解序列模型的训练行为，也为 Transformer 和现代 SSM 的训练提供了方法论基础。

- **涵盖范围**：PyTorch/TensorFlow 中的 RNN API 使用、自定义 LSTM/GRU 实现、训练稳定性技巧、梯度管理、混合精度训练、分布式训练、模型导出与服务化。

## 2. 核心框架与实现

### 2.1 PyTorch 中的 RNN API

PyTorch 提供了高度优化的 cuDNN 后端 RNN 实现：

```python
import torch
import torch.nn as nn

# LSTM: input_size=输入维度, hidden_size=隐藏状态维度, num_layers=层数
lstm = nn.LSTM(input_size=128, hidden_size=512, num_layers=2,
               batch_first=True, dropout=0.3, bidirectional=True)

# GRU
gru = nn.GRU(input_size=128, hidden_size=512, num_layers=2,
             batch_first=True, dropout=0.3)

# 输入格式: (batch, seq_len, input_size)
x = torch.randn(32, 100, 128)

# LSTM 输出: output (所有时刻的隐藏状态), (h_n, c_n) 最后时刻的状态
output, (h_n, c_n) = lstm(x)

# GRU 输出: output, h_n (无细胞状态)
output, h_n = gru(x)
```

关键参数说明：
- `batch_first=True`：输入/输出张量的第一维为 batch，与大多数 NLP 模型的输入格式一致
- `bidirectional=True`：自动创建正向和反向两个独立的 RNN，输出维度为 `hidden_size * 2`
- `dropout`：仅在多层（`num_layers > 1`）时生效，在层间施加 Dropout

### 2.2 从零实现 LSTM（教学参考）

```python
class LSTMCell(nn.Module):
    def __init__(self, input_size, hidden_size):
        super().__init__()
        self.input_size = input_size
        self.hidden_size = hidden_size
        # 四个门合并为一个大的线性变换（GPU 上更高效）
        self.W = nn.Linear(input_size + hidden_size, 4 * hidden_size)

    def forward(self, x, state):
        h_prev, c_prev = state
        # 拼接输入和隐藏状态
        combined = torch.cat([x, h_prev], dim=-1)
        # 一次性计算四个门
        gates = self.W(combined)
        f, i, c_tilde, o = gates.chunk(4, dim=-1)
        # 激活
        f = torch.sigmoid(f)
        i = torch.sigmoid(i)
        c_tilde = torch.tanh(c_tilde)
        o = torch.sigmoid(o)
        # 细胞状态和隐藏状态更新
        c = f * c_prev + i * c_tilde
        h = o * torch.tanh(c)
        return h, (h, c)
```

### 2.3 PackedSequence 处理变长序列

变长序列输入是 RNN 的常见需求（如一批句子长度不一）。PyTorch 提供了 `pack_padded_sequence` 和 `pad_packed_sequence` 机制：

```python
from torch.nn.utils.rnn import pack_padded_sequence, pad_packed_sequence

# sentences: list of tensors with different lengths
lengths = torch.tensor([len(s) for s in batch])
padded = pad_sequence(batch, batch_first=True)  # padding to max len

# Pack: 将 padding 后的序列打包，RNN 跳过 padding 帧
packed = pack_padded_sequence(padded, lengths, batch_first=True, enforce_sorted=False)

# RNN 前向
packed_output, (h_n, c_n) = lstm(packed)

# Unpack: 恢复为 padded 格式
output, output_lengths = pad_packed_sequence(packed_output, batch_first=True)
```

**为什么使用 PackedSequence**：
- 避免在 padding 帧上进行无意义的计算
- 防止 RNN 的隐藏状态受 padding 帧污染
- GPU 上可节省约 20%~50% 的实际计算时间（取决于 batch 内长度差异）

## 3. 训练稳定性技巧

### 3.1 梯度裁剪（Gradient Clipping）

RNN 训练中梯度裁剪是**强制性的**，而非可选的：

```python
# 全局 L2 范数裁剪（推荐）
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=5.0)

# 逐参数值裁剪（备选）
torch.nn.utils.clip_grad_value_(model.parameters(), clip_value=1.0)
```

- 阈值建议：LSTM/GRU 通常 5~10，简单 RNN 通常 1~5
- 监控指标：训练时应定期检查梯度范数分布，如果频繁触发裁剪，可能需调整学习率或初始化

### 3.2 遗忘门偏置初始化

LSTM 的遗忘门偏置 $b_f$ 应初始化为正值（如 1.0），使模型在训练初期倾向于"记住"：

```python
def init_lstm_weights(lstm, forget_gate_bias=1.0):
    for name, param in lstm.named_parameters():
        if 'bias' in name:
            n = param.size(0)
            # 遗忘门偏置初始化为 1.0
            param.data[n//4:n//2].fill_(forget_gate_bias)
            # 其余偏置初始化为 0
            param.data[:n//4].fill_(0)
            param.data[n//2:].fill_(0)
```

### 3.3 Layer Normalization 在 RNN 中

```python
class LayerNormLSTM(nn.Module):
    def __init__(self, input_size, hidden_size):
        super().__init__()
        self.lstm_cell = nn.LSTMCell(input_size, hidden_size)
        self.ln_h = nn.LayerNorm(hidden_size)
        self.ln_c = nn.LayerNorm(hidden_size)

    def forward(self, x, state=None):
        h, c = state if state else (torch.zeros(...), torch.zeros(...))
        h_new, c_new = self.lstm_cell(x, (h, c))
        h_new = self.ln_h(h_new)
        c_new = self.ln_c(c_new)
        return h_new, (h_new, c_new)
```

LayerNorm 在深层 LSTM（≥3 层）中对训练稳定性有显著帮助，ELMo 和 GNMT 均使用了 LayerNorm LSTM。

### 3.4 Truncated BPTT

```python
def train_truncated_bptt(model, data, chunk_size=100):
    hidden = None
    total_loss = 0

    for i in range(0, len(data) - 1, chunk_size):
        chunk = data[i:i + chunk_size]
        target = data[i+1:i + chunk_size + 1]

        # 分离隐藏状态的计算图（阻断跨 chunk 梯度）
        if hidden is not None:
            hidden = (h.detach() for h in hidden)

        output, hidden = model(chunk, hidden)
        loss = criterion(output, target)
        loss.backward()
        optimizer.step()
        optimizer.zero_grad()

        total_loss += loss.item()
    return total_loss
```

### 3.5 学习率调度

RNN 训练对学习率敏感，推荐配置：

```python

optimizer = torch.optim.Adam(model.parameters(), lr=0, betas=(0.9, 0.98), eps=1e-9)

# Warmup + 余弦退火（大模型训练标准）
scheduler = torch.optim.lr_scheduler.SequentialLR(
    optimizer,
    [
        torch.optim.lr_scheduler.LinearLR(optimizer, start_factor=1e-3, total_iters=4000),
        torch.optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=96000)

    milestones=[4000]
```

## 4. 高级训练技术

### 4.1 混合精度训练

```python
scaler = torch.cuda.amp.GradScaler()

with torch.cuda.amp.autocast():
    output = model(x)
    loss = criterion(output, target)

scaler.scale(loss).backward()
scaler.unscale_(optimizer)
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=5.0)
scaler.step(optimizer)
scaler.update()
```

混合精度在 GPU（Volta 及以上架构）上可提供 1.5~2x 的训练加速和 30~50% 的显存节省。注意：RNN 的某些操作（如 cuDNN LSTM）可能不完全支持 `autocast`，需在具体硬件上验证。

### 4.2 梯度累积

在显存不足时，通过梯度累积模拟大 batch 训练：

```python
accumulation_steps = 4
for i, batch in enumerate(dataloader):
    loss = model(batch) / accumulation_steps
    loss.backward()

    if (i + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()
```

### 5.4 推理模式的序列状态管理

```python
class StatefulLSTM(nn.Module):
    """保持状态跨越多个推理调用，适用于流式语音处理"""
    def __init__(self, lstm):
        super().__init__()
        self.lstm = lstm
        self._state = None

    def forward(self, x, reset=False):
        if reset or self._state is None:
            self._state = None
        output, self._state = self.lstm(x, self._state)
        return output
```

## 6. 常见问题与调试

| 问题 | 症状 | 解决方案 |
|:---|:---|:---|
| **梯度爆炸** | loss 突然变为 NaN/Inf | 梯度裁剪 (`max_norm=5.0`)，降低学习率 |
| **梯度消失** | 训练 loss 不下降 | 检查 LSTM 遗忘门初始化，确认使用 LSTM/GRU 而非简单 RNN |
| **过拟合** | 训练 loss 远低于验证 loss | Dropout (0.3~0.5)，LayerNorm，减小 hidden_size，早停 |
| **长序列内存溢出** | CUDA OOM | 减小 batch_size，截断 BPTT 长度，梯度累积，混合精度 |
| **变长序列处理错误** | 输出全为 0 或异常 | 使用 PackedSequence，检查序列长度排序和 enforce_sorted 参数 |
| **cuDNN 与自定义实现不一致** | 输出差异大 | cuDNN 使用特定的数值精度和算法，在某些情况下可通过 `torch.backends.cudnn.deterministic = True` 确保确定性 |

## 8. 与其他技术关系

- **与深度学习框架**：PyTorch 的 `nn.LSTM`/`nn.GRU` 底层使用 cuDNN 加速，TensorFlow 使用 cuDNN 或 Eigen 实现。了解这些底层实现有助于性能调优
- **与训练技巧**：梯度裁剪、学习率调度、权重初始化等训练技巧从 RNN 时代积累的经验直接继承到 Transformer 训练中
- **与模型部署**：RNN 的推理优化（状态管理、批处理策略）对 SSM 部署有直接参考价值，因为两者都以逐 token 状态更新为推理模式
