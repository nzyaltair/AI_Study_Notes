# 大模型推理与优化

## 1. 概述

大语言模型（LLM）推理优化是指对模型部署后的推理阶段进行系统性优化，以在保持生成质量的前提下降低延迟、减少显存占用、提升吞吐量和降低部署成本。

LLM 推理面临三大核心瓶颈：

- **计算瓶颈**：自回归生成需逐 token 产出，每步仅产生一个 token，GPU 计算能力无法充分利用
- **内存瓶颈**：模型参数本身占用大量显存（70B FP16 约 140GB），加上 [[KV-Cache]] 随序列长度线性增长
- **带宽瓶颈**：自回归推理每步需读取全部权重，显存带宽成为速度上限（memory-bandwidth bound）

推理优化技术围绕这三大瓶颈展开，是 LLM 从研究走向生产落地的关键环节。

## 3. 核心概念

- **Prefill 与 Decode 两阶段**：Prefill 阶段并行处理输入 prompt（计算密集型），Decode 阶段逐 token 生成（带宽密集型）
- **[[KV-Cache]]**：缓存已计算的 Key/Value 避免重复计算，是自回归推理的核心优化
- **[[量化]]**：将 FP16 权重压缩到 INT8/INT4/FP8，减少显存和带宽需求
- **[[剪枝与稀疏化]]**：移除冗余参数或利用稀疏结构减少计算量
- **连续批处理（Continuous Batching）**：动态加入/移除请求，最大化 GPU 利用率
- **Speculative Decoding**：小模型草拟 + 大模型验证，打破逐 token 串行依赖
- **Prefill-Decode 分离（Disaggregated Inference）**：将计算密集的 Prefill 和带宽密集的 Decode 分配到不同 GPU 集群
- **MoE 稀疏激活**：每次推理只激活部分专家参数，以小计算量获得大模型容量

## 4. 技术原理

### 4.1 KV-Cache 显存占用

$$M_{\text{KV}} = 2 \times n_{\text{layers}} \times \text{batch} \times n_{\text{KV\_heads}} \times \text{seq\_len} \times d_{\text{head}} \times \text{dtype\_size}$$

以 Llama-2-70B（FP16, 80层, 64头, $d_{head}=128$, seq=4096, batch=1）为例，KV-Cache 约占 5.4GB；而 batch=64 时则需约 345GB。

### 4.2 量化原理

将浮点权重映射到低精度整数：

$$w_{\text{quant}} = \text{round}(w / s) \times s$$

其中 $s$ 为缩放因子。GPTQ 基于二阶 Hessian 信息逐列量化，AWQ 通过激活感知保护重要权重。

### 4.3 投机解码原理

小模型（Draft Model）生成候选 token 序列，大模型（Target Model）并行验证。接受概率：

$$P_{\text{accept}} = \min\left(1, \frac{\pi_{\text{target}}(y_i)}{\pi_{\text{draft}}(y_i)}\right)$$

期望加速比取决于 Draft 模型的接受率，通常可获得 2-3x 加速。

### 4.4 连续批处理

传统批处理需等待同 batch 所有请求完成，GPU 利用率仅 30-40%。连续批处理在每步动态调度，GPU 利用率提升至 80-90%，吞吐量提升 3-4x。

## 6. 优势与局限

**优势**：
- 显著降低部署成本：INT4 量化可使 70B 模型在单张 A100 上运行
- 大幅提升吞吐量：vLLM 比原生 HuggingFace 高 10-20x
- 保持生成质量：投机解码和优质量化几乎无质量损失
- 支持长上下文：KV-Cache 压缩使 128K+ 上下文成为可能

**局限**：
- 量化对数学推理等敏感任务仍有精度影响
- 投机解码依赖草稿模型质量，加速比不稳定
- 系统级优化（PagedAttention、连续批处理）工程复杂度高
- 长上下文的质量衰减问题仍未完全解决

## 7. 应用场景

- **LLM API 服务**：vLLM/TensorRT-LLM + 连续批处理 + 量化，支撑高并发推理
- **本地/边缘部署**：llama.cpp + GGUF 量化，在消费级 CPU/GPU 上运行
- **长上下文应用**：RAG、代码库分析、长文档摘要，依赖 KV-Cache 压缩与位置外推
- **实时对话系统**：Speculative Decoding + 低延迟框架，满足 P99 < 500ms
- **Agent 应用**：结构化解码 + 高并发推理，支撑工具调用与多步推理

## 9. 前沿发展

- **潜在注意力（MLA）**：DeepSeek-V2 将 KV-Cache 压缩到潜在向量，减少 93.3%，成为新的架构趋势
- **分离式推理（Disaggregated Inference）**：Prefill 和 Decode 分配到不同 GPU 集群，各自优化
- **多 token 预测**：Medusa/EAGLE 在单步内预测多个 token，超越传统投机解码
- **无草稿模型并行解码**：Lookahead Decoding 无需独立草稿模型，通过 Jacobi 迭代并行解码
- **FP8 原生推理**：NVIDIA Hopper/Blackwell 架构硬件支持 FP8，精度损失极小
- **MoE 推理优化**：Mixtral 8x7B、DeepSeek-MoE 等稀疏模型的专用推理加速
- **端侧推理**：苹果 MLX、高通 AI Engine 等推动 LLM 在手机端运行
- **推理时计算扩展（Test-Time Compute Scaling）**：OpenAI o1 模式，通过推理时多步搜索提升质量
