# 模型选择与调参

## 背景

[[机器学习]]中，选择合适的模型和优化超参数是提高性能的关键步骤。不同模型适用于不同数据分布和任务类型，超参数调优直接影响模型泛化能力。

## 核心思想

模型选择是在候选模型中选取最适合数据和任务的模型；调参是寻找最优超参数组合使模型泛化性能最大化。

## 技术原理

### 交叉验证

- **K折交叉验证**：数据分K份，每次用K-1份训练、1份验证，循环K次取平均
- **分层K折**：保持每折中类别比例一致
- **留一法（LOO）**：K=N的极端情况，计算量大但数据利用率高
- **时间序列CV**：前向链验证，保证时间顺序

### 超参数调优方法

- **网格搜索**（Grid Search）：穷举所有参数组合
  - 优点：全面；缺点：计算量大
- **随机搜索**（Random Search）：随机采样参数组合
  - 比网格搜索更高效，尤其在高维参数空间
- **贝叶斯优化**：利用历史评估结果构建代理模型，智能选择下一组参数
  - TPE（Tree-structured Parzen Estimator）
  - 高斯过程
  - 工具：Optuna、Hyperopt

### 评估指标

| 任务类型 | 指标 | 说明 |
|---------|------|------|
| 分类 | Accuracy | 正确预测比例 |
| 分类 | Precision/Recall | 精确率/召回率 |
| 分类 | F1-Score | Precision和Recall的调和平均 |
| 分类 | AUC-ROC | ROC曲线下面积 |
| 回归 | MSE/RMSE | 均方误差 |
| 回归 | MAE | 平均绝对误差 |
| 回归 | R² | 决定系数 |

### 模型选择策略

1. **基线模型**：先用简单模型（线性回归/逻辑回归）建立基线
2. **复杂度递增**：逐步尝试更复杂的模型
3. **集成方法**：多个模型组合往往优于单一模型
4. **No Free Lunch**：没有 universally 最优的模型，需根据数据和任务选择
