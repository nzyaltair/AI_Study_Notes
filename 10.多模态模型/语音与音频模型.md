# 语音与音频模型

## 背景

语音与音频AI涵盖语音识别（ASR）、语音合成（TTS）和音频生成。Whisper（2022）实现了多语言鲁棒语音识别，现代TTS模型实现了自然度极高的语音合成，音频生成模型开始创作音乐和音效。

## 核心思想

通过在超大规模音频数据上训练Transformer模型，实现语音识别、合成和音频理解生成的统一。

## 技术原理

### 语音识别（ASR）

#### Whisper架构

- **编码器-解码器Transformer**
- 输入：Mel频谱图 → 卷积编码 → Transformer编码器
- 输出：Transformer解码器自回归生成文本
- **多任务**：转录 + 翻译 + 语言识别 + 时间戳
- **训练数据**：68万小时多语言弱标注数据

#### 关键技术

- **Mel频谱图**：将音频转换为时频表示
- **CTC损失**：处理变长对齐
- **SpecAugment**：频谱图数据增强
- **流式识别**：分块处理实现低延迟

### 语音合成（TTS）

| 方法 | 原理 | 代表模型 |
|------|------|---------|
| **拼接式** | 拼接预录语音片段 | 传统TTS |
| **参数式** | 声学参数合成 | HMM-based |
| **神经网络声码器** | WaveNet等自回归生成 | WaveNet, WaveGlow |
| **端到端** | 文本直接生成语音 | Tacotron, FastSpeech |
| **扩散/Flow** | 扩散模型生成语音 | NaturalSpeech, VITS |

#### 现代TTS流程

```
文本 → 文本前端（音素转换）→ 声学模型 → 声码器 → 语音波形
```

#### 零样本语音克隆

- 给定几秒参考音频，克隆说话人音色
- 方法：Speaker Embedding + 声学模型条件化
- 代表：XTTS, OpenVoice

### 音频生成

| 任务 | 方法 | 代表 |
|------|------|------|
| **音乐生成** | 自回归/扩散 | MusicGen, AudioLDM |
| **音效生成** | 文本到音频 | AudioGen |
| **语音转换** | 改变音色保持内容 | So-VITS-SVC |

### 统一语音模型

趋势：一个模型同时处理ASR、TTS和语音理解：
- **AudioLM**：音频自回归生成
- **SPIRIT-LM**：语音-文本统一模型
- **Qwen-Audio**：音频理解大模型

## 发展演进

HMM-GMM（传统ASR）→ DNN-HMM → RNN/CTC → Conformer → **Whisper**（2022）→ 统一语音模型

TTS：拼接式 → 参数式 → 神经网络（WaveNet）→ 端到端 → 零样本克隆

## 应用领域

- **语音助手**：Siri、小爱同学（ASR+TTS）
- **会议记录**：实时转写
- **有声内容**：有声书、新闻播报
- **多语言翻译**：实时语音翻译
- **音乐创作**：AI作曲、编曲
- **无障碍**：为视障人士朗读、为听障人士转写

## 与其他技术关系

- 语音模型基于[[Transformer架构详解|Transformer]]架构
- Whisper的编码器-解码器与[[预训练语言模型|T5]]类似
- 语音能力扩展了[[多模态模型|多模态]]模型的输入输出
- 语音交互是[[Agent智能体|Agent]]的重要交互方式
- 语音合成中的声码器与[[图像生成模型|扩散模型]]有技术交叉
