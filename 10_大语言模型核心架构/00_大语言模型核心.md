# 大语言模型核心

大语言模型（LLM）是基于[[Transformer]]的超大规模[[预训练语言模型]]，通过在海量文本上自监督预训练获得通用语言能力，再通过指令微调和对齐技术成为实用的AI助手。GPT-3（2020）证明了"Scale is all you need"，ChatGPT（2022）展示了RLHF的威力，开启了LLM时代。

## 子主题导航

### [[LLM整体架构]]

Decoder-only Transformer架构、Scaling Laws、RoPE位置编码、混合专家模型（MoE）、长上下文机制。

### [[注意力机制]]

缩放点积注意力、多头注意力、因果掩码、MHA/MQA/GQA/MLA对比、FlashAttention。

### [[位置编码]]

正弦编码、可学习编码、相对位置编码、RoPE旋转位置编码、ALiBi、长度外推方法。

### [[分词]]

BPE/SentencePiece分词、词表设计、特殊token处理、Token效率。

### [[Embedding]]

Word2Vec/GloVe传统词嵌入、BERT上下文感知嵌入、对比学习嵌入（SimCSE）、RAG场景嵌入选择。

### [[混合专家模型]]

MoE基本原理、门控网络与路由机制、Switch Transformer/Mixtral/DeepSeek-MoE、负载均衡。

### [[长上下文与记忆机制]]

位置编码扩展（YaRN/NTK）、Flash Attention、KV-Cache优化、RAG外部记忆、SSM线性注意力。

### [[大模型设计模式]]

架构模式（Decoder-only/RMSNorm/SwiGLU）、训练模式（预训练→SFT→对齐）、推理模式（KV-Cache/量化/解码策略）、应用模式（CoT/RAG/Agent）。

### [[思维链]]

Chain-of-Thought推理、Self-Consistency、Tree of Thoughts、o1推理模型、ReAct框架。

## 训练流程全景

```
数据收集与清洗 → 预训练（Next Token Prediction）
  → 监督微调（SFT，指令跟随）
  → 偏好对齐（RLHF/DPO，人类偏好对齐）
  → 安全对齐（Red Teaming，价值对齐）
  → 部署与推理优化
```

## 与其他技术关系

- LLM基于[[Transformer]]架构和[[预训练语言模型]]范式
- 训练依赖分布式训练技术
- 推理优化与系统工程化相关
- LLM是Agent和多模态的核心引擎
