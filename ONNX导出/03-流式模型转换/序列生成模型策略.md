# 序列生成模型策略

序列生成模型（如GPT、T5、BART等自回归模型）在ONNX转换过程中面临独特挑战。这些模型通常采用**自回归生成**机制，需要动态处理可变长度的序列，并妥善管理KV Cache以实现高效推理。

## 核心挑战

### 1. 自回归特性处理

自回归模型在生成过程中，每个时间步的输入依赖于前一个时间步的输出：

```
for t in range(max_length):
    output = model(input_ids[:, t:t+1], past_key_values)
    next_token = sample(output)
    input_ids = torch.cat([input_ids, next_token], dim=1)
```

这种**迭代生成**模式在ONNX中需要特殊处理，因为：
- 动态序列长度
- 状态（KV Cache）需要在推理步骤间保持
- 需要避免在每一步都重复计算注意力

### 2. KV Cache管理

Transformer模型中的KV Cache用于缓存之前时间步的键值对，避免重复计算：

```python
# KV Cache结构示例
# past_key_values: tuple of (key, value) tensors, shape [batch, num_heads, seq_len, head_dim]
past_key_values = (
    (key_0, value_0),  # layer 0
    (key_1, value_1),  # layer 1
    ...
)
```

在ONNX中，KV Cache通常作为**显式输入/输出**处理：

```python
import torch
import torch.nn as nn

class GPT2WithCache(nn.Module):
    def __init__(self, model):
        super().__init__()
        self.model = model

    def forward(self, input_ids, attention_mask, past_key_values):
        # past_key_values: tuple of tuples
        outputs = self.model(
            input_ids=input_ids,
            attention_mask=attention_mask,
            past_key_values=past_key_values,
            use_cache=True
        )
        return outputs.logits, outputs.past_key_values
```

### 3. 动态序列长度处理

流式生成过程中，序列长度从1逐步增长到最大长度：

```python
# PyTorch中的动态长度示例
def generate_streaming(model, input_ids, max_len=50):
    past_key_values = None
    for i in range(max_len):
        if past_key_values is None:
            # 第一步：处理完整序列
            outputs = model(input_ids, use_cache=True)
            logits = outputs.logits[:, -1, :]
            past_key_values = outputs.past_key_values
        else:
            # 后续步骤：只处理最后一个token
            outputs = model(
                input_ids[:, -1:],
                past_key_values=past_key_values,
                use_cache=True
            )
            logits = outputs.logits[:, -1, :]
            past_key_values = outputs.past_key_values

        next_token = torch.argmax(logits, dim=-1, keepdim=True)
        input_ids = torch.cat([input_ids, next_token], dim=1)
```

## ONNX转换策略

### 基本转换方法

对于需要KV Cache的自回归模型，需要定义**自定义的ONNX符号**来处理动态形状：

```python
import torch
from transformers import GPT2LMHeadModel, GPT2Config
import onnx
from onnx import helper, numpy_helper, TensorProto

def convert_gpt2_with_kv_cache(model_path, output_path):
    # 加载预训练模型
    model = GPT2LMHeadModel.from_pretrained(model_path)
    model.eval()

    # 创建包装器，显式处理past_key_values
    class GPT2OnnxWrapper(torch.nn.Module):
        def __init__(self, model):
            super().__init__()
            self.model = model

        def forward(self, input_ids, attention_mask, *past_key_values):
            # 重塑past_key_values为模型期望的格式
            past = tuple(
                (past_key_values[i*2], past_key_values[i*2+1])
                for i in range(self.model.config.num_hidden_layers)
            )

            outputs = self.model(
                input_ids=input_ids,
                attention_mask=attention_mask,
                past_key_values=past,
                use_cache=True
            )

            # 分离logits和新的past_key_values
            logits = outputs.logits

            # 展平past_key_values作为输出
            new_past = outputs.past_key_values
            flattened_past = tuple(pk for layer in new_past for pk in layer)

            return (logits,) + flattened_past

    wrapper = GPT2OnnxWrapper(model)

    # 准备示例输入
    batch_size = 1
    seq_len = 1  # 流式推理时通常为1
    past_seq_len = 5  # 之前的缓存序列长度
    num_layers = model.config.num_hidden_layers
    num_heads = model.config.num_attention_heads
    head_dim = model.config.hidden_size // num_heads

    input_ids = torch.randint(0, model.config.vocab_size, (batch_size, seq_len))
    attention_mask = torch.ones(batch_size, seq_len + past_seq_len)

    # 创建初始past_key_values（第一次调用时可能为空）
    past_key_values = []
    for _ in range(num_layers):
        past_key_values.extend([
            torch.randn(batch_size, num_heads, past_seq_len, head_dim),
            torch.randn(batch_size, num_heads, past_seq_len, head_dim)
        ])

    # 导出到ONNX
    torch.onnx.export(
        wrapper,
        (input_ids, attention_mask, *past_key_values),
        output_path,
        input_names=['input_ids', 'attention_mask'] +
                    [f'past_key_{i}' for i in range(num_layers)] +
                    [f'past_value_{i}' for i in range(num_layers)],
        output_names=['logits'] +
                    [f'present_key_{i}' for i in range(num_layers)] +
                    [f'present_value_{i}' for i in range(num_layers)],
        dynamic_axes={
            'input_ids': {0: 'batch_size', 1: 'seq_length'},
            'attention_mask': {0: 'batch_size', 1: 'total_length'},
            **{f'past_key_{i}': {1: 'num_heads', 2: 'past_seq_length', 3: 'head_dim'}
               for i in range(num_layers)},
            **{f'past_value_{i}': {1: 'num_heads', 2: 'past_seq_length', 3: 'head_dim'}
               for i in range(num_layers)},
            **{f'present_key_{i}': {1: 'num_heads', 2: 'total_seq_length', 3: 'head_dim'}
               for i in range(num_layers)},
            **{f'present_value_{i}': {1: 'num_heads', 2: 'total_seq_length', 3: 'head_dim'}
               for i in range(num_layers)},
            'logits': {0: 'batch_size', 1: 'seq_length'}
        },
        opset_version=14,
        do_constant_folding=True
    )

    print(f"ONNX模型已保存到: {output_path}")
```

### 使用ONNX Runtime进行流式推理

转换后，可以使用ONNX Runtime进行高效的流式推理：

```python
import onnxruntime as ort
import numpy as np

class ONNXStreamingGenerator:
    def __init__(self, model_path, device='cuda'):
        self.session = ort.InferenceSession(
            model_path,
            providers=['CUDAExecutionProvider', 'CPUExecutionProvider']
            if device == 'cuda' else ['CPUExecutionProvider']
        )

        # 获取输入输出名称
        self.input_names = [inp.name for inp in self.session.get_inputs()]
        self.output_names = [out.name for inp in self.session.get_outputs()]

    def generate(self, input_ids, max_length=50, temperature=1.0, top_k=50):
        """流式生成文本"""
        batch_size = input_ids.shape[0]
        num_layers = (len(self.input_names) - 2) // 2

        # 初始past_key_values为None（使用空缓存）
        past_key_values = {
            name: np.zeros(
                (batch_size, 0, model_config.num_attention_heads, model_config.hidden_size // model_config.num_attention_heads),
                dtype=np.float32
            )
            for name in self.input_names if 'past_' in name
        }

        generated_tokens = []
        for step in range(max_length):
            # 准备当前步的输入
            inputs = {
                'input_ids': input_ids[:, -1:] if step > 0 else input_ids,
                'attention_mask': np.ones((batch_size, input_ids.shape[1])),
                **past_key_values
            }

            # ONNX推理
            outputs = self.session.run(self.output_names, inputs)
            logits = outputs[0]  # 第一个输出是logits
            present_kv = outputs[1:]  # 后续输出是KV cache

            # 采样下一个token
            next_token_logits = logits[:, -1, :] / temperature
            top_k_logits, top_k_indices = torch.topk(
                torch.from_numpy(next_token_logits), top_k
            )
            probs = torch.softmax(top_k_logits, dim=-1)
            next_token_id = top_k_indices[0, torch.multinomial(probs, 1)[0]].item()

            generated_tokens.append(next_token_id)

            # 更新past_key_values
            past_key_values = {
                name: present_kv[i]
                for i, name in enumerate([k for k in self.input_names if 'past_' in k])
            }

            # 准备下一步输入
            input_ids = np.concatenate(
                [input_ids, [[next_token_id]]],
                axis=1
            )

        return generated_tokens
```

## Beam Search实现考虑

对于需要beam search的场景，需要处理多个beam路径：

```python
def expand_past_key_values(past_kv, beam_size):
    """为beam search扩展past_key_values"""
    expanded = {}
    for name, tensor in past_kv.items():
        # shape: [batch, num_heads, seq_len, head_dim]
        batch, heads, seq_len, dim = tensor.shape
        expanded[name] = tensor.repeat_interleave(beam_size, dim=0)
    return expanded

def select_past_key_values(past_kv, beam_indices):
    """根据beam选择对应的past_key_values"""
    selected = {}
    for name, tensor in past_kv.items():
        selected[name] = tensor[beam_indices]
    return selected
```

## 性能优化建议

1. **缓存复用**：确保KV Cache在不同推理步骤间正确传递
2. **内存布局**：使用连续内存布局避免不必要的复制
3. **batch维度优化**：利用batch维度进行并行推理
4. **算子融合**：在导出时启用`opset_version >= 13`以利用更多融合优化

## 最佳实践总结

| 场景 | 推荐策略 |
|------|---------|
| 文本生成 | KV Cache + 动态sequence length |
| 语音识别 | RNN结构配合state输入输出 |
| 机器翻译 | 编码器-解码器分离 + KV Cache |
| 对话系统 | 上下文窗口管理 + cache修剪 |

## 常见陷阱

1. **past_key_values维度不匹配**：确保模型输入输出维度与ONNX定义一致
2. **attention_mask更新**：流式推理中需要正确更新attention mask
3. **序列截断**：长序列需要适当的缓存管理策略
4. **opset版本过低**：使用opset >= 13以支持更好的动态形状处理
