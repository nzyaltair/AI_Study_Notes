# 机器人控制与ROS

## 一句话理解

将规划生成的轨迹转化为电机指令执行，并通过反馈控制消除误差——是机器人从"计划"到"行动"的最后一步；ROS（机器人操作系统）是支撑这一过程的软件框架。

## 背景与发展

机器人控制理论源于经典控制理论，从简单的PID到现代的模型预测控制（MPC）和强化学习控制。ROS自2007年发布以来，已成为机器人软件开发的事实标准。

| 年代 | 里程碑 | 作者/机构 | 意义 |
|:---|:---|:---|:---|
| 1922 | PID控制器 | Minorsky | 反馈控制基础 |
| 1950s | 最优控制 | Bellman/Pontryagin | 现代控制理论 |
| 1980s | 阻抗控制 | Hogan | 力控交互 |
| 1986 | Subsumption架构 | Brooks | 行为主义机器人 |
| 2007 | ROS 1.0 | Stanford/Willow Garage | 机器人软件标准化 |
| 2010s | MPC普及 | 多家 | 实时最优控制 |
| 2017 | ROS 2.0 | Open Robotics | 支持实时/分布式 |
| 2020s | RL控制 | 多家 | 学习型运动控制 |

## 核心概念

### PID控制

最广泛使用的反馈控制算法：

$$u(t) = K_p e(t) + K_i \int_0^t e(\tau) d\tau + K_d \frac{de(t)}{dt}$$

- $K_p$（比例）：减小当前误差
- $K_i$（积分）：消除稳态误差
- $K_d$（微分）：预测误差趋势，改善动态响应

PID简单、通用，但不适合强非线性系统和多变量耦合系统。

### 模型预测控制（MPC）

在每个时刻求解有限时域最优控制问题：

$$\min_{\mathbf{u}_{0:N-1}} \sum_{k=0}^{N-1} L(\mathbf{x}_k, \mathbf{u}_k) + V_f(\mathbf{x}_N)$$

$$\text{s.t.} \quad \mathbf{x}_{k+1} = f(\mathbf{x}_k, \mathbf{u}_k), \quad \mathbf{x}_0 = \hat{\mathbf{x}}(t)$$

只执行第一个控制输入 $\mathbf{u}_0$，下一时刻重新求解。

优点：可处理约束、多变量耦合、前瞻规划。缺点：在线求解计算开销大。

### 阻抗控制

机器人与环境交互时的力控方法：

$$M_d \ddot{\mathbf{x}} + B_d \dot{\mathbf{x}} + K_d (\mathbf{x} - \mathbf{x}_d) = \mathbf{F}_{ext}$$

使机器人表现为虚拟弹簧-阻尼系统，$M_d, B_d, K_d$ 为期望惯性、阻尼和刚度。适合接触式操作（打磨、装配）。

### 行为架构（Subsumption Architecture）

Brooks提出的行为主义方法——将复杂行为分解为简单行为的层次叠加：

```
高层行为（探索）
    ↓ 抑制
中层行为（避障）
    ↓ 抑制
低层行为（前进）
```

每个行为独立运行，高层行为可抑制低层行为。与现代端到端学习方法形成对比。

## 工作原理：ROS

### ROS架构

ROS（Robot Operating System）不是真正的操作系统，而是机器人软件的中间件框架：

```
    应用层：导航/抓取/感知节点
         ↕ 通信
    ROS中间层：话题/服务/动作
         ↕
    操作系统：Linux/实时内核
         ↕
    硬件：传感器/执行器
```

### 核心概念

| 概念 | 说明 |
|:---|:---|
| 节点（Node） | 独立运行的进程，执行特定功能 |
| 话题（Topic） | 异步发布/订阅消息通道 |
| 服务（Service） | 同步请求/响应通信 |
| 动作（Action） | 长时任务的异步反馈通信 |
| 参数服务器 | 全局参数存储 |
| TF（Transform Tree） | 坐标变换树管理 |

### 常用ROS包

| 包 | 功能 |
|:---|:---|
| `move_base` / `Nav2` | 导航栈（路径规划+局部控制） |
| `AMCL` | 自适应蒙特卡洛定位 |
| `Gazebo` | 3D物理仿真器 |
| `RViz` | 3D可视化工具 |
| `MoveIt` | 机械臂运动规划框架 |
| `cartographer` | Google开源SLAM |

### ROS 1 vs ROS 2

| 维度 | ROS 1 | ROS 2 |
|:---|:---|:---|
| 中间件 | 自定义TCPROS | DDS（Data Distribution Service） |
| 实时性 | 不支持 | 支持实时QoS |
| 多机通信 | 需要roscore | 去中心化发现 |
| 安全性 | 有限 | 支持DDS安全层 |
| 适用 | 研究/原型 | 生产/工业 |

## 应用场景

| 场景 | 控制方法 |
|:---|:---|
| 工业机械臂 | PID/阻抗控制+轨迹跟踪 |
| 无人机 | PID/MPC姿态和位置控制 |
| 自动驾驶 | MPC路径跟踪 |
| 四足机器人 | MPC全身控制 |
| 协作机器人 | 阻抗控制+力感知 |
| 服务机器人 | 导航栈（AMCL+move_base） |

## 优势与不足

### 优势
- PID简单可靠，适用面广
- MPC可处理约束和多变量耦合
- ROS提供完善的软件生态和工具链
- 开源社区活跃，资源丰富

### 不足
- PID不适合强非线性系统
- MPC在线计算开销大，需要高性能硬件
- ROS 1缺乏实时性保证
- ROS学习曲线陡峭，配置复杂
