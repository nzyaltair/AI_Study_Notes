# 分词与子词切分

## 1. 概述

分词（Tokenization）是 NLP 的第一步——将连续的自然语言文本切分为计算机可处理的最小语言单位（Token）。分词质量直接决定下游任务的性能上限。英文等以空格分隔的语言看似简单，但缩写（don't）、复合词（state-of-the-art）、URL 等仍需特殊处理。中文、日文等词间无空格的语言则面临更根本的分词歧义问题。

子词切分（Subword Tokenization）是深度学习时代的主流方案——在"全词"和"字符"之间找到平衡点，既避免词表过大（OOV 问题），又避免字符序列过长（语义信息稀疏）。BPE、WordPiece、Unigram 等算法已成为现代大语言模型的标准分词组件。

## 3. 核心概念

### 3.1 词（Word）vs 子词（Subword）vs 字符（Character）

| 粒度 | 示例（"unhappiness"） | 优点 | 缺点 |
|:---|:---|:---|:---|
| 词 | ["unhappiness"] | 语义完整 | OOV 严重，词表大 |
| 子词 | ["un", "happiness"] | 平衡 OOV 和语义 | 需要学习切分策略 |
| 字符 | ["u","n","h","a",...] | 无 OOV | 序列长，语义稀疏 |

### 3.2 关键术语

| 术语 | 含义 |
|:---|:---|
| Vocabulary / Vocab | 词表，所有可用 token 的集合 |
| OOV（Out-of-Vocabulary） | 词表外的词，传统分词的痛点 |
| Type vs Token | 词表中的唯一词 vs 文本中的实例 |
| Morpheme | 语素，最小语义单位（如 un-、-happy、-ness） |
| Normalization | 文本标准化（大小写、全半角、Unicode） |

### 3.3 中文分词的歧义问题

中文没有空格分隔，分词面临多种歧义：

- **交集型歧义**："结婚的和尚未结婚的" → "结婚/的/和/尚未/结婚" vs "结婚/的/和尚/未/结婚"
- **组合型歧义**："羽毛球" → "羽毛/球" vs "羽毛球"
- **未登录词**：人名、新词、专有名词不在词典中

## 4. 技术原理

### 4.1 传统分词方法

#### 最大匹配法（Maximum Matching）

- **正向最大匹配（FMM）**：从左到右，取最长词典匹配
- **逆向最大匹配（BMM）**：从右到左，逆向取最长匹配
- **双向最大匹配**：比较 FMM 和 BMM 结果，选择分词数更少或歧义更少的

简单高效，但无法处理未登录词，且不考虑上下文。

#### 统计序列标注方法

将分词建模为序列标注问题——对每个字符标注 B（词首）、M（词中）、E（词尾）、S（单字）：

```
字符：  我  爱  自  然  语  言  处  理
标签：  S   S   B   E   B   E   B   E
分词：  我  爱  自然      语言      处理
```

使用 HMM 或 CRF 进行标注，能利用上下文消解歧义。参见 [[05_序列标注模型]]。

### 4.2 BPE（Byte Pair Encoding）

BPE 是一种贪心的子词切分算法，通过迭代合并最高频的符号对来构建词表。

**训练算法**：

```
1. 初始化词表为所有字符
2. 统计所有相邻符号对的频率
3. 合并频率最高的符号对为新 token
4. 将新 token 加入词表
5. 重复步骤 2-4 直到词表达到目标大小
```

**合并公式**：

$$\text{merge}_i = \arg\max_{(a, b)} \text{count}(a, b)$$

**示例**（"low"、"lower"、"newest"、"widest"）：

```
初始：l o w </w>   l o w e r </w>   n e w e s t </w>   w i d e s t </w>
合并 e s → es：    l o w </w>  l o w e r </w>  n e w es t </w>  w i d es t </w>
合并 es t → est：  l o w </w>  l o w e r </w>  n e w est </w>   w i d est </w>
...
```

**编码时**：对输入文本按训练得到的合并规则依次应用，将未见过的词分解为已知子词。

### 4.3 WordPiece

WordPiece 与 BPE 类似，但合并标准不同——BPE 合并频率最高的对，WordPiece 合并使语言模型似然增益最大的对：

$$\text{score}(a, b) = \frac{P(ab)}{P(a) \cdot P(b)}$$

选择使 $\text{score}(a,b)$ 最大的对进行合并。直觉上，如果 $ab$ 的概率远高于 $a$ 和 $b$ 独立出现的概率乘积，则 $ab$ 应作为一个单元。

BERT 使用 WordPiece，词表大小约 30K。

### 4.4 Unigram Language Model

Unigram LM 与 BPE/WordPiece 的思路相反——不是从字符逐步合并，而是从一个大词表开始逐步删减。

**训练算法**：

```
1. 从大量候选子词初始化大词表
2. 用 EM 算法估计每个子词的概率 P(w)
3. 计算删除每个子词后对语料似然的损失
4. 删除损失最小的子词（保留必要的）
5. 重复直到词表达到目标大小
```

**语言模型**：

$$P(\text{sentence}) = \prod_{i} P(\text{subword}_i)$$

对于同一个词，可能有多种子词切分方式，Unigram LM 选择概率最高的切分。SentencePiece 默认使用 Unigram 算法。

### 4.5 SentencePiece

SentencePiece 是 Google 开源的分词框架，具有以下关键特性：

- **语言无关**：将文本视为 Unicode 字符序列，不依赖空格分隔，适用于任何语言
- **可逆性**：分词结果可以无损还原为原始文本
- **支持 BPE 和 Unigram**：两种算法可选
- **特殊 token 支持**：如 `<s>`、`</s>`、`<unk>`

T5、ALBERT、XLNet、LLaMA 等模型使用 SentencePiece。

### 4.6 BBPE（Byte-level BPE）

BBPE 将文本编码为 UTF-8 字节序列后再做 BPE，使词表完全覆盖所有可能的字符——包括 emoji、罕见 Unicode 字符等，从根本上消除 OOV。

GPT-2/3/4 使用 BBPE，词表大小约 50K。

## 5. 关键方法/模型对比

| 方法 | 合并/选择标准 | 典型词表大小 | 代表模型 |
|:---|:---|:---|:---|
| BPE | 最高频符号对 | 30K-50K | GPT-2、LLaMA |
| WordPiece | 最大似然增益 | 30K | BERT、DistilBERT |
| Unigram LM | 最小似然损失 | 30K-50K | T5、ALBERT |
| BBPE | 字节级最高频对 | 50K | GPT-2/3/4 |
| SentencePiece | 框架（BPE/Unigram） | 30K-50K | T5、LLaMA |

### 不同分词器对比

```
输入文本: "I love natural language processing"

BPE:      ["I", " love", " natural", " language", " processing"]
WordPiece: ["i", "love", "natural", "language", "processing"]  (需要预分词)
SentencePiece: ["▁I", "▁love", "▁natural", "▁language", "▁processing"]
BBPE:     ["I", " love", " natural", " language", " processing"]  (字节级)
```

## 6. 优势与局限

### 子词切分的优势

1. **解决 OOV**：未见词可分解为已知子词（如 "unhappiness" → "un" + "happiness"）
2. **词表可控**：通过词表大小平衡覆盖率和模型效率
3. **形态学感知**：自动学习词缀等形态学单元
4. **跨语言通用**：SentencePiece 等框架语言无关

### 局限

1. **切分非唯一**：同一文本在不同分词器下结果不同，影响可比性
2. **中文粒度问题**：中文子词切分可能导致语义割裂
3. **多语言词表偏置**：词表中英语 token 占比通常过高，导致非英语压缩率低
4. **大小写信息丢失**：部分分词器统一小写化，丢失命名实体线索

## 7. 应用场景

- **大语言模型**：所有 LLM 都内置分词器，分词质量影响训练效率和推理速度
- **机器翻译**：子词切分使源语言和目标语言共享词表，减少 OOV
- **搜索引擎**：分词是检索系统的前端组件，影响召回率
- **输入法**：中文输入法核心依赖分词
- **语音识别**：分词器决定 ASR 的输出单位

## 8. 与其他技术关系

- **与序列标注模型的关系**：传统分词使用 HMM/CRF 建模为序列标注问题。参见 [[05_序列标注模型]]。
- **与大语言模型的关系**：分词器是 LLM 的基础组件，影响 token 效率、上下文长度利用和多语言能力。参见 [[10_大语言模型核心架构/04_分词与Tokenization]]。
- **与预训练语言模型的关系**：BERT（WordPiece）、GPT（BBPE）、T5（SentencePiece）的不同分词选择影响模型表现。参见 [[09_预训练语言模型/00_预训练语言模型_综述]]。
- **与信息检索的关系**：分词质量直接影响 BM25 等检索方法的召回率。参见 [[06_信息检索与文本挖掘]]。

## 9. 前沿发展

- **多语言词表均衡**：解决 LLM 中英语 token 占比过高的问题（如 LLaMA-3 扩展词表至 128K）
- **token 压缩率优化**：减少非英语语言的 token 消耗（如 Qwen 优化中文 token 效率）
- **动态分词**：根据上下文动态调整切分粒度
- **字节级模型**：直接处理字节序列，完全消除分词（如 ByT5、 MegaByte）
- **多模态 token**：将图像/音频 token 与文本 token 统一（如 VQ-VAE + SentencePiece）
