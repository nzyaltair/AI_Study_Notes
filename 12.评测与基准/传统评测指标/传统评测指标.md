# 传统评测指标

## 核心定位
本文聚焦 NLP 与生成式 AI 领域中经实践验证的传统自动评测指标，阐明其设计逻辑、适用边界与工程价值。传统指标虽与现代评测方法（如 [[LLM-as-a-Judge]]）存在差异，但在当代工作流中仍发挥着快速迭代、基线评估与自动化监控的重要作用。

## 系统性梳理

### 机器翻译任务

#### [[BLEU]]
- **数学定义**：基于 n-gram 精度的几何平均值，结合 brevity penalty
- **计算逻辑**：统计候选翻译与参考翻译的 n-gram 匹配率，惩罚过短翻译
- **输入要求**：候选翻译文本、多个参考翻译文本（推荐 4-5 个）
- **开源实现**：`sacrebleu`（推荐）、`nltk.translate.bleu_score`
- **典型阈值**：机器翻译：0.25-0.40（优秀）；文本生成：0.10-0.20

#### [[METEOR]]
- **数学定义**：考虑同义词、词干、形态变化的匹配度计算
- **计算逻辑**：基于精确匹配、词干匹配、同义词匹配的综合评分
- **输入要求**：候选翻译文本、参考翻译文本
- **开源实现**：`nltk.translate.meteor_score`
- **典型阈值**：机器翻译：0.30-0.45（优秀）

#### [[TER]] (Translation Edit Rate)
- **数学定义**：将候选翻译转换为参考翻译所需的编辑操作比率
- **计算逻辑**：计算插入、删除、替换操作的数量占参考翻译长度的比例
- **输入要求**：候选翻译文本、参考翻译文本
- **开源实现**：`ter-com` 工具
- **典型阈值**：机器翻译：0.20-0.35（优秀，值越低越好）

### 文本摘要任务

#### [[ROUGE]] 系列
- **数学定义**：基于召回率的评估指标，包括 ROUGE-N、ROUGE-L、ROUGE-W、ROUGE-S
- **计算逻辑**：统计摘要与参考摘要的 n-gram、最长公共子序列等重叠度
- **输入要求**：生成摘要、多个参考摘要
- **开源实现**：`rouge-score`、`nltk.translate.rouge_score`
- **典型阈值**：新闻摘要：ROUGE-1≈0.40，ROUGE-L≈0.35（优秀）

#### [[CIDEr]]
- **数学定义**：基于 TF-IDF 加权的 n-gram 相似度
- **计算逻辑**：计算生成摘要与参考摘要的语义相似度，强调信息量
- **输入要求**：生成摘要、多个参考摘要
- **开源实现**：`coco-caption` 工具包
- **典型阈值**：图像描述：0.80-1.20（优秀）

### 语言建模任务

#### [[Perplexity]]
- **数学定义**：模型预测概率的几何平均值的倒数
- **计算逻辑**：PPL = exp(-(1/N) * sum(log P(w_i | w_1...w_{i-1})))
- **输入要求**：模型、测试语料库
- **开源实现**：`transformers` 库的模型评估方法
- **典型阈值**：英语新闻语料：10-30（优秀，值越低越好）

### 分类任务

#### Accuracy
- **数学定义**：正确预测的样本数占总样本数的比例
- **计算逻辑**：(TP + TN) / (TP + TN + FP + FN)
- **输入要求**：预测标签、真实标签
- **开源实现**：`sklearn.metrics.accuracy_score`
- **典型阈值**：因任务而异，一般 >0.80 为良好

#### Precision & Recall
- **数学定义**：
  - Precision = TP / (TP + FP)
  - Recall = TP / (TP + FN)
- **计算逻辑**：分别衡量预测为正例的样本中实际为正例的比例，以及实际为正例的样本中被正确预测的比例
- **输入要求**：预测标签、真实标签
- **开源实现**：`sklearn.metrics.precision_score`、`sklearn.metrics.recall_score`
- **典型阈值**：因任务而异，需根据具体场景权衡

#### [[F1-score]]
- **数学定义**：Precision 和 Recall 的调和平均值
- **计算逻辑**：2 * (Precision * Recall) / (Precision + Recall)
- **输入要求**：预测标签、真实标签
- **开源实现**：`sklearn.metrics.f1_score`
- **典型阈值**：因任务而异，一般 >0.75 为良好

### 对话系统任务

#### Distinct-n
- **数学定义**：不同 n-gram 占总 n-gram 的比例
- **计算逻辑**：distinct-n = 唯一 n-gram 数 / 总 n-gram 数
- **输入要求**：对话回复文本
- **开源实现**：自定义实现
- **典型阈值**：distinct-1 > 0.4，distinct-2 > 0.8（表示多样性良好）

#### Coherence
- **数学定义**：基于话语间语义关联的评分
- **计算逻辑**：可通过连贯性模型或基于 [[BERT]] 的相似度计算
- **输入要求**：对话历史、当前回复
- **开源实现**：自定义实现或使用预训练连贯性模型
- **典型阈值**：因评估方法而异

## 实用性强化

### Python 代码片段

#### BLEU 计算示例
```python
# 库版本：sacrebleu==2.3.1
import sacrebleu

# 候选翻译
candidates = ["The cat sat on the mat"]
# 参考翻译（支持多个参考）
references = [["The cat is on the mat", "There is a cat on the mat"]]

# 计算 BLEU
bleu = sacrebleu.corpus_bleu(candidates, references)
print(f"BLEU score: {bleu.score:.2f}")
```

#### ROUGE 计算示例
```python
# 库版本：rouge-score==0.1.2
from rouge_score import rouge_scorer

scorer = rouge_scorer.RougeScorer(['rouge1', 'rougeL'], use_stemmer=True)
generated_summary = "The cat sat on the mat. It was happy."
reference_summary = "The cat is on the mat and it appears to be happy."

scores = scorer.score(reference_summary, generated_summary)
print(f"ROUGE-1: {scores['rouge1'].fmeasure:.2f}")
print(f"ROUGE-L: {scores['rougeL'].fmeasure:.2f}")
```

### 指标特性对比

| 指标 | 适用任务 | 敏感维度 | 常见陷阱 | 与人工评分相关性 |
|------|----------|----------|----------|------------------|
| BLEU | 机器翻译 | 词序、长度 | 无法评估语义、惩罚创造性表达 | 中（机器翻译）/ 低（创意文本） |
| ROUGE | 文本摘要 | 信息覆盖 | stemming 影响、对词序不敏感 | 中（摘要任务） |
| Perplexity | 语言建模 | 语法流畅度 | 与人类偏好弱相关 | 低（生成质量评估） |
| F1-score | 分类任务 | 类别平衡 | 对不平衡数据集敏感 | 高（分类任务） |
| Distinct-n | 对话系统 | 表达多样性 | 无法评估语义质量 | 中（多样性评估） |

### 典型误用案例

- **BLEU 误用**：用 BLEU 评估诗歌、创意写作等注重表达多样性的文本
- **ROUGE 误用**：忽略 `use_stemmer` 参数设置，导致不同评估结果不可比
- **Perplexity 误用**：直接用 Perplexity 评估生成文本的质量，忽略语义连贯性
- **分类指标误用**：在类别不平衡任务中仅使用 Accuracy，导致评估偏差

## 前瞻性延伸

### 传统指标的局限性
- 无法评估事实性：传统指标无法检测生成内容中的事实错误
- 无法评估安全性：无法识别有害内容、偏见或不当表达
- 无法评估推理链：对需要多步推理的任务评估能力有限
- 与人类偏好弱相关：特别是在创造性、连贯性等主观维度

### 与现代评测的互补关系
- **传统指标**：用于快速迭代、基线评估、自动化监控
- **[[人工评测]]**：用于终版验证、质量保证、用户体验评估
- **[[LLM-as-a-Judge]]**：用于维度扩展、复杂任务评估、多维度评分

### 研究引用
- ACL 2023 研究表明："BLEU 与人类偏好的相关性在低资源语言中显著下降"
- EMNLP 2024 研究发现："传统指标在评估长文本生成时表现不佳，建议结合 [[LLM-as-a-Judge]] 使用"
