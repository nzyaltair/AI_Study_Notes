# 大模型推理与优化

## 背景与发展

大语言模型（LLM）从研究走向生产部署时，推理阶段面临三大核心挑战：显存占用大（模型参数加 [[KV-Cache]]）、计算延迟高（自回归逐 token 生成）、部署成本高昂（GPU 资源消耗）。随着模型规模从 7B 扩展到 175B+，这些瓶颈日益突出，推理优化技术成为 LLM 落地的关键。

早期的推理优化主要集中在模型压缩（量化、剪枝）上。2023 年后，以 [[vLLM]] 的 PagedAttention、FlashAttention、Speculative Decoding 为代表的系统级优化成为主流，推理吞吐量提升了数倍至数十倍。

## 核心思想

推理优化围绕四个核心方向展开：

- **减少重复计算**：[[KV-Cache]] 缓存注意力键值对，避免自回归生成中的冗余计算
- **降低显存占用**：[[量化]] 将 FP16 权重压缩到 INT4/INT8，大幅减少模型占用
- **提升吞吐量**：连续批处理（Continuous Batching）和并行策略最大化 GPU 利用率
- **控制生成质量**：解码与采样策略在确定性与多样性之间权衡

## 技术原理

### KV-Cache 机制

自回归生成时缓存已计算的 Key 和 Value，生成第 $t$ 个 token 时只需计算新 token 的 Q 与缓存中所有 K 的注意力。显存占用公式：

$$M_{\text{KV}} = 2 \times n_{\text{layers}} \times n_{\text{heads}} \times d_{\text{head}} \times \text{seq\_len} \times \text{batch} \times \text{dtype\_size}$$

GQA/MQA 通过减少 KV 头数降低缓存大小，PagedAttention 通过分页管理减少碎片。

### 量化技术

将 FP16 权重映射到低精度表示：$w_{\text{quant}} = \text{round}(w / s) \times s$，其中 $s$ 为缩放因子。

| 方法 | 精度 | 特点 | 工具 |
|------|------|------|------|
| GPTQ | 4-bit | 逐层量化，基于二阶信息 | AutoGPTQ |
| AWQ | 4-bit | 激活感知量化，保护重要权重 | AutoAWQ |
| GGUF | 2-8 bit | CPU/GPU 混合，llama.cpp 生成 | llama.cpp |
| INT8 | 8-bit | 简单高效，精度损失小 | bitsandbytes |
| FP8 | 8-bit 浮点 | 硬件原生支持，H100 | TensorRT-LLM |

### 推理加速框架

| 框架 | 核心技术 | 适用场景 |
|------|---------|---------|
| vLLM | PagedAttention、连续批处理 | 通用 LLM 服务 |
| TensorRT-LLM | 内核融合、INT8/FP8 | NVIDIA 最优性能 |
| llama.cpp | CPU/GPU 混合、GGUF 量化 | 本地部署、边缘设备 |
| SGLang | RadixAttention、结构化生成 | Agent 应用 |

### 解码与采样策略

| 策略 | 原理 | 特点 |
|------|------|------|
| Greedy | 每步选概率最高 token | 确定性，易重复 |
| Temperature | $P' = \text{softmax}(\text{logits} / T)$ | T<1 更确定，T>1 更多样 |
| Top-K | 只从概率最高 K 个 token 采样 | 控制多样性 |
| Top-P | 从累积概率 >= P 的最小集合采样 | 自适应多样性 |
| Beam Search | 维护多个候选序列 | 适合翻译，不适合对话 |
| Speculative Decoding | 小模型草拟，大模型验证 | 加速 2-3x |

## 发展演进

KV-Cache -> FlashAttention -> 量化（GPTQ/AWQ/GGUF）-> PagedAttention/vLLM -> Speculative Decoding -> FP8 推理

## 关键算法·模型

- **PagedAttention**（vLLM）：分页 KV-Cache 管理，类似 OS 虚拟内存
- **FlashAttention**：分块计算注意力，减少 HBM 读写
- **SmoothQuant**：将激活量化难度转移到权重
- **StreamingLLM**：Attention Sink + 滑动窗口，支持无限长度
- **YaRN**：分频段 RoPE 外推，扩展上下文长度

## 应用场景

- **LLM 服务**：API 服务（OpenAI、vLLM 部署）
- **本地部署**：llama.cpp 在消费级 GPU/CPU 上运行
- **边缘部署**：手机端 LLM（GGUF 量化）
- **高并发场景**：连续批处理提升服务吞吐
- **长上下文应用**：KV-Cache 压缩与注意力优化

## 与其他技术关系

- 推理优化依赖 [[注意力机制]] 和 [[大语言模型]] 架构的理解
- [[量化]] 与混合精度训练的技术原理相通
- [[KV-Cache]] 是自回归推理的核心优化机制
- 解码策略影响 [[Agent]] 的输出质量
- 长上下文扩展涉及位置编码外推与 KV-Cache 管理
