# 大模型训练优化

## 1. 概述

随着 [[00_大语言模型核心架构_综述|大语言模型]] 参数量从亿级增长到千亿甚至万亿级，传统优化方法面临显存不足、训练不稳定、通信开销巨大等挑战。大模型训练优化技术通过分层自适应学习率（LARS/LAMB）、混合精度训练、梯度累积、ZeRO 内存优化等手段，使超大规模模型训练成为可能。这些技术是现代 LLM 训练基础设施的核心组件。

## 2. 发展历史

| 年代 | 里程碑 | 意义 |
|------|--------|------|
| 2017 | LARS (You et al.) | 分层自适应学习率，支持超大 batch |
| 2017 | Mixed Precision Training (Micikevicius et al.) | FP16 混合精度训练 |
| 2018 | LAMB (You et al.) | Adam 的分层版本，支持 64K+ batch |
| 2019 | ZeRO (Rajbhandari et al.) | 零冗余优化器，消除内存冗余 |
| 2020 | FSDP (PyTorch) | 完全分片数据并行 |
| 2021 | 8-bit Optimizer (Dettmers et al.) | 优化器状态量化 |
| 2022 | BF16 普及 | 替代 FP16，无需损失缩放 |
| 2023 | Flash Attention | 注意力计算内存优化 |
| 2024 | 1-bit Adam / 8-bit Adam | 通信和存储压缩 |

## 3. 核心概念

- **Batch Size 扩展**：增大 batch 可以提高并行度和吞吐量，但需要调整学习率和优化策略
- **混合精度训练**：用低精度（FP16/BF16）计算前向和反向，高精度（FP32）维护主权重
- **梯度累积**：通过多次小 batch 的梯度累加来模拟大 batch 训练
- **内存墙**：模型参数 + 梯度 + 优化器状态的总和超过单 GPU 显存
- **通信墙**：多 GPU 间梯度同步的通信开销成为瓶颈

### 大模型训练的内存构成

以 Adam 优化器为例，训练一个 $N$ 参数的模型需要存储：

| 组件 | 精度 | 内存 |
|------|------|------|
| 模型参数 | FP32 | $4N$ |
| 梯度 | FP32 | $4N$ |
| Adam 一阶矩 $m$ | FP32 | $4N$ |
| Adam 二阶矩 $v$ | FP32 | $4N$ |
| **总计** | | **$16N$** |

7B 参数模型需要 $16 \times 7 \times 10^9 = 112$ GB，远超单 GPU 80GB 显存。

混合精度（BF16 参数 + FP32 主权重）下：

| 组件 | 精度 | 内存 |
|------|------|------|
| 模型参数 | BF16 | $2N$ |
| 主权重 | FP32 | $4N$ |
| 梯度 | BF16 | $2N$ |
| Adam $m$ | FP32 | $4N$ |
| Adam $v$ | FP32 | $4N$ |
| **总计** | | **$16N$** |

混合精度主要节省激活值内存，优化器状态仍占大头。

## 4. 技术原理

### 4.1 LARS（Layer-wise Adaptive Rate Scaling）

You et al. (2017) 提出，为网络的每一层独立计算学习率：

$$\lambda_l = \frac{\eta \cdot \|w_l\|}{\|g_l\| + \beta \|w_l\|}$$

$$w_l \leftarrow w_l - \lambda_l \cdot g_l$$

其中 $w_l$ 和 $g_l$ 分别是第 $l$ 层的权重和梯度，$\beta$ 是小常数（如 $10^{-4}$）。

**核心思想**：每层的"信任比"（gradient-to-weight ratio）不同，用各自的比例缩放学习率，使所有层以相似的相对速度更新。

**效果**：支持将 batch size 扩展到 32K+ 而不损失精度，在 ImageNet 训练中将 ResNet-50 从 90 epoch 压缩到几分钟。

### 4.2 LAMB（Layer-wise Adaptive Moments for Batch optimization）

You et al. (2019) 提出，将 LARS 的分层思想与 Adam 结合：

$$r_l = \frac{\|w_l\|}{\|\hat{m}_l / (\sqrt{\hat{v}_l} + \epsilon)\| + \beta \|w_l\|}$$

$$w_l \leftarrow w_l - \eta \cdot r_l \cdot \frac{\hat{m}_l}{\sqrt{\hat{v}_l} + \epsilon}$$

**效果**：支持 64K+ 的超大 batch 训练 BERT，将预训练时间从 3 天缩短到 76 分钟。是超大 batch 训练的关键技术。

### 4.3 混合精度训练

Micikevicius et al. (2017) 提出，用 FP16 计算前向和反向传播，用 FP32 维护主权重：

```
FP32 主权重 → 拷贝为 FP16 → FP16 前向 → FP16 反向 → FP32 梯度更新 → FP32 主权重
```

**损失缩放（Loss Scaling）**：FP16 的精度范围有限（$[6 \times 10^{-8}, 65504]$），小梯度会下溢。将损失乘以 $S$ 倍使梯度放大到可表示范围，更新时再除以 $S$：

$$\text{scaled\_loss} = \text{loss} \times S$$
$$\text{grad} = \text{scaled\_grad} / S$$

**BF16 替代**：BF16 有 8 位指数（与 FP32 相同）+ 7 位尾数，动态范围与 FP32 相同但精度更低。使用 BF16 **无需损失缩放**，大幅简化训练流程。Ampere/Hopper 架构 GPU 原生支持 BF16。

### 4.4 梯度累积

当 GPU 显存不足以容纳大 batch 时，通过多次小 batch 前向+反向累积梯度：

```python
optimizer.zero_grad()
for i, (x, y) in enumerate(loader):
    loss = model(x, y) / accumulation_steps
    loss.backward()
    if (i + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()
```

**等效效果**：$B_{\text{eff}} = B_{\text{mini}} \times \text{accumulation\_steps}$

**局限**：前向传播仍需逐步执行，不能真正加速训练，只是用时间换取显存。

### 4.5 ZeRO（Zero Redundancy Optimizer）

Rajbhandari et al. (2019) 提出，通过将训练状态分片到多个 GPU 上消除冗余：

| 阶段 | 分片内容 | 内存（$N$ 参数，$K$ GPU） | 适用场景 |
|------|---------|--------------------------|---------|
| ZeRO-1 | 优化器状态 | $4N + 12N/K$ | 基础场景 |
| ZeRO-2 | 优化器状态 + 梯度 | $2N + (2+12)N/K$ | 中等规模 |
| ZeRO-3 | 优化器状态 + 梯度 + 参数 | $16N/K$ | 超大模型 |

**关键思想**：传统数据并行中每个 GPU 都保存完整的优化器状态副本，ZeRO 将这些状态均匀分片，需要时再通过 All-Gather 收集。

**通信开销**：ZeRO-1/2 不增加额外通信（梯度同步改为 Reduce-Scatter）；ZeRO-3 需要额外的 All-Gather 收集参数，通信量约为数据并行的 1.5 倍。

### 4.6 FSDP（Fully Sharded Data Parallel）

PyTorch 原生实现的 ZeRO-3 等价方案：

- 参数在计算前通过 All-Gather 收集
- 计算完成后释放非本地参数
- 梯度通过 Reduce-Scatter 同步并分片

```python
from torch.distributed.fsdp import FullyShardedDataParallel as FSDP

model = Model()
model = FSDP(model)  # 自动分片
```

**优势**：原生集成 PyTorch DDP，支持混合精度、梯度检查点、CPU offload。

### 4.7 梯度检查点（Gradient Checkpointing）

以时间换空间的激活值内存优化技术：

- 前向传播时不保存中间激活值，只保存检查点
- 反向传播时从最近的检查点重新计算激活值

**效果**：将激活值内存从 $O(L)$ 降到 $O(\sqrt{L})$（$L$ 为层数），代价是增加约 33% 的前向计算。

### 4.8 CPU Offloading

将不活跃的参数或优化器状态卸载到 CPU 内存甚至 NVMe SSD：

- **ZeRO-Offload**：优化器状态和梯度在 CPU 上更新
- **ZeRO-Infinity**：利用 NVMe SSD 进一步扩展可用内存

**权衡**：PCIe 带宽（~32 GB/s）远低于 GPU 显存带宽（~900 GB/s），通信成为瓶颈。

## 5. 大模型训练优化器配置

### GPT-3 / LLaMA 级别典型配置

| 组件 | 配置 |
|------|------|
| 优化器 | AdamW |

| 权重衰减 | 0.1 |
| 梯度裁剪 | 1.0 |
| Batch Size | 4M tokens |
| 精度 | BF16 |
| 梯度累积 | 视 GPU 数和显存而定 |

### 训练不稳定问题

大模型训练中常见的不稳定问题及对策：

| 问题 | 原因 | 对策 |
|------|------|------|
| Loss Spike | 梯度爆炸/数据异常 | 降低学习率、跳过异常 batch |
| 梯度爆炸 | 深层网络梯度连乘 | 梯度裁剪 |
| 精度溢出 | FP16 下溢 | 损失缩放或使用 BF16 |
| 训练发散 | 学习率过大 | 更长 warmup、降低学习率 |
| 死神经元 | 激活值溢出 | 降低学习率、使用更稳定初始化 |

## 6. 优势与局限

### 优势

- 使千亿级参数模型的训练成为可能
- 混合精度训练显著提升计算吞吐量（2-3 倍）
- ZeRO/FSDP 消除数据并行的内存冗余
- LARS/LAMB 支持超大 batch 训练，提升并行效率

### 局限

- 混合精度可能引入数值不稳定
- ZeRO-3/FSDP 增加通信开销
- 梯度检查点增加约 33% 计算时间
- CPU Offloading 受 PCIe 带宽限制
- 大 batch 训练的泛化性能可能下降（需调整学习率/正则化）

## 9. 前沿发展

- **1-bit Adam / 0/1 Adam**：梯度通信压缩到 1 bit，减少通信量 50 倍
- **8-bit Optimizer**：将 Adam 状态量化为 8 bit，内存减半
- **Sophia / Muon**：二阶优化器在大模型中的实用化
- **Flash Attention**：IO 感知的注意力计算，减少 HBM 读写
- **Ring Attention**：跨设备分布式注意力计算，支持超长序列
- **训练崩溃恢复**：异步检查点和状态恢复机制
- **弹性训练**：节点动态加入/退出时的训练容错
