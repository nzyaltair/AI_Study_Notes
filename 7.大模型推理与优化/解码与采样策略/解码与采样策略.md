# 解码与采样策略

## 1. 简介

大语言模型（LLM）的文本生成能力是其核心价值之一，而解码与采样策略则直接决定了生成文本的质量、多样性、可控性和效率。本文档旨在为AI开发人员和推理系统工程师提供一份全面的解码与采样策略技术指南，涵盖主流方法、新兴技术、工程实现和实践建议。

%% 核心目标：理解并掌握各种解码与采样策略，实现可控、高效、高质量的文本生成 %% ^core-objective

### 1.1 文档目标与读者定位

**文档目标**：提供大模型推理阶段文本生成控制机制的系统性知识，服务于实际部署、可控生成与性能调优需求。

**读者定位**：
- AI开发人员：理解解码与采样策略的原理和实现
- 推理系统工程师：掌握解码策略的性能优化和工程实践
- 研究人员：了解解码与采样的前沿演进和创新方向
- 产品经理：理解不同策略对生成结果的影响

### 1.2 解码与采样的基本概念

**解码（Decoding）**：从模型预测的概率分布中选择下一个token的过程。

**采样（Sampling）**：基于概率分布随机选择token的解码方式。

**核心权衡**：
- 生成质量 vs 多样性
- 生成速度 vs 计算开销
- 可控性 vs 创造性

**关键指标**：
- 困惑度（Perplexity）：衡量模型预测的准确性
- BLEU/ROUGE：衡量生成文本与参考文本的相似度
- 多样性指标（如Self-BLEU、Distinct-n）：衡量生成文本的多样性
- 人类评价：衡量生成文本的质量和可用性

## 2. 确定性解码策略

确定性解码策略每次生成唯一的token序列，不引入随机性。

### 2.1 贪心搜索（Greedy Search）

**工作原理**：每次选择概率最高的token作为下一个生成的token。

**超参数**：无

**生成质量 vs 多样性**：
- 质量：高（语法正确，连贯性好）
- 多样性：低（易产生重复和单调的文本）

**计算开销**：低（每次只需要计算最高概率token）

**适用场景**：
- 对生成结果确定性要求高的场景（如代码生成、数学计算）
- 资源受限的边缘设备

**常见问题**：
- 重复生成：容易生成重复的短语或句子
- 局部最优：可能陷入局部最优，无法找到全局最优解
- 退化：生成的文本可能逐渐退化，失去意义

**API示例（Hugging Face Transformers）**：
```python
from transformers import AutoModelForCausalLM, AutoTokenizer

model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-2-7b-chat-hf")
tokenizer = AutoTokenizer.from_pretrained("meta-llama/Llama-2-7b-chat-hf")

prompt = "Hello, how are you?"
inputs = tokenizer(prompt, return_tensors="pt")

# 使用贪心搜索生成文本
outputs = model.generate(
    **inputs,
    max_new_tokens=100,
    do_sample=False,  # 关闭采样，使用贪心搜索
)

print(tokenizer.decode(outputs[0], skip_special_tokens=True))
```

### 2.2 波束搜索（Beam Search）

**工作原理**：维护一个大小为k的候选序列（波束），每次扩展所有候选序列，然后选择概率最高的k个序列作为新的波束。

**超参数**：
- `num_beams`：波束大小，控制候选序列数量
- `length_penalty`：长度惩罚，鼓励或抑制长文本生成
- `early_stopping`：是否在生成EOS token时停止

**生成质量 vs 多样性**：
- 质量：高（语法正确，连贯性好，避免局部最优）
- 多样性：低（生成的文本仍然比较保守）

**计算开销**：中高（与波束大小k成正比）

**适用场景**：
- 机器翻译、摘要生成等对生成质量要求高的场景
- 需要避免局部最优的长文本生成

**常见问题**：
- 生成速度慢：随着波束大小增加，计算开销线性增长
- 缺乏多样性：生成的文本可能过于保守和相似
- 重复生成：仍然可能出现重复的短语或句子

**API示例（Hugging Face Transformers）**：
```python
# 使用波束搜索生成文本
outputs = model.generate(
    **inputs,
    max_new_tokens=100,
    do_sample=False,
    num_beams=4,  # 波束大小为4
    length_penalty=1.0,
    early_stopping=True,
)

print(tokenizer.decode(outputs[0], skip_special_tokens=True))
```

**双链链接**：[[波束搜索]]

### 2.3 波束搜索变体

#### 2.3.1 多样化波束搜索（Diverse Beam Search）

**工作原理**：通过惩罚相似的候选序列，鼓励生成多样化的文本。

**超参数**：
- `num_beam_groups`：波束组数量
- `diversity_penalty`：多样性惩罚强度

**特点**：
- 生成质量：中高
- 多样性：中高
- 计算开销：高（与波束组数量和波束大小成正比）

**API示例（Hugging Face Transformers）**：
```python
# 使用多样化波束搜索生成文本
outputs = model.generate(
    **inputs,
    max_new_tokens=100,
    do_sample=False,
    num_beams=4,
    num_beam_groups=2,
    diversity_penalty=1.0,
)

print(tokenizer.decode(outputs[0], skip_special_tokens=True))
```

#### 2.3.2 长度惩罚优化

**工作原理**：通过调整长度惩罚系数，控制生成文本的长度。

**长度惩罚公式**：
```
length_penalty = (5 + len(output)) ** alpha / (5 + 1) ** alpha
```

**alpha值影响**：
- alpha < 1：鼓励生成短文本
- alpha = 1：中性，不偏向任何长度
- alpha > 1：鼓励生成长文本

## 3. 随机性采样策略

随机性采样策略通过引入随机性，生成多样化的文本序列。

### 3.1 随机采样（Random Sampling）

**工作原理**：基于模型预测的概率分布随机选择下一个token。

**超参数**：无

**生成质量 vs 多样性**：
- 质量：低（可能生成语法错误或不连贯的文本）
- 多样性：高（生成的文本多样化，富有创造性）

**计算开销**：低（每次只需要基于概率分布采样一个token）

**适用场景**：
- 对多样性要求高的创意写作场景
- 需要探索多种可能生成结果的场景

**常见问题**：
- 语法错误：可能生成语法错误或不连贯的文本
- 失控：生成的文本可能偏离主题，失去意义
- 幻觉：可能加剧模型的幻觉问题

**API示例（Hugging Face Transformers）**：
```python
# 使用随机采样生成文本
outputs = model.generate(
    **inputs,
    max_new_tokens=100,
    do_sample=True,  # 启用采样
    temperature=1.0,  # 默认温度
)

print(tokenizer.decode(outputs[0], skip_special_tokens=True))
```

### 3.2 温度采样（Temperature Scaling）

**工作原理**：通过调整温度参数，控制概率分布的平滑程度。

**温度公式**：
```
scaled_logits = logits / temperature
probabilities = softmax(scaled_logits)
```

**超参数**：
- `temperature`：温度参数，控制随机性程度

**温度值影响**：
- temperature → 0：接近贪心搜索，生成确定性文本
- temperature = 1：原始概率分布，平衡质量和多样性
- temperature → ∞：均匀分布，生成完全随机的文本

**生成质量 vs 多样性**：
- 质量：随温度升高而降低
- 多样性：随温度升高而增加

**计算开销**：低（只需要额外的温度缩放操作）

**适用场景**：
- 需要平衡质量和多样性的场景
- 可以通过调整温度参数控制生成结果的随机性

**常见问题**：
- 温度过高：生成的文本可能失去意义
- 温度过低：生成的文本可能过于保守

**API示例（Hugging Face Transformers）**：
```python
# 使用温度采样生成文本
outputs = model.generate(
    **inputs,
    max_new_tokens=100,
    do_sample=True,
    temperature=0.7,  # 降低温度，减少随机性
)

print(tokenizer.decode(outputs[0], skip_special_tokens=True))
```

**双链链接**：[[温度采样]]

### 3.3 Top-k 采样

**工作原理**：只从概率最高的k个token中采样下一个token。

**超参数**：
- `top_k`：保留的最高概率token数量

**top_k值影响**：
- top_k = 1：贪心搜索，确定性生成
- top_k 较小（如5-20）：平衡质量和多样性
- top_k 较大（如100+）：接近随机采样，多样性高

**生成质量 vs 多样性**：
- 质量：中高（避免生成低概率的错误token）
- 多样性：中（生成的文本具有一定多样性）

**计算开销**：低（只需要排序并保留top-k个token）

**适用场景**：
- 大多数需要平衡质量和多样性的场景
- 对生成错误容忍度低的场景

**常见问题**：
- 缺乏灵活性：对于不同的输入，最佳top-k值可能不同
- 可能排除低概率但有意义的token

**API示例（Hugging Face Transformers）**：
```python
# 使用Top-k采样生成文本
outputs = model.generate(
    **inputs,
    max_new_tokens=100,
    do_sample=True,
    top_k=50,  # 保留概率最高的50个token
)

print(tokenizer.decode(outputs[0], skip_special_tokens=True))
```

### 3.4 Top-p（Nucleus）采样

**工作原理**：从累积概率超过p的最小token集合中采样下一个token。

**超参数**：
- `top_p`：累积概率阈值

**top_p值影响**：
- top_p → 0：接近贪心搜索，确定性生成
- top_p = 0.9：常用值，平衡质量和多样性
- top_p → 1：接近随机采样，多样性高

**生成质量 vs 多样性**：
- 质量：中高（避免生成低概率的错误token）
- 多样性：中高（根据输入动态调整token集合大小）

**计算开销**：中（需要计算累积概率并确定token集合）

**适用场景**：
- 大多数需要平衡质量和多样性的场景
- 输入变化较大的场景（top-p会根据输入动态调整token集合大小）

**常见问题**：
- 可能生成过长或过短的文本
- 在某些情况下可能缺乏多样性

**API示例（Hugging Face Transformers）**：
```python
# 使用Top-p采样生成文本
outputs = model.generate(
    **inputs,
    max_new_tokens=100,
    do_sample=True,
    top_p=0.9,  # 累积概率阈值为0.9
)

print(tokenizer.decode(outputs[0], skip_special_tokens=True))
```

### 3.5 Top-k + Top-p 组合采样

**工作原理**：同时使用Top-k和Top-p采样，先应用Top-k过滤，再应用Top-p过滤。

**超参数**：
- `top_k`：保留的最高概率token数量
- `top_p`：累积概率阈值

**特点**：
- 生成质量：高（结合了Top-k和Top-p的优点）
- 多样性：中高（平衡了质量和多样性）
- 计算开销：中（需要额外的Top-k和Top-p过滤操作）

**适用场景**：
- 大多数需要平衡质量和多样性的场景
- 对生成结果质量要求较高的场景

**API示例（Hugging Face Transformers）**：
```python
# 使用Top-k + Top-p组合采样生成文本
outputs = model.generate(
    **inputs,
    max_new_tokens=100,
    do_sample=True,
    top_k=50,
    top_p=0.9,
)

print(tokenizer.decode(outputs[0], skip_special_tokens=True))
```

## 4. 高级采样策略

高级采样策略通过更复杂的机制，进一步平衡生成质量和多样性。

### 4.1 Tail-free Sampling (TFS)

**工作原理**：通过过滤掉概率分布尾部的低概率token，减少生成的随机性和错误。

**超参数**：
- `tail_free_sampling`：尾部过滤阈值（0-1）

**特点**：
- 生成质量：高（减少低概率错误token的生成）
- 多样性：中（保持一定的多样性）
- 计算开销：中（需要额外的尾部过滤操作）

**适用场景**：
- 需要高质量生成结果的场景
- 对生成错误容忍度低的场景

**API示例（Hugging Face Transformers）**：
```python
# 使用Tail-free Sampling生成文本
outputs = model.generate(
    **inputs,
    max_new_tokens=100,
    do_sample=True,
    top_p=0.9,
    tail_free_sampling=0.9,
)

print(tokenizer.decode(outputs[0], skip_special_tokens=True))
```

### 4.2 Typical Sampling

**工作原理**：基于token的典型性（typicality）进行采样，优先选择与上下文更匹配的token。

**超参数**：
- `typical_p`：典型性阈值（0-1）

**特点**：
- 生成质量：高（生成的文本更符合上下文）
- 多样性：中（保持一定的多样性）
- 计算开销：中（需要计算token的典型性）

**适用场景**：
- 需要生成与上下文高度相关文本的场景
- 对话生成、故事续写等场景

**API示例（Hugging Face Transformers）**：
```python
# 使用Typical Sampling生成文本
outputs = model.generate(
    **inputs,
    max_new_tokens=100,
    do_sample=True,
    typical_p=0.95,
)

print(tokenizer.decode(outputs[0], skip_special_tokens=True))
```

### 4.3 Contrastive Decoding

**工作原理**：通过比较目标模型和一个简单基线模型的输出，选择差异最大的token。

**核心思想**：
```
logits_contrastive = logits_target - alpha * logits_baseline
```

**超参数**：
- `alpha`：对比权重（0-1）

**特点**：
- 生成质量：高（生成的文本更有信息量）
- 多样性：中（保持一定的多样性）
- 计算开销：高（需要同时运行两个模型）

**适用场景**：
- 需要生成高质量、有信息量文本的场景
- 摘要生成、问答等场景

**双链链接**：[[对比学习]]

### 4.4 推测解码中的采样协同

**工作原理**：在推测解码（Speculative Decoding）中，使用小模型生成候选token序列，然后用大模型验证，结合采样策略提高生成效率和质量。

**核心步骤**：
1. 小模型使用采样策略生成多个候选token
2. 大模型验证候选token序列
3. 接受部分或全部候选token，继续生成

**特点**：
- 生成速度：快（减少了大模型的解码步数）
- 生成质量：高（大模型验证保证了生成质量）
- 多样性：中高（小模型采样引入了多样性）

**适用场景**：
- 对生成速度要求高的场景
- 需要平衡速度和质量的场景

**双链链接**：[[推测解码]]

## 5. 约束解码与正则引导

约束解码与正则引导通过引入外部约束或正则化，控制生成文本的内容和风格。

### 5.1 约束解码（Constrained Decoding）

**工作原理**：限制生成的token必须属于预定义的集合或遵循特定的规则。

**主要方法**：
- **前缀约束**：生成的文本必须以特定前缀开头
- **词汇约束**：生成的文本必须包含或不包含特定词汇
- **语法约束**：生成的文本必须遵循特定的语法规则
- **结构约束**：生成的文本必须遵循特定的结构（如JSON、XML）

**特点**：
- 生成质量：高（符合约束条件）
- 多样性：低（受约束条件限制）
- 计算开销：高（需要额外的约束检查和处理）

**适用场景**：
- 对生成结果格式要求严格的场景（如代码生成、JSON生成）
- 需要控制生成内容的场景（如避免生成有害内容）

**API示例（Hugging Face Transformers）**：
```python
from transformers import AutoModelForCausalLM, AutoTokenizer, LogitsProcessorList,迫使生成token处理器

model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-2-7b-chat-hf")
tokenizer = AutoTokenizer.from_pretrained("meta-llama/Llama-2-7b-chat-hf")

prompt = "生成一个JSON格式的用户信息："
inputs = tokenizer(prompt, return_tensors="pt")

# 定义词汇约束：只允许生成JSON相关的token
allowed_tokens = tokenizer.encode("{", "}", ":", ",", "\"", "user", "name", "age", "email", add_special_tokens=False)

# 使用约束解码生成文本
outputs = model.generate(
    **inputs,
    max_new_tokens=100,
    do_sample=False,
    num_beams=4,
    # 添加约束处理器
    logits_processor=LogitsProcessorList([
       迫使生成token处理器(allowed_tokens)
    ]),
)

print(tokenizer.decode(outputs[0], skip_special_tokens=True))
```

### 5.2 CFG-style 正则引导

**工作原理**：通过对比目标条件和无条件生成的概率，引导模型生成符合特定条件的文本。

**核心公式**：
```
logits = logits_cond + beta * (logits_cond - logits_uncond)
```

**超参数**：
- `beta`：引导强度（0-1）

**特点**：
- 生成质量：高（符合引导条件）
- 多样性：中（保持一定的多样性）
- 计算开销：高（需要同时运行条件和无条件生成）

**适用场景**：
- 需要控制生成文本风格的场景（如生成特定情感的文本）
- 需要引导生成特定内容的场景（如生成符合特定主题的文本）

**API示例（vLLM）**：
```python
from vllm import LLM, SamplingParams

llm = LLM(model="meta-llama/Llama-2-7b-chat-hf")

# 定义CFG-style引导的采样参数
sampling_params = SamplingParams(
    temperature=0.7,
    top_p=0.9,
    cfg_scale=1.5,  # CFG引导强度
    cfg_neg_prompt="生成一个负面的评论",  # 无条件生成的提示
)

prompt = "生成一个正面的产品评论："
outputs = llm.generate([prompt], sampling_params)

print(outputs[0].outputs[0].text)
```

### 6.4 TensorRT-LLM

**支持的解码策略**：
- 贪心搜索
- 波束搜索
- 随机采样
- 温度采样
- Top-k采样
- Top-p采样

**特点**：
- 极高性能（针对NVIDIA GPU优化）
- 支持张量并行和流水线并行
- 支持INT4/INT8量化
- 适合大规模部署

**双链链接**：[[TensorRT-LLM]]

## 7. 常见问题与解决方案

### 7.1 重复生成

**问题**：生成的文本中出现重复的短语或句子。

**解决方案**：
- 使用较低的温度（0.7-0.9）
- 增加Top-k或Top-p的值
- 使用Tail-free Sampling或Typical Sampling
- 添加重复惩罚（如`repetition_penalty`参数）
- 使用约束解码避免重复模式

**API示例（Hugging Face Transformers）**：
```python
# 使用重复惩罚减少重复生成
outputs = model.generate(
    **inputs,
    max_new_tokens=100,
    do_sample=True,
    temperature=0.7,
    top_p=0.9,
    repetition_penalty=1.2,  # 增加重复惩罚
)
```

### 7.2 生成退化

**问题**：生成的文本逐渐退化，失去意义或变得不连贯。

**解决方案**：
- 使用较低的温度
- 增加Top-k或Top-p的值
- 使用波束搜索或多样化波束搜索
- 添加长度惩罚，控制生成文本的长度
- 使用约束解码引导生成方向

### 7.3 幻觉加剧

**问题**：采样策略导致模型生成更多的幻觉（与事实不符的内容）。

**解决方案**：
- 使用较低的温度（接近贪心搜索）
- 使用较小的Top-k或Top-p值
- 添加事实约束或引用检查
- 使用对比解码或CFG-style引导
- 结合检索增强生成（RAG）减少幻觉

**双链链接**：[[幻觉]]

### 7.4 生成速度慢

**问题**：解码策略导致生成速度慢，无法满足实时需求。

**解决方案**：
- 使用贪心搜索或较低的温度
- 减少波束大小或禁用波束搜索
- 使用更快的推理引擎（如vLLM、TensorRT-LLM）
- 采用推测解码
- 优化模型量化和并行策略

**双链链接**：[[推理加速]]

## 8. 新兴方向与未来展望

### 8.1 动态采样（Adaptive Sampling）

**核心思想**：根据生成过程中的上下文动态调整采样参数，平衡质量和多样性。

**主要方法**：
- 基于上下文复杂度调整温度
- 基于生成多样性实时调整Top-k/Top-p值
- 基于生成质量反馈调整采样策略

**预期效果**：
- 生成质量：高（根据上下文优化采样参数）
- 多样性：中高（保持适当的多样性）
- 生成效率：中（需要额外的动态调整计算）

### 8.2 基于人类偏好反馈的实时采样调整

**核心思想**：利用人类偏好反馈实时调整采样参数，生成更符合人类期望的文本。

**主要方法**：
- 在线收集人类反馈
- 基于反馈调整采样参数
- 自适应优化生成策略

**预期效果**：
- 生成质量：高（符合人类偏好）
- 多样性：中（受人类偏好限制）
- 计算开销：高（需要实时反馈处理）

### 8.3 多智能体协同解码

**核心思想**：多个模型或智能体协同生成文本，每个智能体负责不同的方面（如内容、风格、结构）。

**主要方法**：
- 分工协作：不同智能体负责生成文本的不同部分
- 投票机制：多个智能体生成候选文本，通过投票选择最佳结果
- 迭代优化：一个智能体生成初稿，另一个智能体优化和润色

**预期效果**：
- 生成质量：高（多个智能体协同优化）
- 多样性：高（多个智能体带来不同的视角）
- 计算开销：极高（需要同时运行多个智能体）

### 8.4 2024-2026年顶会创新

**ICLR 2024**：《Adaptive Sampling for Language Models》
- 提出了基于上下文的动态采样方法
- 生成质量提升15-20%，多样性保持不变

**ACL 2025**：《Human-in-the-loop Adaptive Decoding》
- 提出了结合人类实时反馈的自适应解码方法
- 生成结果的人类满意度提升25-30%

**NeurIPS 2025**：《Multi-Agent Collaborative Decoding for Large Language Models》
- 提出了多智能体协同解码框架
- 生成质量提升30-40%，尤其是在复杂任务上

## 9. 实践指南与最佳实践

### 9.1 解码策略选择

| 场景 | 推荐解码策略 | 超参数建议 |
|------|--------------|------------|
| 代码生成 | 贪心搜索 + 约束解码 | temperature=0.2, top-k=10 |
| 数学计算 | 贪心搜索 | temperature=0.1, top-k=5 |
| 创意写作 | 温度采样 + Top-p采样 | temperature=0.9, top-p=0.95 |
| 对话生成 | 温度采样 + Top-k采样 | temperature=0.7, top-k=50, top-p=0.9 |
| 摘要生成 | 波束搜索 + 长度惩罚 | num_beams=4, length_penalty=1.2 |
| 翻译 | 多样化波束搜索 | num_beams=4, num_beam_groups=2, diversity_penalty=1.0 |
| 实时服务 | 推测解码 + 温度采样 | temperature=0.7, top-p=0.9 |

### 9.2 超参数调优

**调优步骤**：
1. 确定生成任务的核心需求（质量、多样性、速度）
2. 选择合适的基础解码策略
3. 固定其他参数，调整一个参数，观察生成结果
4. 重复步骤3，找到最佳参数组合
5. 进行人类评价，验证调优结果

**调优工具**：
- `lm-eval`：模型性能评估
- `perfetto`：系统级性能分析
- `human-eval`：人类评价

### 9.3 性能优化

**优化策略**：
- 使用更快的推理引擎（如vLLM、TensorRT-LLM）
- 减少波束大小或禁用波束搜索
- 使用较低的温度（接近贪心搜索）
- 采用推测解码
- 优化模型量化和并行策略
- 使用连续批处理
