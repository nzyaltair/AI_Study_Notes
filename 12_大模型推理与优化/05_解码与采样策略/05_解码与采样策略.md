# 解码与采样策略

## 背景与发展

大语言模型的文本生成能力取决于解码与采样策略。解码是从模型预测的概率分布中选择下一个 token 的过程，直接决定生成文本的质量、多样性、可控性和效率。

早期的解码策略以贪心搜索和波束搜索为主，适合机器翻译等确定性任务。随着开放式生成任务（对话、创意写作）的兴起，Top-K 采样、Top-P（核采样）等随机性策略成为主流。Speculative Decoding（2023）将解码效率提升到新水平。

## 核心思想

解码策略在以下维度间权衡：

- **质量 vs 多样性**：确定性策略质量高但单调，随机性策略多样但可能失控
- **速度 vs 开销**：简单策略计算快，复杂策略增加计算成本
- **可控性 vs 创造性**：约束解码确保格式，自由采样激发创意

## 技术原理

### 确定性解码

**贪心搜索**（Greedy Search）：每步选概率最高的 token。确定性输出，计算开销低，但易产生重复和退化。

**波束搜索**（Beam Search）：维护 $k$ 个候选序列，每步扩展所有候选并保留概率最高的 $k$ 个：

$$\text{score}(y_{1:t}) = \sum_{i=1}^{t} \log P(y_i | y_{1:i-1}, x)$$

适合翻译、摘要等对质量要求高的任务，但缺乏多样性，不适合开放对话。

### 随机性采样

**温度采样**（Temperature Scaling）：通过温度参数 $T$ 调整概率分布平滑度：

$$P'(w_i) = \frac{\exp(\text{logit}_i / T)}{\sum_j \exp(\text{logit}_j / T)}$$

- $T \to 0$：接近贪心搜索，生成确定
- $T = 1$：原始概率分布
- $T \to \infty$：趋于均匀分布，生成随机

**Top-K 采样**：只从概率最高的 $K$ 个 token 中采样。$K$ 较小时偏向质量，$K$ 较大时偏向多样性。

**Top-P 采样**（核采样，Nucleus Sampling）：从累积概率达到 $P$ 的最小 token 集合中采样。自适应调整候选集大小，比 Top-K 更灵活。

**Top-K + Top-P 组合**：先 Top-K 过滤再 Top-P 过滤，结合两者优点。

### 约束解码

限制生成的 token 必须遵循特定规则（前缀约束、词汇约束、语法约束、JSON 结构约束），通过 LogitsProcessor 实现。

### Classifier-Free Guidance

通过对比条件和无条件生成的概率引导生成方向：

$$\text{logits} = \text{logits}_{\text{cond}} + \beta \cdot (\text{logits}_{\text{cond}} - \text{logits}_{\text{uncond}})$$

### Speculative Decoding

使用小模型生成候选 token 序列，大模型一次性验证并接受或拒绝：

1. 小模型用采样策略生成多个候选 token
2. 大模型并行验证候选序列
3. 接受匹配的 token，在首个拒绝处重新生成

延迟降低 30-50%，生成质量与大模型完全一致。

## 发展演进

贪心搜索 -> 波束搜索 -> 随机采样 -> Top-K/Top-P 采样 -> 约束解码 -> Speculative Decoding -> 动态自适应采样

## 关键算法·模型

### 场景策略选择

| 场景 | 推荐策略 | 参数建议 |
|------|---------|---------|
| 代码生成 | 贪心 + 约束 | temperature=0.2, top_k=10 |
| 数学计算 | 贪心 | temperature=0.1, top_k=5 |
| 创意写作 | 温度 + Top-P | temperature=0.9, top_p=0.95 |
| 对话生成 | 温度 + Top-K + Top-P | temperature=0.7, top_k=50, top_p=0.9 |
| 摘要/翻译 | 波束搜索 | num_beams=4, length_penalty=1.2 |
| 实时服务 | Speculative + 温度 | temperature=0.7, top_p=0.9 |

### 重复惩罚

通过 `repetition_penalty` 参数降低已出现 token 的概率，缓解重复生成问题。

## 应用场景

- **代码生成**：贪心搜索 + 约束解码确保语法正确
- **对话系统**：温度采样 + Top-P 平衡质量和多样性
- **翻译/摘要**：波束搜索保证流畅性
- **实时服务**：Speculative Decoding 降低延迟
- **结构化输出**：约束解码确保 JSON/XML 格式
