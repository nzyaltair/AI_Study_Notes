# 分布式优化与并行训练

## 1. 概述

分布式优化与并行训练是 [[00_大语言模型核心架构_综述|大语言模型]] 训练的工程基础。当单个 GPU 无法容纳整个模型或训练速度无法满足需求时，需要将训练过程分布到多个 GPU 甚至多个节点上。并行策略包括数据并行、模型并行（张量并行、流水线并行）以及它们的组合（3D 并行）。ZeRO 和 FSDP 等技术进一步模糊了数据并行和模型并行的边界，成为大模型训练的标准方案。

## 2. 发展历史

| 年代 | 里程碑 | 意义 |
|------|--------|------|
| 2012 | Parameter Server (Dean et al.) | Google DistBelief 分布式训练框架 |
| 2015 | All-Reduce 并行 (Baidu) | Ring All-Reduce 成为标准 |
| 2016 | Layer-Parallel / Pipeline (PipeDream) | 流水线并行原型 |
| 2019 | Megatron-LM (NVIDIA) | 张量并行训练大模型 |
| 2019 | GPipe (Google) | 同步流水线并行 |
| 2019 | ZeRO (Microsoft) | 零冗余数据并行 |
| 2020 | FSDP (PyTorch) | 原生完全分片数据并行 |
| 2021 | Megatron-LM 3D 并行 | 数据 + 张量 + 流水线组合 |
| 2022 | DeepSpeed Ulysses | 高效序列并行 |
| 2023 | Ring Attention | 跨设备长序列分布式注意力 |

## 3. 核心概念

- **数据并行**：每个 GPU 持有完整模型副本，处理不同数据子集，梯度同步后统一更新
- **模型并行**：模型本身被切分到多个 GPU 上
  - **张量并行**（Tensor Parallelism）：将单个矩阵乘法切分到多个 GPU
  - **流水线并行**（Pipeline Parallelism）：将不同层分配到不同 GPU，数据流水通过
- **3D 并行**：数据并行 + 张量并行 + 流水线并行的组合
- **All-Reduce**：多 GPU 间的集合通信原语，将所有 GPU 的梯度求和并广播
- **通信-计算重叠**：将梯度同步通信与计算重叠以隐藏通信延迟

### 并行度的关系

| 维度 | 切分对象 | 典型规模 |
|------|---------|---------|
| 数据并行 | Batch | 数十~数千 GPU |
| 张量并行 | 矩阵维度 | 通常 2-8 GPU（受 NVLink 限制） |
| 流水线并行 | 模型层数 | 4-16 级 |
| 序列并行 | 序列长度 | 2-8 GPU |

## 4. 技术原理

### 4.1 数据并行（Data Parallelism）

**基本流程**：

1. 每个 GPU 持有完整模型副本
2. 将 mini-batch 均分到各 GPU
3. 各 GPU 独立前向+反向计算梯度
4. All-Reduce 同步梯度（求平均）
5. 各 GPU 独立执行相同的优化器更新

```
GPU 0: [Batch 0] → Model → Grad 0 ─┐
GPU 1: [Batch 1] → Model → Grad 1 ─┤ All-Reduce → 平均梯度
GPU 2: [Batch 2] → Model → Grad 2 ─┤
GPU 3: [Batch 3] → Model → Grad 3 ─┘
                                     ↓
                              各 GPU 更新参数
```

**Ring All-Reduce**：GPU 排成环形，每步与邻居交换部分梯度，$K$ 个 GPU 需 $2(K-1)$ 步完成。通信量为 $\frac{2(K-1)}{K} \cdot |\text{grad}|$，与 GPU 数 $K$ 近似无关。

**PyTorch DDP 实现**：

```python
from torch.nn.parallel import DistributedDataParallel as DDP

model = Model().cuda()
model = DDP(model, device_ids=[local_rank])
# 自动梯度同步
```

**同步 vs 异步**：
- **同步 SGD**：等待所有 GPU 完成再更新，等价于大 batch SGD
- **异步 SGD**：不等待，使用陈旧梯度更新，吞吐更高但可能收敛差

### 4.2 张量并行（Tensor Parallelism）

将单个矩阵乘法 $\mathbf{Y} = \mathbf{X}\mathbf{W}$ 切分到多个 GPU：

**列切分**（Column Parallel）：

$$\mathbf{W} = [\mathbf{W}_1, \mathbf{W}_2, \dots, \mathbf{W}_K]$$

$$\mathbf{Y} = [\mathbf{X}\mathbf{W}_1, \mathbf{X}\mathbf{W}_2, \dots, \mathbf{X}\mathbf{W}_K]$$

每个 GPU 计算输出的不同列，前向无需通信，反向需要 All-Reduce。

**行切分**（Row Parallel）：

$$\mathbf{W} = \begin{bmatrix} \mathbf{W}_1 \\ \mathbf{W}_2 \\ \vdots \\ \mathbf{W}_K \end{bmatrix}, \quad \mathbf{X} = [\mathbf{X}_1, \mathbf{X}_2, \dots, \mathbf{X}_K]$$

$$\mathbf{Y} = \sum_{i=1}^K \mathbf{X}_i \mathbf{W}_i$$

前向需要 All-Reduce，反向无需通信。

**Megatron-LM 的 MLP 层**：交替使用列切分和行切分，使前向仅需一次 All-Reduce：

```
输入 X
  │
  ├─ Column Parallel → GeLU → Row Parallel → Y
  │                    (GPU 间无通信)        │
  └──────────────────────────────────────────┘
                              ↓
                         All-Reduce
```

**适用场景**：需要在 NVLink 域内（通常单节点 8 GPU），因为通信频繁且低延迟要求高。

### 4.3 流水线并行（Pipeline Parallelism）

将模型的不同层分配到不同 GPU：

```
GPU 0: Layer 1-4
GPU 1: Layer 5-8
GPU 2: Layer 9-12
GPU 3: Layer 13-16
```

**朴素流水线的问题**：只有一个 batch 在流水线中，大部分 GPU 空闲（气泡）。

**GPipe 方案**：将 mini-batch 切分为 $M$ 个 micro-batch，依次填充流水线：

```
时间 →
GPU 0: [m1][m2][m3][m4] -- -- -- -- [m1][m2][m3][m4]  ← 反向
GPU 1: -- [m1][m2][m3][m4] -- -- [m1][m2][m3][m4] --
GPU 2: -- -- [m1][m2][m3][m4] [m1][m2][m3][m4] -- --
GPU 3: -- -- -- [m1][m2][m3][m4][m3][m2][m1] -- -- --
```

气泡比例：$\frac{K-1}{M+K-1}$，$K$ 为流水线级数，$M$ 为 micro-batch 数。

**1F1B 调度**（Megatron-LM）：交替执行前向和反向，减少激活值内存：

```
GPU 0: [F1][F2][F3][F4][B1][B2][B3][B4]...
```

**Interleaved 1F1B**：将每级分为多个虚拟级，进一步减少气泡。

### 4.4 序列并行（Sequence Parallelism）

针对长序列场景，将序列维度切分到多个 GPU：

- **DeepSpeed Ulysses**：在注意力计算前后通过 All-to-All 通信在序列维度和注意力头维度间转换
- **Ring Attention**：KV 块在 GPU 间环形传递，无需完整加载序列

**适用场景**：超长上下文（100K+ tokens）训练，突破单 GPU 显存限制。

### 4.5 3D 并行

组合三种并行策略：

```
         ┌─────────────┐
         │ 数据并行 (DP) │  ← 最外层，跨节点
         ├─────────────┤
         │ 流水线并行(PP) │  ← 中间层，跨节点
         ├─────────────┤
         │ 张量并行 (TP) │  ← 最内层，NVLink 域内
         └─────────────┘
```

**典型配置**（175B 模型）：
- TP = 8（单节点 8 GPU，NVLink 互联）
- PP = 16（跨 16 个节点）
- DP = 4（4 路数据并行）
- 总计 512 GPU

### 4.6 ZeRO 与 FSDP

详见 [[07_大模型训练优化]]。ZeRO 将数据并行中的冗余状态（优化器状态、梯度、参数）分片，使数据并行也能训练超大模型。

**ZeRO vs 模型并行**：
- ZeRO-3/FSDP 在数学上等价于一种特殊的模型并行 + 数据并行
- ZeRO 不改变计算图，实现更简单
- ZeRO-3 的通信模式是 All-Gather + Reduce-Scatter，与张量并行的通信不同

### 4.7 通信优化

| 技术 | 原理 | 效果 |
|------|------|------|
| 梯度压缩 | 量化/稀疏化梯度 | 减少通信量 4-8 倍 |
| 梯度累积 | 多步本地累积后同步 | 减少通信频率 |
| 通信-计算重叠 | 异步通信与计算并行 | 隐藏通信延迟 |
| 拓扑感知 | 根据 NVLink/InfiniBand 拓朴分配通信路径 | 提高带宽利用 |
| 1-bit Adam | 梯度符号化传输 | 通信量减少 32 倍 |

## 5. 并行策略选择指南

| 模型规模 | GPU 数 | 推荐策略 |
|---------|--------|---------|
| < 1B | 1-8 | DDP |
| 1B-10B | 8-64 | DDP / FSDP |
| 10B-70B | 64-512 | FSDP / 3D 并行 |
| 70B-175B | 512-2048 | 3D 并行 (TP+PP+DP) |
| 175B+ | 2048+ | 3D 并行 + ZeRO + 序列并行 |

## 6. 优势与局限

### 优势

- 使超大模型训练成为可能（千亿/万亿参数）
- 数据并行线性扩展吞吐量
- 3D 并行充分利用硬件拓扑
- ZeRO/FSDP 降低实现复杂度

### 局限

- 通信开销随 GPU 数增加而增大
- 流水线并行存在气泡，GPU 利用率 < 100%
- 异步 SGD 可能因梯度陈旧影响收敛
- 实现复杂度高，调试困难
- 故障容错挑战：数千 GPU 中单个故障可能导致训练中断

## 9. 前沿发展

- **Expert Parallelism**：MoE 模型的专家并行，将不同专家分配到不同 GPU
- **Ring Attention / Blockwise Ring Attention**：超长序列分布式注意力
- **异构训练**：CPU+GPU+TPU 混合训练
- **弹性训练**：GPU 动态加入/退出，支持抢占式云实例
- **去中心化训练**：跨数据中心训练，应对网络延迟
- **通信压缩**：1-bit Adam、PowerSGD、DeepComp 等梯度压缩方法
- **自动并行**：Alpa、Unity 等自动搜索最优并行策略
