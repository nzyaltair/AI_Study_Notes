# 大模型训练与对齐

## 背景与发展

大语言模型（LLM）的训练范式经历了从传统预训练到多阶段对齐的演进。GPT-1 确立了"预训练+微调"范式，GPT-3 展示了规模化带来的上下文学习能力，InstructGPT 提出三阶段训练流程（SFT→RM→PPO），ChatGPT 将 RLHF 推向主流，DPO（2023）简化了对齐流程，DeepSeek-R1（2025）以 GRPO 推动了推理模型训练。这一系列演进构成了现代大模型训练的核心方法论。

## 核心思想

大模型训练是一个多阶段流程，每个阶段赋予模型不同能力：

- **预训练**：在海量文本上学习通用语言表示，获取世界知识与语言能力
- **监督微调（SFT）**：用指令-回答对教会模型跟随指令格式回答问题
- **偏好对齐**：通过人类或 AI 反馈使模型输出符合偏好与价值观

## 技术原理

### 预训练

- **目标**：Next Token Prediction（自回归语言建模）
  $$L = -\sum_{t=1}^{T} \log P(x_t \mid x_{<t})$$
- **数据**：万亿级 token 的网页文本、书籍、代码
- **训练**：分布式训练（数据并行/张量并行/流水线并行）+ ZeRO 优化
- **关键超参**：学习率（Warmup + Cosine）、Batch Size、序列长度

### 监督微调（SFT）

- **目标**：让模型学会跟随指令格式回答问题
- **数据**：高质量指令-回答对（Alpaca/ShareGPT/人工标注）
- **训练方式**：全量微调或参数高效微调（[[LoRA]]/[[QLoRA]]）
- **核心原则**：数据质量远比数量重要

### 偏好对齐

#### RLHF

三阶段流程：SFT 模型 → 训练奖励模型（RM）→ PPO 优化

- 奖励模型损失：$L_{RM} = -\log\sigma(r(x, y_w) - r(x, y_l))$
- PPO 目标：$\max_\pi \mathbb{E}[r(x,y)] - \beta \, \text{KL}(\pi \| \pi_{SFT})$

#### DPO

无需训练奖励模型，直接从偏好数据优化策略：

$$L_{DPO} = -\log\sigma\left(\beta\log\frac{\pi(y_w|x)}{\pi_{ref}(y_w|x)} - \beta\log\frac{\pi(y_l|x)}{\pi_{ref}(y_l|x)}\right)$$

#### 其他方法

| 方法 | 特点 |
|------|------|
| [[RLAIF]] | 用 AI 反馈代替人类反馈 |
| [[GRPO]] | 组内相对优势，去除奖励模型与 Critic |
| Constitutional AI | 基于宪法的自我修正 |

## 发展演进

GPT-1（预训练+微调）→ GPT-2（零样本）→ GPT-3（上下文学习）→ InstructGPT（SFT+RLHF）→ ChatGPT（RLHF+安全对齐）→ DPO（2023，简化对齐）→ 推理模型 o1（2024，RL+CoT）→ DeepSeek-R1（2025，GRPO）

## 关键算法·模型

- **预训练**：[[LLaMA]]、[[Qwen]]、[[Mistral]]、[[DeepSeek]]
- **对齐算法**：[[RLHF]]、[[DPO]]、[[GRPO]]、[[RLAIF]]
- **PEFT 方法**：[[LoRA]]、[[QLoRA]]、[[Adapter]]
- **分布式框架**：[[DeepSpeed]]、[[Megatron-LM]]、[[FSDP]]

## 应用场景

- **基座模型训练**：LLaMA/Qwen/DeepSeek 的预训练
- **领域微调**：医疗、法律、金融等领域适配
- **对齐安全**：防止有害输出、注入价值观
- **个性化**：基于用户偏好微调

## 与其他技术关系

- 预训练基于 Transformer 架构和预训练范式
- 分布式训练依赖 ZeRO、3D 并行等训练技巧
- LoRA 等 PEFT 方法是低秩近似的应用
- 对齐后的模型用于 Agent 和多模态应用
- 推理优化详见 LLM 推理与优化
