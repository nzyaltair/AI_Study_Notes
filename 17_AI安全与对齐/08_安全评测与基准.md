# 安全评测与基准

## 1. 概述

安全评测与基准是量化AI模型安全性能的标准化方法。与通用能力评测（MMLU、HumanEval等）不同，安全评测聚焦于模型在有害内容生成、越狱鲁棒性、偏见输出、隐私泄露等安全维度的表现。

安全评测面临三大挑战：**安全行为的多样性**（有害内容的定义因文化、法律而异）、**攻击面的开放性**（无法穷举所有攻击路径）、**评判的主观性**（安全与否的判断依赖标准定义）。近年来，AdvBench、HarmBench、ToxiGen等标准化基准的发布使安全评测逐步走向系统化。

## 3. 核心概念

### 安全评测维度

| 维度 | 评测目标 | 代表基准 |
|------|---------|---------|
| 有害内容 | 暴力、非法、自残等内容生成倾向 | RealToxicityPrompts, HarmBench |
| 越狱鲁棒性 | 对抗性提示的防御能力 | AdvBench, HarmBench |
| 偏见与歧视 | 不同人群的输出差异 | BOLD, BBQ |
| 毒性 | 仇恨言论、侮辱性内容 | ToxiGen, Perspective API |
| 隐私安全 | 训练数据提取、PII泄露 | PrivacyBench |
| 诚实性 | 幻觉率、不确定性表达 | HaluEval, TruthfulQA |
| 指令注入 | 提示注入防御能力 | PromptBench, InjecBench |

### 评测方法分类

- **静态基准**：预定义测试集，固定评分标准
- **动态评测**：实时生成测试用例，对抗性评估
- **红队评测**：人工+自动化攻击发现漏洞
- **LLM-as-Judge**：用强模型自动评估安全输出
- **人类评估**：人工标注安全/不安全，金标准

## 4. 技术原理

### 有害内容评测

**RealToxicityPrompts**：包含38K提示，评估模型续写生成有毒内容的倾向。使用Perspective API计算毒性分数：

$$\text{Toxicity Score} = \frac{1}{N} \sum_{i=1}^{N} \max(\text{toxicity}(generation_i))$$

**HarmBench**：标准化有害行为评测框架，包含多种有害内容类别：
- 网络安全（恶意代码生成）
- 化学/生物/放射/核（CBRN）
- 有害内容（暴力、歧视）
- 欺骗/误导

评测流程：给定攻击提示 → 模型生成回复 → LLM-as-Judge判断是否成功生成有害内容 → 计算ASR。

### 越狱鲁棒性评测

**AdvBench**：520条对抗性提示，测试模型对越狱攻击的防御能力。评估指标为拒绝率（Refusal Rate）——模型成功拒绝有害请求的比例。

**评测设计**：
1. 对每个越狱提示，让模型生成回复
2. 用LLM-as-Judge判断回复是否违反安全准则
3. 计算攻击成功率（ASR）和拒绝率

### 偏见评测

**BBQ**（Bias Benchmark for QA）：测量模型在不同人群维度（性别、种族、宗教等）上的偏见。每个问题有target和non-target两组版本，偏见分数为：

$$\text{Bias Score} = \frac{1}{N} \sum_{i=1}^{N} (P(target_i) - P(non\text{-}target_i))$$

### 诚实性评测

**TruthfulQA**：测试模型是否生成常见误解的错误信息。包含817个问题，涵盖健康、法律、金融、阴谋论等。评估模型是否能识别并拒绝常见错误观念。

### 评测挑战与对策

| 挑战 | 描述 | 对策 |
|------|------|------|
| 数据污染 | 安全测试集出现在训练数据中 | 动态生成测试用例 |
| 评判偏差 | LLM-as-Judge自身存在偏见 | 多评判器投票，人工校准 |
| 攻击面开放 | 无法穷举所有攻击路径 | 红队+自动化攻击组合 |
| 文化差异 | 有害内容定义因文化而异 | 多文化安全数据集 |
| 过拟合基准 | 模型针对基准优化但实际安全未提升 | 动态基准，定期更新 |

## 6. 优势与局限

**优势**：
- 标准化基准使安全性能可量化、可比较
- 多维度覆盖有害内容、越狱、偏见等安全维度
- 自动化评测（LLM-as-Judge）降低评估成本
- 安全评测工具链成熟，可集成到CI/CD

**局限**：
- 安全是开放性问题——无法穷举所有攻击路径
- 静态基准易被过拟合——模型优化基准分数但实际安全未提升
- LLM-as-Judge存在已知偏差（偏好长回复、特定格式）
- 有害内容定义的文化差异难以统一
- 安全-有用性权衡——高安全分数可能伴随有用性下降

## 7. 应用场景

- **模型发布前安全评估**：部署前量化安全性能
- **安全对齐迭代**：跟踪安全基准分数指导对齐改进
- **合规审计**：满足法规要求的安全评估文档
- **模型对比**：技术报告中的安全性能对比
- **持续监控**：部署后定期安全检测
