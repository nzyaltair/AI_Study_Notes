# 其他框架转换路径

> **标签**: #mxnet #paddlepaddle #jax
> **相关链接**: [[附录/工具链汇总]]

## 概述

除了主流的 PyTorch 和 TensorFlow，ONNX 生态还支持多种深度学习框架的转换。本文介绍 MXNet、PaddlePaddle、JAX 以及 legacy 框架 Caffe/Caffe2 的转换路径。

## MXNet 转换

### 环境准备

```bash
# 安装 MXNet（CPU 版本）
pip install mxnet

# 或 GPU 版本（CUDA 11.x）
pip install mxnet-cu112

# 安装 ONNX-MXNet（mxnet-onnx）
pip install onnx-mxnet
```

注意：`onnx-mxnet` 是 ONNX 官方维护的 MXNet 后端，支持从 MXNet 到 ONNX 的转换。

### 从 MXNet 符号模型转换

```python
import mxnet as mx
from mxnet import nd
from mxnet.gluon import nn
import onnx
import onnx_mxnet

# 1. 定义 MXNet Gluon 模型
class MXNetModel(nn.HybridBlock):
    def __init__(self, **kwargs):
        super().__init__(**kwargs)
        self.conv1 = nn.Conv2D(32, 3, activation='relu')
        self.pool = nn.MaxPool2D(2)
        self.flatten = nn.Flatten()
        self.fc1 = nn.Dense(128, activation='relu')
        self.fc2 = nn.Dense(10)

    def hybrid_forward(self, F, x):
        x = self.conv1(x)
        x = self.pool(x)
        x = self.flatten(x)
        x = self.fc1(x)
        x = self.fc2(x)
        return x

# 2. 初始化并混合（hybridize）模型
model = MXNetModel()
model.initialize()
model.hybridize()  # 必须调用 hybridize 才能导出

# 3. 准备示例输入
dummy_input = nd.random.uniform(shape=(1, 3, 224, 224))

# 4. 前向传播以构建计算图
_ = model(dummy_input)

# 5. 导出为 ONNX
onnx_path = 'mxnet_model.onnx'
model.export(
    onnx_path,
    epoch=0,
    input_names=['input'],
    output_names=['output'],
    # 指定 opset 版本
    onnx_opset=13
)

print(f"✓ ONNX 模型已导出: {onnx_path}.onnx")
```

### 从 Symbol 模型转换

```python
import mxnet as mx

# 1. 构建 Symbol 模型（兼容旧版 MXNet）
data = mx.sym.Variable('data')
conv1 = mx.sym.Convolution(data=data, num_filter=32, kernel=(3, 3), name='conv1')
relu1 = mx.sym.Activation(data=conv1, act_type='relu', name='relu1')
pool1 = mx.sym.Pooling(data=relu1, pool_type='max', kernel=(2, 2), name='pool1')
flatten = mx.sym.flatten(data=pool1, name='flatten')
fc1 = mx.sym.FullyConnected(data=flatten, num_hidden=128, name='fc1')
relu2 = mx.sym.Activation(data=fc1, act_type='relu', name='relu2')
fc2 = mx.sym.FullyConnected(data=relu2, num_hidden=10, name='fc2')
output = mx.sym.SoftmaxOutput(data=fc2, name='softmax')

# 2. 保存为符号文件
sym_path = 'model-symbol.json'
param_path = 'model-0000.params'
output.save(sym_path)
# 参数需要单独保存

# 3. 转换为 ONNX（使用 onnx-mxnet 的 convert 函数）
from onnx_mxnet.convert import convert

onnx_path = 'mxnet_symbol.onnx'
convert(sym_path, param_path, onnx_path, input_shape=[(1, 3, 224, 224)])

print(f"✓ Symbol 模型已转换: {onnx_path}")
```

### MXNet 转换注意事项

| 问题 | 解决方案 |
|------|---------|
| HybridBlock 未 hybridize | 必须调用 `model.hybridize()` |
| 动态形状不支持 | MXNet 导出为固定形状 |
| 自定义算子 | 需要实现 symbolic 函数并注册到 onnx-mxnet |
| BatchNorm 导出问题 | 确保处于推理模式（`model.collect_params().initialize()` 后冻结） |

## PaddlePaddle 转换

### 环境准备

```bash
# 安装 PaddlePaddle（CPU）
pip install paddlepaddle

# GPU 版本（CUDA 11.2）
pip install paddlepaddle-gpu

# 安装 Paddle2ONNX
pip install paddle2onnx
```

Paddle2ONNX 是 PaddlePaddle 官方推荐的转换工具，支持动态轴和大部分算子。

### 从 PaddlePaddle 模型转换

```python
import paddle
import paddle2onnx

# 方式1: 从动态图（Eager）模型转换
class PaddleModel(paddle.nn.Layer):
    def __init__(self):
        super().__init__()
        self.conv1 = paddle.nn.Conv2D(3, 32, 3, padding=1, activation='relu')
        self.pool = paddle.nn.MaxPool2D(2)
        self.flatten = paddle.nn.Flatten()
        self.fc1 = paddle.nn.Linear(32 * 112 * 112, 128)
        self.fc2 = paddle.nn.Linear(128, 10)

    def forward(self, x):
        x = self.conv1(x)
        x = self.pool(x)
        x = self.flatten(x)
        x = self.fc1(x)
        x = self.fc2(x)
        return x

# 创建模型
model = PaddleModel()
model.eval()  # 设置为 eval 模式

# 准备输入
dummy_input = paddle.randn([1, 3, 224, 224])

# 转换为 ONNX（使用 paddle2onnx.export）
onnx_model = paddle2onnx.export(
    model,
    (dummy_input,),
    input_spec=[paddle.static.InputSpec(shape=[None, 3, 224, 224], dtype='float32')],
    opset_version=13
)

# 保存
with open('paddle_model.onnx', 'wb') as f:
    f.write(onnx_model)

print("✓ PaddlePaddle 模型转换完成")
```

### 从静态图模型转换

```python
import paddle
import paddle2onnx

# 1. 构建静态图模型
paddle.enable_static()

# 创建程序
start_prog = paddle.static.Program()
main_prog = paddle.static.Program()

with paddle.static.program_guard(main_prog, start_prog):
    # 输入占位符
    x = paddle.static.data(name='x', shape=[None, 3, 224, 224], dtype='float32')

    # 网络结构
    conv1 = paddle.static.nn.conv2d(x, num_filters=32, filter_size=3, padding=1, act='relu')
    pool1 = paddle.static.nn.pool2d(conv1, pool_size=2, pool_type='max')
    flatten = paddle.flatten(pool1, start_axis=1)
    fc1 = paddle.static.nn.fc(flatten, size=128, activation='relu')
    fc2 = paddle.static.nn.fc(fc1, size=10)

    # 指定输入输出
    model = paddle.static.Program()

# 2. 转换
onnx_model = paddle2onnx.export(
    main_prog,
    model_path=None,  # 已加载的程序设为 None
    params_path=None, # 参数文件路径
    input_spec=[paddle.static.InputSpec(shape=[None, 3, 224, 224], dtype='float32')],
    opset_version=13
)

# 保存
with open('paddle_static.onnx', 'wb') as f:
    f.write(onnx_model)
```

### Paddle 转换常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 算子不支持 | 使用小众 Paddle 算子 | 使用 `--enable_dev_version` 标志或更新 Paddle2ONNX |
| 输出命名混乱 | Paddle 内部节点名 | 使用 `output_spec` 参数指定输出 |
| 动态形状失败 | 输入 shape 固定 | 在 `InputSpec` 中使用 `None` 表示动态维度 |
| 量化模型转换 | INT8 算子兼容性 | 转换为 FP32 后再量化 |

## JAX 转换

JAX 本身没有官方的 ONNX 导出工具，但可通过以下路径转换：

### 方案1: 使用 ONNX 脚本（ONNX-Script）

```python
import jax
import jax.numpy as jnp
import onnx
from onnx import helper, TensorProto
import onnxruntime as ort

# 手动实现 JAX 模型的 ONNX 转换
# 步骤1: 将 JAX 参数转换为 ONNX 初始化器

def jax_to_onnx_weights(params):
    """将 JAX 参数字典转换为 ONNX 初始化器列表"""
    initializers = []
    for name, value in params.items():
        # JAX 数组转 numpy
        value_np = jax.device_get(value)

        # 创建 ONNX TensorProto
        tensor = helper.make_tensor(
            name=name,
            data_type=TensorProto.DataType.FLOAT,
            dims=value_np.shape,
            vals=value_np.flatten().tolist()
        )
        initializers.append(tensor)
    return initializers

# 步骤2: 构建计算图节点（需要手动定义）
# 由于 JAX 无计算图，需使用 ONNX 算子手动组装
# 这是复杂且易错的方法，仅适用于简单模型
```

**缺点**: 手动实现复杂，不推荐用于大型模型。

### 方案2: 通过 PyTorch 中转

```python
import jax
import jax.numpy as jnp
import torch
import torch.onnx

# 1. 定义 JAX 模型
def jax_model_fn(params, x):
    return jnp.dot(x, params['w']) + params['b']

# 2. 训练好的参数
rng = jax.random.PRNGKey(0)
params = {
    'w': jax.random.normal(rng, (10, 5)),
    'b': jnp.zeros(5)
}

# 3. 转换为 NumPy，再构建 PyTorch 模型
class TorchModel(torch.nn.Module):
    def __init__(self, jax_params):
        super().__init__()
        self.weight = torch.nn.Parameter(
            torch.from_numpy(jax.device_get(jax_params['w'])).float()
        )
        self.bias = torch.nn.Parameter(
            torch.from_numpy(jax.device_get(jax_params['b'])).float()
        )

    def forward(self, x):
        return torch.matmul(x, self.weight.T) + self.bias

torch_model = TorchModel(params)

# 4. 导出 ONNX
dummy_input = torch.randn(1, 10)
torch.onnx.export(
    torch_model,
    dummy_input,
    'jax_via_torch.onnx',
    input_names=['input'],
    output_names=['output'],
    opset_version=13
)

print("✓ JAX 模型已通过 PyTorch 中转转换")
```

### 方案3: via TensorFlow（JAX2TF）

```python
# 使用 jax2tf 将 JAX 转换为 TF SavedModel，再用 tf2onnx
import jax
import jax.numpy as jnp
import tensorflow as tf
from jax.experimental.jax2tf import convert

# 1. JAX 函数
def jax_fn(x):
    return jax.nn.relu(x)

# 2. 转换为 TF 函数
tf_fn = convert(jax_fn, polymorphic_shapes=['(b, ...)'])

# 3. 保存为 SavedModel
@tf.function
def tf_model(x):
    return tf_fn(x)

# 需要具体化输入 shape
spec = tf.TensorSpec(shape=[None, 10], dtype=tf.float32)
tf_model = tf_model.get_concrete_function(spec)

# 保存
tf.saved_model.save(tf_model, 'jax_tf_model', signatures=tf_model)

# 4. 使用 tf2onnx 转换
import tf2onnx
model_proto, _ = tf2onnx.convert.from_saved_model(
    'jax_tf_model',
    opset=13
)
with open('jax_model.onnx', 'wb') as f:
    f.write(model_proto)
```

**局限**: JAX 特有算子（如 `jax.lax.scan`）可能无法正确转换。

## Caffe 和 Caffe2 转换（Legacy）

> **注意**: Caffe 和 Caffe2 已进入维护模式，新项目建议迁移到其他框架。

### Caffe 转换

```bash
# 使用 onnx-caffe2 工具（已废弃，仅作参考）
# 旧方式（TF 1.x 时代）
python -m onnx_caffe2.frontend.caffe.caffe2onnx \
    --caffe_model model.caffemodel \
    --caffe_prototxt model.prototxt \
    --output model.onnx
```

### Caffe2 转换

```python
import caffe2.python.onnx as onnx_caffe2
from caffe2.python import core, workspace

# 1. 加载 Caffe2 模型
init_net = 'model_init_net.pb'
predict_net = 'model_predict_net.pb'

# 2. 转换为 ONNX
onnx_model = onnx_caffe2.caffe2onnx.convert(
    init_net,
    predict_net,
    input_dims={
        'data': [1, 3, 224, 224]  # 输入维度
    }
)

# 3. 保存
with open('caffe2_model.onnx', 'wb') as f:
    f.write(onnx_model)

print("✓ Caffe2 模型转换完成")
```

**推荐迁移路径**:
- Caffe → PyTorch/TensorFlow → ONNX
- Caffe2 → PyTorch → ONNX

## 转换质量评估

### 输出对比测试

```python
import numpy as np
import onnxruntime as ort

def compare_framework_outputs(framework_outputs, onnx_path, onnx_input_names):
    """比较原始框架和 ONNX 输出"""
    sess = ort.InferenceSession(onnx_path)

    # 准备 ONNX 输入
    onnx_inputs = {}
    for i, name in enumerate(onnx_input_names):
        onnx_inputs[name] = framework_outputs[i]  # 假设 inputs 顺序一致

    # ONNX 推理
    onnx_outputs = sess.run(None, onnx_inputs)

    # 对比每一路输出
    for i, (fw_out, onnx_out) in enumerate(zip(framework_outputs, onnx_outputs)):
        # 确保数据类型一致
        if isinstance(fw_out, np.ndarray):
            fw_np = fw_out
        else:
            fw_np = fw_out.numpy() if hasattr(fw_out, 'numpy') else np.array(fw_out)

        max_diff = np.abs(fw_np - onnx_out).max()
        mse = np.mean((fw_np - onnx_out) ** 2)

        print(f"输出 {i}:")
        print(f"  原始形状: {fw_np.shape}")
        print(f"  ONNX 形状: {onnx_out.shape}")
        print(f"  最大误差: {max_diff:.2e}")
        print(f"  均方误差: {mse:.2e}")

        if max_diff > 1e-3:
            print("  ⚠ 警告: 误差超过阈值!")
        else:
            print("  ✓ 通过")
```

## 选择转换路径的建议

### 决策树

```text
需要转换模型?
  │
  ├─ 是否是 PyTorch 模型? → 是 → 使用 torch.onnx.export
  │
  ├─ 是否是 TensorFlow/Keras 模型? → 是 → 使用 tf2onnx
  │
  ├─ 是否是 PaddlePaddle 模型? → 是 → 使用 paddle2onnx
  │
  ├─ 是否是 MXNet 模型?
  │    ├─ Gluon HybridBlock → 使用 model.export() + onnx_opset
  │    └─ Symbol → 使用 onnx-mxnet.convert
  │
  ├─ 是否是 JAX 模型?
  │    ├─ 简单模型 → 通过 PyTorch 中转
  │    ├─ 复杂算子 → 使用 jax2tf + tf2onnx
  │    └─ 必须原生 → 暂时不支持，需手动实现
  │
  ├─ 是否是 Caffe/Caffe2 模型?
  │    └─ 建议先转换为 PyTorch/TensorFlow，再导出 ONNX
  │
  └─ 其他框架 → 检查 ONNX 官方支持的框架列表
```

### 优先级建议

1. **primary**: 使用框架官方支持的导出工具（PyTorch/TF/Paddle）
2. **fallback**: 通过 Intermediate 框架转换（JAX → PyTorch）
3. **manual**: 对于 legacy 框架，考虑手动重写模型
