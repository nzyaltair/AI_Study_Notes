# 基于模型的强化学习

## 1. 概述

基于模型的强化学习（Model-Based RL）学习环境的动态模型（状态转移 $P$ 和奖励 $R$），并在学到的模型中进行规划或生成数据。与无模型方法相比，基于模型的方法具有更高的样本效率——因为模型可以从少量真实交互中学习，然后在模型中"免费"生成大量模拟经验。

基于模型 RL 的核心权衡是**模型偏差（Model Bias）**：学到的模型与真实环境的差异会导致在模型中规划出的策略在真实环境中表现退化。如何平衡样本效率与模型精度是基于模型 RL 的核心挑战。

## 3. 核心概念

### 3.1 环境模型

环境模型是对 MDP 动态的内部表征：

- **转移模型** $\hat{P}(s'|s,a)$：预测下一状态
- **奖励模型** $\hat{R}(s,a)$ 或 $\hat{R}(s,a,s')$：预测奖励

模型可以是表格形式（小状态空间）或参数化函数（神经网络/高斯过程）。

### 3.2 规划（Planning）

给定模型，通过以下方式进行规划：

- **值函数规划**：在模型上运行动态规划/值迭代
- **搜索规划**：蒙特卡洛树搜索（MCTS）、滚动时域搜索
- **数据增强**：在模型中生成模拟经验，辅助无模型学习（Dyna 架构）

### 3.3 模型偏差

学到的模型 $\hat{P}, \hat{R}$ 与真实 $P, R$ 之间的差异。在模型中优化得越好，越可能**过拟合到模型的错误**——策略在模型中表现优异但在真实环境中退化。

缓解策略：
- 模型不确定性量化（集成模型、贝叶斯模型）
- 短程规划（减少模型误差累积）
- 模型与无模型混合（Dyna 架构）

## 4. 技术原理

### 4.1 Dyna 架构

**论文**：Sutton, *Dyna, an Integrated Architecture for Learning, Planning, and Reacting* (1990)

Dyna 将模型学习和无模型学习统一：

```
1. 真实交互：用当前策略与环境交互，获得 (s, a, r, s')
2. 直接学习：用真实经验更新 Q（无模型）
3. 模型学习：用真实经验更新模型 P̂, R̂
4. 模拟规划：从模型中采样 (s, a, r, s')，用无模型方法更新 Q
   （重复 n 次）
```

Dyna 的核心洞察：模型学习后可以"免费"生成模拟经验，加速无模型学习。

### 4.2 蒙特卡洛树搜索（MCTS）

MCTS 是基于采样的前向搜索，在已知或学到的模型中进行：

```
1. 选择（Selection）：从根节点用 UCT 公式选择子节点
   UCT = Q(s,a) + c * sqrt(ln(N(s)) / N(s,a))
2. 扩展（Expansion）：在叶节点扩展新动作
3. 模拟（Simulation/Rollout）：用快速策略模拟到终止，获得回报
4. 回传（Backpropagation）：将回报回传更新路径上所有节点
```

MCTS 的优势在于**自适应搜索深度**——不需要展开整个搜索树，而是在有前景的方向深入搜索。AlphaGo 的成功就是 MCTS + 神经网络策略/值函数的结合。

### 4.3 AlphaGo / AlphaZero

**AlphaGo**（Silver et al., 2016）：
- 用人类专家棋谱监督学习初始化策略网络
- 用强化学习自我对弈改进策略网络
- 训练值网络评估局面
- 推理时用 MCTS + 策略/值网络指导搜索

**AlphaZero**（Silver et al., 2018）：
- **完全从零自对弈**——无需人类棋谱
- 统一策略和值网络为单一网络 $f_\theta(s) \to (p, v)$
- MCTS 生成的搜索概率 $\pi$ 作为策略改进目标
- 循环迭代：自对弈 → MCTS 搜索 → 训练网络 → 更强自对弈

**AlphaZero 的损失函数**：

$$\mathcal{L} = (z - v)^2 - \pi^T \log p + c\|\theta\|^2$$

其中 $z$ 是实际结局，$v$ 是值网络预测，$\pi$ 是 MCTS 搜索概率，$p$ 是策略网络预测。

### 4.4 MuZero

**论文**：Schrittwieser et al., *Mastering Atari, Go, chess and shogi by planning with a learned model* (Nature, 2020)

MuZero 的突破在于**不需要已知环境规则**——它学习一个**隐式世界模型**，将状态映射到潜在空间并在其中规划。

**三个网络**：
- **表示网络** $h(o) \to s$：将观测 $o$ 编码到潜在状态 $s$
- **动态网络** $g(s, a) \to (s', r)$：在潜在空间中预测下一状态和奖励
- **预测网络** $f(s) \to (p, v)$：预测策略和值函数

**规划**：在潜在空间中用 MCTS 搜索 $k$ 步前向，生成策略 $\pi$ 和值 $v$。

**训练**：同时优化策略预测、值预测、奖励预测和状态转移预测。

MuZero 统一了棋类（完全信息、已知规则）和 Atari（部分可观测、未知规则）——它不需要任何先验的游戏规则知识。

### 4.5 World Models

**论文**：Ha & Schmidhuber, *World Models* (2018)

World Models 将"世界模型"概念推到极致：

1. **视觉模型（V）**：用 VAE 将高维观测编码到潜在空间
2. **记忆模型（M）**：用 LSTM/RNN 预测潜在状态的时序演化
3. **控制器（C）**：在潜在空间中用简单策略（如线性策略或 CMA-ES）决策

核心洞察：如果世界模型足够准确，可以在纯潜在空间中训练策略，大幅降低计算成本。

### 4.6 Dreamer 系列

**Dreamer**（Hafner et al., 2020）/ **DreamerV2**（2021）/ **DreamerV3**（2023）

Dreamer 在学到的潜在世界模型中通过**反向传播梯度**进行长程规划（而非 MCTS 的前向搜索）：

1. **世界模型学习**：RSSM（Recurrent State-Space Model）学习潜在动态
2. **Actor-Critic 在模型中训练**：在模型生成的想象轨迹上用策略梯度训练 Actor 和 Critic
3. **策略部署**：将学到的策略直接部署到真实环境

**DreamerV3 的突破**：固定超参数在 150+ 任务上取得 SOTA，无需逐任务调参。

### 4.7 模型预测控制（MPC）

MPC 是经典控制理论与 RL 的交汇点。在每个时间步：

1. 用模型预测未来 $H$ 步的状态和奖励
2. 优化这 $H$ 步的动作序列使累积奖励最大化
3. 只执行第一个动作，然后重新规划（滚动时域）

**TD-MPC**（Hansen et al., 2022）/ **TD-MPC2**（2023）将 MPC 与深度 RL 结合：学习潜在世界模型，在模型中用 MPC 进行规划，同时用 TD 学习辅助值函数估计。

## 5. 基于模型 vs 无模型

| 维度 | 基于模型 | 无模型 |
|:---|:---|:---|
| 样本效率 | 高 | 低 |
| 渐近性能 | 受模型偏差限制 | 可达最优 |
| 计算开销 | 高（需训练模型+规划） | 低 |
| 适用性 | 需要可学模型 | 通用 |
| 模型偏差 | 有 | 无 |

实践中常采用**混合策略**：先用基于模型方法快速学习（高样本效率），再切换到无模型方法精细优化（高渐近性能）。

## 6. 优势与局限

### 优势

1. **样本效率高**：模型可复用，在数据稀缺场景中至关重要
2. **可规划性**：能进行前向搜索/规划，处理长程决策
3. **可迁移**：学到的模型可迁移到相关任务
4. **安全性**：在模型中预评估策略，降低真实环境风险

### 局限

1. **模型偏差**：学到的模型不完美，误差累积影响规划
2. **计算成本高**：模型训练 + 规划的计算开销大
3. **模型学习困难**：高维/复杂环境的模型学习具有挑战性
4. **探索-利用冲突**：需要平衡探索以改善模型 vs 利用模型规划

## 7. 应用场景

| 场景 | 方法 | 说明 |
|:---|:---|:---|
| 棋类游戏 | AlphaZero | 自对弈 + MCTS |
| Atari | MuZero / DreamerV3 | 无需已知规则的规划 |
| 机器人控制 | TD-MPC / Dreamer | 样本效率要求高 |
| 化学分子设计 | 世界模型 + 规划 | 真实实验成本高 |
| 自动驾驶 | MPC + 学习模型 | 安全约束下的规划 |

## 8. 与其他技术关系

- **与动态规划的关系**：DP 在已知模型上做规划，基于模型 RL 在学到的模型上做规划。参见 [[02_值函数与动态规划]]。
- **与无模型方法的关系**：Dyna 架构将两者统一；MuZero/Dreamer 在模型中训练无模型策略。参见 [[03_无模型预测与控制]]。
- **与最优控制的关系**：MPC 是经典最优控制的延伸，RL 为其提供了数据驱动的模型学习。参见 [[01_强化学习基础与MDP]]。
- **与搜索的关系**：MCTS 是基于采样的搜索算法，与 Alpha-Beta 剪枝等经典搜索互补。参见 [[01_强化学习基础与MDP]]。
- **与大模型的关系**：世界模型的概念被推广到 LLM 的"世界知识"建模；MuZero 式的自对弈训练思想被用于推理模型训练。参见 [[10_强化学习与大模型]]。

## 9. 前沿发展

- **DreamerV3**：固定超参数跨域 SOTA，降低调参门槛
- **EfficientZero**：样本高效的 MuZero 变体，1 小时内掌握 Atari
- **Transformer 世界模型**：用 Transformer 替代 RNN 建模长程动态（IRIS、TWM）
- **扩散模型作为世界模型**：用扩散模型建模环境的随机性和多模态性
- **Foundation World Models**：大规模预训练的通用世界模型，跨任务迁移
- **基于物理的模型**：将物理先验嵌入世界模型（可微物理引擎）
