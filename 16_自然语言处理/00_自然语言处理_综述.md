# 自然语言处理综述

## 1. 概述

自然语言处理（Natural Language Processing, NLP）是人工智能、计算机科学与语言学的交叉学科，研究**让计算机理解、生成和处理人类语言**的理论与方法。NLP 的核心目标是跨越人机交互的语言鸿沟——使机器能够阅读文本、理解语义、进行对话、翻译语言、回答问题，甚至生成流畅自然的新文本。

- **解决的问题**：人类语言具有高度的歧义性、层次性和开放性，同一个句子在不同语境下可能有截然不同的含义。NLP 旨在让计算机系统性地处理这些语言特性，完成从基础词法分析到高级语义理解与文本生成的任务链。
- **核心价值**：NLP 是"语言智能"的技术基石，也是大语言模型时代所有文本处理能力的学科根源。从搜索引擎到机器翻译，从智能助手到代码生成，NLP 技术无处不在。理解 NLP 的发展脉络和技术体系，是理解现代 AI——特别是大语言模型——如何"理解语言"的必要前提。
- **知识体系范围**：分词与子词切分 → 词法分析（词性标注/NER）→ 句法分析（依存/成分）→ 语义表示与理解 → 序列标注模型（HMM/CRF/BiLSTM-CRF）→ 信息检索与文本挖掘 → 文本分类与情感分析 → 机器翻译 → 信息抽取与知识图谱 → 文本生成与摘要 → 问答与对话系统 → NLP 评估与工程实践 → 大模型时代的 NLP 范式变革。

### 范式划分

NLP 的发展经历了四个主要范式：

| 范式 | 时期 | 核心思想 | 代表方法 |
|:---|:---|:---|:---|
| 基于规则 | 1950s–1980s | 人工编写语法规则 | 正则表达式、上下文无关文法 |
| 基于统计 | 1990s–2010s | 从语料中学习概率分布 | HMM、CRF、IBM MT、n-gram |
| 深度学习 | 2014–2018 | 端到端神经网络 | RNN/LSTM、Seq2Seq+Attention |
| 预训练与LLM | 2018–至今 | 大规模预训练+微调/提示 | BERT、GPT、In-Context Learning |

## 3. 核心概念

### 3.1 NLP 的层次结构

NLP 的处理通常分为多个层次，从底层的字符处理到高层的语义理解：

```
语音 → 文本 → 分词 → 词法分析 → 句法分析 → 语义理解 → 语用分析 → 应用
        ↑        ↑         ↑          ↑          ↑
     Tokenization POS     Parsing    Semantic   Pragmatics
```

| 层次 | 任务 | 典型方法 |
|:---|:---|:---|
| 语音层 | 语音识别（ASR）、语音合成（TTS） | 声学模型、Tacotron、VITS |
| 形态层 | 分词、词形还原、词干提取 | BPE、Porter Stemmer、WordPiece |
| 词法层 | 词性标注、命名实体识别 | HMM、CRF、BiLSTM-CRF |
| 句法层 | 依存分析、成分分析 | PCFG、Transition-based、Transformer |
| 语义层 | 词义消歧、语义角色标注 | 基于知识库、BERT |
| 语用层 | 指代消解、意图识别 | 端到端模型 |
| 应用层 | 翻译、问答、摘要、对话 | Seq2Seq、LLM |

### 3.2 语言模型的演进

语言模型是 NLP 的核心——几乎所有现代 NLP 任务都可以归结为语言建模问题。

| 阶段 | 方法 | 公式 | 特点 |
|:---|:---|:---|:---|
| 统计 n-gram | 最大似然估计 | $P(w_t|w_{t-n+1}^{t-1}) = \frac{c(w_{t-n+1}^t)}{c(w_{t-n+1}^{t-1})}$ | 简单但数据稀疏 |
| 神经语言模型 | 前馈/循环网络 | $P(w_t|context) = \text{softmax}(W h_t + b)$ | 分布式表示，缓解稀疏 |
| 预训练语言模型 | 自监督训练 | $L = -\sum_t \log P(w_t | \tilde{w})$ | 上下文相关表示 |
| 大语言模型 | 超大规模自回归 | $P(x) = \prod_t P(x_t | x_{<t}; \theta)$ | 涌现能力，通用智能 |

### 3.3 关键术语

| 术语 | 含义 |
|:---|:---|
| Token / Tokenization | 语言单位及其切分过程 |
| Lemma / Stem | 词元（词典形式）/ 词干（截断形式） |
| POS Tag | 词性标注（名词、动词、形容词等） |
| NER | 命名实体识别（人名、地名、机构名等） |
| Dependency Parse | 依存句法分析（词与词之间的修饰关系） |
| Constituency Parse | 成分句法分析（短语的层次结构） |
| SRL | 语义角色标注（谁对谁做了什么） |
| Coreference | 共指消解（指代消解） |
| NLI | 自然语言推理（蕴含/矛盾/中立） |

## 4. 核心公式速览

### 4.1 n-gram 语言模型

$$P(w_1^n) = \prod_{t=1}^{n} P(w_t | w_1^{t-1}) \approx \prod_{t=1}^{n} P(w_t | w_{t-n+1}^{t-1})$$

**加一平滑（Laplace Smoothing）**：

$$P_{\text{add1}}(w_t | w_{t-n+1}^{t-1}) = \frac{c(w_{t-n+1}^t) + 1}{c(w_{t-n+1}^{t-1}) + |V|}$$

### 4.2 TF-IDF → BM25

$$\text{TF-IDF}(t, d) = \text{TF}(t, d) \times \log\frac{N}{\text{df}(t)}$$

$$\text{BM25}(t, d) = \text{IDF}(t) \cdot \frac{\text{TF}(t, d) \cdot (k_1 + 1)}{\text{TF}(t, d) + k_1 \cdot (1 - b + b \cdot |d|/\text{avgdl})}$$

对 TF 做饱和处理，对文档长度归一化。$k_1 \approx 1.2$, $b \approx 0.75$。

### 4.3 HMM 序列标注

$$P(\mathbf{x}, \mathbf{y}) = \prod_{t=1}^{T} P(y_t | y_{t-1}) \cdot P(x_t | y_t)$$

Viterbi 算法解码：$\hat{\mathbf{y}} = \arg\max_{\mathbf{y}} P(\mathbf{y} | \mathbf{x})$

### 4.4 线性链 CRF

$$P(\mathbf{y}|\mathbf{x}) = \frac{1}{Z(\mathbf{x})} \exp\left(\sum_{t=1}^T \left(\sum_k \lambda_k f_k(y_t, \mathbf{x}, t) + \sum_l \mu_l g_l(y_{t-1}, y_t, \mathbf{x}, t)\right)\right)$$

配分函数 $Z(\mathbf{x})$ 用前向算法高效计算。

### 4.5 Word2Vec 负采样

$$P_n(w) \propto \text{count}(w)^{3/4}$$

高频词采样概率降低，低频词提高，指数 3/4 取得平衡。

### 4.6 BLEU 评估

$$\text{BLEU} = \text{BP} \cdot \exp\left(\sum_{n=1}^{N} w_n \log p_n\right)$$

其中 BP 为长度惩罚（Brevity Penalty）：$\text{BP} = \min(1, e^{1-r/c})$。

### 4.7 BPE 合并规则

$$\text{merge}_i = \arg\max_{(a, b)} \text{count}(a, b)$$

迭代合并最高频符号对，解决 OOV 问题。

## 5. 方法分类总览

| 任务类别 | 传统方法 | 深度学习方法 | LLM 方法 |
|:---|:---|:---|:---|
| 分词 | 最大匹配、词典 | BPE/WordPiece/SentencePiece | 直接使用分词器 |
| 词性标注 | HMM、规则 | BiLSTM-CRF、BERT | 提示工程 |
| NER | CRF、字典匹配 | BiLSTM-CRF、BERT-CRF | Few-shot + 提示 |
| 句法分析 | PCFG、Shift-Reduce | Transition-based NN、BERT | 直接生成 |
| 语义角色标注 | 规则+统计 | 端到端神经网络 | 直接生成 |
| 文本分类 | SVM、朴素贝叶斯 | TextCNN、BERT | Zero/Few-shot |
| 机器翻译 | IBM Model、Phrase-based | Seq2Seq+Attn、Transformer | LLM 直接翻译 |
| 信息检索 | TF-IDF、BM25 | DPR、ColBERT | 混合检索+LLM |
| 摘要 | 抽取式（TextRank） | 抽取+生成式（BART） | LLM 直接生成 |
| 问答 | 规则+模板 | DrQA、BERT QA | RAG + LLM |
| 对话 | 有限状态机 | 模块化系统 | LLM 端到端 |

## 7. 与其他方向关系

- **与预训练语言模型的关系**：BERT/GPT/T5 等 PLM 是 NLP 从"特定任务模型"走向"通用基础模型"的关键里程碑，本方向侧重 NLP 任务和方法论，PLM 侧重模型架构与训练。参见 [[09_预训练语言模型/00_预训练语言模型_综述]]。
- **与 Transformer 的关系**：注意力机制最初在 NLP 的机器翻译中诞生（Bahdanau 2015），Transformer 则成为现代 NLP 的统一基础架构。参见 [[08_Transformer与注意力机制/00_Transformer与注意力机制_综述]]。
- **与大语言模型核心架构的关系**：LLM 是 NLP 技术的集大成者，分词、位置编码、注意力等都是 NLP 基础组件。参见 [[10_大语言模型核心架构/00_大语言模型核心架构_综述]]。
- **与序列模型的关系**：RNN/LSTM/GRU 是 NLP 处理序列数据的基础架构。参见 [[06_序列模型/00_序列模型]]。
- **与 LLM 应用工程的关系**：RAG、提示工程、Agent 等都是 NLP 任务在大模型时代的新实践形式。参见 [[19_LLM应用工程/00_LLM应用工程]]。
- **与机器学习的关系**：CRF、SVM、朴素贝叶斯等传统 ML 方法曾是 NLP 的主力工具。参见 [[03_机器学习]]。
- **与数学基础的关系**：概率论、信息论（熵、互信息）是统计 NLP 的数学根基。参见 [[01_数学基础]]。
