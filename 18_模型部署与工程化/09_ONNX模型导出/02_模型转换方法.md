# 模型转换方法

## 1. 概述

将训练好的深度学习模型从框架特定格式（如 PyTorch 的 `.pt`、TensorFlow 的 SavedModel）转换为 ONNX 格式，是实现跨平台部署的第一步。各主流框架均提供了官方或社区维护的 ONNX 导出工具，其中 PyTorch 的 `torch.onnx.export` 功能最完整、使用最广泛。

模型转换解决的核心问题：
- **框架锁定**：训练框架与推理引擎的耦合
- **格式不兼容**：不同框架的模型格式不互通
- **部署灵活性**：需要将模型部署到多种推理引擎和硬件平台

转换的本质是将框架的计算图映射为 ONNX IR：框架中的每个算子通过符号函数映射到对应的 ONNX 算子，框架的权重参数序列化为 ONNX TensorProto 初始器。

## 2. 发展历史

| 年代 | 里程碑 | 意义 |
|:---|:---|:---|
| 2017 | torch.onnx.export | PyTorch 首个 ONNX 导出工具 |
| 2018 | tf2onnx | TensorFlow 转 ONNX 工具 |
| 2019 | JIT Tracing 改进 | 动态图追踪能力增强 |
| 2020 | Opset 11+ 动态形状 | dynamic_axes 支持 |
| 2021 | paddle2onnx | PaddlePaddle 官方导出工具 |
| 2022 | 符号推导完善 | 更多算子的 ONNX 映射 |
| 2023 | PyTorch 2.x torch.export | 基于 TorchDynamo 的新导出路径 |
| 2024 | 自动化转换工具 | 减少手动算子处理 |

## 3. 核心概念

### 3.1 JIT Tracing vs JIT Scripting

| 方式 | 原理 | 优点 | 缺点 |
|:---|:---|:---|:---|
| **Tracing** | 记录前向传播执行路径 | 简单、兼容性好 | 控制流被固化为特定分支 |
| **Scripting** | 解析 Python 源码生成图 | 保留控制流语义 | 代码风格约束多 |

### 3.2 符号函数（Symbolic Function）

PyTorch 通过注册在 `torch.onnx.symbolic_helper` 中的映射函数，将 ATen 算子转换为 ONNX 算子组合。当无对应映射时，需通过 `register_custom_op_symbolic` 注册自定义符号函数。

### 3.3 动态轴（Dynamic Axes）

指定 ONNX 模型的哪些维度可变，语法为 `{tensor_name: {dim_index: dim_name}}`。

### 3.4 常量折叠（Constant Folding）

在导出时预计算常量操作，减少推理时计算量。默认开启。

## 4. 技术原理

### 4.1 PyTorch：torch.onnx.export

```python
torch.onnx.export(
    model,                    # PyTorch 模型（须 eval 模式）
    args,                     # 示例输入（tensor 或 tuple）
    f,                        # 输出路径
    export_params=True,       # 是否导出权重
    opset_version=14,         # ONNX Opset 版本
    do_constant_folding=True, # 常量折叠优化
    input_names=['input'],    # 输入张量名
    output_names=['output'],  # 输出张量名
    dynamic_axes={            # 动态轴配置
        'input': {0: 'batch_size'},
        'output': {0: 'batch_size'}
    }
)
```

**关键参数**：
- `dynamic_axes`：指定可变维度，命名应与 input_names/output_names 一致
- `opset_version`：推荐 13 或 14，兼顾兼容性和功能
- `training`：须设为 `TrainingMode.EVAL`，否则 Dropout/BatchNorm 行为不正确

### 4.2 ResNet50 动态 batch 导出

```python
import torch, torchvision.models as models

model = models.resnet50(pretrained=True).eval()
dummy_input = torch.randn(4, 3, 224, 224)

torch.onnx.export(
    model, dummy_input, 'resnet50.onnx',
    input_names=['image'], output_names=['probs'],
    dynamic_axes={
        'image': {0: 'batch_size', 2: 'height', 3: 'width'},
        'probs': {0: 'batch_size'}
    },
    opset_version=13, do_constant_folding=True
)
```

### 4.3 自定义算子处理

```python
from torch.onnx import register_custom_op_symbolic

def my_op_symbolic(g, x, y):
    return g.op('Mul', x, y)  # 映射到已有 ONNX 算子

register_custom_op_symbolic('::my_op', my_op_symbolic, 9)
```

### 4.4 TensorFlow：tf2onnx

```bash
python -m tf2onnx.convert \
    --saved-model saved_model_dir \
    --output model.onnx \
    --opset 13
```

Keras 模型可直接转换：

```python
import tf2onnx
spec = (tf.TensorSpec((None, 224, 224, 3), tf.float32, name="input"),)
model_proto, _ = tf2onnx.convert.from_keras(model, input_signature=spec, opset=13)
```

### 4.5 PaddlePaddle：paddle2onnx

```python
import paddle2onnx
onnx_model = paddle2onnx.export(
    model, (dummy_input,),
    input_spec=[paddle.static.InputSpec(shape=[None, 3, 224, 224], dtype='float32')],
    opset_version=13
)
```

### 4.6 框架转换工具对比

| 框架 | 工具 | 动态轴 | 导出速度 | 维护状态 |
|:---|:---|:---|:---|:---|
| PyTorch | `torch.onnx` | 原生支持 | 快（~3.8s/ResNet50） | 活跃 |
| TensorFlow | `tf2onnx` | 支持 | 较慢（~5.2s） | 活跃 |
| PaddlePaddle | `paddle2onnx` | 支持 | 快 | 活跃 |
| MXNet | `onnx-mxnet` | 不支持 | — | 维护模式 |
| JAX | via PyTorch/TF 中转 | 依赖中转 | — | 社区 |

## 5. 关键方法与模型

### 5.1 算子映射优先级

1. 直接映射：框架算子有对应 ONNX 算子
2. 组合映射：用多个 ONNX 算子组合实现
3. 自定义符号函数：手动注册映射关系
4. onnxruntime-extensions：加载扩展算子库
5. GraphSurgeon 子图替换：图级别替换


## 6. 优势与局限

### 6.1 优势

- **工具链成熟**：主流框架均有官方导出工具
- **动态形状支持**：dynamic_axes 灵活配置可变维度
- **社区活跃**：常见问题有成熟解决方案

### 6.2 局限

- **算子覆盖不全**：部分框架特有算子需手动处理
- **控制流限制**：Tracing 模式下控制流被固化
- **大模型内存**：导出时可能 OOM
- **LLM 适用性差**：大语言模型更适合专用推理引擎
