# 安全与对齐基础

## 1. 概述

AI安全与对齐致力于回答一个根本问题：**如何确保AI系统做我们想让它做的事，而不做我们不想让它做的事？** 这包含两个层面——安全性（系统可靠、鲁棒、隐私保护）和对齐（行为符合人类价值观与意图）。

对齐问题之所以困难，原因在于：人类价值观本身是模糊的、冲突的、演变的；模型能力增长可能超越人类监督能力；优化目标与真实意图之间存在偏差。

## 2. 发展历史

- **2014-2016**：对抗样本发现（Szegedy, Goodfellow），AI安全从理论讨论进入实证研究
- **2016-2019**：算法公平性研究兴起（COMPAS争议、Amazon招聘工具），隐私攻击（成员推断）系统化
- **2017-2019**：对齐问题学术化——Stuart Russell提出"有益、可逆、透明"三原则；Anthropic成立，聚焦AI安全
- **2020-2021**：训练数据提取（Carlini从GPT-2提取数据），证明大模型的隐私风险
- **2022**：InstructGPT/ChatGPT将RLHF工程化，对齐从理论走向产业实践
- **2022-2023**：LLM安全新范式——提示注入（OWASP LLM Top 10 #1）、越狱攻击、Constitutional AI
- **2023-2024**：国际治理共识——布莱切利园宣言、EU AI Act通过、NIST AI RMF推广
- **2024-2025**：超级对齐研究（弱到强泛化）、机制可解释性工程化、可扩展监督

## 3. 核心概念

### 安全性维度（CIA三元组 + AI扩展）

| 维度 | 含义 | AI特有挑战 |
|------|------|-----------|
| 机密性（Confidentiality） | 防止训练数据/系统信息泄露 | 模型记忆导致训练数据提取 |
| 完整性（Integrity） | 防止恶意篡改模型行为 | 提示注入覆盖系统指令 |
| 可用性（Availability） | 防止服务中断 | 资源耗尽、对抗性负载 |
| 鲁棒性（Robustness） | 输入扰动下保持正确 | 对抗样本、分布外泛化 |
| 公平性（Fairness） | 无歧视性输出 | 数据偏见导致系统性歧视 |
| 可问责性（Accountability） | 决策可追溯 | 黑箱模型难以归责 |

### 对齐的核心定义

**价值对齐**（Value Alignment）：使AI系统的目标函数、行为策略和输出结果与人类价值观一致。包含三个层次：

1. **意图对齐**（Intent Alignment）：模型做用户想要它做的事
2. **偏好对齐**（Preference Alignment）：模型输出符合人类偏好数据
3. **价值对齐**（Value Alignment）：模型行为符合深层人类价值观和社会规范

### 对齐难题

- **规范歧义**（Specification Problem）：人类难以精确描述想要什么，奖励函数只是真实意图的近似
- **奖励黑客**（Reward Hacking）：模型优化代理目标但偏离真实意图（如刷分但不解决实际问题）
- **分布偏移**：训练环境与部署环境不一致导致对齐失效
- **可扩展监督**（Scalable Oversight）：模型能力超越人类评估能力时如何监督
- **权力寻求**（Power-Seeking）：高级系统可能寻求资源以确保目标完成

### 威胁模型分类

| 攻击者能力 | 攻击类型 | 典型场景 |
|-----------|---------|---------|
| 白盒 | 梯度攻击（FGSM/PGD） | 本地模型对抗攻击 |
| 黑盒 | 查询攻击、迁移攻击 | API模型攻击 |
| 灰盒 | GCG对抗后缀 | 已知模型架构但无权重 |
| 无访问 | 社会工程、间接注入 | 通过外部内容攻击Agent |

## 4. 技术原理

### 安全纵深防御架构

```
用户输入 → [输入护栏] → [模型推理] → [输出护栏] → 响应
              │              │              │
         模式检测        对齐训练        内容审核
         语义分析        安全RLHF       PII过滤
         注入检测        宪法审查        水印嵌入
```

### 对齐训练流水线

```
预训练模型 → SFT（指令微调）→ 偏好对齐（RLHF/DPO/RLAIF）→ 安全微调 → 部署
                                    ↓
                            红队测试 → 安全数据收集 → 迭代
```

## 5. 关键方法与模型

| 方法 | 核心思想 | 适用阶段 |
|------|---------|---------|
| RLHF | 人类偏好训练奖励模型，强化学习优化 | 对齐训练 |
| RLAIF | AI反馈替代人类反馈，降低标注成本 | 对齐训练 |
| Constitutional AI | 宪法原则驱动的自我审查与修正 | 对齐训练 |
| DPO | 直接偏好优化，无需强化学习 | 对齐训练 |
| 对抗训练 | 对抗样本参与训练提升鲁棒性 | 安全训练 |
| 红队测试 | 系统化攻击发现安全漏洞 | 安全评估 |
| 差分隐私 | 梯度加噪提供形式化隐私保证 | 隐私保护 |

## 6. 优势与局限

**优势**：
- 系统性框架覆盖从训练到部署全链路
- 对齐方法已证明能有效降低有害输出
- 纵深防御提供多层安全保障

**局限**：
- 对齐是近似而非解决——无法保证完美对齐
- 安全-能力存在权衡（过度对齐降低有用性）
- 评估困难——无法穷举所有有害行为
- 价值观的多元性和文化差异难以统一

## 7. 应用场景

- **LLM产品安全**：聊天机器人、代码助手、文档处理的安全防护
- **自动驾驶**：对抗鲁棒性确保感知系统可靠性
- **医疗AI**：公平性确保不同人群诊断准确率一致
- **金融风控**：可解释性满足合规审计要求
- **内容平台**：自动化有害内容检测与过滤
