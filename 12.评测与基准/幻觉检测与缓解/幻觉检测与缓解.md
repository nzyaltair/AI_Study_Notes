# 幻觉检测与缓解

## 基础界定

### 幻觉类型定义 ^hallucination-definition
- **事实性幻觉**：生成与客观事实不符的内容（例："爱因斯坦发明了互联网"）
- **逻辑性幻觉**：内容存在逻辑矛盾（例："他既是医生又是律师，但从未上过大学"）
- **上下文冲突**：与对话历史或给定上下文不一致（例：用户问"北京天气"，模型回答"纽约今天晴朗"）
- **虚构引用**：编造不存在的来源或引用（例："根据2024年NASA研究报告显示..."）

### 概念边界
- 与 [[模型不确定性]] 的区别：幻觉是错误的确定性输出，不确定性是对输出的置信度表达
- 与 [[可信AI]] 的关系：幻觉检测是可信AI的重要组成部分，但可信AI还包括安全性、隐私性等维度

## 检测体系

### 自动化方法

#### FactScore
- **解决问题**：客观事实性验证
- **核心逻辑**：将生成文本分解为事实断言，通过检索验证每个断言的真实性
- **适用场景**：需要高事实准确性的任务（如问答、信息提取）
- **计算成本**：中高（需要外部检索）
- **误报率**：中等（可能错过隐性事实错误）
- **伪代码**：
```python
def calculate_factscore(generated_text, retriever):
    claims = extract_claims(generated_text)
    scores = []
    for claim in claims:
        evidence = retriever.retrieve(claim, top_k=3)
        score = evaluate_evidence_match(claim, evidence)
        scores.append(score)
    return sum(scores) / len(scores) if scores else 0.0
```

#### SelfCheckGPT
- **解决问题**：模型自我验证能力
- **核心逻辑**：让模型生成多个候选答案，通过一致性分析检测幻觉
- **适用场景**：资源受限环境，无法使用外部检索
- **计算成本**：高（需要多次生成）
- **误报率**：低（利用模型自身知识）
- **与竞品对比**：比单纯依赖检索的方法更能处理隐性知识

#### RAGAS 幻觉指标
- **解决问题**：检索增强生成中的幻觉检测
- **核心逻辑**：评估生成内容与检索上下文的一致性
- **适用场景**：RAG系统评估与优化
- **计算成本**：中（依赖检索质量）
- **误报率**：低（直接关联检索上下文）
- **配置要点**：需设置合适的相似度阈值和上下文窗口大小

#### 基于检索的验证流程
- **解决问题**：实时生成内容的事实性验证
- **核心逻辑**：生成后立即进行检索验证，标记可疑内容
- **适用场景**：需要实时反馈的应用（如客服、教育）
- **计算成本**：中（需要低延迟检索）
- **误报率**：中等（依赖检索系统覆盖度）

### 人工评测

#### 设计指南
- 明确标注任务：事实性检查、逻辑性检查、上下文一致性检查
- 提供详细的标注指南和示例
- 设计多维度评分体系（0-5分制）

#### 标注规范
- **事实性**：0=严重错误，5=完全正确
- **逻辑性**：0=逻辑混乱，5=逻辑严密
- **一致性**：0=完全不符，5=完全一致
- **引用真实性**：0=伪造引用，5=真实可靠

#### Kappa 一致性计算
- **解决问题**：评估标注者间的一致性
- **计算方法**：Cohen's Kappa（两个标注者）、Fleiss' Kappa（多个标注者）
- **阈值参考**：0.6-0.8=中等一致，>0.8=高度一致

### 工具链

#### TruLens
- **核心功能**：RAG应用的可观测性和幻觉检测
- **配置要点**：
```python

from trulens_eval.feedback import Groundedness

# 配置幻觉检测反馈函数
grounded = Groundedness(groundedness_provider=OpenAI())
groundedness_feedback = Feedback(grounded.groundedness_measure).on(
    context=lambda x: x.get('context', ''),
    response=lambda x: x.get('response', '')

```
- **适用场景**：生产级RAG系统监控

#### LangSmith
- **核心功能**：LLM应用的追踪、评估和幻觉检测
- **配置要点**：需设置适当的评估数据集和评分标准
- **适用场景**：开发和生产环境的全链路监控

#### 开源检测库
- **HaluEval**：专注于对话系统的幻觉检测
- **HallucinationBench**：多任务幻觉评估基准
- **适用场景**：模型开发和基准测试

## 缓解策略

### 训练阶段

#### 对比学习
- **解决问题**：减少模型生成事实错误的倾向
- **核心方法**：对比正确和错误的生成样本，让模型学习区分
- **适用边界**：需要高质量的对比数据
- **与竞品对比**：比单纯的监督学习更能捕捉错误模式

#### 事实约束损失
- **解决问题**：强制模型生成符合事实的内容
- **核心方法**：在损失函数中加入事实一致性惩罚项
- **适用边界**：需要外部事实验证器
- **实现要点**：平衡生成质量和事实准确性

#### 知识增强预训练
- **解决问题**：提升模型的事实知识储备
- **核心方法**：在预训练中融入结构化知识
- **适用边界**：计算资源需求高
- **与竞品对比**：比微调更能提升长期事实记忆

### 推理阶段

#### [[RAG]] 优化 ^rag-mitigation
- **解决问题**：通过外部知识减少幻觉
- **核心方法**：优化检索策略、上下文整合、重排序机制
- **配置要点**：
  - 检索器：选择合适的嵌入模型和索引策略
  - 上下文窗口：平衡信息丰富度和噪声
  - 提示工程：明确要求模型基于检索内容生成
- **适用边界**：依赖检索系统的质量和覆盖度

#### 置信度校准
- **解决问题**：让模型的输出概率与实际准确性匹配
- **核心方法**：后处理校准（如温度缩放）、动态提示调整
- **适用场景**：需要可靠性评估的任务
- **实现要点**：使用验证集进行校准

#### 后验验证模块
- **解决问题**：生成后进行事实性检查
- **核心方法**：集成外部验证器，对可疑内容进行标记
- **适用场景**：高风险应用（如医疗、法律）
- **与竞品对比**：比单纯依赖生成时约束更灵活

#### 拒绝生成机制
- **解决问题**：避免在不确定情况下生成错误内容
- **核心方法**：设置置信度阈值，低于阈值时拒绝回答
- **适用场景**：需要高可靠性的场景
- **实现要点**：平衡拒绝率和用户体验

### 系统设计

#### 人类反馈闭环
- **解决问题**：持续改进模型的事实性
- **核心方法**：收集用户反馈，标记幻觉实例，用于模型微调
- **适用场景**：生产环境的持续优化
- **实现要点**：设计便捷的反馈机制和数据处理流程

#### 多模型交叉验证架构
- **解决问题**：通过模型间共识提高可靠性
- **核心方法**：多个模型独立生成，通过投票或加权融合确定最终输出
- **适用场景**：高风险任务的最终决策
- **实现要点**：选择互补性强的模型组合

## 评估与演进

### 量化指标
- **幻觉率**：生成内容中包含幻觉的比例
- **事实一致性分数**：生成内容与事实的匹配程度
- **用户信任度量表**：用户对生成内容的信任程度评分

### 前沿方向（2024–2026）
- **神经符号结合验证**：将符号推理与神经网络结合，提高验证准确性
- **多模态幻觉检测**：扩展到图像、视频等多模态内容的幻觉检测
- **因果推理缓解**：通过因果建模减少逻辑幻觉
- **LLM-as-Judge 优化**：提升模型自我评估的准确性和可靠性

### 交叉研究趋势
- 与 [[大模型安全]] 的交叉：幻觉可能导致安全风险
- 与 [[可解释性]] 的交叉：可解释性技术有助于理解幻觉产生的原因
- 与 [[模型校准]] 的交叉：校准良好的模型通常幻觉率更低
