# 语音与音频

## 背景与发展

语音与音频 AI 涵盖语音识别（ASR）、语音合成（TTS）和音频生成。Whisper（2022）实现了多语言鲁棒语音识别，现代 TTS 模型实现了自然度极高的语音合成，音频生成模型开始创作音乐和音效。

语音 AI 从早期的 HMM-GMM 统计模型，发展到 DNN-HMM、RNN/CTC，再到 Transformer 架构（Conformer、Whisper）。端到端 TTS 从拼接式发展到零样本语音克隆，扩散模型也引入到语音合成中。

## 核心思想

通过在超大规模音频数据上训练 Transformer 模型，实现语音识别、合成和音频理解生成的统一。语音 AI 的趋势是走向统一语音模型，一个模型同时处理 ASR、TTS 和语音理解。

## 技术原理

### 语音识别（ASR）

**Whisper 架构**（编码器-解码器 Transformer）：

- 输入：Mel 频谱图 -> 卷积编码 -> Transformer 编码器
- 输出：Transformer 解码器自回归生成文本
- 多任务：转写 + 翻译 + 语言识别 + 时间戳
- 训练数据：68 万小时多语言弱标注数据

**关键技术**：

- **Mel 频谱图**：将音频转换为时频表示
- **CTC 损失**：处理变长对齐
- **SpecAugment**：频谱图数据增强
- **流式识别**：分块处理实现低延迟

### 语音合成（TTS）

| 方法 | 原理 | 代表模型 |
|------|------|---------|
| 拼接式 | 拼接预录语音片段 | 传统 TTS |
| 参数式 | 声学参数合成 | HMM-based |
| 神经网络声码器 | WaveNet 等自回归生成 | WaveNet, WaveGlow |
| 端到端 | 文本直接生成语音 | Tacotron, FastSpeech |
| 扩散/Flow | 扩散模型生成语音 | NaturalSpeech, VITS |

**现代 TTS 流程**：

```
文本 -> 文本前端（音素转换）-> 声学模型 -> 声码器 -> 语音波形
```

**零样本语音克隆**：给定几秒参考音频，克隆说话人音色。方法为 Speaker Embedding + 声学模型条件化。代表：XTTS, OpenVoice。

### 音频生成

| 任务 | 方法 | 代表 |
|------|------|------|
| 音乐生成 | 自回归/扩散 | MusicGen, AudioLDM |
| 音效生成 | 文本到音频 | AudioGen |
| 语音转换 | 改变音色保持内容 | So-VITS-SVC |

### 统一语音模型

一个模型同时处理 ASR、TTS 和语音理解：

- **AudioLM**：音频自回归生成
- **SPIRIT-LM**：语音-文本统一模型
- **Qwen-Audio**：音频理解大模型

## 发展演进

ASR：HMM-GMM -> DNN-HMM -> RNN/CTC -> Conformer -> Whisper（2022）-> 统一语音模型

TTS：拼接式 -> 参数式 -> 神经网络（WaveNet）-> 端到端 -> 零样本克隆

## 关键算法·模型

- **Whisper**（OpenAI 2022）：68 万小时多语言训练，编码器-解码器 Transformer
- **WaveNet**（DeepMind 2016）：自回归神经声码器
- **FastSpeech**：非自回归 TTS，并行生成
- **VITS**：端到端 + 变分推理 + flow
- **MusicGen**（Meta 2023）：自回归音乐生成

## 应用场景

- **语音助手**：Siri、小爱同学（ASR + TTS）
- **会议记录**：实时语音转写
- **有声内容**：有声书、新闻播报
- **多语言翻译**：实时语音翻译
- **音乐创作**：AI 作曲、编曲
- **无障碍**：为视障人士朗读、为听障人士转写

## 与其他技术关系

- 语音模型基于 Transformer 架构
- Whisper 的编码器-解码器与 T5 类似
- 语音能力扩展了 [[多模态AI]] 模型的输入输出
- 语音交互是 [[Agent]] 的重要交互方式
- 语音合成中的声码器与 [[扩散模型]] 有技术交叉
