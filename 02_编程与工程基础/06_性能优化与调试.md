# 性能优化与调试

## 背景与发展

AI 模型规模从百万参数增长到千亿参数，训练成本和推理延迟成为关键挑战。性能优化贯穿模型全生命周期：训练阶段追求 GPU 利用率与显存效率，推理阶段追求低延迟与高吞吐量。

## 核心思想

性能优化遵循"测量优先"原则：先用 profiling 工具定位瓶颈，再针对性优化。优化涉及算法、代码、系统、硬件多个层次，需在精度、速度、显存三者间权衡。

## 技术原理

### 性能分析

- **Python profiler**：`cProfile` 定位慢函数，`line_profiler` 逐行分析
- **PyTorch profiler**：`torch.profiler` 分析 GPU kernel 执行时间和数据加载瓶颈
- **关键指标**：
  - 训练：吞吐量（samples/sec）、GPU 利用率、显存峰值
  - 推理：延迟（ms）、吞吐量（tokens/sec）、首 token 延迟（TTFT）

### GPU 训练优化

**混合精度训练**：FP16 前向计算 + FP32 主权重，减少显存占用并利用 Tensor Core 加速。

```python
from torch.cuda.amp import autocast, GradScaler
scaler = GradScaler()
with autocast():
    loss = criterion(model(input), target)
scaler.scale(loss).backward()
scaler.step(optimizer)
scaler.update()
```

**梯度累积**：多次小 batch 前向传播后统一反向传播，等效大 batch 训练。

**梯度检查点**（Gradient Checkpointing）：前向传播时不保存中间激活值，反向传播时重新计算，以额外计算换取显存节省。

### 显存优化

- **及时释放**：`del tensor; torch.cuda.empty_cache()`
- **ZeRO 优化**（DeepSpeed）：优化器状态、梯度、参数分片到多 GPU
- **CPU offload**：将不活跃参数卸载到 CPU 内存
- **模型并行**：将大模型拆分到多个 GPU

### 推理优化

- **量化**：FP16/INT8/INT4 降低精度，减少显存和计算量
- **KV-Cache**：缓存注意力机制的 Key-Value 对，避免重复计算
- **Flash Attention**：IO-aware 的注意力计算，减少 HBM 读写
- **Speculative Decoding**：小模型预测候选 token，大模型并行验证，加速自回归生成
- **批处理**：合并多个请求提高 GPU 利用率，Continuous Batching 动态管理请求队列

### 调试方法

- **梯度检查**：`torch.autograd.gradcheck` 验证自定义梯度正确性
- **NaN/Inf 检测**：`torch.isnan(tensor).any()` 定位数值异常
- **形状断言**：关键位置添加 `assert tensor.shape == expected`
- **过拟合测试**：在小数据集上验证模型能否达到零训练误差，排除实现错误
- **可视化监控**：TensorBoard / W&B 观察损失曲线、梯度分布、学习率变化

### 分布式训练

- **数据并行（DDP）**：每 GPU 完整模型副本，处理不同数据，梯度 AllReduce 同步
- **张量并行**：将单个线性层拆分到多 GPU（如 Megatron-LM）
- **流水线并行**：将不同层分配到不同 GPU，微批次流水线执行
- **混合并行**：3D 并行 = 数据并行 + 张量并行 + 流水线并行

## 发展演进

单 GPU 训练 → DataParallel → DistributedDataParallel → 混合并行（Megatron + DeepSpeed）→ 大模型训练框架（3D 并行 + ZeRO + CPU offload）。推理侧从朴素自回归到 KV-Cache、Flash Attention、量化、Speculative Decoding 等系统级优化。

## 关键算法·模型

- **混合精度**：FP16 计算 + FP32 主权重，GradScaler 动态调整 loss scale 防止梯度下溢
- **ZeRO**：$R = \frac{M}{N_d}$，将 $M$ 参数分片到 $N_d$ 设备，通信量不变但显存线性降低
- **Flash Attention**：分块计算注意力，IO 复杂度从 $O(n^2)$ HBM 访问降为 $O(n^2/s)$，$s$ 为块大小
- **A2A 梯度同步**：AllReduce 通信原语，带宽最优环形算法

## 应用场景

- 大模型训练：GPU 集群利用率优化、显存管理、分布式通信
- 推理服务：延迟优化、批处理、量化部署
- 边缘部署：模型压缩、INT8 量化
- 训练调试：梯度异常、数值不稳定、收敛问题排查

## 与其他技术关系

- 混合精度训练依赖数值表示与浮点运算理论
- 分布式训练依赖 [[Linux与开发环境]] 中的 GPU 集群与网络配置
- 推理量化是模型部署链路的关键环节
- 显存优化与 PyTorch autograd 的计算图机制直接相关
