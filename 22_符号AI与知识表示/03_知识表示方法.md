# 知识表示方法

## 一句话理解

将世界知识编码为机器可存储、检索和推理的结构化形式——是符号AI的核心问题，决定了推理系统的能力和效率。

## 背景与发展

知识表示（Knowledge Representation, KR）是AI的核心问题之一。从最早的语义网络到框架系统，再到本体和描述逻辑，知识表示方法不断演进。每种方法在表达能力和推理效率之间做出不同的权衡。

| 年代 | 里程碑 | 作者 | 意义 |
|:---|:---|:---|:---|
| 1956 | 语义网络 | Quillian | 图结构知识表示 |
| 1968 | 框架理论 | Minsky | 面向对象的知识结构 |
| 1974 | 产生式规则 | Newell & Simon | IF-THEN规则表示 |
| 1985 | KL-ONE | Brachman & Schmolze | 描述逻辑起源 |
| 1993 | 本体定义 | Gruber | 知识工程的形式化 |
| 2004 | OWL标准 | W3C | Web本体语言标准化 |
| 2012 | Knowledge Graph | Google | 工业级大规模应用 |

## 核心概念

### 知识表示的权衡

任何知识表示方法都面临以下权衡：

| 维度 | 说明 |
|:---|:---|
| 表达能力 | 能表示多复杂的知识 |
| 推理效率 | 推理的计算复杂度 |
| 可扩展性 | 知识规模增长后的可维护性 |
| 可理解性 | 人类是否能理解表示内容 |
| 不确定性处理 | 能否处理不精确或概率性知识 |

表达能力强通常意味着推理效率低——这是知识表示的根本矛盾。

### 语义网络（Semantic Network）

用有向图表示知识：节点代表概念/实体，边代表关系。

```
  猫 ──IS-A──→ 哺乳动物 ──IS-A──→ 动物
  │                              │
  HAS                            HAS
  │                              │
  爪子                           皮肤
```

关键关系类型：
- **IS-A**（实例/子类关系）：支持继承推理
- **HAS-A**（部分/属性关系）：支持属性传播
- **Instance-of**：实体与概念的关系

优势：直观、支持继承推理。不足：缺乏形式语义，推理可能产生歧义。

### 框架（Frames）

Minsky提出的面向对象知识表示。每个框架是一个数据结构：

```
框架: 哺乳动物
  IS-A: 动物
  皮肤: 有毛
  繁殖: 胎生
  体温: 恒温

框架: 猫
  IS-A: 哺乳动物
  体型: 小
  食性: 肉食
  默认行为: 抓老鼠
```

框架支持：
- **继承**：子框架继承父框架的属性
- **默认值**：属性可以有默认值
- **附加过程**：属性可绑定计算过程（如 demon）

框架是面向对象编程的概念先驱。

### 产生式规则（Production Rules）

IF-THEN形式的知识表示：

```
IF 动物有羽毛 THEN 动物是鸟
IF 动物会飞 AND 动物产卵 THEN 动物是鸟
IF 动物是鸟 AND 动物不会飞 AND 动物体型大 THEN 动物是鸵鸟
```

特点：
- 模块化——每条规则独立
- 可解释——推理链清晰
- 前向/反向链推理

缺点：规则数量大时维护困难，规则间可能冲突。参见 [[04_专家系统]]。

### 本体（Ontology）

对特定领域概念体系的**形式化描述**，包括：
- **类/概念**：领域中的实体类别
- **层次关系**：子类-父类（subClassOf）
- **属性关系**：概念间的关联
- **约束/公理**：对属性和关系的限制

Gruber的经典定义：**本体是共享概念化的明确规范**（"Ontology is an explicit specification of a conceptualization"）。

本体的形式化基础是**描述逻辑**（Description Logic），它是一阶逻辑的可判定子集。参见 [[02_形式逻辑与推理]]。

### 知识表示与现代方法

现代知识表示已超越纯符号方法：

| 方法 | 类型 | 特点 |
|:---|:---|:---|
| 知识图谱 | 符号 | 大规模三元组存储 |
| 知识嵌入（TransE等） | 神经 | 将实体/关系映射到向量空间 |
| 大语言模型 | 神经 | 参数化隐式知识存储 |
| 神经符号AI | 混合 | 结合符号推理与神经学习 |

## 应用场景

| 场景 | 知识表示方法 |
|:---|:---|
| 专家系统 | 产生式规则、框架 |
| 语义网 | RDF/OWL本体 |
| 知识图谱 | 三元组、本体 |
| 医疗决策支持 | 本体+规则 |
| 智能问答 | 知识图谱+推理 |
| 自然语言理解 | 语义角色标注、知识图谱 |

## 优势与不足

### 优势
- 结构化表示支持精确推理
- 可解释——推理过程可追溯
- 支持知识共享和复用

### 不足
- 知识获取瓶颈——构建大规模知识库成本高
- 难以处理模糊性和不确定性
- 维护成本随规模增长
