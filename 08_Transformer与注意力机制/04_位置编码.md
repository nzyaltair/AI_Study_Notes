# 位置编码

## 1. 概述

位置编码（Position Encoding, PE）是 Transformer 中为自注意力注入位置信息的技术。自注意力机制本身是排列不变的（Permutation Invariant）——打乱输入序列的顺序不影响注意力计算结果，因此模型无法区分"猫追狗"和"狗追猫"。位置编码通过向输入嵌入添加位置信号或在注意力计算中引入位置偏置，使模型能够感知 token 在序列中的位置和相对顺序。

- **解决的问题**：自注意力的排列不变性导致模型无法建模序列顺序，而顺序是语言、时间序列等数据的核心特征。
- **核心挑战**：位置编码需要在计算效率、外推能力（训练长度之外保持性能）和架构兼容性之间取得平衡。

## 3. 核心概念

### 3.1 排列不变性

自注意力的注意力分数 $QK^T$ 仅取决于 token 的内容嵌入，与位置无关。因此，交换两个 token 的位置不会改变注意力计算结果（除了位置编码的影响）。位置编码的作用是打破这种排列不变性。

### 3.2 绝对位置编码 vs 相对位置编码

| 类型 | 原理 | 优势 | 劣势 |
|:---|:---|:---|:---|
| **绝对位置编码** | 为每个位置分配唯一的编码向量 | 实现简单 | 外推能力差 |
| **相对位置编码** | 编码 token 间的相对距离 | 外推友好，更符合语言直觉 | 实现复杂度较高 |

### 3.3 长度外推（Length Extrapolation）

模型在训练长度 $L_{train}$ 之外（$L_{test} > L_{train}$）保持性能的能力。这是位置编码设计的核心考量之一——现代 LLM 需要支持 4K → 128K 甚至更长的上下文扩展。

## 4. 技术原理

### 4.1 正弦/余弦编码

原始 Transformer 提出的固定位置编码：

$$PE(pos, 2i) = \sin\left(\frac{pos}{10000^{2i/d_{model}}}\right)$$

$$PE(pos, 2i+1) = \cos\left(\frac{pos}{10000^{2i/d_{model}}}\right)$$

- 不同维度使用不同频率的正弦/余弦波（从 $2\pi$ 到 $10000 \cdot 2\pi$）
- 位置编码与嵌入相加：$x_{pos} = \text{embed}(token) + PE(pos)$
- **理论优点**：$\sin(\alpha + \beta) = \sin\alpha\cos\beta + \cos\alpha\sin\beta$，使位置 $pos + k$ 可表示为 $pos$ 的线性变换，理论上支持相对位置
- **实际局限**：正弦编码的外推能力有限，超出训练长度后性能下降

### 4.2 可学习位置嵌入

将位置编码作为可训练参数表 $PE \in \mathbb{R}^{L_{max} \times d_{model}}$：

$$x_{pos} = \text{embed}(token) + PE[pos]$$

- **优势**：前向传播快，模型可自由学习最优位置表示
- **劣势**：最大长度固定（$L_{max}$），无法外推
- **应用**：BERT（512）、GPT-2（1024）、ViT

### 4.3 相对位置编码

直接建模 token 间的相对位置关系，而非注入绝对位置。

**Shaw 方法（2018）**：在注意力计算中加入相对位置嵌入 $R_{i-j}$：

$$e_{ij} = (x_i W_Q)(x_j W_K + R_{i-j})^T$$

**Transformer-XL（2019）**：引入相对位置编码 $K_R$ 和可学习向量 $u, v$：

$$e_{ij} = (x_i W_Q)(x_j W_K + R_{i-j})^T + u^T R_{i-j}$$

**T5 偏置（2019）**：使用可学习的桶化偏置矩阵 $B_{i-j}$：

$$e_{ij} = q_i^T k_j + B_{i-j}$$

T5 将相对距离分桶（log 分桶：近距离精细、远距离粗略），每个桶一个可学习参数。

**DeBERTa（2020）**：解耦内容注意力和位置注意力：

$$e_{ij} = (x_i W_Q)(x_j W_K)^T + (x_i P_Q)(P_{i-j})^T + (x_j W_K)(P_{i-j})^T$$

### 4.4 旋转位置编码（RoPE）

**论文**：Su et al., *RoFormer: Enhanced Transformer with Rotary Position Embedding* (2021)

RoPE 通过**旋转矩阵**将位置信息注入 Q 和 K，使点积注意力自动包含相对位置信息。

**核心思想**：对于二维向量 $q = (q_0, q_1)$，在位置 $m$ 处旋转角度 $\theta_m = m\theta$：

$$q_m' = R_m q_m, \quad R_m = \begin{pmatrix} \cos m\theta & -\sin m\theta \\ \sin m\theta & \cos m\theta \end{pmatrix}$$

**关键性质**：

$$q_m'^T k_n' = (R_m q_m)^T (R_n k_n) = q_m^T R_m^T R_n k_n = q_m^T R_{n-m} k_n$$

注意力分数仅依赖相对位置 $n - m$，而非绝对位置。

**多维扩展**：将 $d$ 维向量分成 $d/2$ 个二维子空间，每个子空间使用不同频率 $\theta_i = 10000^{-2i/d}$：

$$q_m' = R_m q_m, \quad R_m = \text{diag}(R_{m,\theta_0}, R_{m,\theta_1}, \ldots, R_{m,\theta_{d/2-1}})$$

**实现**（高效形式，无需构造稀疏旋转矩阵）：

```python
def apply_rope(q, k, cos, sin):
    # q, k: (B, num_heads, T, d_k)
    # cos, sin: (T, d_k/2)
    q_rot = (q * cos) + (rotate_half(q) * sin)
    k_rot = (k * cos) + (rotate_half(k) * sin)
    return q_rot, k_rot

def rotate_half(x):
    x1, x2 = x.chunk(2, dim=-1)
    return torch.cat((-x2, x1), dim=-1)
```

**优势**：
- 相对位置编码，无需额外参数
- 与各种注意力机制兼容（MHA、GQA、MLA）
- 计算高效，仅涉及逐元素乘法
- 现代LLM标配（LLaMA、Qwen、Mistral、DeepSeek）

### 4.5 ALiBi（Attention with Linear Biases）

**论文**：Press et al., *Train Short, Test Long: Attention with Linear Biases Enables Input Length Extrapolation* (2021)

不使用显式位置嵌入，在注意力分数中添加线性偏置：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T + m \cdot B}{\sqrt{d_k}}\right) V$$

其中 $B_{i,j} = -(i - j)$（当 $i \geq j$ 时），$m$ 是头特定的斜率因子：

$$m_h = \frac{1}{2^{8/h}} \quad \text{(如 h=8 时: 1/2, 1/4, ..., 1/256)}$$

**特点**：
- 无需位置嵌入参数
- 距离越远偏置越大（惩罚远距离注意力）
- **原生外推**：训练 1024 长度可在 2048+ 长度上保持性能
- 适合 Decoder-only 架构
- 代表模型：BLOOM、MPT

### 4.6 长度外推方法

当需要将训练长度 $L_{train}$ 的模型扩展到 $L_{test}$ 时，需要对 RoPE 进行调整：

| 方法 | 原理 | 扩展倍数 | 代表应用 |
|:---|:---|:---|:---|
| **位置插值（PI）** | 线性压缩位置索引：$pos' = pos \cdot L_{train} / L_{test}$ | 4-8x | LLaMA 2 Long |
| **NTK-aware** | 调整 RoPE 基频 $\theta$：$\theta' = \theta \cdot s^{d/(d-2)}$ | 8-32x | Qwen 1.5 |
| **YaRN** | 分频段处理：高频外推、低频插值 | 16-64x | LLaMA 3 |
| **Dynamic NTK** | 推理时根据当前长度动态调整 $\theta$ | 任意 | vLLM 默认 |
| **xPos** | RoPE + 指数衰减因子 | - | 原论文 |
| **LongRoPE** | 非均匀插值，进化搜索最优频率 | 128x+ | 微软 |

**YaRN 原理**：

$$\theta_i' = \begin{cases} \theta_i & \text{高频段（外推，保持不变）} \\ \theta_i / s & \text{低频段（插值，线性压缩）} \\ \text{插值过渡} & \text{中频段} \end{cases}$$

YaRN 还引入注意力缩放补偿：$t = 0.1 \log(s) + 1$，对注意力分数乘以 $1/t$。

### 5.2 ViT 中的位置编码

ViT 使用可学习位置嵌入编码图像块的位置。由于图像是二维的，ViT 将 patch 序列化后使用一维位置嵌入（实验表明一维可学习嵌入已足够）。Swin Transformer 等后续工作引入了二维相对位置偏置。

## 6. 优势与局限

### 优势

1. **打破排列不变性**：使自注意力能感知序列顺序
2. **相对位置建模**：RoPE、ALiBi 等方法使注意力直接编码相对距离，更符合语言直觉
3. **外推能力**：ALiBi 原生外推、YaRN/NTK 后处理外推，支持 4K → 128K 扩展
4. **参数效率**：RoPE 无额外参数，ALiBi 仅需每头一个斜率参数

### 局限

1. **外推不完美**：即使 YaRN，超长上下文（如 1M token）仍面临挑战
2. **注意力稀释**：长序列中注意力分散到更多位置，有效注意力减弱
3. **位置编码与注意力耦合**：位置编码的选择影响注意力行为，修改位置编码需要重新训练
4. **二维/多维位置**：视觉、视频等场景需要多维位置编码，设计更复杂

## 7. 应用场景

| 场景 | 推荐方案 | 说明 |
|:---|:---|:---|
| 短文本 NLP | 可学习嵌入 / RoPE | 固定长度，无需外推 |
| 长上下文 LLM | RoPE + YaRN/NTK | 需要外推到训练长度之外 |
| 视觉 Transformer | 可学习 2D 嵌入 / 相对偏置 | 图像块位置编码 |
| 语音识别 | 相对位置编码 | 语音信号时长变化大 |
| 多模态 | 分离的位置编码 | 不同模态使用独立位置空间 |

## 9. 前沿发展

- **超长上下文**：从 128K 到 1M+ token 的位置编码方案（如 StreamingLLM 的注意力_sink + 滑动窗口）
- **无位置编码（NoPE）**：Kazemnejad et al. (2023) 发现 Decoder-only 模型中因果掩码本身已隐含绝对位置信息，短序列上完全不用位置编码也能取得良好性能，但长序列外推能力仍待验证
- **LongRoPE**：通过进化搜索找到最优非均匀频率插值方案，支持 128x+ 外推（Microsoft, 2024）
- **Self-Extend**：推理时无需微调的位置扩展方法
- **连续位置编码**：处理非整数位置（如语音帧、视频帧），支持任意精度的时间对齐
- **3D 位置编码**：视频理解中的三维位置（时间 + 空间 H/W）
- **位置编码与 KV-Cache 交互**：Dynamic NTK 在推理时根据缓存长度动态调整
- **无位置编码方案**：Mamba 等 SSM 架构通过递归状态天然编码位置，无需显式位置编码
