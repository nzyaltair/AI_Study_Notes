# Adapter

## 背景与发展

Adapter（适配器）由 Houlsby et al. (2019) 提出，是最早的参数高效微调方法之一。Adapter 通过在 Transformer 层中插入小型可训练模块实现任务适配，为后续的 LoRA、Prefix Tuning 等 PEFT 方法奠定了基础。虽然 LoRA 在多数场景下已取代 Adapter，但 Adapter 在多任务学习和模块化部署中仍有独特价值。

## 核心思想

在预训练模型的特定层之间插入小型瓶颈结构模块（适配器），仅训练适配器参数而保持原始模型参数不变。适配器采用"降维-激活-升维"的瓶颈结构，通过残差连接保持原始信息流。

## 技术原理

### 瓶颈结构

适配器的核心是瓶颈结构：

1. **降维层**：将 $d$ 维输入降维到 $r$ 维（$r \ll d$，如 $r = d/16$）
2. **非线性激活**：通常使用 GELU
3. **升维层**：将 $r$ 维恢复到 $d$ 维
4. **残差连接**：$y = x + f(x)$，保证梯度流

### 插入位置

原始 Adapter（Houlsby 2019）在每个 Transformer 层的两个位置插入：

- **注意力层后**（Adapter 1）
- **FFN 层后**（Adapter 2）

后续研究表明，仅在 FFN 层后插入适配器（Adapter-Fusion）即可达到接近的性能，且减少推理延迟。

### 参数量分析

每个适配器参数量为 $2dr + d$（两个线性层 + 归一化），约为模型总参数的 0.5%-3%。当 $r = d/16$ 时，每个适配器约 $d^2/8$ 参数。

### 前向传播

$$h = x + W_{up} \cdot \text{GELU}(W_{down} \cdot \text{LayerNorm}(x))$$

其中 $W_{down} \in \mathbb{R}^{r \times d}$，$W_{up} \in \mathbb{R}^{d \times r}$。

### 与 LoRA 的区别

| 特征 | Adapter | LoRA |
|------|---------|------|
| 结构 | 瓶颈模块（降维+激活+升维） | 低秩矩阵 $BA$ |
| 插入方式 | 串行（层后插入） | 并行（加到权重上） |
| 推理延迟 | 有（额外计算层） | 无（可合并到权重） |
| 非线性 | 有（GELU 激活） | 无（纯线性） |
| 参数量 | ~3% | ~1% |

## 发展演进

Adapter (2019, Houlsby) → AdapterFusion (2020，多任务融合) → Compacter (2021，低秩+共享参数) → Parallel Adapter (2022，并行连接) → MAD-X (跨语言迁移)

## 关键算法·模型

- **Adapter**：原始瓶颈结构适配器
- **AdapterFusion**：学习融合多个预训练适配器的输出
- **Compacter**：低秩分解 + 跨层共享参数，进一步减少参数量
- **Parallel Adapter**：并行连接，减少推理延迟
- **MAD-X**：语言特定 + 任务特定的双层适配器结构

## 应用场景

- **多任务学习**：为每个任务训练独立适配器，共享基座模型
- **跨语言迁移**：使用 MAD-X 实现零样本跨语言迁移
- **持续学习**：通过添加新适配器学习新任务，避免灾难性遗忘
- **模块化部署**：动态加载/切换不同任务的适配器

## 与其他技术关系

- Adapter 是 [[参数高效微调]] 的重要方法之一
- [[LoRA]] 在多数场景下性能优于 Adapter，但 Adapter 的非线性结构在特定任务有优势
- AdapterFusion 支持多适配器知识融合，是 LoRA 不具备的能力
- Adapter 可与模型并行、分布式训练结合
