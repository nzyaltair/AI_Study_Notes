# 特征工程

## 1. 概述

特征工程（Feature Engineering）是将原始数据转化为模型可有效利用的特征表示的过程。在[[00_深度学习基础_综述|深度学习]]兴起前，特征工程的质量往往比模型选择更能决定最终性能——好的特征能让简单模型达到复杂模型的效果。

Pedro Domingos 在《A Few Useful Things to Know About Machine Learning》（2012）中指出：**"特征工程是机器学习成功的关键"**。尽管现代[[00_深度学习基础_综述|深度学习]]通过端到端学习自动从非结构化数据中提取特征，但在表格数据、可解释性要求高的场景中，精心设计的特征工程依然是不可替代的竞争优势。

| 年代 | 里程碑 | 意义 |

## 3. 核心概念

### 3.1 特征工程六步法

```
原始数据 → 数据清洗 → 特征构造 → 特征变换 → 特征选择 → 特征评估
```

1. **数据清洗**：处理缺失值、异常值、重复数据
2. **特征构造**：从原始字段创建新特征
3. **特征变换**：标准化、归一化、编码
4. **特征选择**：筛选最有价值的特征子集
5. **特征评估**：验证特征对模型性能的贡献

### 3.2 特征类型与处理策略

| 特征类型 | 示例 | 处理方法 |
|---------|------|---------|
| **数值连续** | 价格、温度、年龄 | 标准化/归一化、分箱、对数变换 |
| **数值离散** | 计数（访问量）、评分（1-5） | 可保留也可 One-Hot |
| **类别名义** | 城市、品牌（无顺序） | One-Hot、目标编码、嵌入编码 |
| **类别有序** | 学历、评级（有顺序） | Label Encoding、Ordinal Encoding |
| **时间戳** | 交易时间、注册日期 | 提取周期特征（小时/周几/月份） |
| **文本** | 评论、描述 | TF-IDF、词嵌入、预训练模型 |
| **图像** | 照片、截图 | CNN 特征提取、预训练模型 |
| **地理** | 经纬度 | Geohash、距离计算 |

## 4. 技术原理

### 4.1 数值特征处理

**标准化与归一化**：

| 方法 | 公式 | 输出范围 | 适用场景 |
|------|------|---------|---------|
| **Z-score 标准化** | $x' = \frac{x - \mu}{\sigma}$ | $\approx [-3, 3]$ | 假设高斯分布，SVM/PCA/逻辑回归 |
| **Min-Max 归一化** | $x' = \frac{x - \min}{\max - \min}$ | $[0, 1]$ | 神经网络、需固定范围 |
| **Robust Scaling** | $x' = \frac{x - \text{median}}{\text{IQR}}$ | 无固定 | 含异常值数据 |
| **MaxAbs Scaling** | $x' = \frac{x}{\max(\|x\|)}$ | $[-1, 1]$ | 稀疏数据 |

**关键规则**：基于距离/相似度的模型（SVM、KNN、K-means、PCA）**必须**标准化；树模型（决策树、随机森林、XGBoost）**不需**标准化。

**偏态处理**：

- **对数变换**：$x' = \log(x + c)$，处理右偏分布（如收入、价格）
- **Box-Cox 变换**：$x' = (x^\lambda - 1)/\lambda$（$\lambda \neq 0$），参数化偏态修正
- **Yeo-Johnson 变换**：Box-Cox 的可处理负值和零值的推广

**分箱（Binning/Discretization）**：
- 等宽分箱：固定区间宽度
- 等频分箱：每个箱中样本数大致相等
- 基于决策树分箱：用目标变量的信息增益确定分界点

### 4.2 类别特征编码

| 编码方法 | 做法 | 优点 | 缺点 |
|---------|------|------|------|
| **One-Hot** | 每类别一个二进制位 | 无偏、简单 | 高基数时维度爆炸 |
| **Label** | 类别映射为整数 | 紧凑 | 引入虚假序关系 |
| **Ordinal** | 按序映射整数 | 保留序信息 | 仅适合有序类别 |
| **目标编码** | 用目标变量均值替换 | 捕获与目标的关系 | 容易过拟合/泄露 |
| **Count/Frequency** | 用出现频率替换 | 简单、信息量高 | 丢失语义 |
| **Hash 编码** | 哈希函数映射到固定维度 | 在高基数时高效 | 可能碰撞 |
| **嵌入编码** | 学习低维稠密向量 | 语义丰富、维度可控 | 需神经网络训练 |

**目标编码防泄露**：
- 必须用 K 折交叉验证计算：对当前折的样本，用其他折的数据计算目标均值
- 加入平滑项：$\text{Enc} = \frac{n_{\text{cat}} \times \text{mean}_{\text{cat}} + m \times \text{mean}_{\text{global}}}{n_{\text{cat}} + m}$，其中 $m$ 为平滑参数

### 4.3 特征选择

三大类方法：

| 类型 | 方法 | 特点 |
|------|------|------|
| **过滤法（Filter）** | 方差阈值、相关系数、互信息、卡方检验 | 独立于模型，快速 |
| **包装法（Wrapper）** | RFE（递归特征消除）、前向/后向选择 | 依赖模型，有偏 |
| **嵌入法（Embedded）** | L1 正则化（Lasso）、树模型特征重要性 | 训练时自动选择 |

**常用实践**：
1. 先用方差阈值去掉常量/近常量特征
2. 用相关系数/MI 做快速初筛
3. 用树模型的特征重要性精筛
4. 用 RFE + 交叉验证做最终确认

**互信息（Mutual Information）**：
$$I(X; Y) = \sum_{x,y} P(x,y) \log\frac{P(x,y)}{P(x)P(y)}$$

衡量特征与目标之间的依赖程度，可捕获非线性关系（优于相关系数）。

### 4.4 特征构造

**通用技巧**：

1. **交叉特征**：$x_1 \times x_2$、$x_1 / x_2$，捕获特征间交互
2. **多项式特征**：$x, x^2, x^3$，拟合非线性
3. **聚合特征**：按 group 计算的统计量（mean、max、min、std、count）
4. **时间窗口特征**：滑动窗口的均值/方差/趋势
5. **比率特征**：$x_1/(x_2 + \epsilon)$，归一化
6. **差分特征**：$x_t - x_{t-1}$，捕获变化量
7. **日期特征**：从时间戳提取年/月/日/周几/季度/是否周末
8. **文本特征**：TF-IDF、n-gram 统计、主题模型

### 4.5 自动特征工程

**Featuretools**：基于"深度特征合成"（Deep Feature Synthesis），自动生成聚合和变换特征：
- 定义实体（Entities）及关系（如 customers → transactions）
- 自动生成特征的统计聚合（SUM、MEAN、MAX 等跨关系操作）
- 自动生成变换特征（时间差、数值运算）

**AutoFeat**：选择能最大化与目标关系的非线性特征组合。

**树模型特征重要性**：用 XGBoost/LightGBM 训练后，基于 Gain/Cover 筛选 Top-K 特征。

## 6. 优势与局限

### 6.1 优势

- **精度提升**：好的特征是模型性能的最强杠杆
- **可解释性**：精心设计的特征提供数据洞察
- **通用性**：同一组特征可用于多种模型
- **降成本**：更好的特征 → 更简单的模型 → 更快的推理

### 6.2 局限

- **人工成本高**：传统特征工程依赖领域专家
- **领域限定**：一组好特征难以迁移到新领域
- **过拟合风险**：特征过多或目标编码不当会引发过拟合
- **信息损失**：手工特征可能丢失原始数据中的信息

## 7. 应用场景

- **金融风控**：从交易记录构造用户画像特征（消费习惯、风险评分）
- **推荐系统**：用户/物品侧特征交叉、时间窗口行为序列
- **点击率预估**：大规模稀疏特征 + 特征交叉（FFM、DeepFM）
- **时间序列预测**：滞后特征、滑动窗口统计、周期性特征
- **医学诊断**：从原始指标构造临床评分、复合指标

## 9. 前沿发展

- **自动特征工程（AutoFE）**：基于强化学习或进化算法的自动特征搜索
- **嵌入特征**：用预训练模型（LLM、视觉 Transformer）提取通用特征表示
- **因果特征选择**：基于因果推断选择对目标有因果效应（而非仅关联）的特征
- **联邦特征工程**：在隐私保护约束下的分布式特征构造
- **特征存储（Feature Store）**：Feast、Tecton 等工具管理特征生命周期，实现在线/离线特征一致性
- **多模态特征融合**：将表格、文本、图像特征统一表示
