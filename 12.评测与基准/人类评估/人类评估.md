# 人类评估

## 简介

人类评估在大模型/生成式 AI 评测中具有不可替代的价值，能够弥补自动指标的盲区，如对话质量、创造性、安全性、价值观对齐等主观维度。它与 [[自动评估]] 形成互补关系：自动评估适合快速迭代和客观指标，人类评估适合终版验证和主观质量判断。

## 全流程拆解

### 评估目标与维度设计

> "人类评估需明确回答：我们究竟在评估什么？" —— Belz et al. (2023)

- **核心维度**：`[[评估维度]]`（相关性、事实性、流畅度、有害性等）
- **设计原则**：
  - 维度需可操作化（如"相关性"定义为"与用户查询直接相关的程度"）
  - 维度间应相互独立，避免重叠
  - 根据任务类型选择合适的维度组合（如对话任务强调连贯性，问答任务强调事实性）
- **应用场景映射**：
  - 对话质量：相关性、连贯性、自然度
  - 创造性任务：原创性、多样性、实用性
  - 安全性：有害性、偏见、不当内容
  - 价值观对齐：公平性、尊重、社会责任

### 评估者策略

- **招募标准**：
  - 人口统计多样性（年龄、性别、教育背景等）
  - 领域专业知识（如医疗领域需相关背景）
  - 语言能力（多语言评估需母语者）
- **培训方案**：
  - 详细的评估指南学习
  - 示例标注练习
  - 反馈与校准环节
- **资质校验**：
  - 测试集评估（与专家标注对比）
  - 通过率阈值设置（如≥80%）
- **反偏见机制**：
  - 盲评设计（隐藏模型信息）
  - 多样化评估者团队
  - 偏见检测与修正流程

### 任务设计

- **成对比较（A/B Test）**：
  - 适用场景：比较两个模型的相对表现
  - 优势：减少绝对评分的主观性
  - 设计要点：平衡展示顺序，避免顺序偏差
- **Likert 量表**：
  - 常用尺度：5点或7点量表
  - 适用场景：需要量化单一维度表现
  - 设计要点：明确端点定义，避免模糊表述
- **开放式反馈**：
  - 适用场景：收集定性改进建议
  - 优势：发现定量评估未覆盖的问题
  - 设计要点：提供结构化提示，引导具体反馈
- **情景模拟**：
  - 适用场景：评估模型在特定场景下的表现
  - 优势：更接近真实使用场景
  - 设计要点：场景描述详细，包含必要上下文

### 工具链

- **主流平台**：
  - Scale AI：大规模众包评估，支持复杂任务设计
  - Labelbox：数据标注与评估一体化平台，适合企业级应用
  - Appen：全球评估者网络，支持多语言评估
- **自建系统要点**：
  - 任务分发机制（负载均衡、质量控制）
  - 评估者管理系统（培训、资质、绩效）
  - 数据安全措施（加密、访问控制）
- **数据安全规范**：
  - 敏感数据脱敏
  - 评估数据匿名化
  - 合规性检查（GDPR、CCPA等）

### 数据分析

- **评分者间信度**：
  - `[[Kappa系数]]`：评估分类一致性
  - ICC (Intraclass Correlation)：评估量表一致性
  - 可接受阈值：Kappa≥0.6，ICC≥0.7
- **显著性检验**：
  - 配对 t 检验：比较两个模型的差异
  - ANOVA：比较多个模型的差异
  - p值阈值：通常为0.05
- **结果可视化**：
  - 箱线图：展示评分分布
  - 雷达图：展示多维度表现
  - 热力图：展示模型在不同场景下的表现

### 伦理与合规

- **知情同意**：
  - 明确告知评估目的、流程、数据使用方式
  - 获得评估者的书面同意
- **隐私保护**：
  - 评估数据匿名化
  - 保护评估者个人信息
  - 数据存储与传输加密
- **报酬公平性**：
  - 合理的时薪标准（参考当地最低工资）
  - 及时支付
  - 基于任务复杂度和质量的奖励机制
- **参考**：`[[AI伦理]]` 中的相关原则

## 实战锦囊

### 可复用评分表示例

**对话质量评估表**
| 维度 | 1（很差） | 2（较差） | 3（一般） | 4（良好） | 5（优秀） |
|------|-----------|-----------|-----------|-----------|-----------|
| 相关性 | 完全不相关 | 部分相关 | 基本相关 | 高度相关 | 完全相关 |
| 事实性 | 严重事实错误 | 较多事实错误 | 少量事实错误 | 基本正确 | 完全正确 |
| 流畅度 | 严重语法错误 | 较多语法错误 | 基本流畅 | 流畅自然 | 非常流畅 |
| 有害性 | 严重有害内容 | 部分有害内容 | 无明显有害内容 | 积极健康 | 非常积极健康 |

### 任务说明模板

**成对比较任务说明**
1. 请阅读用户查询和两个模型回复（A和B）
2. 基于以下维度比较两个回复：相关性、事实性、流畅度、整体质量
3. 选择你认为在该维度上表现更好的回复
4. 如果两个回复表现相当，请选择"平局"
5. 请在评论框中简要说明你的选择理由

### 常见陷阱清单

- **锚定效应**：评估者受先前评分影响
- **顺序偏差**：评估者对先展示的样本有偏好
- **光环效应**：评估者因某一维度表现而影响整体评分
- **极端响应偏差**：评估者倾向于使用量表的极端值
- **中间倾向偏差**：评估者倾向于使用量表的中间值

### 关键指标计算逻辑

**Kappa系数计算**
```python
def calculate_cohen_kappa(rater1_scores, rater2_scores):
    # 构建混淆矩阵
    confusion_matrix = build_confusion_matrix(rater1_scores, rater2_scores)
    # 计算观察一致率
    observed_agreement = calculate_observed_agreement(confusion_matrix)
    # 计算期望一致率
    expected_agreement = calculate_expected_agreement(confusion_matrix)
    # 计算Kappa系数
    kappa = (observed_agreement - expected_agreement) / (1 - expected_agreement)
    return kappa
```

**显著性检验**
```python
def perform_significance_test(model_a_scores, model_b_scores):
    from scipy import stats
    # 配对t检验
    t_stat, p_value = stats.ttest_rel(model_a_scores, model_b_scores)
    # 判断显著性
    is_significant = p_value < 0.05
    return t_stat, p_value, is_significant
```

## 前沿演进

### LLM-as-a-Judge 的校准与局限

- **校准方法**：
  - 与人类评分对比，调整LLM判断标准
  - 使用黄金数据集微调判断模型
  - 多LLM法官投票机制
- **局限性**：
  - 可能存在与原模型相同的偏见
  - 对创造性任务的评估能力有限
  - 缺乏人类的生活经验和常识判断
- **与 [[自动评估]] 的关系**：LLM-as-a-Judge 是自动评估的高级形式，仍需人类评估作为最终校准

### 混合评估框架

- **核心思路**：
  - 人类反馈引导自动指标优化
  - 自动评估筛选，人类评估重点审查
  - 循环迭代：人类评估结果用于改进自动评估模型
- **优势**：
  - 平衡成本与质量
  - 提高评估的可扩展性
  - 减少人类评估的疲劳效应

### 跨文化评估挑战

- **主要挑战**：
  - 文化差异导致的价值观差异
  - 语言表达习惯的不同
  - 评估标准的文化适应性
- **应对策略**：
  - 招募具有跨文化背景的评估者
  - 设计文化适应性评估指南
  - 对评估结果进行文化差异分析
- **研究引用**：
  - ACL 2024 研究："Cross-Cultural Evaluation of Large Language Models"
  - CHI 2025 研究："Cultural Biases in Human Evaluation of AI Systems"

### 动态长期评估趋势

- **关键方向**：
  - 模型行为的长期追踪评估
  - 环境变化对模型表现的影响
  - 用户反馈驱动的持续评估
- **技术方案**：
  - 建立评估数据时间序列
  - 设计自适应评估框架
  - 结合用户反馈与专家评估
