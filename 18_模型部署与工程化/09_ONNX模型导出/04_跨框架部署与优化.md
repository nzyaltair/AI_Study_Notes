# 跨框架部署与优化

## 背景与发展

将 [[ONNX]] 模型部署到生产环境时，需要在多种推理引擎间做出选择。[[ONNX Runtime]] 提供跨平台通用推理能力，[[TensorRT]] 在 NVIDIA GPU 上提供极致性能，OpenVINO 面向 Intel 硬件优化。不同引擎的算子支持范围、优化策略和部署复杂度各异，跨框架部署需要综合考虑性能、兼容性和维护成本。

## 核心思想

ONNX 作为中间格式，解耦了训练框架和推理引擎：一次导出，多处部署。推理引擎通过 Execution Provider（EP）机制适配不同硬件后端，每个 EP 针对特定硬件做算子级优化。选择推理引擎时遵循"最佳性能优先"原则——按 EP 列表顺序尝试，首个支持的 EP 被使用。

## 技术原理

### ONNX → TensorRT 优化

TensorRT 在加载 ONNX 模型后执行多级优化：

1. **层融合**：将 Conv+BatchNorm+ReLU 等连续算子融合为单个 kernel，减少 kernel 启动开销和中间结果存储
2. **精度校准**：FP16 无需校准数据，INT8 需要代表性数据集确定激活值量化范围
3. **Kernel 自动调优**：针对目标 GPU 架构从候选 kernel 中选择最优实现
4. **动态形状优化**：通过 min/opt/max shapes 配置预编译多个 kernel

```bash
# trtexec 命令行转换
trtexec --onnx=model.onnx --saveEngine=model.engine \
    --fp16 --workspace=4096 \
    --minShapes=input:1x3x224x224 \
    --optShapes=input:8x3x224x224 \
    --maxShapes=input:32x3x224x224
```

**精度模式对比**：

| 模式 | 内存占用 | 推理速度 | 精度损失 |
|------|---------|---------|---------|
| FP32 | 1.0x | 1.0x | 0% |
| FP16 | ~0.6x | 1.5-2x | <1% |
| INT8 | ~0.5x | 2-4x | 1-2% |

### ONNX Runtime 配置

```python
import onnxruntime as ort

sess_options = ort.SessionOptions()
sess_options.graph_optimization_level = ort.GraphOptimizationLevel.ORT_ENABLE_ALL

providers = [
    ('CUDAExecutionProvider', {
        'device_id': 0,
        'cudnn_conv_algo_search': 'DEFAULT',
        'gpu_mem_limit': 2 * 1024 * 1024 * 1024,
    }),
    'CPUExecutionProvider'  # Fallback
]

session = ort.InferenceSession("model.onnx", sess_options=sess_options, providers=providers)
```

### 框架对比

| 维度 | PyTorch | TensorFlow |
|------|---------|-----------|
| 转换 API | `torch.onnx.export()` | `tf2onnx.convert()` |
| 动态形状 | 原生支持，`dynamic_axes` | 需 `--inputs` 参数配置 |
| 导出速度 | 快（约 3.8s/ResNet50） | 较慢（约 5.2s） |
| 算子覆盖 | Opset 17 达 96% | Opset 17 达 93% |
| 调试体验 | 错误信息较详细 | 需查文档 |
| 生产稳定性 | 8/10 | 9/10 |

### 算子兼容性处理

ONNX 转换中最常见的挑战是算子不兼容。处理方案按优先级排列：

1. **提升 Opset 版本**：新 Opset 可能已支持目标算子
2. **onnxruntime-extensions**：加载扩展算子库
   ```python
   from onnxruntime_extensions import get_library_path
   sess_options = ort.SessionOptions()
   sess_options.register_custom_ops_library(get_library_path())
   session = ort.InferenceSession("model.onnx", sess_options)
   ```
3. **算子替换（模型重构）**：用标准 ONNX 算子组合替代不兼容算子，如 `repeat_interleave` → `expand` + `reshape`
4. **ONNX GraphSurgeon 子图替换**：在图级别替换不兼容节点
5. **自定义算子注册**：实现 C++ kernel 并注册到 ONNX Runtime

## 发展演进

ONNX Runtime 从 1.0 版本逐步发展出多 EP 架构，支持 CUDA、TensorRT、OpenVINO、CoreML、DirectML 等后端。TensorRT 8.x 引入对动态形状的原生支持，大幅降低了动态 batch 模型的优化难度。TensorRT-LLM 专为 LLM 推理优化，提供 attention 算子融合、KV Cache 量化等高级特性。

## 关键算法·模型

- **Conv-BN-ReLU 融合**：将三层算子合并为单个卷积 kernel，BatchNorm 参数折叠到 Conv 权重中
- **QKV 矩阵融合**：将 Attention 中三个独立的 Q/K/V 投影矩阵合并为一次 MatMul，减少 kernel 启动
- **Timing Cache**：TensorRT 首次构建时缓存 kernel 计时数据，后续构建复用可加速 50%+
- **EntropyCalib2**：INT8 量化校准算法，通过最大化输出分布熵来确定激活值量化范围

### 性能对比（ResNet50, V100 GPU）

| 运行时 | Batch=1 | Batch=32 | 加速比 |
|--------|---------|----------|--------|
| PyTorch 原生 | 2.1 ms | 15.2 ms | 1.0x |
| ONNX Runtime (CUDA) | 1.8 ms | 11.2 ms | 1.2-1.4x |
| TensorRT FP16 | 1.2 ms | 6.8 ms | 1.8-2.2x |
| TensorRT INT8 | 0.9 ms | 5.2 ms | 2.3-2.9x |

## 应用场景

- **NVIDIA GPU 生产部署**：ONNX → TensorRT FP16/INT8，极致推理性能
- **多平台通用部署**：ONNX Runtime + CPU/GPU EP，一套代码覆盖多硬件
- **Intel 硬件优化**：ONNX → OpenVINO IR，利用 Intel CPU/GPU/VPU 加速
- **移动端部署**：ONNX → TensorFlow Lite 或 ONNX Runtime Mobile
- **Apple Silicon**：ONNX Runtime + CoreML EP，利用 ANE 神经引擎
