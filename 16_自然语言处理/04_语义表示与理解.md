# 语义表示与理解

## 1. 概述

语义理解（Semantic Understanding）是 NLP 的核心目标——让计算机不仅识别文字的表面形式，更理解其背后的**意义**。从词级语义（一个词的含义是什么）到句子级语义（这句话表达了什么意思）再到篇章级语义（这段话的逻辑关系是什么），语义理解贯穿 NLP 处理的各个层次。

语义表示经历了从"符号化知识库"到"分布式向量表示"再到"上下文感知预训练表示"的演进。Word2Vec 的分布式假说、BERT 的上下文嵌入，以及大语言模型的隐式语义理解，逐步将语义表示从人工定义转向数据驱动学习。

## 3. 核心概念

### 3.1 语义的层次

| 层次 | 研究内容 | 典型任务 |
|:---|:---|:---|
| 词级语义 | 词义、同义词、反义词、上下位关系 | 词义消歧（WSD）、词相似度 |
| 句子级语义 | 句子含义、语义角色、蕴含关系 | 语义角色标注（SRL）、NLI |
| 篇章级语义 | 篇章结构、指代关系、逻辑关系 | 共指消解、篇章分析 |

### 3.2 分布式语义假说

**"一个词的意义由其上下文决定"**（You shall know a word by the company it keeps, Firth, 1957）。

这是现代语义表示的理论基础——通过统计词的共现模式来捕捉语义关系：

- 上下文相似的词 → 语义相似
- 共现频率高的词 → 语义相关

### 3.3 词义消歧（Word Sense Disambiguation, WSD）

同一个词在不同上下文中可能有不同含义：

- "bank"：河岸 vs 银行
- "苹果"：水果 vs 公司
- "打"：打球 vs 打电话 vs 打人

WSD 的目标是根据上下文确定词的正确含义。

### 3.4 语义角色标注（Semantic Role Labeling, SRL）

识别句子中的"谁对谁做了什么"——即谓词的论元结构：

```
句子：张三昨天在北京给李四买了一本书

谓词：买
施事（Agent）：张三
受事（Patient）：一本书
受益者（Beneficiary）：李四
时间（Time）：昨天
地点（Location）：北京
```

### 3.5 自然语言推理（Natural Language Inference, NLI）

判断前提（Premise）与假设（Hypothesis）之间的逻辑关系：

| 关系 | 含义 | 示例 |
|:---|:---|:---|
| 蕴含（Entailment） | 前提成立则假设成立 | 前提："一个男人在跑步" → 假设："一个男人在运动" |
| 矛盾（Contradiction） | 前提与假设不能同时成立 | 前提："一个男人在跑步" → 假设："一个男人在睡觉" |
| 中立（Neutral） | 既不蕴含也不矛盾 | 前提："一个男人在跑步" → 假设："男人穿着红色衣服" |

## 4. 技术原理

### 4.1 词向量：从 Word2Vec 到 BERT

#### Word2Vec（静态词向量）

Word2Vec 通过语言模型任务学习词的分布式表示。核心思想是用向量空间的几何关系编码语义关系：

$$\vec{king} - \vec{man} + \vec{woman} \approx \vec{queen}$$

两种训练目标：
- **CBOW**：用上下文预测中心词
- **Skip-gram**：用中心词预测上下文

负采样目标函数：

$$L = -\log \sigma(v_w^T v_c) - \sum_{w_i \sim P_n} \log \sigma(-v_{w_i}^T v_c)$$

**局限**：静态向量——同一个词在所有上下文中表示相同，无法处理多义词。

#### BERT（上下文词向量）

BERT 通过双向 Transformer 编码器生成上下文相关的词向量：

```
"苹果很好吃" → BERT("苹果") ≠ BERT 中的 "苹果" in "苹果发布了iPhone"
```

BERT 解决了多义性问题——词的表示随上下文动态变化。

### 4.2 句子语义表示

#### Sentence-BERT

标准 BERT 获取句子表示需要将两个句子同时输入，计算复杂度为 $O(n^2)$。Sentence-BERT 通过孪生网络结构，使每个句子独立编码：

$$\text{sim}(s_1, s_2) = \cos(\text{pool}(BERT(s_1)), \text{pool}(BERT(s_2)))$$

支持高效的语义相似度计算和向量检索。

#### SimCSE（Simple Contrastive Sentence Embedding）

通过对比学习增强句子表示质量：

- **无监督 SimCSE**：同一句子的两次 dropout 作为正例对，不同句子作为负例
- **有监督 SimCSE**：利用 NLI 数据集的蕴含/矛盾关系作为正/负例

对比学习目标：

$$L = -\log \frac{e^{\text{sim}(h_i, h_i^+)/\tau}}{\sum_j e^{\text{sim}(h_i, h_j^+)/\tau}}$$

### 4.3 语义角色标注

SRL 通常分为两步：

1. **论元识别**：识别句子中的谓词和候选论元
2. **论元分类**：为每个论元分配语义角色（施事、受事、时间等）

端到端神经网络方法将两步统一为序列标注或 span 标注问题。

### 4.4 共指消解（Coreference Resolution）

识别文本中指向同一实体的不同表述：

```
"张三[i]买了一辆车。他[i]非常开心。这辆车[j]是红色的。它[j]很漂亮。"
```

传统方法使用 Mention-Pair 分类器判断两个 mention 是否共指；端到端方法（如 SpanBERT）直接预测 span 之间的共指关系。

## 5. 关键方法/模型

| 方法 | 类型 | 核心思想 | 代表场景 |
|:---|:---|:---|:---|
| WordNet | 知识库 | 人工定义同义词集和语义关系 | 词义消歧、语义相似度 |
| Word2Vec / GloVe | 静态词向量 | 分布式假说 | 词相似度、聚类 |
| ELMo | 上下文词向量 | BiLSTM 语言模型 | 上下文相关表示 |
| BERT | 预训练表示 | 双向 Masked LM | 几乎所有语义任务 |
| Sentence-BERT | 句子嵌入 | 孪生 BERT | 语义搜索 |
| SimCSE | 对比学习 | 对比学习增强表示 | 句子相似度 |
| LLM | 端到端理解 | 大规模自回归 | 通用语义理解 |

## 6. 优势与局限

### 优势

1. **从符号到向量**：分布式表示使语义计算可操作化（相似度计算、向量运算）
2. **上下文感知**：BERT 等模型解决了多义词问题
3. **迁移学习**：预训练语义表示可迁移到多种下游任务

### 局限

1. **常识缺失**：统计/神经网络方法缺乏世界知识和常识推理能力
2. **组合性问题**：向量运算难以精确表示复杂逻辑组合
3. **可解释性差**：向量表示的语义维度难以人工解释
4. **幻觉问题**：LLM 可能生成语义看似合理但事实错误的内容

## 7. 应用场景

- **语义搜索**：基于语义相似度而非关键词匹配的检索
- **问答系统**：语义匹配问题和答案
- **文本蕴含**：判断陈述之间的逻辑关系
- **对话理解**：理解用户意图和槽位
- **推荐系统**：基于语义相似度推荐相关内容

## 9. 前沿发展

- **LLM 作为语义引擎**：大模型端到端处理语义理解任务，传统任务（WSD、SRL）逐渐被 LLM 隐式处理
- **检索增强语义**：RAG 结合外部知识库增强 LLM 的语义理解能力
- **多模态语义**：视觉-语言对齐（CLIP）将语义表示扩展到多模态空间
- **可解释语义**：研究 LLM 内部的语义表示机制，理解"黑箱"中的语义编码
- **因果语义**：超越相关性，研究因果关系层面的语义理解
