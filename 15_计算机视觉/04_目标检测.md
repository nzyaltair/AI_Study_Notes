# 目标检测

## 1. 概述

目标检测（Object Detection）是计算机视觉的核心任务之一：不仅要识别图像中有哪些物体（"是什么"），还要定位它们的位置（"在哪里"）。输出是一组 $(bbox, class, confidence)$ 三元组，其中 bbox 通常用矩形框 $(x_1, y_1, x_2, y_2)$ 或 $(x, y, w, h)$ 表示。

目标检测是许多实际应用的基础——自动驾驶需要检测车辆和行人，安防需要检测入侵者，机器人需要检测操作对象。它是 CV 领域研究最密集、进展最快的方向之一。

本篇从 CV 任务视角梳理检测的发展脉络、核心方法、评估指标和工程实践。关于 CNN backbone 细节，参见 [[05_卷积神经网络]]。

## 3. 核心概念

### 3.1 任务定义

给定图像 $I$，检测器输出 $N$ 个检测结果 $\{(b_i, c_i, s_i)\}_{i=1}^{N}$：
- $b_i = (x_1, y_1, x_2, y_2)$：边界框坐标
- $c_i$：类别标签
- $s_i$：置信度分数

### 3.2 IoU（交并比）

IoU 是衡量预测框与真实框重叠程度的指标：

$$\text{IoU} = \frac{|B_p \cap B_{gt}|}{|B_p \cup B_{gt}|}$$

- IoU > 0.5：通常视为正确检测
- IoU = 1.0：完全重合

### 3.3 mAP（平均精度均值）

mAP 是目标检测的标准评估指标：

1. 对每个类别，按置信度降序排列检测结果
2. 计算不同阈值下的 Precision-Recall 曲线
3. PR 曲线下面积 = AP（Average Precision）
4. 所有类别 AP 的平均 = mAP

| 指标 | 说明 |
|:---|:---|
| mAP@0.5 | IoU 阈值为 0.5 时的 mAP |
| mAP@0.5:0.95 | IoU 从 0.5 到 0.95（步长 0.05）的平均 mAP，更严格 |
| AP_small/medium/large | 按目标大小分组的 AP |

### 3.4 锚框（Anchor）

预定义的参考框，作为检测的先验：

- 在特征图每个位置放置多个不同尺度/比例的锚框
- 模型预测锚框的偏移量和类别
- 锚框设计影响检测性能（Faster R-CNN/RPN 使用 9 个锚框/位置）

### 3.5 NMS（非极大值抑制）

去除对同一目标的重复检测：

1. 按置信度降序排列所有检测框
2. 选择置信度最高的框，移除与其 IoU 超过阈值（如 0.5）的其余框
3. 对剩余框重复步骤 2

**Soft-NMS**：不直接移除，而是降低重叠框的置信度 $s \leftarrow s \cdot f(\text{IoU})$。

## 4. 技术原理

### 4.1 两阶段检测（Two-Stage）

先生成候选区域，再分类和回归。

#### R-CNN 系列

```
图像 → CNN backbone → 特征图
                      ↓
                 RPN（区域提议网络）→ 候选区域（RoI）
                      ↓
                 RoI Pooling/Align → 固定尺寸特征
                      ↓
                 分类头 + 回归头 → (class, bbox)
```

**RPN**：在特征图每个位置用锚框预测"是否有物体"和"框偏移"，生成候选区域。

**RoI Pooling**：将不同尺寸的候选区域池化为固定尺寸（如 $7 \times 7$）。

**RoI Align**（Mask R-CNN）：用双线性插值替代量化，消除 RoI Pooling 的量化误差。

#### 损失函数

$$\mathcal{L} = \mathcal{L}_{cls} + \lambda_{box} \mathcal{L}_{box} + \lambda_{rpn} \mathcal{L}_{rpn}$$

- $\mathcal{L}_{cls}$：分类交叉熵
- $\mathcal{L}_{box}$：Smooth L1 损失（或 GIoU/DIoU/CIoU）
- $\mathcal{L}_{rpn}$：RPN 的二分类 + 回归损失

### 4.2 单阶段检测（One-Stage）

直接从特征图预测检测结果，无需区域提议。

#### YOLO 系列

```
图像 → CNN backbone → 多尺度特征图
                      ↓
                 每个位置直接预测 (bbox, obj, class)
                      ↓
                 NMS → 最终检测
```

**YOLO 损失函数**：

$$\mathcal{L} = \lambda_{coord} \mathcal{L}_{box} + \mathcal{L}_{obj} + \lambda_{noobj} \mathcal{L}_{noobj} + \mathcal{L}_{cls}$$

- 边界框坐标损失（使用 CIoU 损失）
- 含目标/不含目标的置信度损失
- 分类损失

#### SSD（Single Shot MultiBox Detector）

- 多尺度特征图检测（$38 \times 38, 19 \times 19, \ldots, 1 \times 1$）
- 不同层使用不同尺度的锚框
- Hard Negative Mining 平衡正负样本

#### RetinaNet / Focal Loss

解决单阶段检测的正负样本不平衡问题：

$$\mathcal{L}_{FL} = -\alpha_t (1 - p_t)^\gamma \log(p_t)$$

- $\gamma > 0$：降低易分样本（$p_t$ 大）的损失权重
- $\alpha$：平衡正负样本频率
- 使单阶段检测首次达到两阶段精度

### 4.3 无锚框检测（Anchor-Free）

消除锚框设计的需求：

#### FCOS（Fully Convolutional One-Stage）

逐像素预测：
- 中心性（Centerness）：$c^* = \sqrt{\frac{\min(l,r)}{\max(l,r)} \cdot \frac{\min(t,b)}{\max(t,b)}}$
- 回归：预测每个像素到边界框四边的距离 $(l, t, r, b)$
- 分类：预测类别

#### CenterNet

将目标检测建模为关键点检测：
- 预测目标中心点热力图
- 回归中心点到边界框尺寸的偏移

### 4.4 DETR（Detection Transformer）

用 Transformer 实现端到端检测，无需 NMS 和锚框：

```
图像 → CNN backbone → 特征图 → Transformer Encoder
                                      ↓
                              Transformer Decoder (N 个 object queries)
                                      ↓
                              N 组 (class, bbox) 预测
```

**二分图匹配损失**：将预测集与真实集进行匈牙利匹配：

$$\hat{\sigma} = \arg\min_{\sigma \in \mathfrak{S}_N} \sum_i^N \mathcal{L}_{match}(y_i, \hat{y}_{\sigma(i)})$$

$$\mathcal{L}_{match} = -\mathbb{1}_{c_i \neq \varnothing} \hat{p}_{\sigma(i)}(c_i) + \mathbb{1}_{c_i \neq \varnothing} \mathcal{L}_{box}(b_i, \hat{b}_{\sigma(i)})$$

**DETR 的问题**：训练收敛慢（500 epochs），小目标性能差。

**改进方向**：
- **Deformable DETR**：可变形注意力，只关注参考点附近的少量采样点
- **DAB-DETR**：将 anchor box 作为 query 的内容
- **DN-DETR**：去噪训练加速收敛
- **DINO**：对比去噪 + 混合查询选择，成为 COCO SOTA

### 5.3 边界框回归损失演进

| 损失 | 公式 | 改进 |
|:---|:---|:---|
| Smooth L1 | $\begin{cases} 0.5x^2 & |x|<1 \\ |x|-0.5 & \text{otherwise}\end{cases}$ | 基础回归损失 |
| IoU Loss | $1 - \text{IoU}$ | 尺度不变 |
| GIoU | $1 - \text{IoU} + \frac{|C \setminus (A \cup B)|}{|C|}$ | 处理不相交情况 |
| DIoU | $1 - \text{IoU} + \frac{\rho^2(b, b^{gt})}{c^2}$ | 加中心距离 |
| CIoU | DIoU + $\alpha v$ | 加宽高比一致性 |

## 6. 优势与局限

### 优势

1. **定位 + 分类一体化**：同时回答"是什么"和"在哪里"
2. **实际应用广泛**：自动驾驶、安防、工业检测等核心需求
3. **评估体系完善**：mAP 等指标标准化
4. **工具链成熟**：MMDetection/Detectron2 提供完整训练部署流程

### 局限

1. **密集场景困难**：重叠、遮挡物体检测精度下降
2. **小目标检测困难**：小目标在特征图上信息不足
3. **类别不平衡**：长尾分布导致罕见类检测差
4. **开放词汇检测**：传统检测限于预定义类别集

## 9. 前沿发展

- **开放词汇检测**：Grounding DINO、OWLv2 等，检测任意文本描述的类别
- **3D 目标检测**：从点云或多视图检测 3D 边界框（BEVFormer/FCOS3D）
- **视频目标检测**：利用时序信息提升检测稳定性
- **少样本检测**：Few-Shot Detection，仅少量样本学习新类
- **基础检测模型**：SAM/Grounding DINO 等通用检测器，零样本迁移
- **端侧高效检测**：轻量化检测器在移动端实时运行（YOLOv8n/NanoDet）
