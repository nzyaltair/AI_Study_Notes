# 语音与音频

## 1. 概述

语音与音频 AI 涵盖语音识别（ASR）、语音合成（TTS）、音频理解和音频生成。Whisper（2022）实现了多语言鲁棒语音识别，现代 TTS 模型实现了自然度极高的语音合成，音频生成模型开始创作音乐和音效。当前趋势是走向**统一语音模型**——一个模型同时处理 ASR、TTS 和语音理解，以及实时端到端语音交互（GPT-4o）。

语音与音频 AI 解决的核心问题：
- **语音识别（ASR）**：将语音转换为文本，支持转录、翻译、命令识别
- **语音合成（TTS）**：将文本转换为自然语音，支持个性化音色
- **音频理解**：理解音频内容（音乐分析、声事件检测、情感识别）
- **音频生成**：从文本描述生成音乐、音效、环境音
- **实时语音交互**：低延迟端到端语音对话

## 3. 核心概念

### 3.1 音频表示

音频信号的基础表示形式：

| 表示 | 描述 | 用途 |
|:---|:---|:---|
| 波形 | 时域采样值 $x(t) \in [-1, 1]$ | 原始信号，声码器输出 |
| Mel 频谱图 | 时频表示，Mel 刻度 | ASR/TTS 标准输入 |
| MFCC | Mel 频率倒谱系数 | 传统 ASR 特征 |
| STFT | 短时傅里叶变换 | 时频分析基础 |
| 离散 token | 音频量化编码 | 自回归音频生成 |

**Mel 频谱图计算流程**：
```
音频波形 → 预加重 → 分帧 → 加窗 → FFT → Mel 滤波器组 → 对数 → Mel 频谱图
```

### 3.2 ASR 基础

语音识别的核心挑战：
- **变长对齐**：语音和文本长度不同，需要处理时间-字符对齐
- **噪声鲁棒性**：真实环境中背景噪声、混响、口音等干扰
- **多语言**：支持多种语言的识别和翻译
- **流式处理**：低延迟实时识别

**关键损失函数**：

- **CTC（Connectionist Temporal Classification）**：无需帧级标注的对齐方法
  $$\mathcal{L}_{CTC} = -\log \sum_{a \in \mathcal{A}(x, y)} \prod_t p(a_t | x_{\leq t})$$
  通过 blank 标签和合并机制处理变长对齐

- **RNN-T（RNN Transducer）**：编码器-解码器 + CTC 思想，支持流式识别
- **Seq2Seq + Attention**：编码器提取声学特征，解码器自回归生成文本

### 3.3 TTS 基础

语音合成的传统流程：

```
文本 → 文本前端(音素转换/分词/韵律) → 声学模型(文本→声学特征) → 声码器(声学特征→波形)
```

现代 TTS 的关键演进：
- **声码器**：从波形直接生成（WaveNet）到并行生成（WaveGlow, HiFi-GAN）
- **端到端**：从文本直接到语音（Tacotron, FastSpeech, VITS）
- **零样本克隆**：几秒参考音频即可克隆音色

### 3.4 音频 Token 化

将连续音频离散化为 token 序列，使自回归模型能处理音频：

| 方法 | 原理 | 代表 |
|:---|:---|:---|
| SoundStream | RVQ 量化编码 | Google |
| EnCodec | 残差向量量化 | Meta |
| DAC | 高保真音频编码 | Descript Audio Codec |
| WavTokenizer | 单码本高语义编码 | 2024 |

## 4. 技术原理

### 4.1 Whisper 架构

Whisper 是 OpenAI 发布的多语言语音识别模型：

```
音频 → Mel 频谱图 → 卷积编码 → Transformer 编码器 → Transformer 解码器 → 文本
```

**架构特点**：
- 编码器-解码器 Transformer（类似 [[09_预训练语言模型/03_BERT与编码器路线|编码器-解码器]]）
- 输入：80 维 Mel 频谱图
- 编码器：3 层 1D 卷积 + 12-32 层 Transformer
- 解码器：12-32 层 Transformer，自回归生成

**多任务能力**：
- 语音转写（Transcription）
- 语音翻译（Translation，多语言→英语）
- 语言识别（Language Identification）
- 时间戳预测（Timestamp Prediction）
- 语音活动检测（VAD）

**训练数据**：68 万小时多语言弱标注数据，从互联网收集。

### 4.2 Conformer 架构

Conformer 是 ASR 的经典编码器架构，融合卷积和 Transformer 的优势：

```
输入 → Feed-Forward → Multi-Head Self-Attention → Convolution → Feed-Forward → 输出
```

- **Self-Attention**：捕获全局依赖
- **卷积模块**：捕获局部模式
- **Macaron 结构**：前后各一层 FFN

### 4.3 现代 TTS 架构

#### VITS：端到端 TTS

VITS 将声学模型和声码器统一为一个端到端模型：

```
文本 → 文本编码器 → 对齐 → 解码器(flow + 声码器) → 波形
                ↑
        变分后验 q(z|x)
```

关键组件：
- **后验编码器**：从波形提取潜在表示
- **流（Flow）**：可逆变换，增强表达能力
- **解码器**：从潜在表示生成波形（HiFi-GAN）
- **随机时长预测器**：处理文本-语音时长对齐

#### NaturalSpeech 3：扩散 TTS

利用扩散模型进行语音合成：
- 将语音分解为内容、韵律、音色、声学四个因子
- 对每个因子分别建模
- 扩散模型生成声学特征
- 实现零样本语音克隆，自然度接近真人

#### FastSpeech：非自回归 TTS

解决自回归 TTS 速度慢的问题：
- **时长预测器**：预测每个音素的帧数
- **并行生成**：一次性生成所有帧的声学特征
- **自回归蒸馏**：从自回归教师模型学习对齐

### 4.4 音频生成

#### MusicGen：自回归音乐生成

```
文本描述 → T5 文本编码 → LM(自回归生成音频 token) → 声码器解码 → 音乐波形
```

- 使用 EnCodec 将音频编码为离散 token
- Transformer 语言模型自回归生成音频 token
- 层级结构：语义 token + 声学 token

#### AudioLDM：扩散音频生成

```
文本 → CLAP 文本编码 → 条件嵌入 → 潜在扩散模型去噪 → VAE 解码 → 音频
```

- 类似图像生成的 LDM 思路
- 在音频潜在空间执行扩散
- 支持文本到音频、文本到音乐生成

### 4.5 统一语音模型

一个模型同时处理多种语音任务的趋势：

**SPIRIT-LM（Meta）**：
- 将文本和语音 token 混合训练
- 支持语音→文本、文本→语音、语音→语音
- 保留 LLM 的语言推理能力

**GPT-4o Voice**：
- 端到端语音到语音，不经过文本中间表示
- 实时交互，延迟约 200ms
- 能感知语音中的情感、语调、环境音

## 6. 优势与局限

### 6.1 优势

- **自然度突破**：现代 TTS 合成语音已接近真人水平
- **多语言支持**：Whisper 等模型支持 90+ 语言
- **零样本克隆**：几秒参考音频即可克隆任意音色
- **实时交互**：GPT-4o 实现端到端实时语音对话
- **开源生态**：Whisper、VITS 等模型开源可用

### 6.2 局限

- **噪声环境**：ASR 在高噪声、多人说话场景仍不理想
- **情感表达**：TTS 的情感自然度仍有提升空间
- **低资源语言**：小语种语音识别和合成质量差
- **实时性**：高自然度 TTS 推理速度仍有瓶颈
- **安全风险**：语音克隆可用于诈骗、伪造
- **评估困难**：语音质量评估主观性强，自动指标不完善

## 9. 前沿发展

- **端到端语音交互**：GPT-4o 式的端到端语音到语音模型，消除文本中间表示
- **统一语音模型**：一个模型同时处理 ASR、TTS、语音翻译、语音理解
- **实时性**：流式 ASR 和低延迟 TTS 实现真正的实时交互
- **情感感知与表达**：识别和生成带情感的语音
- **零样本多语言克隆**：一个模型支持任意语言任意音色克隆
- **音频大模型**：将音频作为 LLM 的原生输入/输出模态
- **音乐理解与生成统一**：不仅生成音乐，还能分析、推荐、改编
- **高保真音频生成**：48kHz 高保真音乐和音效生成
- **空间音频**：生成带空间方位信息的三维音频
- **安全防御**：语音水印、深度伪造检测技术
