# 强化学习与大模型

## 1. 概述

强化学习与大语言模型（LLM）的融合是近年来 AI 领域最重要的突破之一。从 2022 年 InstructGPT 将 RLHF 引入大模型对齐训练，到 2024-2025 年 GRPO 和 RL 驱动的推理模型（o1/R1），强化学习已经成为大模型训练流程中不可或缺的环节。

本篇从**强化学习视角**审视 RL 与 LLM 的结合，重点讨论 RL 算法在 LLM 对齐和推理训练中的具体应用方式、技术挑战和最新进展。具体的对齐方法细节（RLHF/RLAIF/DPO/GRPO 的完整推导和工程实现）参见 [[11_大模型训练与对齐]]。

## 3. 核心概念

### 3.1 RL 作为 LLM 微调工具

在 LLM 训练流程中，RL 出现在**对齐阶段**（SFT 之后）：

```
预训练 → 监督微调(SFT) → 偏好对齐(RLHF/DPO/GRPO)
```

RL 对齐的目标：让 LLM 的输出更符合人类偏好（有用、无害、诚实），同时不损害预训练和 SFT 获得的能力。

### 3.2 LLM 作为 RL 策略

LLM 的自回归生成天然对应 RL 框架：

| RL 概念 | LLM 对应 |
|:---|:---|
| 状态 $s_t$ | 当前 token 序列 $(x, y_{<t})$ |
| 动作 $a_t$ | 下一个 token $y_t$ |
| 策略 $\pi_\theta(a_t\|s_t)$ | LLM 的 token 概率 $P_\theta(y_t \| x, y_{<t})$ |
| 轨迹 $\tau$ | 完整生成序列 $(y_1, y_2, \ldots, y_T)$ |
| 奖励 $R$ | 奖励模型对完整输出的评分 |
| 动作空间 $\mathcal{A}$ | 词表（vocab，通常 3万-15万） |

LLM 的 RL 微调本质上是**极高维动作空间的策略优化**。

### 3.3 奖励模型（Reward Model）

RLHF 的核心组件。从人类偏好数据中训练一个奖励模型 $r_\phi(x, y)$，对输入 $x$ 和输出 $y$ 给出标量奖励分数：

$$\mathcal{L}_{RM} = -\mathbb{E}_{(x, y_w, y_l)} \left[ \log \sigma(r_\phi(x, y_w) - r_\phi(x, y_l)) \right]$$

其中 $y_w$ 是偏好（赢）的输出，$y_l$ 是不偏好（输）的输出。

奖励模型的质量直接决定 RLHF 的效果——"垃圾进，垃圾出"。

## 4. 技术原理

### 4.1 RLHF 的 RL 阶段

**论文**：Ouyang et al., *Training language models to follow instructions with human feedback (RLHF)* (2022)

RLHF 的 RL 阶段使用 PPO 优化 LLM 策略：

**目标函数**：

$$\max_{\pi_\theta} \mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\theta(\cdot|x)} \left[ r_\phi(x, y) - \beta \cdot \text{KL}(\pi_\theta(\cdot|x) \| \pi_{\text{ref}}(\cdot|x)) \right]$$

- $r_\phi(x, y)$：奖励模型评分
- $\beta \cdot \text{KL}$：KL 散度惩罚，防止策略偏离参考模型 $\pi_{\text{ref}}$（SFT 模型）太远，保持语言能力

**PPO 更新**：

$$r_t(\theta) = \frac{\pi_\theta(y_t | x, y_{<t})}{\pi_{\text{old}}(y_t | x, y_{<t})}$$

$$\mathcal{L}^{CLIP} = \mathbb{E}\left[\min(r_t \hat{A}_t, \text{clip}(r_t, 1-\epsilon, 1+\epsilon) \hat{A}_t)\right]$$

其中 $\hat{A}_t$ 由 Critic（价值模型）估计。

**RLHF 的工程挑战**：
1. **价值模型训练**：Critic 与 LLM 同等规模，显存翻倍
2. **KL 控制**：$\beta$ 太大限制学习，太小导致"奖励黑客"（reward hacking）
3. **训练不稳定**：LLM 的高维策略空间使 PPO 训练敏感
4. **奖励黑客**：策略找到奖励模型的漏洞而非真正改善质量

### 4.2 RLAIF

**论文**：Anthropic, *Constitutional AI: Harmlessness from AI Feedback* (2022023)

RLAIF（RL from AI Feedback）用 **AI 模型替代人类**进行偏好标注：

1. 给定宪法（Constitution）——一组行为准则
2. AI 模型根据宪法评估和修正输出
3. 用 AI 偏好数据训练奖励模型
4. 后续 RL 流程与 RLHF 相同

**优势**：降低人工标注成本，可大规模生成偏好数据。

### 4.3 GRPO

**论文**：DeepSeek-AI, *DeepSeekMath: Pushing the Limits of Math Reasoning* (2024)

GRPO 去除 PPO 中的 Critic（价值模型），用**组内相对优势**替代：

**流程**：
1. 对每个问题 $q$，采样 $G$ 个回答 $\{o_1, \ldots, o_G\} \sim \pi_{\theta_{old}}(\cdot|q)$
2. 用奖励模型对每个回答评分 $r_1, \ldots, r_G$
3. 计算组内标准化优势：$A_i = \frac{r_i - \text{mean}(r)}{\text{std}(r)}$
4. 用 PPO clipped 目标更新策略

**关键优势**：
- **去除价值模型**：省去与 LLM 同等规模的 Critic，显存减半
- **无偏基线**：组内均值天然提供基线，无需学习
- **适合推理任务**：推理任务的奖励（对/错）天然适合组内比较

参见 [[11_大模型训练与对齐/08_GRPO]]。

### 4.4 DPO（RL 的"绕过"）

**论文**：Rafailov et al., *Direct Preference Optimization* (2023)

DPO 的核心洞察：RLHF 的最优策略有闭式解：

$$\pi^*(y|x) \propto \pi_{\text{ref}}(y|x) \exp\left(\frac{r(x,y)}{\beta}\right)$$

由此反解出奖励：$r(x,y) = \beta \log \frac{\pi^*(y|x)}{\pi_{\text{ref}}(y|x)} + \beta \log Z(x)$

代入 Bradley-Terry 偏好模型，得到无需显式 RL 的损失：

$$\mathcal{L}_{DPO} = -\mathbb{E}_{(x, y_w, y_l)} \left[ \log \sigma\left(\beta \log \frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)}\right) \right]$$

DPO 将 RLHF 转化为简单的分类损失，不需要奖励模型、价值模型或在线采样。参见 [[11_大模型训练与对齐/06_DPO]]。

### 4.5 RL 驱动的推理模型

**OpenAI o1**（2024）/ **DeepSeek-R1**（2025）

推理模型标志着 RL 与 LLM 融合的新阶段——用 RL 训练 LLM 进行**长链推理**（Chain-of-Thought）：

**DeepSeek-R1 的训练范式**：
1. **R1-Zero**：直接在基座模型上用 GRPO + 可验证奖励（数学题正确性、代码运行结果）训练
2. **R1**：在 R1-Zero 基础上加入 SFT 冷启动数据和多阶段 RL

**关键发现**：
- 纯 RL（无 SFT）即可激发推理能力（R1-Zero）
- 可验证奖励（verifiable rewards）比主观偏好奖励更适合推理任务
- RL 训练后模型自发产生"反思""验证"等推理行为（aha moment）

**推理任务中 RL 的优势**：
- 推理任务有**可验证的正确答案**（数学/代码），奖励信号客观
- 长链推理的探索空间巨大，RL 的试错搜索天然适合
- 组内比较（GRPO）天然适配"多解中选最优"的推理场景

## 5. RLHF vs DPO vs GRPO 对比

| 维度 | RLHF (PPO) | DPO | GRPO |
|:---|:---|:---|:---|
| 是否需要 RL | 是（PPO） | 否（监督学习） | 是（简化 PPO） |
| 奖励模型 | 需要 | 不需要（隐式） | 需要（或可验证奖励） |
| 价值模型 | 需要 | 不需要 | 不需要 |
| 在线采样 | 需要 | 不需要 | 需要 |
| 显存开销 | 最高（3 个大模型） | 最低（1 个模型） | 中等（2 个模型） |
| 适合场景 | 通用对齐 | 简单偏好对齐 | 推理任务对齐 |
| 训练稳定性 | 较差 | 好 | 中等 |

## 6. 优势与局限

### 优势

1. **人类偏好对齐**：RL 是让 LLM 符合人类偏好的最直接方法
2. **推理能力激发**：RL 能激发 LLM 的推理潜力，超越 SFT
3. **可验证奖励**：推理/代码任务有客观奖励，RL 信号清晰
4. **探索与发现**：RL 的试错搜索能发现 SFT 数据中不存在的策略

### 局限

1. **训练成本高**：RLHF/GRPO 需要大量推理和采样计算
2. **奖励黑客**：策略可能利用奖励模型漏洞而非真正改善
3. **训练不稳定**：高维策略空间的 PPO 训练敏感
4. **KL 约束的两难**：过于严格保持基座能力，过于宽松导致退化
5. **奖励设计困难**：开放域对话的奖励难以客观定义

## 9. 前沿发展

- **过程奖励模型（PRM）**：对推理的每一步给奖励，而非只对最终答案，提供更细粒度的学习信号
- **Self-Play + LLM**：LLM 自对弈生成训练数据，类似 AlphaZero 的范式在语言领域的应用
- **多轮交互 RL**：将 RL 扩展到多轮对话/Agent 交互场景
- **RL + 工具使用**：训练 LLM 在推理中使用工具（搜索、计算器、代码执行）
- **可验证奖励的扩展**：从数学/代码扩展到更多有可验证标准的领域
- **推理时计算扩展**：o1 类模型在推理时进行更多搜索/验证，RL 训练使模型学会高效利用推理时计算
- **RLAIF 的改进**：更强大的 AI 评估者（如 GPT-4 级别）提供更高质量偏好数据
