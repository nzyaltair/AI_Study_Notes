# 模型转换方法

## 背景与发展

将训练好的深度学习模型从框架特定格式（如 PyTorch 的 `.pt`、TensorFlow 的 SavedModel）转换为 [[ONNX]] 格式，是实现跨平台部署的第一步。各主流框架均提供了官方或社区维护的 ONNX 导出工具，其中 PyTorch 的 `torch.onnx.export` 功能最完整、使用最广泛。

## 核心思想

转换的本质是将框架的计算图映射为 ONNX IR：框架中的每个算子通过**符号函数（Symbolic Function）**映射到对应的 ONNX 算子，框架的权重参数序列化为 ONNX TensorProto 初始器。转换过程需要指定示例输入（dummy input）以触发追踪或脚本化，从而生成静态或动态计算图。

## 技术原理

### PyTorch：torch.onnx.export

```python
torch.onnx.export(
    model,                    # PyTorch 模型（须 eval 模式）
    args,                     # 示例输入（tensor 或 tuple）
    f,                        # 输出路径
    export_params=True,       # 是否导出权重
    opset_version=14,         # ONNX Opset 版本
    do_constant_folding=True, # 常量折叠优化
    input_names=['input'],    # 输入张量名
    output_names=['output'],  # 输出张量名
    dynamic_axes={            # 动态轴配置
        'input': {0: 'batch_size'},
        'output': {0: 'batch_size'}
    }
)
```

**关键参数说明**：

- `dynamic_axes`：指定哪些维度可变。语法为 `{tensor_name: {dim_index: dim_name}}`。命名应与 `input_names`/`output_names` 中的键一致。
- `opset_version`：影响算子覆盖范围。推荐 13 或 14，兼顾兼容性和功能。
- `do_constant_folding`：预计算常量操作，减少推理时计算量。默认开启。
- `training`：须设为 `TrainingMode.EVAL`，否则 Dropout/BatchNorm 行为不正确。

**ResNet50 动态 batch 导出示例**：

```python
import torch, torchvision.models as models

model = models.resnet50(pretrained=True).eval()
dummy_input = torch.randn(4, 3, 224, 224)

torch.onnx.export(
    model, dummy_input, 'resnet50.onnx',
    input_names=['image'], output_names=['probs'],
    dynamic_axes={
        'image': {0: 'batch_size', 2: 'height', 3: 'width'},
        'probs': {0: 'batch_size'}
    },
    opset_version=13, do_constant_folding=True
)
```

**自定义算子处理**：当 PyTorch 算子无对应 ONNX 映射时，可通过 `register_custom_op_symbolic` 注册符号函数：

```python
from torch.onnx import register_custom_op_symbolic

def my_op_symbolic(g, x, y):
    return g.op('Mul', x, y)  # 映射到已有 ONNX 算子

register_custom_op_symbolic('::my_op', my_op_symbolic, 9)
```

### TensorFlow：tf2onnx

TensorFlow 模型需先保存为 SavedModel 格式，再用 `tf2onnx` 转换：

```bash
python -m tf2onnx.convert \
    --saved-model saved_model_dir \
    --output model.onnx \
    --opset 13
```

Python API 方式：

```python
import tf2onnx
model_proto, _ = tf2onnx.convert.from_saved_model(
    'saved_model_dir', opset=13,
    input_names=['input:0'], output_names=['output:0']
)
with open('model.onnx', 'wb') as f:
    f.write(model_proto.SerializeToString())
```

Keras 模型可直接转换：

```python
spec = (tf.TensorSpec((None, 224, 224, 3), tf.float32, name="input"),)
model_proto, _ = tf2onnx.convert.from_keras(model, input_signature=spec, opset=13)
```

### PaddlePaddle：paddle2onnx

```python
import paddle2onnx
onnx_model = paddle2onnx.export(
    model, (dummy_input,),
    input_spec=[paddle.static.InputSpec(shape=[None, 3, 224, 224], dtype='float32')],
    opset_version=13
)
```

## 发展演进

早期 ONNX 导出依赖 TorchScript 追踪（trace），对控制流支持有限。PyTorch 1.x 后逐步改进符号推导，Opset 13+ 支持更完善的动态形状。PaddlePaddle 的 paddle2onnx 在国内生态中重要性持续增长，PaddleOCR 等模型的 ONNX 导出已成为标准流程。

## 关键算法·模型

- **JIT Tracing**：`torch.jit.trace` 记录前向传播执行路径，将动态图固化为静态图。控制流（if/for）会被"固化"为特定分支。
- **JIT Scripting**：`torch.jit.script` 解析 Python 源码生成计算图，能保留控制流语义，但对代码风格有约束。
- **符号推导**：PyTorch 通过注册在 `torch.onnx.symbolic_helper` 中的映射函数，将 ATen 算子转换为 ONNX 算子组合。

## 应用场景

- **PyTorch 模型部署**：研究模型训练完成后导出 ONNX，用于 ONNX Runtime 或 TensorRT 推理
- **TensorFlow 模型迁移**：将 TF SavedModel 转为 ONNX，统一到 ONNX Runtime 推理管线
- **PaddlePaddle 模型跨框架**：PaddleOCR、PaddleDetection 等模型导出 ONNX 供其他框架使用
- **量化模型导出**：对量化感知训练（QAT）后的模型导出为 ONNX，Opset 13+ 支持量化算子
