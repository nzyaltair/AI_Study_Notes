# 文本分类与情感分析

## 1. 概述

文本分类（Text Classification）是 NLP 中最基础也最广泛应用的任务——将文本分配到一个或多个预定义的类别中。情感分析（Sentiment Analysis）是文本分类的重要子领域，专注于分析文本中表达的情感极性和情绪。从垃圾邮件过滤到新闻分类，从情感监测到意图识别，文本分类无处不在。

文本分类是 NLP 方法演进的缩影：从朴素贝叶斯、SVM 等传统机器学习方法，到 TextCNN、BERT 等深度学习模型，再到 LLM 的零样本/少样本分类。它也是验证新模型有效性的标准基准任务。

## 3. 核心概念

### 3.1 文本分类的类型

| 类型 | 说明 | 示例 |
|:---|:---|:---|
| 二分类 | 两个互斥类别 | 垃圾邮件/正常邮件 |
| 多分类 | 多个互斥类别 | 新闻分类（体育/科技/政治） |
| 多标签 | 一篇文档可属多个类别 | 文章标签（科技+AI+教育） |
| 层次分类 | 类别有层次结构 | 商品分类（电子>手机>智能机） |

### 3.2 情感分析的层次

| 层次 | 任务 | 说明 |
|:---|:---|:---|
| 文档级 | 整篇文档的情感 | "这部电影很好看" → 正面 |
| 句子级 | 单个句子的情感 | "画面精美，但剧情无聊" → 混合 |
| 方面级（ABSA） | 特定方面的情感 | "食物好，服务差" → 食物:正面, 服务:负面 |

### 3.3 情感词典

基于人工构建或自动挖掘的情感词表进行情感分析：

- **英文**：SentiWordNet、VADER、AFINN
- **中文**：知网情感词典、NTUSD

每个词标注情感极性（正面/负面）和强度。简单快速但无法处理上下文（如"不好"中的否定）和反讽。

## 4. 技术原理

### 4.1 朴素贝叶斯

基于贝叶斯定理和特征独立假设：

$$P(y | \mathbf{x}) = \frac{P(y) P(\mathbf{x} | y)}{P(\mathbf{x})} \propto P(y) \prod_i P(x_i | y)$$

**多项式朴素贝叶斯**（适合文本）：

$$P(x_i | y) = \frac{N_{yi} + \alpha}{N_y + \alpha |V|}$$

- $N_{yi}$：类别 $y$ 下词 $x_i$ 出现的次数
- $\alpha$：平滑参数（Laplace 平滑 $\alpha=1$）

优势：训练快、适合高维稀疏文本特征。

### 4.2 SVM 文本分类

SVM 寻找最大间隔超平面：

$$\min_{\mathbf{w}, b} \frac{1}{2} \|\mathbf{w}\|^2 + C \sum_i \max(0, 1 - y_i(\mathbf{w}^T \phi(\mathbf{x}_i) + b))$$

使用核函数（如线性核或 RBF 核）处理非线性。在 TF-IDF 特征上，线性 SVM 长期是文本分类的 SOTA。

### 4.3 TextCNN

Kim (2014) 提出的卷积文本分类模型：

```
词嵌入矩阵 → 多尺度1D卷积 → Max-pooling → 拼接 → 全连接 → 分类
```

- 不同大小的卷积核捕获不同长度的 n-gram 特征
- Max-pooling 提取最重要的特征
- 结构简单、训练快、效果强

### 4.4 BERT 文本分类

BERT 微调进行文本分类——在 `[CLS]` token 的表示上接线性分类器：

```
[CLS] 文本内容 [SEP] → BERT → h_[CLS] → Dropout → Linear → Softmax → 类别
```

**损失函数**：交叉熵

$$L = -\sum_c y_c \log P(c | \mathbf{x})$$

BERT 微调在多数文本分类基准上达到 SOTA。

### 4.5 LLM 零样本/少样本分类

```
零样本：
"判断以下新闻的类别（体育/科技/政治/娱乐）：
'苹果发布新一代iPhone'
类别：科技"

少样本（In-Context Learning）：
"新闻：'勇士队夺冠' → 类别：体育
新闻：'国会通过新法案' → 类别：政治
新闻：'苹果发布新一代iPhone' → 类别："
```

## 5. 关键方法对比

| 方法 | 准确率（AG News） | 训练数据需求 | 训练速度 | 推理速度 |
|:---|:---|:---|:---|:---|
| 朴素贝叶斯 | ~85% | 少 | 极快 | 极快 |
| SVM + TF-IDF | ~92% | 中 | 快 | 快 |
| TextCNN | ~91% | 中 | 快 | 快 |
| FastText | ~92% | 中 | 极快 | 极快 |
| BERT 微调 | ~95% | 多 | 慢 | 中 |
| LLM Zero-shot | ~88% | 无 | 无需训练 | 慢 |
| LLM Few-shot | ~92% | 极少 | 无需训练 | 慢 |

## 6. 优势与局限

### 优势

1. **应用广泛**：几乎所有 NLP 应用都包含分类环节
2. **评估清晰**：准确率、F1 等指标直观明确
3. **技术成熟**：从传统 ML 到 LLM 全链条方法可选

### 局限

1. **标注数据**：监督方法需要大量标注数据
2. **类别不平衡**：实际场景中类别分布通常不均衡
3. **领域迁移**：训练于一个领域的模型在另一个领域性能下降
4. **长尾类别**：罕见类别分类效果差

## 7. 应用场景

- **垃圾邮件/短信过滤**：二分类
- **新闻分类**：多分类
- **情感监测**：舆情分析、产品评论分析
- **意图识别**：对话系统中的用户意图分类
- **内容审核**：有害内容/违规内容检测
- **法律文本分类**：案件类型、法条匹配
- **医疗文本分类**：疾病编码、病历分类

## 9. 前沿发展

- **LLM 分类器**：用 LLM 替代专门训练的分类器，降低标注需求
- **指令微调分类**：通过指令微调让 LLM 在多种分类任务间泛化
- **方面级情感分析（ABSA）**：细粒度情感分析，识别特定方面的情感
- **多模态分类**：结合文本、图像、视频的多模态内容分类
- **动态类别体系**：不预定义类别，通过聚类和 LLM 自动发现类别
- **因果推断**：超越相关性分类，分析文本与结果之间的因果关系
