# 工程实践与框架

## 1. 概述

知识蒸馏与模型压缩的工程实践是将理论方法转化为可部署方案的关键环节。本章涵盖主流框架工具、实际应用流程、部署方式选择和工程注意事项，帮助从理论理解走向实践落地。

## 2. 发展历史

| 年代 | 里程碑 | 核心意义 |
|:---|:---|:---|
| 2015 | Distiller (Intel) | 早期蒸馏框架 |
| 2019 | HuggingFace DistilBERT | 工业级蒸馏模型发布 |
| 2020 | TextBrewer (华为) | NLP 蒸馏框架 |
| 2021 | MMRazor (OpenMMLab) | 视觉模型压缩工具箱 |
| 2022 | Neural Magic SparseML | 稀疏化+量化工具 |
| 2023 | LCM Pipeline | 扩散模型蒸馏工作流 |
| 2023 | DeepSpeed Compression | 大模型压缩工具包 |
| 2024 | AutoDistill | 自动化蒸馏框架 |

## 3. 核心概念

### 3.1 压缩流程全景

标准模型压缩流程：

```
训练大模型 → 蒸馏 → 量化 → 剪枝 → 部署评估 → 迭代优化
```

各环节可灵活组合：
- **蒸馏+量化**：最常见组合
- **蒸馏+剪枝**：移除冗余结构
- **蒸馏+量化+剪枝**：极致压缩
- **仅量化/仅剪枝**：快速压缩

### 3.2 部署目标选择

| 部署目标 | 约束条件 | 推荐方案 |
|:---|:---|:---|
| 云端 GPU | 显存 40-80GB | 蒸馏 + INT8 量化 |
| 边缘 GPU (Jetson) | 显存 4-8GB | 蒸馏 + INT4 量化 |
| CPU 服务器 | 内存大但无 GPU | GGUF 量化 + 蒸馏 |
| 移动端 | 内存 < 4GB | 极致蒸馏 + INT4 + 剪枝 |
| 浏览器 | WASM 限制 | 小模型蒸馏 + INT8 |

## 4. 主流框架与工具

### 4.1 HuggingFace 生态

**DistilBERT 系列**：
- 直接提供蒸馏好的模型（DistilBERT, DistilGPT2, DistilRoBERTa）
- 使用 `transformers` 库直接加载

```python
from transformers import DistilBertModel
model = DistilBertModel.from_pretrained('distilbert-base-uncased')
```

**自定义蒸馏**：
- 使用 `Trainer` API 实现自定义蒸馏
- 支持响应蒸馏和特征蒸馏

### 4.2 TextBrewer（华为）

专为 NLP 模型蒸馏设计的开源框架：

- 支持 BERT/GPT/RoBERTa 等模型蒸馏
- 内置响应蒸馏、特征蒸馏、注意力蒸馏
- 灵活的蒸馏方案配置
- 支持多教师蒸馏

### 4.3 DeepSpeed Compression

微软 DeepSpeed 的压缩工具包：

- 支持量化、剪枝、蒸馏
- 面向大模型优化
- 与 DeepSpeed 训练框架集成
- 支持 MoE 模型压缩

### 4.4 Neural Magic SparseML

稀疏化+量化工具：

- 支持结构化/非结构化剪枝
- 支持量化感知训练
- 面向推理优化
- 与 ONNX/TensorRT 集成

### 4.5 MMRazor (OpenMMLab)

视觉模型压缩工具箱：

- 支持 CNN/ViT 蒸馏
- 内置 AT/FitNets/RKD 等方法
- 与 OpenMMLab 生态集成
- 支持检测/分割蒸馏

### 4.6 LCM Pipeline

扩散模型蒸馏工作流：

- 从 SD 模型蒸馏 LCM
- 支持 LCM-LoRA 训练
- 与 diffusers 库集成

## 5. 实际应用流程

### 5.1 BERT 蒸馏实践

```
1. 选择教师模型：BERT-base (12层, 110M参数)
2. 设计学生模型：6层, 66M参数
3. 准备数据：
   - 预训练数据：Wikipedia + BookCorpus
   - 任务数据：GLUE 下游任务
4. 两阶段蒸馏：
   a. 通用蒸馏：在预训练数据上蒸馏（响应+特征+注意力）
   b. 任务蒸馏：在任务数据上蒸馏微调
5. 评估：GLUE 基准
6. 进一步压缩：INT8 量化
7. 部署：ONNX 导出 → TensorRT 推理
```

### 5.2 LLM 蒸馏实践

```
1. 选择教师：GPT-4 API
2. 选择学生：LLaMA-7B
3. 数据生成：
   a. 编写种子指令（175条）
   b. 用 GPT-4 自我指令生成（52K条）
   c. 质量过滤
4. 训练：SFT (监督微调)
   - 学习率：2e-5
   - Epoch：3
   - LoRA 微调可选
5. 评估：MT-Bench, AlpacaEval
6. 量化：GPTQ INT4
7. 部署：vLLM / llama.cpp
```

### 5.3 扩散模型蒸馏实践

```
1. 选择教师：Stable Diffusion v1.5 (20-50步)
2. 蒸馏方法：LCM
3. 训练配置：
   - 潜在空间一致性蒸馏
   - CFG 快速估计
   - EMA 教师更新
4. LoRA 适配器：减少训练参数
5. 评估：FID, CLIP Score
6. 部署：diffusers pipeline 1-4步生成
```

### 5.4 组合压缩实践

```
1. 训练教师模型（大型，全精度）
2. 蒸馏为学生模型（小型，FP16）
3. 结构化剪枝（移除冗余通道/层）
4. INT8/INT4 量化（PTQ 或 QAT）
5. ONNX 导出
6. 推理引擎优化（TensorRT/OpenVINO）
7. 性能评估和迭代
```

## 6. 工程注意事项

### 6.1 蒸馏超参数调优

| 超参数 | 推荐范围 | 说明 |
|:---|:---|:---|
| 温度 $\tau$ | 2-10 | 分类任务通常 4-8 |
| 蒸馏权重 $\alpha$ | 0.5-0.9 | 蒸馏 vs 硬标签 |
| 特征蒸馏层选择 | 中间层 | 浅层和深层各有优势 |
| 学习率 | 教师的 0.1-0.5x | 学生通常用较小学习率 |
| Batch size | 尽量大 | 关系蒸馏需大 batch |

### 6.2 常见问题

| 问题 | 原因 | 解决方案 |
|:---|:---|:---|
| 学生性能远低于教师 | 蒸馏方案不当 | 检查特征对齐/温度设置 |
| 训练不收敛 | 损失权重失衡 | 调整各损失项权重 |
| 显存不足 | 教师和学生同时加载 | 梯度检查点/冻结教师 |
| 蒸馏后量化精度下降 | 量化破坏蒸馏效果 | 量化感知蒸馏 |
| 关系蒸馏效果差 | batch 太小 | 增大 batch 或跨 batch |

### 6.3 评估指标

| 任务 | 指标 |
|:---|:---|
| 分类 | Accuracy, F1 |
| 生成 | BLEU, ROUGE, FID |
| 检索 | Recall@K, NDCG |
| 检测 | mAP |
| 压缩率 | 参数量, FLOPs, 推理延迟 |
| 效率 | 吞吐量 (tokens/s), 显存占用 |

### 6.4 部署优化

- **ONNX 导出**：跨平台推理
- **TensorRT**：GPU 最优推理
- **OpenVINO**：Intel CPU 优化
- **Core ML**：Apple 设备
- **TFLite**：移动端
- **vLLM/llama.cpp**：LLM 专用推理
