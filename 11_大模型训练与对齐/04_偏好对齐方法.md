# 偏好对齐方法

## 1. 概述

偏好对齐（Preference Alignment）是指通过人类或AI反馈调整大语言模型的输出行为，使其与人类偏好、价值判断和安全标准保持一致。偏好对齐是现代大模型训练流水线的第三阶段，处于预训练和监督微调之后：

$$\text{预训练} \to \text{监督微调（SFT）} \to \text{偏好对齐} \to \text{部署}$$

InstructGPT首先提出RLHF方案，DPO（2023）将对齐从强化学习简化为分类问题，GRPO（2024）推动了推理模型的训练。偏好对齐方法的发展趋势是：流程越来越简化，所需资源越来越少，适用场景越来越广。

## 3. 核心概念

### 3.1 偏好数据

偏好数据通常表示为三元组 $(x, y_w, y_l)$，其中：
- $x$ 是输入（提示/指令）
- $y_w$ 是偏好输出（chosen/accepted）
- $y_l$ 是非偏好输出（rejected/rejected）

### 3.2 Bradley-Terry模型

人类对两个输出的偏好可以用相对奖励建模：

$$P(y_w \succ y_l \mid x) = \sigma\left(r(x, y_w) - r(x, y_l)\right)$$

这是大多数偏好对齐方法的数学基础。

### 3.3 方法分类

偏好对齐方法可分为两大类：

- **基于RL的方法**：训练奖励模型 + 强化学习优化（RLHF、GRPO）
- **基于分类的方法**：直接从偏好数据优化策略（DPO及其变体）

## 4. 技术原理

### 4.1 主流方法对比

| 方法 | 反馈来源 | 奖励模型 | 参考模型 | Critic | 训练稳定性 | 计算成本 | 数据格式 |
|------|---------|---------|---------|--------|-----------|---------|---------|
| [[RLHF]] | 人类反馈 | 需要 | 需要 | 需要 | 中（PPO不稳定） | 高 | 偏好对 |
| [[DPO]] | 人类偏好对 | 不需要 | 需要 | 不需要 | 高 | 低 | 偏好对 |
| [[RLAIF]] | AI反馈 | 需要 | 需要 | 需要 | 中 | 中 | 偏好对 |
| [[GRPO]] | 规则/模型 | 不需要 | 需要 | 不需要 | 高 | 低 | 生成+评分 |
| KTO | 二元反馈 | 不需要 | 需要 | 不需要 | 高 | 低 | 好/坏标注 |
| SimPO | 偏好对 | 不需要 | 不需要 | 不需要 | 高 | 最低 | 偏好对 |
| ORPO | 偏好对 | 不需要 | 不需要 | 不需要 | 高 | 低 | SFT+偏好 |

### 4.2 RLHF

三阶段流程：SFT → 训练奖励模型（RM）→ PPO优化

- 奖励模型损失：$L_{RM} = -\log\sigma(r(x, y_w) - r(x, y_l))$
- PPO目标：$\max_\pi \mathbb{E}[r(x,y)] - \beta \, D_{KL}(\pi \| \pi_{SFT})$

详见[[RLHF]]。

### 4.3 DPO

无需训练奖励模型，直接从偏好数据优化策略。核心洞察是RLHF最优策略与奖励函数存在闭式解关系：

$$r(x,y) = \beta \log \frac{\pi^*(y|x)}{\pi_{ref}(y|x)} + C(x)$$

代入Bradley-Terry模型得到DPO损失：

$$L_{DPO} = -\log\sigma\left(\beta\log\frac{\pi(y_w|x)}{\pi_{ref}(y_w|x)} - \beta\log\frac{\pi(y_l|x)}{\pi_{ref}(y_l|x)}\right)$$

详见[[DPO]]。

### 4.4 RLAIF

使用AI模型替代人类提供偏好反馈，降低标注成本。数学形式与RLHF相同，仅反馈数据来源不同。详见[[RLAIF]]。

### 4.5 GRPO

对同一问题生成一组候选回答，通过组内相对比较计算优势值，去除奖励模型和Critic：

$$\tilde{A}_i = \frac{r_i - \text{mean}(r_1, \ldots, r_G)}{\text{std}(r_1, \ldots, r_G)}$$

详见[[GRPO]]。

### 4.6 方法选型决策

```
有明确正确答案（数学/代码）？
  ├── 是 → GRPO（规则奖励，组内比较）
  └── 否 → 有配对偏好数据？
            ├── 是 → 计算资源充足？
            │         ├── 是 → RLHF（性能上限最高）
            │         └── 否 → DPO（简单稳定）
            │         └── 想去除参考模型？→ SimPO
            └── 否（只有好/坏标注）→ KTO
                  └── 想在SFT阶段同时对齐？→ ORPO
```

## 6. 优势与局限

### 优势

- **方法多样化**：从复杂的RLHF到简单的SimPO，覆盖不同场景
- **效果显著**：对齐后的模型在安全性、有用性上显著提升
- **灵活适配**：可根据数据格式、计算资源选择合适方法
- **持续演进**：方法不断简化，门槛持续降低

### 局限

- **偏好数据获取难**：高质量人类偏好数据稀缺且昂贵
- **奖励模型偏差**：RM可能被Hacking，泛化能力有限
- **对齐税**：对齐可能损害模型的部分能力（如创造力）
- **评估困难**：对齐效果难以用单一指标全面衡量
- **文化差异**：不同文化背景下的偏好标准不同

## 7. 应用场景

- **对话系统**：确保对话内容安全、友好、有帮助
- **内容生成**：生成符合人类偏好的文本和代码
- **推理模型训练**：GRPO在DeepSeek-R1中的推理能力对齐
- **安全防护**：防止有害输出、注入价值观
- **个性化**：基于用户偏好微调

## 8. 与其他技术关系

- 偏好对齐在[[监督微调]]之后进行
- 对齐效果依赖[[数据收集与清洗]]中偏好数据的质量
- RLHF中的PPO来源于强化学习（方向13）
- 安全对齐与价值对齐是偏好对齐的特定应用
- 对齐后的模型用于Agent和多模态应用

## 9. 前沿发展

### 9.1 流程简化

- **去除参考模型**：SimPO直接使用策略概率，无需参考模型前向传播
- **SFT+对齐一体化**：ORPO在SFT阶段同时进行偏好优化
- **去除奖励模型**：DPO/GRPO直接从数据学习

### 9.2 在线与迭代

- **Iterative DPO**：多轮DPO，每轮用新模型生成偏好数据
- **Online RLHF**：在线收集反馈并实时更新
- **自训练循环**：模型生成 → 评估 → 筛选 → 训练

### 9.3 推理模型训练

- **GRPO + 规则奖励**：DeepSeek-R1证明可用规则奖励训练推理能力
- **过程奖励模型（PRM）**：逐步评估推理质量
- **测试时计算Scaling**：通过更长推理链提升性能

### 9.4 多模态对齐

- 将偏好对齐扩展到视觉/音频等多模态场景
- 多模态偏好数据收集
- 跨模态对齐评估
