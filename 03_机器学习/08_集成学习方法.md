# 集成学习方法

## 1. 概述

[[08_集成学习方法|集成学习]]（Ensemble Learning）通过构建并结合多个基学习器来完成学习任务，其核心哲学是：**多个弱学习器的组合可以构成一个强学习器**。集成学习是 Kaggle 竞赛和工业表格数据领域的统治性方法论，几乎所有冠军方案都使用了某种形式的集成。

从偏差-方差的角度看：
- **Bagging** 主要**降低方差**（并行独立训练减少过拟合）
- **Boosting** 主要**降低偏差**（串行修正前序模型的错误）

## 2. 发展历史

| 年代 | 里程碑 | 意义 |
|------|--------|------|
| 1990 | Schapire 证明弱可学习 ⇔ 强可学习 | Boosting 理论基础 |
| 1996 | Breiman 提出 Bagging | Bootstrap 聚合降低方差 |
| 1997 | Freund & Schapire 提出 AdaBoost | 首个实用的 Boosting 算法，获 2003 Gödel 奖 |
| 1999 | Friedman 提出 GBDT（梯度提升） | 函数空间梯度下降统一 Boosting 框架 |
| 2001 | Breiman 提出随机森林 | Bagging + 随机特征选择 |
| 2003 | Friedman 发表 Greedy Function Approximation | GBDT 系统论述 |
| 2014 | Chen & Guestrin 发布 XGBoost | 二阶梯度 + 正则化 + 系统优化 |
| 2017 | Microsoft 发布 LightGBM | GOSS + EFB 极致效率 |
| 2017 | Yandex 发布 CatBoost | 原生类别特征处理 + Ordered Boosting |

## 3. 核心概念

### 3.1 为什么集成有效？

设基学习器的错误率为 $\epsilon$，且错误相互独立，则 $T$ 个学习器多数投票的错误率满足：

$$P(\text{集成错误}) = \sum_{k=0}^{\lfloor T/2 \rfloor} \binom{T}{k} \epsilon^k (1-\epsilon)^{T-k}$$

当 $\epsilon < 0.5$ 且 $T$ 增大时，集成错误率指数级下降。

**现实限制**：基学习器不可能完全独立。因此核心矛盾在于——既需要基学习器有一定**准确性**（accuracy），又要保证足够的**多样性**（diversity）。

### 3.2 偏差-方差视角

| 集成方法 | 主要降低 | 机制 |
|---------|---------|------|
| **Bagging** | 方差 | 并行独立训练，平均化消除随机波动 |
| **Boosting** | 偏差 | 串行修正，每步在前序基础上优化 |
| **Stacking** | 两者 | 元学习器学习如何最优组合 |

### 3.3 弱学习器与强学习器

**弱学习器**（Weak Learner）：性能仅略好于随机猜测（准确率 $> 50\%$）的分类器。

Kearns & Valiant（1988）提出的开放问题：弱可学习是否等价于强可学习？Schapire（1990）通过构造性证明给出了肯定答案，奠定了 Boosting 的理论基础。

## 4. 技术原理

### 4.1 Bagging（Bootstrap Aggregating）

**核心机制**：从训练集中有放回抽取 $n$ 个样本（Bootstrap 采样），重复 $T$ 次得到 $T$ 个不同的训练集，在每个训练集上训练基学习器，对预测结果投票（分类）或平均（回归）。

**Bootstrap 采样的性质**：
- 每个样本被选中的概率：$1 - (1 - 1/n)^n \approx 1 - 1/e \approx 63.2\%$
- 约 36.8% 的样本从未被选中（Out-Of-Bag, OOB）
- OOB 样本可作为免费验证集，给出泛化误差的无偏估计

**随机森林（Random Forest）**：

在 Bagging 基础上引入双重随机性：
1. **样本随机**：Bootstrap 采样训练每棵树
2. **特征随机**：每次分裂时只从随机选取的 $m$ 个特征中选择最优分裂特征

- 分类：$m = \sqrt{d}$（$d$ 为总特征数）
- 回归：$m = d/3$

**三重随机性的效果**：
- 大大增加树之间的多样性
- 降低方差而不显著增加偏差
- 对缺失值和异常值鲁棒

**特征重要性评估**：
- **MDI**（Mean Decrease in Impurity）：统计某特征在所有树中作为分裂特征时的不纯度减少总量
- **排列重要性（Permutation Importance）**：随机打乱某特征的值，观察模型性能下降幅度——模型无关的特征重要性度量

### 4.2 Boosting

Boosting 的核心思想是**串行训练基学习器，每个新模型重点关注前序模型的错误**。

#### AdaBoost

**核心机制**：
1. 初始化样本权重：$w_i^{(1)} = 1/n$
2. 第 $t$ 轮：基于权重训练弱分类器 $h_t$，计算加权错误率 $\epsilon_t$
3. 计算分类器权重：$\alpha_t = \frac{1}{2}\ln\left(\frac{1-\epsilon_t}{\epsilon_t}\right)$
4. 更新样本权重：$w_i^{(t+1)} = w_i^{(t)} \cdot \exp(-\alpha_t y_i h_t(x_i))$，归一化
5. 最终模型：$H(x) = \text{sign}\left(\sum_t \alpha_t h_t(x)\right)$

**关键性质**：
- 损失函数为指数损失 $L = \exp(-yH(x))$
- 错分样本权重指数增长，正确分类样本权重指数衰减
- 对噪声敏感（指数损失会极大关注离群点）

#### 梯度提升（Gradient Boosting）

Friedman（1999）将 Boosting 统一为**函数空间中的梯度下降**：

设当前集成模型为 $F_{m-1}(x)$，梯度提升在第 $m$ 步让新基学习器 $h_m(x)$ 拟合**残差的负梯度方向**：

$$r_{i}^{(m)} = -\left[\frac{\partial L(y_i, F(x_i))}{\partial F(x_i)}\right]_{F = F_{m-1}}$$

更新：$F_m(x) = F_{m-1}(x) + \eta \cdot h_m(x)$，其中 $\eta$ 为学习率。

**梯度提升 = 函数空间的梯度下降**：每个基学习器是函数空间中的"一步更新"。

**损失函数与对应负梯度**：

| 损失函数 | 负梯度（伪残差） | 适用场景 |
|---------|-----------------|---------|
| 平方损失 | $y_i - F(x_i)$ | 回归 |
| 绝对损失（MAE） | $\text{sign}(y_i - F(x_i))$ | 回归（对异常值鲁棒） |
| Huber 损失 | 分段：小残差用 L2，大残差用 L1 | 回归（兼顾效率与鲁棒） |
| 对数损失（LogLoss） | $y_i - P(y_i = 1)$ | 二分类 |

### 4.3 现代梯度提升框架

#### XGBoost（eXtreme Gradient Boosting）

**核心创新**：

**目标函数**（含正则化）：

$$\mathcal{L} = \sum_{i=1}^{n} l(y_i, \hat{y}_i) + \sum_{k=1}^{K} \Omega(f_k)$$

$$\Omega(f) = \gamma T + \frac{1}{2}\lambda \|\mathbf{w}\|^2$$

- 第一项：训练损失
- $\gamma T$：每片叶子的固定惩罚（控制树复杂度）
- $\frac{1}{2}\lambda\|\mathbf{w}\|^2$：叶子权重的 L2 惩罚（防权重过大）

**二阶 Taylor 展开**：对损失函数做二阶近似，使用梯度 $g_i$ 和 Hessian $h_i$：

$$\mathcal{L}^{(t)} \approx \sum_{i=1}^{n} \left[g_i f_t(x_i) + \frac{1}{2}h_i f_t^2(x_i)\right] + \Omega(f_t)$$

二阶信息使得分裂点搜索更精确，也支持自定义损失函数。

**系统优化**：
- 列块（Column Block）并行：预排序特征 + 缓存感知访问
- 近似分裂查找：加权分位数草图处理大规模数据
- 稀疏感知：自动学习缺失值的最优默认方向

#### LightGBM（Light Gradient Boosting Machine）

**核心创新**：

1. **Leaf-wise 生长**（vs XGBoost 的 Level-wise）：每次选择增益最大的叶子分裂，相同叶子数下损失更低。但需限制最大深度防过拟合

2. **GOSS**（Gradient-based One-Side Sampling）：
   - 保留全部高梯度样本（对训练更重要的样本）
   - 随机采样低梯度样本
   - 采样后的梯度需做重要性校正
   - 效果：在几乎不损失精度的情况下大幅减少数据量

3. **EFB**（Exclusive Feature Bundling）：
   - 将互斥（不同时非零）的稀疏特征捆绑为一个特征
   - 大幅降低特征维度，几乎无损

4. **直方图算法**：将连续特征值离散化为 $b$ 个 bin，分裂点搜索从 $O(n)$ 降至 $O(b)$

**优势总结**：训练速度极快、内存占用少、支持类别特征、天然支持大规模数据。

#### CatBoost

**核心创新**：

1. **原生的类别特征处理**：使用 Ordered Target Statistics 编码，结合排序原则防止目标泄露

2. **Ordered Boosting**：经典的逐样本残差计算会引入预测偏移（Prediction Shift），Ordered Boosting 通过对每个样本使用"不包括该样本的历史数据"训练的模型来估计残差，本质上是一种在线学习的残差计算方式

3. **对称树**（Oblivious Trees）：每层使用相同的分裂条件和特征 → 推理速度极快

**适用场景**：类别特征大量存在的场景（如点击率预估）、需要开箱即用的快速基线。

### 4.4 Stacking 与 Blending

**Stacking**（堆叠泛化，Wolpert, 1992）：
1. 将训练集分为 $K$ 折
2. 用 $K-1$ 折训练第一层的 $M$ 个不同基学习器
3. 在 hold-out 折上预测，拼接所有折的预测得到"元特征"
4. 用元特征训练第二层的元学习器（通常为逻辑回归或简单线性模型）

**关键技巧**：
- 使用 $K$ 折交叉验证生成元特征，避免数据泄露
- 第一层基学习器应**多样化**（不同算法族、不同参数）
- 元学习器应**简单**（避免在元特征上过拟合）

**Blending**：用 hold-out 验证集（而非 K 折 CV）生成元特征，更简单但数据利用率低。

## 5. 关键方法/模型

### 5.1 方法对比

| 特性 | 随机森林 | XGBoost | LightGBM | CatBoost |
|------|---------|---------|----------|----------|
| 训练策略 | 并行 Bagging | 串行 Boosting | 串行 Boosting | 串行 Boosting |
| 树生长 | Level-wise | Level-wise | Leaf-wise | Symmetric |
| 正则化 | 树的随机化 | $\gamma T + \lambda\|w\|^2$ | 深度/叶子数限制 | Ordered Boosting |
| 类别特征 | 不支持（需编码） | 不支持（需编码） | 支持 | 原生支持 |
| 缺失值 | 可处理 | 自动学习 | 自动处理 | 自动处理 |
| 适用规模 | 中大规模 | 中大规模 | 超大规模 | 类别特征多 |

### 5.2 选择指南

- **默认首选**：XGBoost 或 LightGBM —— 在大多数表格数据任务上是 SOTA
- **类别特征极多**：CatBoost —— 开箱即用，无需手动编码
- **数据量极大、追求速度**：LightGBM —— 训练最快，内存最少
- **需要并行训练**：随机森林 —— Bagging 天然可并行
- **追求可解释性**：随机森林 + 特征重要性分析

## 6. 优势与局限

### 6.1 优势

- **表格数据 SOTA**：XGBoost/LightGBM 在结构化数据上难以被深度学习超越
- **鲁棒性**：对异常值、缺失值、特征尺度不敏感
- **特征重要性**：天然提供特征重要性排序，用于特征选择和理解
- **无需特征缩放**：树模型天然不受特征尺度影响
- **较少超参数**：相比深度学习，需要调的超参数少得多

### 6.2 局限

- **外推能力弱**：树模型在训练数据范围外的预测不可靠（尤其回归）
- **模型体积大**：数百棵树组成，推理内存占用大
- **非平滑预测**：基于树的方法产生分段常数预测，导数不连续
- **无法处理非结构化数据**：图像、文本等需要[[00_深度学习基础_综述|深度学习]]
- **Boosting 对噪声敏感**：尤其是 AdaBoost，会在噪声样本上过度关注

## 7. 应用场景

| 领域 | 应用 | 常用方法 |
|------|------|---------|
| 金融 | 信用评分、欺诈检测、风险定价 | XGBoost、LightGBM |
| 广告 | CTR 预估、出价优化 | LightGBM、CatBoost |
| Kaggle 竞赛 | 几乎所有表格数据竞赛 | XGBoost/LightGBM + Stacking |
| 医疗 | 疾病预测、患者分层 | 随机森林（可解释性） |
| 工业 | 预测性维护、质量检测 | XGBoost、LightGBM |
| 推荐系统 | 特征工程、排序模型 | LightGBM Ranker |

## 8. 与其他技术关系

- 集成学习的基学习器最常用 [[06_监督学习方法|决策树]]，也可用其他算法
- Boosting 的梯度下降思想与 [[00_深度学习基础_综述|深度学习]] 的优化方法相通
- Stacking 的交叉验证思想来自 [[03_模型选择与调参|模型选择与调参]]
- 特征重要性可用于 [[02_特征工程|特征选择]]
- 集成方法需要 [[03_模型选择与调参|调参]] 来控制复杂度（树数量、深度、学习率）
- 在大模型时代，LLM + XGBoost 的混合架构开始出现（用 LLM 生成特征）

## 9. 前沿发展

- **GPU 加速**：XGBoost 和 LightGBM 已支持 GPU 训练，CatBoost 原生 GPU 支持优秀
- **联邦梯度提升**：隐私保护下的分布式 Boosting 训练
- **在线 Boosting**：流式数据上的增量 Boosting 更新
- **NGBoost**（Natural Gradient Boosting）：输出概率预测（含不确定性）的梯度提升
- **Explainable Boosting Machines（EBM）**：Glass-box 模型，兼具 GBDT 精度和线性模型可解释性
- **深度集成**：将神经网络与树模型组合（如 DeepGBM）
- **TabTransformer / FT-Transformer**：Transformer 应用于表格数据，开始挑战 XGBoost 的统治地位（但目前多数场景仍不如）
