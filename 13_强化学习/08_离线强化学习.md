# 离线强化学习

## 1. 概述

离线强化学习（Offline RL），又称批量强化学习（Batch RL），研究从**预先收集的固定数据集**中学习策略，无需与环境进行任何在线交互。这与在线 RL 形成对比——在线 RL 中智能体通过交互不断产生新数据来改进策略。

离线 RL 的核心动机是**安全性和成本**：在医疗、自动驾驶、推荐系统等领域，在线探索可能产生危险或昂贵的后果，但从已有历史数据中学习是可行的。离线 RL 的核心挑战是**分布偏移（Distribution Shift）**——数据集由行为策略产生，与目标策略的数据分布不同，导致 off-policy 估值产生外推误差（Extrapolation Error）。

## 3. 核心概念

### 3.1 分布偏移与外推误差

离线 RL 的核心困难。数据集 $\mathcal{D}$ 由行为策略 $\beta$ 产生，目标策略 $\pi$ 的分布可能不同。当 $\pi$ 选择 $\beta$ 未执行过的动作（OOD 动作）时，Q 函数对这些动作的估计可能过高（因为缺乏数据纠正），导致策略被错误的高估值引导。

**外推误差的三种来源**：

1. **数据不足**：某些 $(s,a)$ 在数据集中很少出现
2. **OOD 动作**：目标策略选择数据集中从未出现的动作
3. **误差传播**：Q 值的自举使 OOD 误差沿状态-动作链传播

### 3.2 在线 vs 离线 vs Off-policy

| 设定 | 数据来源 | 数据分布 | 探索能力 |
|:---|:---|:---|:---|
| 在线 RL | 实时交互 | 随策略变化 | 可探索 |
| Off-policy 在线 | 回放池（实时更新） | 近似策略分布 | 可探索 |
| 离线 RL | 固定数据集 | 行为策略 $\beta$ | 不可探索 |

off-policy 是技术手段（可用不同策略的数据），离线是设定约束（不能用新数据）。离线 RL 是 off-policy 的极端情况——数据分布完全固定。

## 4. 技术原理

### 4.1 BCQ（Batch-Constrained Q-Learning）

**论文**：Fujimoto et al., *Off-Policy Deep RL without Exploration* (2019)

BCQ 的核心思想：**目标策略只在数据集支撑集内选择动作**——即选择行为策略 $\beta$ 实际执行过的动作。

**架构**：
- **VAE 生成模型**：学习行为策略 $\beta(a|s)$，生成数据集中出现过的动作
- **扰动模型**：对生成的动作做小幅扰动 $\xi(a|s)$
- **Q 网络**：在生成+扰动的动作集合中选择 Q 值最高的

$$a = \arg\max_{a_i + \xi} Q(s, a_i + \xi), \quad a_i \sim \beta(a|s)$$

BCQ 通过约束动作在数据支撑集内，避免 OOD 估值问题。

### 4.2 BEAR（Bootstrapping Error Accumulation Reduction）

**论文**：Kumar et al., *Stabilizing Off-Policy Q-Learning via Bootstrapping Error Reduction* (2019)

BEAR 要求学到的策略 $\pi$ 在行为策略 $\beta$ 的**支撑集**（support）内，通过 MMD（Maximum Mean Discrepancy）约束：

$$\min_\pi \text{MMD}(\pi(\cdot|s), \beta(\cdot|s))$$

BEAR 比 BCQ 更宽松——不要求匹配 $\beta$，只要求在 $\beta$ 的支撑范围内。

### 4.3 CQL（Conservative Q-Learning）

**论文**：Kumar et al., *Conservative Q-Learning for Offline RL* (2020)

CQL 的核心思想：**保守地低估 OOD 动作的 Q 值**，防止策略被高估的 OOD 值误导。

CQL 修改 Q 学习目标，在标准 TD 损失上添加正则项：

$$\mathcal{L}_{CQL} = \alpha \left[ \mathbb{E}_{s \sim \mathcal{D}, a \sim \mu} Q(s,a) - \mathbb{E}_{(s,a) \sim \mathcal{D}} Q(s,a) \right] + \mathcal{L}_{TD}$$

- 第一项：最小化从任意分布 $\mu$ 采样的动作的 Q 值（压低 OOD 值）
- 第二项：最大化数据集中 $(s,a)$ 的 Q 值（保持已知值准确）
- $\alpha$：保守程度超参数

**效果**：学到的 Q 函数对数据集中的动作给出合理估计，对 OOD 动作给出保守低估值，从而策略 $\pi(s) = \arg\max_a Q(s,a)$ 自然选择数据支撑集内的动作。

### 4.4 IQL（Implicit Q-Learning）

**论文**：Kostrikov et al., *Offline RL with Implicit Q-Learning* (2022)

IQL 的核心创新：**完全避免对 OOD 动作的查询**。

IQL 不查询 $\max_a Q(s,a)$，而是用分位数回归从数据集内的动作中估计上分位数：

$$\mathcal{L}_V = \mathbb{E}_{(s,a) \sim \mathcal{D}} \left[ L_2^\tau(Q_{\bar{\theta}}(s,a) - V_\psi(s)) \right]$$

$$\mathcal{L}_Q = \mathbb{E}_{(s,a) \sim \mathcal{D}} \left[ (r + \gamma V_\psi(s') - Q_\theta(s,a))^2 \right]$$

其中 $L_2^\tau(x) = |\tau - \mathbf{1}(x < 0)| \cdot x^2$ 是分位数回归损失，$\tau > 0.5$ 取上分位数（乐观但仍在数据集内）。

**策略提取**：用 Advantage-Weighted Regression 从数据中提取策略：

$$\mathcal{L}_\pi = -\mathbb{E}_{(s,a) \sim \mathcal{D}} \left[ \exp(\beta(Q_\theta(s,a) - V_\psi(s))) \log \pi_\theta(a|s) \right]$$

IQL 简洁高效，不需要显式约束或对抗训练，是离线 RL 的强基线。

### 4.5 TD3+BC

**论文**：Fujimoto & Gu, *A Minimalist Approach to Offline RL* (2021)

TD3+BC 在 TD3 的目标函数中加入行为克隆正则化：

$$\mathcal{L} = \mathcal{L}_{TD3} + \lambda \cdot \mathbb{E}\left[ (a - \pi_\theta(s))^2 \right]$$

极其简单——只需一行代码的修改，但在 D4RL 基准上与 CQL/IQL 性能相当。$\lambda$ 通过归一化 Q 值自适应调节。

### 4.6 Decision Transformer

**论文**：Chen et al., *Decision Transformer: Reinforcement Learning via Sequence Modeling* (2021)

Decision Transformer 将 RL 重新表述为**序列建模**问题：

- 输入序列：$(R_1, s_1, a_1, R_2, s_2, a_2, \ldots)$，其中 $R_t$ 是 return-to-go
- 用 Transformer 自回归预测动作 $a_t$
- 推理时给定目标回报 $R^*$，模型生成达到该回报的动作

**洞察**：在足够大的离线数据集上，序列建模可以隐式地学习到策略，无需显式的 Bellman 备份或 Q 函数。

**局限**：在次优数据集上表现受限（因为是从数据中"模仿"而非"优化"）。

## 5. 方法分类

| 类别 | 代表方法 | 核心思想 |
|:---|:---|:---|
| 策略约束 | BCQ, BEAR | 限制策略在数据支撑集内 |
| 值函数保守化 | CQL, IQL | 低估 OOD 动作值 |
| 行为克隆正则化 | TD3+BC, AWAC | 用 BC 约束策略 |
| 序列建模 | Decision Transformer, TT | RL → 序列预测 |
| 扩散策略 | Diffusion-QL | 用扩散模型表示多模态策略 |

## 6. 优势与局限

### 优势

1. **安全**：无需在线探索，适合高风险领域
2. **数据高效**：充分利用已有历史数据
3. **可复现**：固定数据集，消除环境随机性
4. **实用**：直接对接实际系统积累的海量日志数据

### 局限

1. **数据质量依赖**：策略质量受限于行为策略的质量和数据覆盖度
2. **OOD 问题**：分布偏移是根本性挑战
3. **无法超越数据上限**：离线 RL 能发现数据中的最优策略，但不能发现数据中不存在的策略
4. **评估困难**：如何在不与环境交互的情况下评估离线策略是开放问题

## 8. 与其他技术关系

- **与 off-policy RL 的关系**：离线 RL 是 off-policy 的极端情况。Off-policy 在线方法（DQN/SAC）在离线设定下可能因分布偏移而失败。参见 [[03_无模型预测与控制]]、[[05_深度强化学习]]。
- **与模仿学习的关系**：行为克隆是离线 RL 的退化形式（只模仿不优化）。AWAC/TD3+BC 是两者的结合。
- **与探索的关系**：离线 RL 无法探索，探索问题转化为 OOD 动作的处理。参见 [[07_探索与利用]]。
- **与 Transformer 的关系**：Decision Transformer 直接使用 Transformer 架构做序列建模。参见 [[08_Transformer与注意力机制]]。
- **与大模型的关系**：离线 RL 的思想被用于 LLM 对齐——从人类偏好数据中离线学习策略（如 DPO）。参见 [[10_强化学习与大模型]]。

## 9. 前沿发展

- **扩散策略**：用扩散模型表示多模态策略，处理离线数据中的多模态行为
- **基础模型 + 离线 RL**：用预训练视觉/语言模型提供状态表示，提升离线 RL 性能
- **离线到在线微调**：先离线预训练再在线微调，兼顾安全性和性能上限
- **离线策略评估（OPE）**：在不与环境交互的情况下准确评估策略，是离线 RL 落地的关键
- **D4RL 基准**：标准化的离线 RL 评测平台，推动方法比较和进步
- **大规模离线 RL**：从互联网规模数据中学习通用策略
