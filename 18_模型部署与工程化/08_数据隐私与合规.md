# 数据隐私与合规

## 1. 概述

大语言模型在训练和推理过程中接触大量用户数据，数据隐私保护与合规性成为不可忽视的挑战。LLM 的隐私风险与传统软件不同：模型可能记忆训练数据中的个人信息，推理请求中的敏感信息可能通过日志泄露，模型输出可能无意中生成训练数据中的隐私内容。

数据隐私与合规解决的核心问题：
- **训练数据合规**：确保训练数据的采集和使用有合法基础
- **推理隐私保护**：防止用户请求中的敏感信息泄露
- **模型记忆风险**：防止模型无意中输出训练数据中的隐私
- **法规遵从**：满足全球各地区数据保护法规要求

建立覆盖大模型全生命周期的隐私合规体系——从数据采集、预处理、训练微调到推理部署——结合法规框架和技术方法，确保模型在合法合规的前提下运行。

## 3. 核心概念

### 3.1 全球主要法规框架

| 法规 | 地区 | 核心要求 | 对 LLM 的约束 |
|:---|:---|:---|:---|
| **GDPR** | 欧盟 | 数据最小化、目的限制、72h 泄露通知 | 训练数据合法性、数据主体权利、DPIA |
| **CCPA/CPRA** | 加州 | 知情权、删除权、退出权 | 告知数据用途、提供退出机制 |
| **个人信息保护法** | 中国 | 单独同意、数据本地化、出境安全评估 | 敏感数据需单独同意、数据出境受限 |
| **AI Act** | 欧盟 | 风险分级监管 | 高风险 AI 需合规评估、可解释性 |

### 3.2 全生命周期隐私风险

- **数据采集**：无合法基础收集、超范围收集、未告知数据主体
- **模型训练**：训练数据泄露、模型记忆个人信息、微调引入新风险
- **推理服务**：请求泄露敏感信息、输出包含隐私、API 日志记录敏感数据
- **部署运维**：部署环境不安全、第三方依赖风险、模型更新泄露

### 3.3 隐私保护技术谱系

| 技术 | 原理 | 适用阶段 | 代价 |
|:---|:---|:---|:---|
| 差分隐私 | 添加数学噪声 | 训练 | 模型性能下降 |
| 联邦学习 | 数据不出本地 | 训练 | 通信开销 |
| 可信执行环境 | 硬件隔离 | 推理 | 性能开销 |
| 私有化部署 | 数据不出域 | 推理 | 硬件成本 |
| 数据脱敏 | 预处理去除敏感信息 | 全流程 | 信息损失 |

## 4. 技术原理

### 4.1 差分隐私

通过向数据或模型梯度添加精心设计的噪声，提供数学上可证明的隐私保护。$(\epsilon, \delta)$-差分隐私定义为：

$$\Pr[\mathcal{M}(D) \in S] \leq e^\epsilon \cdot \Pr[\mathcal{M}(D') \in S] + \delta$$

其中 $D$ 和 $D'$ 仅差一条记录，$\epsilon$ 越小隐私保护越强。在 LLM 训练中，Opacus 等工具通过梯度裁剪和噪声注入实现差分隐私训练。

拉普拉斯机制添加噪声 $\text{Lap}(b)$，其中 $b = \Delta f / \epsilon$，$\Delta f$ 为函数敏感度。

### 4.2 联邦学习

数据不出本地，各客户端在本地数据上训练模型，仅上传模型参数更新到中央服务器聚合。适用于数据分布在多个机构、隐私要求高、数据难以集中处理的场景。

**安全聚合（Secure Aggregation）**：客户端上传加密梯度，服务器无法看到单个客户端的更新，只能解密聚合结果。

### 4.3 可信执行环境（TEE）

硬件级安全隔离区域，保护数据和代码不被外部访问：

- **Intel SGX**：软件防护扩展，enclave 级隔离
- **AMD SEV**：安全加密虚拟化
- **AWS Nitro Enclaves**：云端 TEE 实例
- **NVIDIA Confidential Computing**：GPU 级可信执行环境

适用于敏感数据推理和隐私计算场景，代价是性能开销和开发复杂度。

### 4.4 私有化部署

将模型部署在客户本地环境，数据不出边界，完全控制数据和模型。通过 Docker 容器化实现标准化部署，满足数据本地化要求。

### 4.5 数据脱敏

| 方法 | 示例 | 适用场景 |
|:---|:---|:---|
| 数据掩码 | `user***@example.com` | 日志、展示 |
| 数据泛化 | 年龄 25 → 20-30 范围 | 统计分析 |
| 数据扰动 | 添加随机噪声 | 数据发布 |
| 日志匿名化 | 移除或加密敏感字段 | 运维日志 |
| Token 化 | 敏感数据替换为 Token | API 传输 |

## 5. 关键方法与模型

### 5.1 模型记忆与提取攻击

研究表明 LLM 会记忆训练数据中的内容，攻击者可通过特定 Prompt 诱导模型输出训练数据。缓解措施：
- 差分隐私训练
- 训练数据去重和清洗
- 输出过滤和审查

### 5.2 知识蒸馏隐私保护

用大模型（教师）生成合成数据训练小模型（学生），学生模型不直接接触原始训练数据，降低隐私泄露风险。

### 5.3 同态加密推理

允许在加密数据上直接执行推理计算，服务器无需解密即可返回加密结果。当前主要受限于计算开销，适用于高安全要求的简单模型推理。

## 6. 优势与局限

### 6.1 隐私保护的价值

- **合规**：满足法规要求，避免法律风险和罚款
- **信任**：增强用户对 AI 服务的信任
- **安全**：降低数据泄露风险
- **竞争力**：隐私保护能力成为商业竞争优势

### 6.2 隐私保护的代价

| 技术 | 代价 | 程度 |
|:---|:---|:---|
| 差分隐私 | 模型性能下降 | 5-15% |
| 联邦学习 | 通信开销、客户端异质性 | 中等 |
| TEE | 性能开销 | 10-30% |
| 私有化部署 | 硬件成本、运维复杂度 | 高 |
| 同态加密 | 计算开销巨大 | 100-1000x |

## 7. 应用场景

| 场景 | 隐私需求 | 推荐方案 |
|:---|:---|:---|
| 医疗 AI | 患者数据保护，HIPAA 合规 | 私有化部署 + 数据脱敏 |
| 金融 AI | 交易数据保护，数据本地化 | 私有化部署 + TEE |
| 企业内部 LLM | 员工交互数据不出企业边界 | 私有化部署 |
| 跨境 AI 服务 | 多地区合规 | 数据本地化 + 分区部署 |
| 多方协作训练 | 数据不可共享 | 联邦学习 + 安全聚合 |
| 高安全推理 | 推理数据不可见 | 同态加密 / TEE |
