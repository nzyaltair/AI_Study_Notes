# 结构化输出

## 1. 概述

### 1.1 目标定位
本文档聚焦大语言模型（LLM）在实际应用中如何**可靠、高效地生成结构化输出**（如 JSON、XML、Protobuf、数据库记录、函数调用参数等），解决"自由文本 → 可编程数据"的关键工程挑战。结构化输出是连接LLM与外部系统的桥梁，使LLM能够与API、数据库、函数调用等编程组件无缝集成。

### 1.2 核心挑战
- **非确定性输出**：LLM天然倾向于生成自由文本，难以保证严格的结构化格式
- **格式多样性**：不同应用场景需要不同的结构化格式（JSON、XML、Protobuf等）
- **嵌套结构复杂性**：深层嵌套结构容易出现字段缺失、类型错误
- **性能与准确性权衡**：严格的格式约束可能影响生成速度和内容质量
- **跨模型兼容性**：不同模型对结构化输出的支持程度差异较大

## 2. 结构化输出方法体系

### 2.1 方法分类框架

| 方法类型 | 实现层面 | 核心机制 | 代表工具/技术 | 适用场景 |
|----------|----------|----------|---------------|----------|
| **提示工程** | 输入层 | 通过提示词引导LLM生成结构化输出 | Schema-guided prompting | 快速原型、简单结构 |
| **输出格式约束** | 输出层 | 定义严格的格式规则限制输出 | JSON Schema、Grammar-based Decoding | 复杂结构、高可靠性要求 |
| **解码层控制** | 解码层 | 在生成过程中实时控制token选择 | Constrained Decoding、Guided Generation | 高性能要求、严格格式约束 |
| **后处理与校验** | 后处理 | 验证和修正生成的结构化输出 | 正则过滤、LLM-as-a-Judge | 兼容性要求高、多模型场景 |
| **工具集成** | 框架层 | 专用工具链简化结构化输出实现 | Outlines、Guidance、lm-format-enforcer | 生产环境、复杂应用 |

## 3. 主流结构化输出方法

### 3.1 提示工程

#### 3.1.1 强约束Prompt
通过明确的指令和格式示例引导LLM生成结构化输出。

**示例**：
```python
prompt = """请将以下产品信息转换为JSON格式：
产品名称：智能手表
价格：1999元
品牌：TechX
上市时间：2026-01-15

要求：
1. 严格按照JSON格式输出，不要添加任何解释
2. 字段名使用中文
3. 价格字段转换为数字类型
4. 上市时间格式为YYYY-MM-DD

输出："""
```

**优缺点**：
- ✅ 实现简单，无需额外工具
- ✅ 兼容所有LLM
- ❌ 准确性依赖模型能力
- ❌ 复杂结构容易出错
- ❌ 缺乏严格的格式保证

#### 3.1.2 Schema-guided Prompting
在提示中嵌入JSON Schema或类似的结构定义，指导LLM生成符合Schema的输出。

**示例**：
```python
import json

schema = {
    "type": "object",
    "properties": {
        "产品名称": {"type": "string"},
        "价格": {"type": "number"},
        "品牌": {"type": "string"},
        "上市时间": {"type": "string", "format": "date"}
    },
    "required": ["产品名称", "价格", "品牌", "上市时间"]
}

prompt = f"""请将以下产品信息转换为符合JSON Schema的JSON格式：
产品名称：智能手表
价格：1999元
品牌：TechX
上市时间：2026-01-15

JSON Schema：
{json.dumps(schema, indent=2)}

要求：严格按照Schema输出，不要添加任何额外内容。
输出："""
```

### 3.2 输出格式约束

#### 3.2.1 JSON Schema约束
利用模型内置的JSON Schema支持，直接传递Schema给模型，确保输出符合格式要求。

**OpenAI API示例**：
```python
from openai import OpenAI
import json

client = OpenAI()

response = client.chat.completions.create(
    model="gpt-4-turbo",
    messages=[
        {"role": "system", "content": "你是一个专业的结构化数据生成器。"},
        {"role": "user", "content": "请帮我查询北京2026-01-26的天气"}
    ],
    functions=[{
        "name": "get_weather",
        "description": "获取指定城市的天气信息",
        "parameters": {
            "type": "object",
            "properties": {
                "city": {"type": "string", "description": "城市名称"},
                "date": {"type": "string", "description": "日期，格式为YYYY-MM-DD"}
            },
            "required": ["city", "date"]
        }
    }],
    function_call={"name": "get_weather"}
)

function_args = json.loads(response.choices[0].message.function_call.arguments)
print(function_args)
```

#### 3.2.2 Grammar-based Decoding
通过定义形式文法（如CFG、BNF）来约束模型生成的token序列，确保输出符合特定语法规则。

**示例（使用Llama.cpp的grammar功能）**：
```bash
# 定义JSON语法规则
cat > json.grammar << EOF
root ::= object

object ::= "{" ws (pair (ws "," ws pair)*)? ws "}"
pair ::= string ws ":" ws value
array ::= "[" ws (value (ws "," ws value)*)? ws "]"
value ::= string | number | object | array | "true" | "false" | "null"
string ::= '"' ([^"\\] | '\\' ["\\/bfnrt] | '\\u' [0-9a-fA-F]{4})* '"'
number ::= [+-]? ("0" | [1-9] [0-9]*) ("." [0-9]*)? ([eE] [+-]? [0-9]+)?
ws ::= [ \t\r\n]*
EOF

# 使用grammar进行解码
./main -m model.gguf -p "请将以下产品信息转换为JSON：智能手表，价格1999元，品牌TechX，2026-01-15上市" --grammar-file json.grammar
```

### 3.3 解码层控制

#### 3.3.1 Constrained Decoding
在解码过程中实时过滤不符合格式要求的token，只允许生成符合约束的token序列。

**示例（概念性代码）**：
```python
from transformers import AutoTokenizer, AutoModelForCausalLM

# 加载模型和分词器
tokenizer = AutoTokenizer.from_pretrained("meta-llama/Llama-3-8b-hf")
model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-3-8b-hf")

# 简单的JSON约束函数
def is_valid_json_so_far(text):
    # 检查当前文本是否符合JSON语法约束
    # 实际实现需要更复杂的逻辑
    return True

def generate_structured_output(prompt, max_new_tokens=128):
    inputs = tokenizer(prompt, return_tensors="pt")
    generated = inputs["input_ids"]
    
    for _ in range(max_new_tokens):
        # 生成下一个token的logits
        outputs = model(generated)
        logits = outputs.logits[:, -1, :]
        
        # 获取所有可能的下一个token
        all_tokens = logits.argsort(dim=-1, descending=True)[0]
        
        # 找到第一个能产生有效JSON的token
        for token in all_tokens:
            next_text = tokenizer.decode(generated[0].tolist() + [token.item()])
            if is_valid_json_so_far(next_text):
                generated = tokenizer(next_text, return_tensors="pt")["input_ids"]
                break
        
        # 检查是否生成了完整的JSON
        if "}" in next_text and next_text.rstrip().endswith("}"):
            break
    
    return next_text
```

### 3.4 后处理与校验

#### 3.4.1 正则过滤与解析
使用正则表达式和解析库验证和修正生成的结构化输出。

**示例**：
```python
import json
import re

def validate_and_fix_json(text):
    # 1. 提取JSON部分
    json_match = re.search(r'\{[\s\S]*\}', text)
    if not json_match:
        return None
    
    json_str = json_match.group()
    
    # 2. 尝试解析JSON
    try:
        return json.loads(json_str)
    except json.JSONDecodeError as e:
        # 3. 简单修复示例：添加缺失的引号
        fixed_json = re.sub(r'(\w+):', r'"\1":', json_str)
        try:
            return json.loads(fixed_json)
        except json.JSONDecodeError:
            return None

# 使用示例
generated_text = "产品信息：{产品名称: 智能手表, 价格: 1999, 品牌: TechX}"
valid_json = validate_and_fix_json(generated_text)
print(valid_json)
```

#### 3.4.2 LLM-as-a-Judge
使用另一个LLM验证和修正生成的结构化输出。

**示例**：
```python
from openai import OpenAI
import json

client = OpenAI()

def fix_json_with_llm(generated_text, schema):
    prompt = f"""请修复以下JSON，使其符合给定的JSON Schema：

生成的JSON：
{generated_text}

JSON Schema：
{json.dumps(schema, indent=2)}

要求：
1. 严格按照Schema修复JSON
2. 只输出修复后的JSON，不要添加任何解释
3. 保持原始数据的准确性

修复后的JSON："""
    
    response = client.chat.completions.create(
        model="gpt-3.5-turbo",
        messages=[{"role": "user", "content": prompt}],
        temperature=0
    )
    
    return response.choices[0].message.content

# 使用示例
schema = {
    "type": "object",
    "properties": {
        "产品名称": {"type": "string"},
        "价格": {"type": "number"}
    }
}
generated_json = '{"产品名称": "智能手表", "价格": "1999"}'
fixed_json = fix_json_with_llm(generated_json, schema)
print(fixed_json)
```

### 3.5 工具集成

#### 3.5.1 Outlines
一个专注于结构化输出的LLM框架，提供强大的格式约束能力。

**示例**：
```python
import outlines
import json

# 定义结构化输出模型
model = outlines.models.transformers("meta-llama/Llama-3-8b-hf")

# 定义JSON Schema
product_schema = {
    "type": "object",
    "properties": {
        "产品名称": {"type": "string"},
        "价格": {"type": "number"},
        "品牌": {"type": "string"},
        "上市时间": {"type": "string", "format": "date"}
    },
    "required": ["产品名称", "价格", "品牌", "上市时间"]
}

# 创建结构化生成函数
structured_generate = outlines.generate.json(model, product_schema)

# 生成结构化输出
prompt = "请将以下产品信息转换为JSON：智能手表，价格1999元，品牌TechX，2026-01-15上市"
result = structured_generate(prompt)
print(result)
```

#### 3.5.2 Guidance
Microsoft开发的结构化输出框架，支持复杂的条件生成和格式约束。

**示例**：
```python
import guidance

# 加载模型
guidance.llm = guidance.llms.OpenAI("gpt-4-turbo")

# 定义结构化生成模板
product_template = guidance("""
{{#system}}
你是一个专业的结构化数据生成器。
{{/system}}

{{#user}}
请将以下产品信息转换为JSON：{{product_info}}
{{/user}}

{{#assistant}}
{{#block format="json"}}
{
  "产品名称": "{{gen 'product_name'}}",
  "价格": {{gen 'price' pattern='\\d+'}},
  "品牌": "{{gen 'brand'}}",
  "上市时间": "{{gen 'launch_date' pattern='\\d{4}-\\d{2}-\\d{2}'}}"
}
{{/block}}
{{/assistant}}
""")

# 生成结构化输出
result = product_template(product_info="智能手表，价格1999元，品牌TechX，2026-01-15上市")
print(result)
```

## 4. 方法对比与选择指南

### 4.1 方法对比矩阵

| 方法 | 准确性 | 延迟 | 灵活性 | 模型兼容性 | 实现复杂度 | 适用场景 |
|------|--------|------|--------|------------|------------|----------|
| 强约束Prompt | 中 | 低 | 高 | 所有模型 | 低 | 快速原型、简单结构 |
| Schema-guided Prompting | 中高 | 低 | 高 | 所有模型 | 中 | 中等复杂度结构 |
| JSON Schema约束 | 高 | 中 | 中 | 支持的模型 | 低 | 依赖模型内置支持的场景 |
| Grammar-based Decoding | 高 | 中 | 低 | 支持的模型 | 高 | 严格格式要求的场景 |
| Constrained Decoding | 高 | 高 | 中 | 支持的模型 | 高 | 高性能要求的场景 |
| 后处理与校验 | 中高 | 低 | 高 | 所有模型 | 中 | 多模型兼容场景 |
| Outlines | 高 | 中 | 中 | 支持的模型 | 中 | 生产环境、复杂结构 |
| Guidance | 高 | 中 | 高 | 支持的模型 | 中 | 复杂条件生成场景 |

### 4.2 选择决策树
1. **如果是快速原型开发**：强约束Prompt → Schema-guided Prompting
2. **如果模型支持内置JSON Schema**：JSON Schema约束
3. **如果需要严格格式保证**：Grammar-based Decoding → Constrained Decoding
4. **如果需要高性能**：vLLM Logits Processors → Constrained Decoding
5. **如果是生产环境**：Outlines → Guidance → lm-format-enforcer
6. **如果需要多模型兼容**：后处理与校验 → 强约束Prompt

## 5. 常见失败模式与应对策略

### 5.1 字段缺失
- **症状**：生成的结构化输出缺少必填字段
- **应对策略**：
  - 使用严格的Schema约束
  - 添加字段完整性校验
  - 实现自动重试机制
  - 在Prompt中明确要求所有字段

### 5.2 类型错误
- **症状**：字段类型不符合要求（如字符串类型的价格字段）
- **应对策略**：
  - 使用JSON Schema的类型约束
  - 添加类型转换和验证
  - 使用Grammar-based Decoding确保数字格式正确
  - 在Prompt中明确指定字段类型

### 5.3 嵌套结构错乱
- **症状**：深层嵌套结构出现括号不匹配、层级错误
- **应对策略**：
  - 使用Grammar-based Decoding
  - 实现嵌套结构验证
  - 将复杂结构拆分为多个简单步骤
  - 使用专用结构化输出工具

### 5.4 格式污染
- **症状**：结构化输出中混入了额外的解释或注释
- **应对策略**：
  - 使用严格的输出格式约束
  - 添加后处理过滤
  - 在Prompt中明确要求"只输出JSON，不要添加任何解释"
  - 使用模型的stop sequences

## 6. 真实应用场景最佳实践

### 6.1 API调用

**场景**：LLM生成API调用参数

**最佳实践**：
- 使用JSON Schema定义API参数结构
- 结合工具调用机制（如OpenAI的Function Calling）
- 添加参数验证和类型检查
- 实现API调用失败后的重试逻辑

**示例**：
```python
from openai import OpenAI
import requests
import json

client = OpenAI()

def call_api_with_llm(prompt, api_schema):
    # 生成API调用参数
    response = client.chat.completions.create(
        model="gpt-4-turbo",
        messages=[{"role": "user", "content": prompt}],
        functions=[{
            "name": "call_api",
            "description": "调用外部API",
            "parameters": api_schema
        }],
        function_call={"name": "call_api"}
    )
    
    # 提取并验证参数
    function_args = json.loads(response.choices[0].message.function_call.arguments)
    
    # 调用API
    api_response = requests.get("https://api.example.com/endpoint", params=function_args)
    return api_response.json()
```

## 7. 新兴方向与前瞻性探讨

### 7.1 原生支持结构化输出的模型训练

**Structured LMs**：专门训练用于生成结构化输出的LLM，如Google的Structured LM、Microsoft的CodeLLaMA等。这些模型在预训练阶段就融入了结构化输出能力，能够更高效、准确地生成各种结构化格式。

### 7.2 编译器式解码

借鉴编译器的设计思想，将结构化输出生成分解为词法分析、语法分析和语义分析等阶段，实现更严格、高效的格式控制。

### 7.3 与类型系统的深度集成

将LLM的结构化输出与静态类型系统（如TypeScript、Python类型注解）深度集成，实现编译时类型检查和智能提示，提高开发效率和代码安全性。

### 7.4 多模态结构化输出

扩展结构化输出能力到多模态场景，如从图像或视频中提取结构化信息，生成包含文本、图像描述和元数据的复合结构化输出。

## 10. 总结与建议

结构化输出是LLM应用工程中的关键技术，直接影响LLM与外部系统的集成效果和可靠性。选择合适的结构化输出方法需要考虑准确性、性能、灵活性和模型兼容性等多个因素。

对于AI开发人员和应用工程师，建议：

1. **从简单到复杂**：先使用提示工程方法快速验证，再根据需求升级到更严格的格式约束

3. **实现完善的验证机制**：无论使用哪种方法，都要添加后处理和验证步骤
4. **持续关注最新进展**：结构化输出领域发展迅速，定期评估和采用新技术
5. **结合具体场景选择**：根据应用场景的具体需求选择最合适的结构化输出方法

通过合理运用结构化输出技术，开发人员可以显著提高LLM应用的可靠性和可维护性，实现"自由文本 → 可编程数据"的高效转换，为构建复杂的LLM应用系统奠定坚实基础。
