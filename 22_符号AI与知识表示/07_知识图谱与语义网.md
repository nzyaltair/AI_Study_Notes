# 知识图谱与语义网

## 一句话理解

将实体、关系和属性组织为大规模图结构的知识库，使机器能够以结构化方式存储、查询和推理世界知识。

## 背景与发展

知识图谱是符号AI在工业界最成功的应用之一。从语义网的学术愿景到Google Knowledge Graph的大规模工业应用，知识图谱已经成为搜索引擎、推荐系统和智能问答的基础设施。

| 年代 | 里程碑 | 作者/机构 | 意义 |
|:---|:---|:---|:---|
| 2001 | 语义网愿景 | Berners-Lee | Web上的知识表示 |
| 2004 | RDF/OWL标准 | W3C | 知识表示标准化 |
| 2006 | Linked Data | Berners-Lee | 开放数据互联 |
| 2007 | DBpedia | Leipzig大学 | Wikipedia结构化 |
| 2010 | Freebase | Metaweb | 大规模实体库 |
| 2012 | Knowledge Graph | Google | 工业级知识图谱 |
| 2012 | Wikidata | Wikimedia | 众包知识图谱 |
| 2013 | TransE | Bordes等 | 知识嵌入表示学习 |
| 2015 | WordNet/ConceptNet | 多家 | 常识知识库 |
| 2020s | LLM+知识图谱 | 多家 | 神经符号融合 |

## 核心概念

### RDF（Resource Description Framework）

知识图谱的基本数据模型——**三元组**（Triple）：

$$(\text{subject}, \text{predicate}, \text{object})$$

例如：
```
(北京, 是...的首都, 中国)
(爱因斯坦, 提出, 相对论)
(相对论, 属于, 物理学)
```

RDF将知识表示为有向标记图：节点是实体/字面量，边是关系/属性。

### OWL（Web Ontology Language）

基于描述逻辑的本体语言，在RDF之上增加：
- **类层次**（subClassOf）：概念间的包含关系
- **属性约束**（cardinality, allValuesFrom等）
- **推理规则**（equivalentClass, disjointWith等）

OWL支持自动推理——从显式声明的事实推出隐含知识。

### SPARQL

RDF数据的查询语言，类似SQL但面向图结构：

```sparql
SELECT ?person ?institution
WHERE {
  ?person rdf:type :Scientist .
  ?person :workAt ?institution .
  ?institution :locatedIn :中国 .
}
```

### 知识图谱 vs 数据库

| 维度 | 知识图谱 | 关系数据库 |
|:---|:---|:---|
| 数据模型 | 图（RDF三元组） | 表（关系） |
| Schema | 灵活/可演化 | 固定/预定义 |
| 查询 | SPARQL（图遍历） | SQL（连接） |
| 推理 | 支持自动推理 | 无推理 |
| 适合 | 异构知识整合 | 结构化事务处理 |

### 知识嵌入（Knowledge Embedding）

将实体和关系映射到低维向量空间，使推理可通过向量运算完成：

| 方法 | 评分函数 | 特点 |
|:---|:---|:---|
| TransE | $\|h + r - t\|$ | 简单高效，适合一对一关系 |
| TransH | 投影到关系超平面 | 处理一对多/多对多 |
| TransR | 分离实体/关系空间 | 表达力更强 |
| RotatE | $h \circ r \approx t$（旋转） | 处理对称/反对称关系 |
| ComplEx | 复数空间 | 处理非对称关系 |

知识嵌入是符号方法与神经方法的桥梁——参见 [[20_图神经网络]]。

## 工作原理

### 知识图谱构建流程

```
数据源 → 信息抽取 → 知识融合 → 知识推理 → 质量评估 → 知识存储
```

1. **信息抽取**：从文本中提取实体（NER）、关系（RE）、事件
2. **知识融合**：实体对齐（同一实体的不同表示合并）、消歧
3. **知识推理**：基于本体规则或嵌入方法推理隐含关系
4. **知识存储**：图数据库（Neo4j）、RDF三元组库

### 实体链接（Entity Linking）

将文本中提到的实体链接到知识图谱中的对应条目：

```
文本: "爱因斯坦在普林斯顿去世"
        ↓ 实体识别
    "爱因斯坦" → Einstein (KG: Q937)
    "普林斯顿" → Princeton (KG: Q1384)
```

### 知识推理

从已有知识推出隐含知识：

- **基于规则**：使用本体公理和逻辑规则推理
  - 如：$A$ 是 $B$ 的子类，$x$ 是 $A$ 的实例 → $x$ 是 $B$ 的实例
- **基于嵌入**：通过向量运算预测新关系
  - 如：$\text{score}(h, r, t) = \|h + r - t\|$，分数低则三元组可信
- **基于图神经网络**：用GNN在知识图谱上传播信息

## 应用场景

| 领域 | 应用 |
|:---|:---|
| 搜索引擎 | Google Knowledge Graph增强搜索结果 |

| 电商 | 商品知识图谱增强搜索和推荐 |

## 优势与不足

### 优势
- 结构化知识存储，支持精确查询和推理
- 可与NLP、推荐系统等深度整合
- 人类可理解，支持可解释AI
- 开放世界假设——不要求完全知识

### 不足
- 构建成本高——需要大规模信息抽取和人工验证
- 知识更新滞后——无法实时反映世界变化
- 常识和隐含知识难以覆盖
- 与神经方法的融合仍在发展中
