# 大模型蒸馏

## 1. 概述

大模型蒸馏（LLM Distillation）是将大语言模型（如 GPT-4、Claude）的能力迁移到更小的开源模型（如 LLaMA、Mistral）的技术。与传统 BERT 蒸馏不同，大模型蒸馏的核心挑战在于：教师模型通常是**闭源 API**，无法访问其参数或中间层特征，只能通过输入-输出对进行"黑盒蒸馏"。

大模型蒸馏已成为缩小开源与闭源模型差距的关键路径。Alpaca (2023) 证明仅用 52K 条 GPT-3.5 生成的指令数据微调 LLaMA，即可达到接近 GPT-3.5 的性能。Orca (2023) 进一步通过渐进式复杂任务蒸馏和推理轨迹迁移，使 13B 模型超越 70B 模型。

## 3. 核心概念

### 3.1 黑盒蒸馏 vs 白盒蒸馏

| 类型 | 教师访问方式 | 迁移内容 | 代表方法 |
|:---|:---|:---|:---|
| **黑盒蒸馏** | 仅 API 输入-输出 | 输出文本/指令数据 | Alpaca, Vicuna, Orca |
| **白盒蒸馏** | 可访问模型参数/中间层 | logits、特征、注意力 | MiniLLM, GKD |

大模型蒸馏以黑盒蒸馏为主，因为教师模型（GPT-4）通常不暴露参数。

### 3.2 数据蒸馏（Data Distillation）

大模型蒸馏的主流范式：从"蒸馏模型输出"转向"蒸馏训练数据"。

**流程**：
1. 用教师模型（GPT-4）生成高质量指令-回答对
2. 过滤低质量数据
3. 用过滤后的数据微调学生模型（SFT）

**优势**：实施简单，不需要访问教师内部状态。

### 3.3 推理能力蒸馏

Orca 系列的关键创新：不仅蒸馏答案，还蒸馏**推理过程**。

- 教师生成 Chain-of-Thought 推理链
- 学生学习模仿推理过程
- 渐进式增加任务复杂度

### 3.4 前向 KL vs 反向 KL

| 特性 | 前向 KL $D_{KL}(p^T \| p^S)$ | 反向 KL $D_{KL}(p^S \| p^T)$ |
|:---|:---|:---|
| 行为 | 模式覆盖（mode-covering） | 模式寻求（mode-seeking） |
| 分布 | 平滑、平均化 | 尖锐、集中 |
| 适合任务 | 分类 | 生成 |
| 计算 | 可直接计算 | 需策略梯度近似 |
| 代表 | Hinton KD, Alpaca | MiniLLM, GKD |

## 4. 技术原理

### 4.1 Alpaca——指令跟随蒸馏

Taori et al. (2023) 的极简方案：

**数据生成**：
1. 用 175 条种子指令
2. 用 GPT-3.5 (text-davinci-003) 自我指令生成 52K 条指令-回答对
3. 花费约 $500 API 费用

**训练**：
- 学生模型：LLaMA-7B
- 方法：监督微调（SFT）
- 损失：标准交叉熵（教师输出作为目标）

$$\mathcal{L}_{SFT} = -\sum_t \log p^S(y_t^{teacher} | y_{<t}^{teacher}, x)$$

**效果**：Alpaca-7B 在多数指令跟随任务上接近 GPT-3.5。

### 4.2 Vicuna——对话能力蒸馏

Chiang et al. (2023) 聚焦对话能力：

**数据**：从 ShareGPT 收集 70K 真实用户-ChatGPT 对话
**训练**：LLaMA-13B + 对话微调
**效果**：在 90% 的测试案例中接近 ChatGPT

### 4.3 Orca——渐进式推理蒸馏

Mukherjee et al. (2023) 的核心创新：

**渐进式复杂任务**：
1. 简单任务 → 中等任务 → 复杂任务
2. 教师提供详细的推理过程和解释
3. 学生逐步学习推理能力

**多样化信号**：
- 系统指令：提示教师从不同角度回答
- 推理轨迹：教师展示完整的 CoT
- 解释：教师解释为什么给出这个答案

**效果**：13B Orca 在多项基准上超越 70B Vicuna，达到 GPT-3.5 水平。

### 4.4 Orca-2——推理策略蒸馏

Orca-2 的关键洞察：教小模型**何时使用何种推理策略**：

- 直接回答
- Chain-of-Thought
- 逐步推理
- 先列举再总结

**训练数据**：对同一问题，用不同推理策略生成回答，让学生学习选择最优策略。

**效果**：13B Orca-2 在零样本推理任务上超越 70B 模型。

### 4.5 WizardLM——Evol-Instruct

Xu et al. (2023) 的自动数据进化方法：

**Evol-Instruction**：
1. 从种子指令出发
2. 通过 LLM 自动演化出更复杂的指令（深度进化）
3. 或生成不同类型的指令（广度进化）
4. 过滤低质量结果

**优势**：自动生成大量高质量、高难度指令数据，覆盖面广。

### 4.6 MiniLLM——反向 KL 蒸馏

Gu et al. (2024) 针对白盒 LLM 蒸馏：

$$\mathcal{L}_{MiniLLM} = D_{KL}(p^S_\theta \| p^T)$$

反向 KL 使学生分布更尖锐，避免前向 KL 的模式平均问题。由于反向 KL 需要计算 $\mathbb{E}_{p^S}[\log p^T / p^S]$，使用策略梯度近似：

$$\nabla_\theta \mathcal{L} \approx -\mathbb{E}_{y \sim p^S}\left[(\log p^T(y) - \log p^S(y)) \nabla_\theta \log p^S(y)\right]$$

加上 length normalization 和 single-step 采样优化训练稳定性。

### 4.7 GKD——在线策略蒸馏

Agarwal et al. (2024) 提出在线策略蒸馏：

- 学生在自己的分布上采样
- 教师提供该分布下的指导
- 比离线蒸馏（固定数据集）更有效

$$\mathcal{L}_{GKD} = D_{KL}(p^S_\theta(\cdot | y_{<t}, x) \| p^T(\cdot | y_{<t}, x))$$

## 5. 方法对比

| 方法 | 教师 | 数据量 | 核心创新 | 学生规模 | 效果 |
|:---|:---|:---|:---|:---|:---|
| Alpaca | GPT-3.5 | 52K | 指令数据蒸馏 | 7B | 接近 GPT-3.5 |
| Vicuna | ChatGPT | 70K | 真实对话蒸馏 | 13B | 90% ChatGPT |
| Orca | GPT-4 | 大量 | 渐进推理蒸馏 | 13B | 超 GPT-3.5 |
| Orca-2 | GPT-4 | 大量 | 推理策略蒸馏 | 13B | 超 70B 模型 |
| WizardLM | ChatGPT | 250K | Evol-Instruct | 7B-70B | 高难度任务强 |
| MiniLLM | 大 LLM | — | 反向 KL | 各规模 | 生成质量更好 |
| GKD | 大 LLM | — | 在线策略蒸馏 | 各规模 | 超越离线蒸馏 |

## 6. 优势与局限

### 优势
- **成本低**：Alpaca 仅花 $500 API 费用即可获得接近 GPT-3.5 的模型
- **开源可部署**：蒸馏后的模型完全开源，可本地部署
- **能力迁移显著**：13B 模型可达到甚至超越 70B 模型
- **可定制**：针对特定领域（数学、代码）定向蒸馏

### 局限
- **教师依赖**：学生无法超越教师，且继承教师的偏见和幻觉
- **API 成本**：大规模蒸馏需大量 API 调用
- **版权与合规**：从闭源模型蒸馏可能违反使用条款
- **黑盒限制**：无法利用教师中间层特征
- **数据质量瓶颈**：蒸馏效果高度依赖生成数据的质量

## 7. 应用场景

- **开源生态**：为开源社区提供高质量小模型
- **私有部署**：企业内部部署蒸馏后的模型，避免数据外泄
- **边缘 AI**：在小设备上运行 LLM
- **垂直领域**：数学推理（WizardMath）、代码生成（CodeAlpaca）
- **成本优化**：用小模型替代大模型 API 调用

## 9. 前沿发展

- **蒸馏+RL 结合**：先蒸馏获得初始推理能力，再用 RL 进一步提升（DeepSeek-R1 路线）
- **推理时蒸馏**：在推理时动态从大模型获取指导
- **多轮对话蒸馏**：蒸馏多轮对话中的上下文推理能力
- **多模态蒸馏**：从 GPT-4V 蒸馏到开源 VLM
- **Agent 能力蒸馏**：蒸馏工具调用、规划和多步推理能力
- **自我蒸馏**：模型利用自身生成数据进行自我提升（Self-Play, RLAIF）
- **数据质量优化**：从数量转向质量的蒸馏数据筛选
