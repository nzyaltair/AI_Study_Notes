# 数据处理与工程化

## 背景

在AI开发中，数据质量直接决定模型性能。原始数据通常杂乱、不完整、格式不一，需要经过采集、清洗、转换、特征工程等步骤才能成为高质量的模型输入。数据工程化是将这些步骤自动化、可复现、可扩展的过程。

## 核心思想

数据处理工程化的核心是构建自动化数据管道（Pipeline），将数据从采集到模型输入的整个过程标准化、可监控、可复现。

## 技术原理

### 数据生命周期

1. **数据采集**：从API、数据库、文件、爬虫等来源收集
2. **数据清洗**：处理缺失值、异常值、重复数据
3. **数据转换**：格式标准化、类型转换、编码
4. **特征工程**：提取、选择、构建特征
5. **模型训练**：使用处理后的数据训练
6. **模型部署与监控**：持续监控数据质量

### 数据清洗

- **缺失值处理**：
  - 删除：`df.dropna()`（缺失率低时）
  - 填充：均值/中位数/众数/插值（`df.fillna()`）
  - 预测：用其他特征预测缺失值
- **异常值检测**：
  - 统计方法：3σ准则、IQR方法
  - 距离方法：KNN、LOF
  - 孤立森林
- **数据去重**：`df.drop_duplicates()`

### 特征工程

- **数值特征**：
  - 标准化：$x' = \frac{x - \mu}{\sigma}$
  - 归一化：$x' = \frac{x - \min}{\max - \min}$
  - 对数变换：$x' = \log(x + 1)$（处理偏态分布）
- **类别特征**：
  - One-Hot编码：`pd.get_dummies()`
  - Label Encoding：`sklearn.preprocessing.LabelEncoder`
  - 目标编码：用目标变量均值替换类别
- **时间特征**：提取年月日、周期性、趋势
- **文本特征**：TF-IDF、Word2Vec、BERT Embedding
- **特征选择**：
  - 过滤法：相关系数、互信息、卡方检验
  - 包装法：递归特征消除（RFE）
  - 嵌入法：L1正则化、树模型特征重要性

### 数据管道

```python
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import StandardScaler, OneHotEncoder

# 数值特征和类别特征分别处理
numeric_transformer = Pipeline([
    ('scaler', StandardScaler())
])
categorical_transformer = Pipeline([
    ('encoder', OneHotEncoder(handle_unknown='ignore'))
])

preprocessor = ColumnTransformer([
    ('num', numeric_transformer, numeric_features),
    ('cat', categorical_transformer, categorical_features)
])

full_pipeline = Pipeline([
    ('preprocessor', preprocessor),
    ('model', model)
])
```

### 数据存储

- **关系型数据库**：MySQL、PostgreSQL（结构化数据）
- **NoSQL**：MongoDB（半结构化）、Redis（缓存）
- **数据仓库**：ClickHouse、Snowflake（分析查询）
- **对象存储**：S3、HDFS（大规模文件）
- **向量数据库**：Pinecone、Milvus（[[向量数据库|嵌入向量]]）

### 数据版本管理

- **DVC**：数据版本控制工具，类似Git但用于大数据
- **Delta Lake**：带ACID事务的数据湖
- **数据血缘**：追踪数据来源和变换历史
