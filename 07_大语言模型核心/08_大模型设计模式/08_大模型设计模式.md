# 大模型设计模式

## 背景与发展

围绕LLM的架构设计、训练方法、推理优化和应用开发形成了一系列可复用的设计模式。这些模式是工程实践中的最佳实践总结，能够帮助开发者快速构建高效、可靠的LLM系统。

## 核心思想

大模型设计模式涵盖架构、训练、推理、应用四个层面，每一层都有经过验证的最佳实践组合，形成从模型设计到工程落地的完整方法论。

## 技术原理

### 架构设计模式

| 模式 | 特点 | 代表模型 |
|------|------|---------|
| Decoder-only | 自回归生成，Scaling Laws友好 | GPT、LLaMA、Qwen |
| Encoder-only | 双向注意力，适合理解任务 | BERT、RoBERTa |
| Encoder-Decoder | 编码器理解+解码器生成 | T5、BART、Whisper |
| MoE稀疏架构 | 总参数大，激活参数少 | Mixtral、DeepSeek-MoE |

**现代LLM标配技术栈**：RoPE + SwiGLU + RMSNorm + Pre-Norm + GQA

| 组件 | 演进 | 代表 |
|------|------|------|
| 归一化 | Post-LN → Pre-LN → RMSNorm | LLaMA |
| 激活函数 | ReLU → GELU → SwiGLU | LLaMA |
| 注意力 | MHA → MQA → GQA | LLaMA 2 70B |

### 训练设计模式

**预训练任务**：自回归语言建模（GPT）、掩码语言建模（BERT）、去噪自编码（T5）

**对齐训练三阶段**：

| 模式 | 方法 | 特点 |
|------|------|------|
| SFT | 监督微调 | 指令跟随能力 |
| RLHF | 强化学习+人类反馈 | 对齐人类偏好 |
| DPO | 直接偏好优化 | 无需奖励模型 |
| RLAIF | AI反馈强化学习 | 使用AI代替人类标注 |

**参数高效微调**：

| 模式 | 原理 | 参数量 |
|------|------|--------|
| LoRA | 低秩矩阵分解 | ~0.1% |
| QLoRA | 量化+LoRA | ~0.1%（更少显存） |
| Adapter | 插入小型适配器层 | ~1% |
| Prefix Tuning | 训练输入前缀 | ~0.01% |

**数据策略**：课程学习、数据混合配比、质量过滤、合成数据

### 推理设计模式

**KV-Cache**：缓存已计算的Key和Value，避免重复计算。变体：Prefix Caching、Paged Attention（vLLM）、KV-Cache量化。

**解码策略**：

| 策略 | 原理 | 适用场景 |
|------|------|---------|
| Greedy | 每步取最大概率token | 确定性输出 |
| Beam Search | 保留Top-B候选序列 | 翻译、摘要 |
| Top-k采样 | 从Top-k中采样 | 创意写作 |
| Top-p（核采样） | 累积概率≥p的token中采样 | 通用生成 |
| Temperature | 调整softmax温度 | 控制随机性 |

**量化**：FP16/BF16（2x）→ INT8（4x）→ INT4（8x）；GPTQ、AWQ、GGUF格式。

**推理加速**：连续批处理（vLLM/TGI）、Speculative Decoding（小模型草拟+大模型验证）、张量并行、流水线并行。

### 应用设计模式

**Prompt工程**：Zero-shot、Few-shot、Chain-of-Thought、Self-Consistency、ReAct

**RAG模式**：Naive RAG（简单检索+拼接）→ Advanced RAG（查询改写+重排序）→ Modular RAG（路由+多路检索+融合）

**Agent模式**：ReAct Agent、Plan-and-Execute、Multi-Agent、Tool-Use Agent

**结构化输出**：JSON Mode、Function Calling、Constrained Decoding、Pydantic验证

## 发展演进

架构从Encoder/Decoder/Enc-Dec三分天下 → Decoder-only统一；训练从全量微调 → PEFT → 指令微调+RLHF；推理从Greedy/Beam → 采样策略+量化+批处理；应用从Prompt → RAG → Agent。

## 关键算法·模型

- **LoRA/QLoRA**：参数高效微调的事实标准
- **PagedAttention (vLLM)**：高吞吐推理服务
- **Speculative Decoding**：降低推理延迟
- **ReAct**：Agent推理-行动框架

## 应用场景

- **模型选型**：根据任务选择架构和微调策略
- **推理部署**：量化+批处理+KV-Cache组合优化
- **应用开发**：RAG+Agent+结构化输出的工程实践
- **评估迭代**：自动评估+LLM-as-Judge+A/B测试
