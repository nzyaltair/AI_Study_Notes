# Agent记忆

## 背景与发展

记忆系统是[[Agent]]的核心组件之一，直接影响Agent的决策能力、上下文理解和持续学习能力。受认知科学中人类记忆分类的启发，Agent记忆系统从简单的对话缓存发展为多类型、多层次的复杂架构。MemGPT（Packer et al., 2023）将操作系统内存管理思想引入Agent记忆，标志着记忆系统从被动存储向主动管理的转变。

## 核心思想

Agent记忆系统模拟人类认知的多类型记忆机制，通过短期记忆维持当前任务上下文，长期记忆实现跨会话知识积累，配合记忆的生命周期管理（编码→存储→检索→遗忘/压缩），实现高效的信息管理。

## 技术原理

### 记忆类型分类

**按存储时长**：

| 类型 | 特点 | 实现方式 |
|------|------|----------|
| 短期记忆 | 容量有限，访问快，易遗忘 | 会话缓存、滑动窗口 |
| 工作记忆 | 动态更新，任务导向 | 临时向量索引、上下文窗口 |
| 长期记忆 | 容量大，相对稳定 | [[向量数据库]]、知识图谱 |

**按内容性质**：

| 类型 | 存储内容 | 特点 |
|------|----------|------|
| 情景记忆 | 特定事件、交互记录 | 有时间戳，事件关联 |
| 语义记忆 | 事实知识、概念关系 | 结构化，可泛化 |
| 程序记忆 | 技能、操作流程 | 隐含性，可执行 |

### 典型记忆架构

**基于向量数据库的记忆**：将记忆内容向量化后存储，通过相似度检索获取相关记忆。优势是支持语义检索、可扩展性好。

**基于图结构的记忆网络**：将记忆表示为图节点和边，通过图算法进行检索和推理。优势是支持关系推理、可解释性好。

**混合记忆系统**：结合多种架构，短期记忆用会话缓存，长期记忆用向量数据库，语义记忆用知识图谱。

### 记忆生命周期

**编码（写入）**：内容捕获 → 预处理 → 特征编码 → 元数据添加（时间戳、重要性评分、关联实体）

**检索（查询）**：

| 策略 | 机制 | 适用场景 |
|------|------|----------|
| 相似性检索 | 计算向量相似度 | 语义相关记忆 |
| 时间检索 | 按时间戳过滤 | 会话恢复、历史追溯 |
| 重要性检索 | 按评分排序 | 关键信息提取 |
| 混合检索 | 加权融合多种策略 | 复杂查询 |

**更新（遗忘/压缩/摘要）**：

- 遗忘：基于TTL、访问频率或重要性评分清理低价值记忆
- 压缩：合并相似记忆、删除冗余、降维
- 摘要：LLM生成记忆的简洁表示，保留核心信息

### 记忆摘要模式

将长对话逐步摘要，形成层次化结构：

```
原始会话 → 短期摘要 → 长期摘要 → 核心知识
检索时：长期摘要 → 短期摘要 → 原始会话（逐步扩展）
```

### 基于反思的记忆增强

定期让Agent反思最近的经历，生成新的见解和规则，存入长期记忆供后续决策使用。

## 发展演进

对话缓存（2022）→ 向量数据库记忆（2023）→ MemGPT分层记忆（2023）→ 反思增强记忆（2023）→ 神经符号记忆 → 可学习记忆控制器

## 关键算法·模型

- **MemGPT**：操作系统式分层记忆管理，记忆分页与动态上下文管理（Packer et al., 2023）
- **Reflexion**：基于反思的记忆增强（Shinn et al., 2023）
- **Generative Agents**：斯坦福小镇Agent，记忆流与反思（Park et al., 2023）
- **记忆摘要**：LLM生成层次化摘要
- **记忆检索**：相似度+时间+重要性的混合检索

## 应用场景

- 长期对话助手（跨会话记忆）
- 个性化Agent（用户偏好学习）
- 持续学习Agent（经验积累）
- 多Agent协作（共享记忆）
- 专业领域助手（知识图谱记忆）

## 与其他技术关系

- Agent记忆依赖[[向量数据库]]实现长期存储
- [[RAG]]可视为Agent记忆的一种实现
- [[Agent]]框架的记忆模块基于本技术
- 记忆检索质量影响[[提示工程]]的上下文构造
- 多Agent协作需要记忆共享与对齐机制
