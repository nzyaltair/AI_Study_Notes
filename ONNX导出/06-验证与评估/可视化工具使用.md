# 可视化工具使用

## 概述

可视化工具是ONNX模型调试和理解的关键手段。本文将介绍主流工具的使用方法，涵盖架构可视化、图编辑、性能分析等场景。

## Netron: 模型架构可视化

### 工具简介

**Netron** 是最流行的深度学习模型可视化工具，支持ONNX、PyTorch、TensorFlow、Keras等50+种格式。

### 安装方式

```bash
# 方式1: 直接下载桌面应用
# https://github.com/lutzroeder/netron/releases

# 方式2: Python包（需要GUI环境）
pip install netron

# 方式3: VS Code扩展
# 扩展名: "Netron"
```

### 基本使用

```python
import netron
import onnx

# 方法1: 启动本地Web服务
model_path = "model.onnx"
netron.start(model_path)
# 浏览器自动打开 http://localhost:8080

# 方法2: 在Python脚本中集成
# 适用于Jupyter Notebook
from netron import browse
browse(model_path)

# 方法3: 保存为HTML（支持离线分享）
netron.export_to_html(model_path, "model_visualization.html")
```

### 图层检查技巧

#### 1. 识别输入输出节点

**查看位置**: 左侧边栏的 Inputs/Outputs 标签页

**关键信息**:
- 节点名称（如 `input_1`, `output`）
- 数据形状（如 `[1, 3, 224, 224]`）
- 数据类型（`float32`, `int64`等）

**常见问题**:
```
动态维度 → 模型未设置fixed batch size
类型不匹配 → 需要类型转换或cast操作
缺失维度 → Reshape节点调整
```

#### 2. 检查算子类型

**操作**: 点击任意节点查看详细属性

**输出示例**:
```
Node: conv1
Op: Conv
Attribute:
  kernel_shape: [3, 3]
  strides: [1, 1]
  pads: [1, 1, 1, 1]
  dilations: [1, 1]
Inputs: [input, weight, bias]
Outputs: [conv1_output]
```

**排查方向**:
- 检查算子是否在ONNX Runtime支持列表
- 查看attribute是否符合预期
- 确认weight shape是否正确

#### 3. 追踪数据流

**方法**: 点击节点上方的输入/输出连线

**Netron高亮显示**:
- 相连的所有节点
- 数据形状在各节点的变化

**应用场景**:
- 定位shape mismatch的位置
- 理解模型数据流向
- 确认不必要的transpose/reshape

#### 4. 查看模型统计

**Netron Statistics标签页** 显示:
```
总参数数量: 25.6M
计算量: 4.1 GFLOPS
节点总数: 187
算子类型分布:
  Conv: 53
  BatchNormalization: 52
  Relu: 52
  Add: 24
  MaxPool: 5
```

### 诊断不支持的算子

**问题场景**: 运行时出现`Operator Not Supported`错误

**Netron诊断步骤**:

1. **定位错误涉及的算子**
   ```bash
   # 查看onnxruntime详细错误
   python -c "
   import onnxruntime as ort
   sess = ort.InferenceSession('model.onnx')
   "
   # 错误输出: "DisseminationOp... does not exist"
   ```

2. **在Netron中搜索该算子**
   - 按`Ctrl+F`打开搜索框
   - 输入算子名称（如`GridSample`）
   - 查看所有该算子的位置和使用场景

3. **分析上下文**
   - 前序算子是什么？
   - 输出shape变化是否符合预期？
   - 能否用其他算子组合替换？

4. **对比原始框架**
   - 在PyTorch/TensorFlow中找到对应代码
   - 确认该算子是否被开发者自定义
   - 可能需要添加custom op

### Netron截图检查清单

当分析模型时，重点关注以下截图：

**✓ 截图1: 完整模型图**

**✓ 截图2: 关键算子详情**

**✓ 截图4: 算子类型统计**
- 确认算子分布是否合理
- 检查是否有非常规算子

```python

def list_ops(graph: gs.Graph):
    """统计模型中所有算子类型"""
    op_counts = {}
    for node in graph.nodes:
        op_counts[node.op] = op_counts.get(node.op, 0) + 1

    print("算子统计:")
    for op, count in sorted(op_counts.items(), key=lambda x: -x[1]):
        print(f"  {op}: {count}")

    return op_counts

# 使用示例
graph = gs.import_onnx(onnx.load("model.onnx"))
```

```python
def adjust_conv_params(graph: gs.Graph, node_name: str, **attrs):
    """
    调整卷积节点参数

    示例: 修改padding/stride
    """
    target_node = next((n for n in graph.nodes if n.name == node_name), None)
    if target_node and target_node.op == 'Conv':
        for attr_name, attr_value in attrs.items():
            target_node.attrs[attr_name] = attr_value
        print(f"已修改节点 {node_name}: {attrs}")
    else:
        print(f"未找到节点: {node_name}")

# 使用示例
```

#### 3. 删除节点（谨慎操作）

```python
def remove_node_and_tensors(graph: gs.Graph, node_name: str):
    """删除指定节点及关联的中间张量"""
    node = next((n for n in graph.nodes if n.name == node_name), None)
    if not node:
        print(f"节点不存在: {node_name}")
        return

    # 断开连接
    for inp in node.inputs:
        inp.outputs.remove(node)
    for out in node.outputs:
        out.inputs.remove(node)

    # 删除节点
    graph.nodes.remove(node)

    # 删除不再使用的张量
    unused_tensors = [t for t in graph.tensors.values()
                     if not t.inputs and not t.outputs]
    for tensor in unused_tensors:
        del graph.tensors[tensor.name]

    print(f"已删除节点: {node_name}")

# 使用示例：删除不必要的transpose
remove_node_and_tensors(graph, "Transpose_23")
```

#### 4. 替换算子

```python
def replace_activation(
    graph: gs.Graph,
    old_activation: str,
    new_activation: str
):
    """
    批量替换激活函数
    例如: 将所有Relu替换为LeakyRelu
    """
    count = 0
    for node in graph.nodes:
        if node.op == old_activation:
            node.op = new_activation

            # 如果是LeakyReLU，需要添加alpha属性
            if new_activation == 'LeakyRelu':
                node.attrs['alpha'] = 0.01

            count += 1

    print(f"替换完成: {old_activation} → {new_activation} ({count}个节点)")
    return count
```

#### 5. 常量折叠优化

```python
def fold_constants(graph: gs.Graph):
    """
    执行常量折叠
    合并Conv+BatchNorm+Add的组合为单个Conv
    """
    before_nodes = len(graph.nodes)
    graph = gs.optimizer.fold_constants(graph, True)
    after_nodes = len(graph.nodes)

    print(f"常量折叠前: {before_nodes} 节点")
    print(f"常量折叠后: {after_nodes} 节点")
    print(f"减少: {before_nodes - after_nodes} 节点")

    return graph
```

#### 6. 清理孤立节点

```python
def cleanup_graph(graph: gs.Graph):
    """清理不再使用的节点和张量"""
    before = len(graph.nodes)
    graph.cleanup()
    after = len(graph.nodes)

    print(f"清理前: {before} 节点")
    print(f"清理后: {after} 节点")
    print(f"清理掉: {before - after} 个孤立节点")

    return graph
```

### 完整图编辑工作流

```python
def optimize_model_with_graphsurgeon(
    input_path: str,
    output_path: str
):
    """
    使用Graphsurgeon优化ONNX模型

    操作流程:
    1. 加载模型
    2. 分析算子分布
    3. 执行优化（删除、替换、折叠）
    4. 验证图结构
    5. 保存模型
    """
    print(f"加载模型: {input_path}")
    graph = gs.import_onnx(onnx.load(input_path))

    print("\n原始模型信息:")
    print(f"  节点数: {len(graph.nodes)}")
    print(f"  张量数: {len(graph.tensors)}")
    list_ops(graph)

    print("\n[1] 常量折叠...")
    graph = fold_constants(graph)
    cleanup_graph(graph)

    print("\n[2] 删除某些节点...")
    # 示例: 删除Debug节点
    nodes_to_remove = [n.name for n in graph.nodes if 'Debug' in n.name]
    for node_name in nodes_to_remove:
        remove_node_and_tensors(graph, node_name)

    cleanup_graph(graph)

    print("\n[3] 替换算子...")
    replace_activation(graph, "Relu", "Clip")  # 某些部署需要Clip

    print("\n[4] 最终验证...")
    print(f"  节点数: {len(graph.nodes)}")
    print(f"  张量数: {len(graph.tensors)}")

    list_ops(graph)

    print(f"\n保存模型: {output_path}")
    optimized_model = gs.export_onnx(graph)
    onnx.save(optimized_model, output_path)

    # 验证保存的模型
    try:
        onnx.checker.check_model(output_path)
        print("✓ 模型检查通过")
    except Exception as e:
        print(f"✗ 模型检查失败: {e}")

    return graph
```

```python
def trace_tensor_flow(
    graph: gs.Graph,
    tensor_name: str,
    max_depth: int = 10
):
    """
    追踪张量的上下游依赖链

    用于理解数据流向和定位问题
    """
    if tensor_name not in graph.tensors:
        print(f"张量不存在: {tensor_name}")
        return

    tensor = graph.tensors[tensor_name]

    print(f"张量信息: {tensor_name}")
    print(f"  形状: {tensor.shape}")
    print(f"  类型: {tensor.dtype}")

    print("\n生产者（上游）:")
    if tensor.inputs:
        for inp in tensor.inputs:
            print(f"  {inp.op} -> {tensor_name}")
    else:
        print("  (graph input)")

    print("\n消费者（下游）:")
    if tensor.outputs:
        for out in tensor.outputs:
            print(f"  {tensor_name} -> {out.op}")
    else:
        print("  (graph output)")

# 使用示例
trace_tensor_flow(graph, "conv1_output")
```

```python
from onnxruntime.tools import performance

def run_benchmark(
    model_path: str,
    providers: List[str] = ['CPUExecutionProvider'],
    batch_size: int = 1,
    input_shape: Tuple[int, ...] = (3, 224, 224)
):
    """
    使用onnxruntime-tools进行基准测试
    """
    import time

    session = ort.InferenceSession(
        model_path,
        providers=providers
    )

    input_name = session.get_inputs()[0].name
    input_data = np.random.randn(batch_size, *input_shape).astype(np.float32)

    # Warm-up
    for _ in range(10):
        _ = session.run(None, {input_name: input_data})

    # Benchmark
    num_iterations = 1000
    start = time.time()
    for _ in range(num_iterations):
        _ = session.run(None, {input_name: input_data})
    total_time = time.time() - start

    avg_latency = total_time / num_iterations * 1000  # ms
    throughput = num_iterations / total_time

    print(f"模型: {Path(model_path).name}")
    print(f"平均延迟: {avg_latency:.2f} ms")
    print(f"吞吐量: {throughput:.1f} FPS")

    return {
        'model': model_path,
        'batch_size': batch_size,
        'avg_latency_ms': avg_latency,
        'throughput_fps': throughput
    }
```

```python
def analyze_model_stats(model_path: str):
    """
    分析模型结构和复杂度
    """
    import onnx

    model = onnx.load(model_path)
    graph = model.graph

    print(f"模型文件: {model_path}")

    print(f"\n图信息:")

    op_counts = {}
    total_params = 0

    for node in graph.node:

    for op, count in sorted(op_counts.items(), key=lambda x: -x[1]):

    # 分析输入输出
    print("\n输入:")
    for inp in graph.input:
        shape = [d.dim_value for d in inp.type.tensor_type.shape.dim]
        print(f"  {inp.name}: {shape}, dtype={inp.type.tensor_type.elem_type}")
```

## ONNX Checker: 兼容性检查

### 基础验证

```python
import onnx

def validate_onnx_model(model_path: str, full_check: bool = False):
    """
    验证ONNX模型的格式正确性

    参数:
    - model_path: ONNX模型路径
    - full_check: True=完整验证（较慢），False=基础验证
    """
    print(f"验证模型: {model_path}")

    try:
        # 基础检查
        model = onnx.load(model_path)

        # 快速检查（仅验证结构）
        onnx.checker.check_model(model)
        print("✓ 基础检查通过")

        if full_check:
            # 完整检查（包含形状推断）
            print("\n执行完整验证...")
            onnx.checker.check_model(model, full_check=True)
            print("✓ 完整检查通过")

            # 形状推断
            print("\n形状推断...")
            inferred_model = onnx.shape_inference.infer_shapes(model)
            print("✓ 形状推断完成")

            # 检查推断后形状
            for value_info in inferred_model.graph.value_info:
                shape = [d.dim_value for d in value_info.type.tensor_type.shape.dim]
                print(f"  {value_info.name}: {shape}")

        return True

    except Exception as e:
        print(f"✗ 验证失败: {type(e).__name__}")
        print(f"  错误信息: {str(e)}")
        return False

# 使用示例
validate_onnx_model("model.onnx", full_check=True)
```

### 自定义验证规则

```python
def validate_model_semantics(model_path: str):
    """
    语义验证：检查模型是否符合特定规范
    """
    import onnx

    model = onnx.load(model_path)
    graph = model.graph

    errors = []

    # 规则1: 检查输入shape合理性
    for inp in graph.input:
        shape = [d.dim_value for d in inp.type.tensor_type.shape.dim]
        if any(dim <= 0 for dim in shape):
            errors.append(f"输入 {inp.name} 包含无效维度: {shape}")

    # 规则2: 检查是否存在孤立节点
    node_names = {node.name for node in graph.node}
    tensor_names = {tensor.name for tensor in graph.tensor}

    for node in graph.node:
        for inp in node.input:
            if inp not in tensor_names and inp not in {i.name for i in graph.input}:
                errors.append(f"节点 {node.name} 引用了不存在的张量: {inp}")

    # 规则3: 检查权重shape与算子匹配
    for node in graph.node:
        if node.op_type == 'Conv':
            # Conv应有权重输入
            if len(node.input) < 2:
                errors.append(f"Conv节点 {node.name} 缺少权重输入")

    if errors:
        print("发现以下问题:")
        for error in errors:
            print(f"  ✗ {error}")
        return False
    else:
        print("✓ 语义验证通过")
        return True

validate_model_semantics("model.onnx")
```

## 教程: 使用Netron诊断不支持的算子

### 场景重现

**错误信息**:
```
[ONNXRuntimeError] : 1 : INVALID_GRAPH :
  [0] Invalid Graph: Node DisseminationOp_56 (DisseminationOp)
  does not exist. Please note when contributing a custom
  operator and it must be registered via kernel registration.
```

### 诊断流程

```python
def diagnose_unsupported_op(model_path: str, error_op: str):
    """
    诊断并报告不支持的算子
    """
    import onnxruntime as ort

    print(f"诊断模型: {model_path}")
    print(f"问题算子: {error_op}")

    # 步骤1: 用Netron可视化分析
    print("\n[步骤1] 使用Netron分析...")
    import netron
    netron.start(model_path)
    print("  → 在浏览器中查找该算子，查看使用上下文")

    # 步骤2: 列出所有算子
    import onnx
    model = onnx.load(model_path)
    all_ops = set(node.op_type for node in model.graph.node)

    print(f"\n[步骤2] 模型中的所有算子:")
    for op in sorted(all_ops):
        print(f"  - {op}")

    print(f"\n[步骤3] 检查问题算子位置:")
    for i, node in enumerate(model.graph.node):
        if node.op_type == error_op:
            print(f"  节点 #{i}: {node.name}")
            print(f"    输入: {node.input}")
            print(f"    输出: {node.output}")
            print(f"    属性: {node.attribute}")

    print(f"\n[步骤4] 调研解决方案:")
    print(f"  1. 升级ONNX Runtime到最新版本")
    print(f"  2. 在转换时启用 --enable-{error_op.lower()}")
    print(f"  3. 添加custom op实现")
    print(f"  4. 用标准算子替换（检查框架文档）")

# 运行诊断
diagnose_unsupported_op("model.onnx", "DisseminationOp")
```

```python
def automated_model_checks(model_path: str):
    """
    CI/CD中的自动化检查
    """
    checks = {
        'format_valid': validate_onnx_model(model_path),
        'no_isolated_nodes': check_no_isolated_nodes(model_path),
        'inputs_valid': validate_inputs(model_path),
        'opset_compatible': check_opset_version(model_path),
        'size_reasonable': check_model_size(model_path)
    }

    print("自动化检查结果:")
    for check, passed in checks.items():
        status = "✓" if passed else "✗"
        print(f"  {status} {check}")

    return all(checks.values())
```

### 性能问题排查决策表

| 现象 | 可能原因 | 排查工具 | 解决方案 |
|-----|---------|---------|---------|
| 推理慢 | 算子未optimized | Netron统计 + runtime perf | 启用图优化 |
| 内存占用大 | 模型未量化 | onnxruntime-tools | 应用量化 |
| 算子不支持 | 使用了非常规op | Netron搜索 | 转换时禁用或注册custom op |
| 延迟抖动大 | 动态shape或线程竞争 | 多次基准测试 | 固定batch size或调整线程配置 |
