# 版本管理与回滚

## 背景与发展

大模型版本管理面临超大模型权重（GB/TB 级）、多维度版本依赖（模型、数据、配置、镜像）和快速迭代需求等挑战，与传统软件版本管理有本质差异。Git LFS 和 DVC 将大文件版本控制引入 Git 生态，MLflow Model Registry 和 Weights & Biases 提供模型注册表和实验跟踪能力。Kubernetes 的滚动更新和金丝雀部署机制为模型灰度发布和快速回滚提供了基础设施支撑。

## 核心思想

对模型权重、训练数据、推理配置、服务镜像和部署状态进行全生命周期管理，确保每次变更可追溯、可验证、可回滚。通过模型注册表统一管理版本元数据，通过灰度发布逐步验证新版本，通过一键回滚在异常时快速恢复服务。

## 技术原理

### 版本管理维度

| 维度 | 管理内容 | 工具 |
|------|----------|------|
| **模型权重** | 基础模型、LoRA 适配器、量化模型 | DVC、模型注册表 |
| **训练数据** | 数据集版本、采样策略 | DVC、LakeFS |
| **配置版本** | 超参数、推理配置、Prompt 模板 | Git |
| **服务镜像** | 推理引擎镜像、依赖版本 | Docker Registry |
| **部署状态** | 副本数、资源分配、路由规则 | Kubernetes、IaC |

### 模型注册表

模型注册表提供版本控制、阶段管理（Development → Staging → Production）和元数据存储。MLflow 的核心工作流：

```python
import mlflow

mlflow.set_experiment("Llama-3-Finetuning")
with mlflow.start_run():
    mlflow.log_params({"learning_rate": 1e-5, "lora_r": 16})

    mlflow.pytorch.log_model(
        pytorch_model=model,
        artifact_path="model",
        registered_model_name="Llama-3-8B-Finance"
    )
```

### 灰度发布

- **蓝绿部署**：新旧版本并行运行，通过路由切换流量，回滚即时
- **金丝雀发布**：小流量（如 5%）验证新版本，逐步放量到 100%
- **A/B 测试**：并行运行多版本，基于统计显著性检验对比效果

Kubernetes 金丝雀部署通过 Ingress 注解控制流量分配：

```yaml
annotations:
  nginx.ingress.kubernetes.io/canary: "true"
  nginx.ingress.kubernetes.io/canary-weight: "10"  # 10% 流量到新版本
```

### 回滚机制

回滚流程：触发（手动/自动）→ 流量切回旧版本 → 缓存清理（[[KV-Cache]]、模型缓存）→ 告警暂停 → 验证恢复。

```bash
# Kubernetes 回滚到上一版本
kubectl rollout undo deployment llm-service
# 回滚到特定版本
kubectl rollout undo deployment llm-service --to-revision=3
```

### 大模型特有挑战

- **分片管理**：TB 级模型需分片存储和并行加载，需维护分片元数据一致性
- **LoRA 组合**：多个 LoRA 适配器的组合版本需适配器版本矩阵管理
- **在线学习**：实时数据更新下的增量版本控制和版本快照机制

## 发展演进

Git + 手动管理 → Git LFS / DVC → MLflow Model Registry → GitOps for AI → 不可变模型部署

## 关键算法·模型

- **语义化版本控制**：`<模型>-<功能>-<主版本>.<次版本>.<修订号>`，主版本变更表示不兼容更新
- **A/B 测试统计检验**：对两组样本均值执行 $t$ 检验，$t = \frac{\bar{X}_A - \bar{X}_B}{\sqrt{s_A^2/n_A + s_B^2/n_B}}$，$p < 0.05$ 时认为存在显著差异
- **滚动更新**：Kubernetes 逐个替换 Pod，通过 `maxSurge` 和 `maxUnavailable` 控制更新速率

## 应用场景

- **生产模型迭代**：从 v1 到 v2 的平滑过渡，支持随时回退
- **A/B 测试实验**：并行运行多版本对比业务指标
- **多环境部署**：开发、测试、生产环境的版本同步与隔离
- **合规审计**：记录模型版本变更历史，满足监管追溯要求
