# 文本生成与摘要

## 1. 概述

文本生成（Natural Language Generation, NLG）是 NLP 中"从理解走向创造"的任务——让计算机生成流畅、连贯、有意义的自然语言文本。文本摘要（Text Summarization）是 NLG 的重要子任务，旨在将长文本压缩为简洁的摘要，同时保留核心信息。

文本生成涵盖摘要、对话生成、故事生成、数据到文本生成等多种任务。从基于模板的生成到 Seq2Seq 神经网络生成，再到大语言模型的自回归生成，文本生成技术经历了从"规则拼接"到"概率生成"再到"智能创作"的质变。

## 3. 核心概念

### 3.1 抽取式 vs 生成式摘要

| 类型 | 方法 | 优势 | 局限 |
|:---|:---|:---|:---|
| 抽取式 | 从原文中选择关键句子 | 事实准确、无幻觉 | 不灵活、可能不连贯 |
| 生成式 | 重新组织语言生成摘要 | 灵活、流畅、可整合信息 | 可能幻觉、事实不一致 |
| 混合式 | 抽取+改写 | 兼顾准确性和流畅性 | 实现复杂 |

### 3.2 摘要的类型

| 类型 | 说明 | 示例 |
|:---|:---|:---|
| 单文档摘要 | 一篇文档的摘要 | 论文摘要 |
| 多文档摘要 | 多篇文档的综合摘要 | 新闻聚合摘要 |
| 查询式摘要 | 针对查询生成摘要 | 搜索结果摘要 |
| 标题生成 | 为文档生成标题 | 新闻标题生成 |
| 会议摘要 | 会议记录的摘要 | 会议纪要 |

### 3.3 文本生成的方式

| 方式 | 说明 | 代表模型 |
|:---|:---|:---|
| 模板填充 | 预定义模板+数据填充 | 天气预报、财务报告 |
| 检索-改写 | 检索相关文本+改写 | 早期对话系统 |
| 自回归生成 | 逐词生成 | GPT 系列 |
| Seq2Seq 生成 | 编码-解码 | BART、T5 |
| 约束生成 | 在约束下生成 | 受控文本生成 |

## 4. 技术原理

### 4.1 抽取式摘要

#### TextRank / LexRank

基于图排序的抽取式摘要——将句子作为节点，句子间相似度作为边权重，运行 PageRank 算法：

$$WS(V_i) = (1-d) + d \cdot \sum_{V_j \in In(V_i)} \frac{w_{ji}}{\sum_{V_k \in Out(V_j)} w_{jk}} WS(V_j)$$

得分最高的句子被选为摘要。

#### BERTSUM

用 BERT 编码句子，然后使用分类器或 Transformer 层对句子排序：

```
文档句子 → BERT 编码 → 句子表示 → 分类器/Transformer → 句子重要性分数 → 选 Top-K
```

### 4.2 生成式摘要

#### Seq2Seq + Attention

将摘要建模为序列到序列的生成任务：

```
编码器：文档 → 上下文向量
解码器：上下文向量 → 摘要（逐词生成）
```

#### Pointer-Generator Network

结合生成和复制的混合机制（See et al., 2017）：

$$p_{\text{gen}} \cdot P_{\text{vocab}}(w) + (1 - p_{\text{gen}}) \cdot \sum_{i: w_i = w} \alpha_{t,i}$$

- $p_{\text{gen}}$：生成概率（从词表中生成）
- $1 - p_{\text{gen}}$：复制概率（从源文本中复制）
- 解决 OOV 问题，减少事实错误

#### BART / T5

Seq2Seq 预训练模型，通过去噪自编码器学习生成能力：

- **BART**：对输入施加噪声（删除、替换、打乱），训练模型恢复原文
- **T5**：将所有任务统一为 text-to-text 格式

### 4.3 自回归生成

GPT 等大语言模型通过自回归方式生成文本：

$$P(x_{1:T}) = \prod_{t=1}^{T} P(x_t | x_{<t}; \theta)$$

**解码策略**：

| 策略 | 公式 | 特点 |
|:---|:---|:---|
| Greedy | $x_t = \arg\max P(x_t \| x_{<t})$ | 快速但可能陷入局部最优 |
| Beam Search | 保留 Top-B 假设 | 平衡质量和效率 |
| Top-k 采样 | 从概率最高的 k 个中采样 | 增加多样性 |
| Top-p（核采样） | 从累积概率 ≤ p 的词中采样 | 自适应多样性 |
| Temperature | $P'(w) = \text{softmax}(\logit(w)/T)$ | T<1 更确定，T>1 更随机 |

### 4.4 评估指标

#### ROUGE

摘要评估的标准指标——基于 n-gram 召回率：

$$\text{ROUGE-N} = \frac{\sum_{S \in \text{Ref}} \sum_{g_n \in S} \text{Count}_{\text{match}}(g_n)}{\sum_{S \in \text{Ref}} \sum_{g_n \in S} \text{Count}(g_n)}$$

- **ROUGE-1**：unigram 召回
- **ROUGE-2**：bigram 召回
- **ROUGE-L**：最长公共子序列

## 5. 关键方法/模型

| 方法 | 类型 | ROUGE-L | 优势 | 局限 |
|:---|:---|:---|:---|:---|
| TextRank | 抽取式 | ~30 | 无需训练、快速 | 不灵活 |
| BERTSUM | 抽取式 | ~40 | 预训练表示 | 仅抽取 |
| Pointer-Gen | 生成式 | ~36 | 解决 OOV | 长文档处理差 |
| BART / T5 | 生成式 | ~42 | Seq2Seq 预训练 | 需要微调 |
| LLM 摘要 | 生成式 | ~45+ | 零样本、高质量 | 成本高、幻觉 |

## 6. 优势与局限

### 优势

1. **效率提升**：自动摘要大幅缩短信息获取时间
2. **个性化**：可根据用户需求生成定制化摘要
3. **多文档整合**：整合多源信息的综合摘要

### 局限

1. **幻觉问题**：生成式摘要可能包含原文没有的信息
2. **事实一致性**：生成的摘要可能与原文事实矛盾
3. **长文档处理**：超出上下文窗口的长文档摘要仍有挑战
4. **评估局限**：ROUGE 等指标无法完全反映摘要质量
5. **领域依赖**：训练于新闻的模型在科技/法律领域效果差

## 7. 应用场景

- **新闻摘要**：新闻聚合、快讯
- **论文摘要**：学术论文自动摘要
- **会议纪要**：会议记录自动总结
- **搜索结果摘要**：搜索引擎结果页摘要
- **产品评论摘要**：电商平台评论汇总
- **数据报告**：数据到文本生成（财务报告、天气报告）

## 9. 前沿发展

- **LLM 摘要**：GPT-4 等大模型生成高质量摘要，支持零样本和指令控制
- **事实一致性**：通过事实约束、事实验证减少幻觉
- **长文档摘要**：层次化摘要、检索增强摘要处理超长文档
- **多模态摘要**：结合文本、图像、视频的多模态摘要
- **可控生成**：控制摘要长度、风格、重点
- **实时摘要**：流式文本的实时摘要生成
- **Agent 摘要**：LLM Agent 自主规划、检索、总结，生成综合报告
