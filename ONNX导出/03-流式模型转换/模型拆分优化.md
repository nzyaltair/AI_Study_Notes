# 模型拆分优化

大型深度学习模型（特别是Transformer类模型）往往会超出单个设备的内存限制，或成为推理延迟的瓶颈。模型拆分（Model Splitting）通过将模型分割为多个独立的子模型，可以实现**内存优化**、**并行推理**和**资源利用率最大化**。

## 何时需要拆分模型

### 内存约束场景

| 场景 | 典型模型大小 | 问题 | 解决方案 |
|------|-------------|------|----------|
| 边缘设备 | < 4GB RAM | 模型无法加载 | 按层拆分 + 流式加载 |
| 单GPU推理 | 10-30GB | OOM错误 | 编码器-解码器分离 |
| 多设备推理 | 跨设备 | 单设备内存不足 | Pipeline并行 |

### 性能优化场景

- **降低延迟**：拆分后可以并行处理不同部分
- **提高吞吐**：流水线并行，重叠计算与IO
- **灵活性**：不同部分可以针对不同硬件优化

## 拆分策略分类

### 1. 按层拆分（Layer-wise Splitting）

将Transformer模型按层数分割：

```python
import torch
import torch.nn as nn
from transformers import AutoModel

def split_transformer_by_layers(model_path, num_splits=2):
    """将Transformer模型按层数分割"""
    model = AutoModel.from_pretrained(model_path)
    config = model.config

    # 计算每组的层数
    total_layers = config.num_hidden_layers
    layers_per_split = total_layers // num_splits

    splits = []
    for i in range(num_splits):
        start_layer = i * layers_per_split
        end_layer = (i + 1) * layers_per_split if i < num_splits - 1 else total_layers

        # 创建子模型
        class SubModel(nn.Module):
            def __init__(self, base_model, start, end):
                super().__init__()
                self.embeddings = base_model.embeddings
                self.encoder = nn.ModuleList(
                    base_model.encoder.layer[start:end]
                )
                self.pooler = base_model.pooler

            def forward(self, hidden_states, attention_mask=None, past_key_values=None):
                # 处理embedding（仅第一段）
                if hasattr(self, 'embeddings'):
                    hidden_states = self.embeddings(hidden_states)

                # 处理encoder层
                new_past = []
                for i, layer in enumerate(self.encoder):
                    if past_key_values is not None:
                        layer_outputs = layer(
                            hidden_states,
                            attention_mask=attention_mask,
                            past_key_value=past_key_values[i]
                        )
                    else:
                        layer_outputs = layer(hidden_states, attention_mask=attention_mask)
                    hidden_states = layer_outputs[0]
                    new_past.append(layer_outputs[1] if len(layer_outputs) > 1 else None)

                pooled = self.pooler(hidden_states) if hasattr(self, 'pooler') else None
                return hidden_states, new_past, pooled

        sub_model = SubModel(model, start_layer, end_layer)
        splits.append(sub_model)

    return splits

# 使用示例
splits = split_transformer_by_layers("bert-base-uncased", num_splits=3)
print(f"Created {len(splits)} sub-models")

for i, split in enumerate(splits):
    dummy_input = torch.randn(2, 128, 768)
    torch.onnx.export(
        split,
        dummy_input,
        f"bert_split_{i}.onnx",
        input_names=['hidden_states', 'attention_mask'],
        output_names=['output', 'past_key_values'],
        dynamic_axes={
            'hidden_states': {0: 'batch', 1: 'seq_len'},
            'attention_mask': {0: 'batch', 1: 'seq_len'}
        }
    )
```

### 2. 功能模块拆分（Module-wise Splitting）

将模型按功能模块（Embedding、Encoder、Decoder、Head）拆分：

```python
def split_model_into_modules(model):
    """将模型拆分为独立的功能模块"""
    class EmbeddingModule(nn.Module):
        def __init__(self, base_model):
            super().__init__()
            self.embeddings = base_model.embeddings

        def forward(self, input_ids):
            return self.embeddings(input_ids)

    class EncoderModule(nn.Module):
        def __init__(self, base_model):
            super().__init__()
            self.encoder = base_model.encoder

        def forward(self, hidden_states, attention_mask=None):
            return self.encoder(hidden_states, attention_mask=attention_mask)

    class HeadModule(nn.Module):
        def __init__(self, base_model):
            super().__init__()
            self.pooler = base_model.pooler
            self.classifier = base_model.classifier

        def forward(self, pooled_output):
            pooled = self.pooler(pooled_output)
            logits = self.classifier(pooled)
            return logits

    return {
        'embedding': EmbeddingModule(model),
        'encoder': EncoderModule(model),
        'head': HeadModule(model)
    }

# 使用
modules = split_model_into_modules(model)

# 导出各个模块
torch.onnx.export(
    modules['embedding'],
    torch.randint(0, 30000, (2, 128)),
    "embedding.onnx"
)

torch.onnx.export(
    modules['encoder'],
    torch.randn(2, 128, 768),
    "encoder.onnx"
)
```

### 3. Sequence Split（序列分割）

对于超长序列，可以沿序列维度分割：

```python
def split_by_sequence(input_tensor, chunk_size=512):
    """将序列分割为chunks"""
    batch, seq_len, dim = input_tensor.shape
    chunks = []
    for i in range(0, seq_len, chunk_size):
        chunk = input_tensor[:, i:i+chunk_size, :]
        chunks.append(chunk)
    return chunks

class ChunkedProcessor(nn.Module):
    def __init__(self, chunk_size=512):
        super().__init__()
        self.chunk_size = chunk_size
        self.process_chunk = nn.Linear(768, 768)  # 示例

    def forward(self, x):
        chunks = split_by_sequence(x, self.chunk_size)
        outputs = []
        for chunk in chunks:
            out = self.process_chunk(chunk)
            outputs.append(out)
        return torch.cat(outputs, dim=1)
```

## Pipeline并行推理

拆分后的模型可以流水线化以提高吞吐量：

```python
class PipelineInference:
    def __init__(self, model_paths, device_mapping=None):
        """
        model_paths: ONNX模型文件路径列表
        device_mapping: 每个模型分配到哪个设备（GPU ID）
        """
        from onnxruntime import InferenceSession
        self.sessions = []
        self.device_mapping = device_mapping or list(range(len(model_paths)))

        for i, path in enumerate(model_paths):
            providers = [
                ('CUDAExecutionProvider', {'device_id': self.device_mapping[i]})
            ] if torch.cuda.is_available() else ['CPUExecutionProvider']

            session = InferenceSession(path, providers=providers)
            self.sessions.append(session)

    def run_pipeline(self, input_data):
        """执行流水线推理"""
        # 阶段1
        output1 = self.sessions[0].run(None, {'input': input_data})[0]

        # 阶段2
        output2 = self.sessions[1].run(None, {'input': output1})[0]

        # ... 更多阶段
        final_output = self.sessions[-1].run(None, {'input': output2})[0]

        return final_output

# 多batch流式处理
def pipeline_streaming(pipeline, data_loader):
    """使用流水线并行处理数据流"""
    futures = []
    for batch in data_loader:
        # 异步提交
        future = pipeline.run_async(batch)
        futures.append(future)

    # 收集结果
    results = [f.result() for f in futures]
    return results
```

### 异步流水线实现

```python
import asyncio
from concurrent.futures import ThreadPoolExecutor

class AsyncPipelineInference:
    def __init__(self, model_sessions, max_concurrent=4):
        self.sessions = model_sessions
        self.executor = ThreadPoolExecutor(max_workers=max_concurrent)
        self.semaphore = asyncio.Semaphore(max_concurrent)

    async def process_stage(self, stage_idx, input_data):
        """异步处理单个阶段"""
        async with self.semaphore:
            loop = asyncio.get_event_loop()
            result = await loop.run_in_executor(
                self.executor,
                lambda: self.sessions[stage_idx].run(None, {'input': input_data})[0]
            )
            return result

    async def run_async_pipeline(self, input_data):
        """执行完整流水线"""
        current = input_data
        for stage_idx in range(len(self.sessions)):
            current = await self.process_stage(stage_idx, current)
        return current
```

## 跨设备拆分策略

### CPU + GPU混合推理

```python
def create_cpu_gpu_pipeline(model_paths):
    """创建CPU-GPU混合流水线"""
    class HybridPipeline:
        def __init__(self):
            # 第一层在CPU（内存大，处理输入）
            self.cpu_session = InferenceSession(
                model_paths[0],
                providers=['CPUExecutionProvider']
            )
            # 后续层在GPU（计算密集）
            self.gpu_sessions = [
                InferenceSession(p, providers=['CUDAExecutionProvider'])
                for p in model_paths[1:]
            ]

        def forward(self, x):
            # CPU处理
            stage1 = self.cpu_session.run(None, {'input': x.cpu().numpy()})[0]

            # GPU流水线
            current = stage1
            for sess in self.gpu_sessions:
                current = sess.run(None, {'input': current})[0]

            return torch.from_numpy(current).cuda()
```

### 多GPU数据并行

```python
class DataParallelInference:
    def __init__(self, model_paths, devices=['cuda:0', 'cuda:1']):
        self.sessions = []
        for path, device in zip(model_paths, devices):
            providers = [('CUDAExecutionProvider', {'device_id': int(device.split(':')[-1])})]
            self.sessions.append(InferenceSession(path, providers=providers))

    def forward(self, x, split_batch=True):
        """数据并行推理"""
        batch_size = x.shape[0]
        if split_batch and len(self.sessions) > 1:
            # 将batch分割到不同设备
            chunk_size = batch_size // len(self.sessions)
            results = []

            for i, session in enumerate(self.sessions):
                start = i * chunk_size
                end = (i + 1) * chunk_size if i < len(self.sessions) - 1 else batch_size
                chunk = x[start:end]

                result = session.run(None, {'input': chunk.numpy()})[0]
                results.append(result)

            return np.concatenate(results, axis=0)
        else:
            # 所有数据送同一个设备
            return self.sessions[0].run(None, {'input': x.numpy()})[0]
```

## 实际案例：LLaMA/Mistral模型拆分

```python
def split_llama_model(model_path, num_splits=4):
    """
    拆分LLaMA风格模型用于资源受限环境
    假设原始模型有num_hidden_layers层Transformer
    """
    from transformers import LlamaForCausalLM

    model = LlamaForCausalLM.from_pretrained(model_path)
    config = model.config
    total_layers = config.num_hidden_layers

    layers_per_split = total_layers // num_splits

    # 导出所有模型部分
    for split_idx in range(num_splits):
        start = split_idx * layers_per_split
        end = (split_idx + 1) * layers_per_split if split_idx < num_splits - 1 else total_layers

        class SplitModel(nn.Module):
            def __init__(self, base_model, start_layer, end_layer):
                super().__init__()
                self.embed_tokens = base_model.model.embed_tokens
                self.layers = base_model.model.layers[start_layer:end_layer]
                self.norm = base_model.model.norm if end_layer == total_layers else None
                self.lm_head = base_model.lm_head if split_idx == num_splits - 1 else None

                self.start_layer = start_layer
                self.end_layer = end_layer
                self.total_layers = total_layers

            def forward(self, input_ids, attention_mask=None, past_key_values=None):
                # Embedding
                hidden_states = self.embed_tokens(input_ids)

                # Transformer层
                new_past = []
                for i, layer in enumerate(self.layers):
                    layer_idx = self.start_layer + i
                    if past_key_values is not None:
                        hidden_states, pkv = layer(
                            hidden_states,
                            attention_mask=attention_mask,
                            past_key_value=past_key_values[i]
                        )
                    else:
                        hidden_states, pkv = layer(hidden_states, attention_mask=attention_mask)
                    new_past.append(pkv)

                # 归一化（仅最后一段）
                if self.norm:
                    hidden_states = self.norm(hidden_states)

                # LM Head（仅最后一段）
                if self.lm_head:
                    logits = self.lm_head(hidden_states)
                    return logits, new_past
                else:
                    return hidden_states, new_past

        split_model = SplitModel(model, start, end)

        # 导出
        dummy_input_ids = torch.randint(0, config.vocab_size, (1, 10))
        torch.onnx.export(
            split_model,
            dummy_input_ids,
            f"llama_split_{split_idx}.onnx",
            input_names=['input_ids', 'attention_mask'] + (
                [f'past_{i}' for i in range(layers_per_split)] if split_idx > 0 else []
            ),
            output_names=['output'] + (
                [f'present_{i}' for i in range(layers_per_split)] if split_idx < num_splits - 1 else ['logits']
            ),
            dynamic_axes={
                'input_ids': {0: 'batch', 1: 'seq_len'},
                'attention_mask': {0: 'batch', 1: 'seq_len'},
                **{f'past_{i}': {2: 'seq_len'} for i in range(layers_per_split)}
            },
            opset_version=14
        )
```

## 性能对比

| 拆分策略 | 内存占用 | 延迟 | 吞吐量 | 复杂度 |
|---------|---------|------|--------|--------|
| 不分拆（单模型） | 高 | 低 | 中 | 低 |
| 按层拆分 | 中 | 中 | 高（流水线） | 中 |

## 最佳实践

### 决策流程

```
是否需要拆分？
├─ 是：内存不足？
│  ├─ 是：使用跨设备Pipeline
│  └─ 否：检查序列长度？
│     ├─ 超长：使用序列分割
│     └─ 高斯：按层拆分
└─ 否：保持完整模型
```

### 拆分的黄金法则

1. **平衡层数**：尽量使每个子模型大小相近（避免bubble）
2. **最小化通信**：减少中间状态的序列/维度变化
3. **保留接口一致**：输入输出维度应该清晰定义
4. **测试等价性**：确保拆分后结果与原始模型一致

```python
def validate_split_equivalence(original_model, split_models, dummy_input):
    """验证拆分模型的输出与原始模型一致"""
    # 原始模型输出
    with torch.no_grad():
        original_out = original_model(dummy_input)

    # 拆分模型流水线输出
    current = dummy_input
    for split_model in split_models:
        current = split_model(current)

    # 比较
    torch.testing.assert_close(original_out, current, rtol=1e-4, atol=1e-4)
    print("✓ 拆分模型验证通过")
```

## 自动拆分工具

```python
def auto_split_model(model, max_memory_per_part, strategy='layer'):
    """
    自动拆分模型以符合内存约束

    Args:
        model: PyTorch模型
        max_memory_per_part: 每个部分的最大内存（字节）
        strategy: 'layer' | 'module' | 'auto'
    """
    from collections import OrderedDict

    layers = list(model.named_children())
    parts = []
    current_part = []
    current_memory = 0

    for name, layer in layers:
        layer_memory = sum(p.numel() * p.element_size() for p in layer.parameters())
        if current_memory + layer_memory > max_memory_per_part:
            # 保存当前部分
            if current_part:
                parts.append(current_part)
            current_part = [(name, layer)]
            current_memory = layer_memory
        else:
            current_part.append((name, layer))
            current_memory += layer_memory

    if current_part:
        parts.append(current_part)

    print(f"Auto split into {len(parts)} parts")
    for i, part in enumerate(parts):
        mem = sum(layer[1].numel() * layer[1].element_size()
                  for layer in part) / 1024**2
        print(f"  Part {i}: {mem:.1f} MB, {len(part)} layers")

    return parts
```

## 挑战与限制

| 挑战 | 描述 | 解决方案 |
|------|------|----------|
| KV Cache传递 | 拆分模型的KV Cache需要跨模型传输 | 设计统一的缓存格式，序列化传递 |
| 梯度计算 | 拆分后训练需要跨越多个设备 | 使用模型并行框架（DeepSpeed, Megatron） |
| 部署复杂度 | 需要管理多个ONNX文件 | 使用模型服务器或统一运行时 |
| 设备不均衡 | 不同设备性能差异导致bottleneck | 根据计算能力动态分配层数 |
