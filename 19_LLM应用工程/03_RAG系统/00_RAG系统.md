# RAG系统

## 背景与发展

检索增强生成（Retrieval-Augmented Generation，RAG）是一种将信息检索与[[大语言模型]]结合的技术范式。通过从外部知识库中检索相关信息来增强LLM的生成能力，解决纯LLM的三大限制：知识时效性、事实准确性（减少幻觉）和可追溯性。RAG由Lewis et al.（2020）提出，现已成为LLM应用工程的核心技术。

## 核心思想

RAG = 检索器 + 生成器。在LLM生成回答前，先从外部知识库中检索与用户查询相关的文档片段，将其注入提示上下文，使LLM基于检索到的知识生成更准确、可追溯的回答。

## 技术原理

### 核心架构

```
用户查询 → 查询理解 → 向量/关键词检索 → 文档重排 → 提示构造 → LLM生成 → 答案输出
                ↑                                              ↑
            嵌入模型                                        知识库
```

### 主要内容

- [[基础RAG]]：RAG系统的基本原理、架构和流程
- [[进阶RAG]]：查询重写、多向量检索、融合检索、重排、索引演进
- [[多模态RAG]]：支持图像、视频等多模态内容的检索增强
- [[向量数据库]]：向量存储、相似度计算、ANN索引

### 核心组件

| 组件 | 功能 | 典型实现 |
|------|------|----------|
| 检索器（Retriever） | 从知识库查找相关文档 | 向量检索、关键词检索 |
| 生成器（Generator） | 基于检索结果生成回答 | GPT、Claude、Llama |
| 知识库（Knowledge Base） | 存储待检索文档 | [[向量数据库]]、文档存储 |
| 嵌入模型（Embedding Model） | 文本转向量表示 | BGE、text-embedding-ada |

## 发展演进

基础RAG（2020）→ 混合检索（2023）→ HyDE/查询重写（2023）→ Self-RAG/CRAG（2024）→ Agentic RAG（2024）→ 端到端可训练RAG

## 关键算法·模型

- 检索：向量检索（ANN）、BM25关键词检索、混合检索、RRF融合
- 重排：Cross-Encoder、BGE-Reranker、ColBERT
- 查询优化：HyDE、查询重写、子查询分解
- 嵌入模型：BGE、E5、text-embedding-ada-002
- 评估：RAGAS、TruLens

## 应用场景

- 企业知识库问答系统
- 文档检索与摘要生成
- 客服系统与产品文档问答
- 法律/医疗等专业领域问答
- 多模态内容理解（[[多模态RAG]]）
