## 1. 注意力机制概述

### 1.1 核心概念

**注意力机制**是一种让模型能够自动关注输入序列中重要部分的机制，它允许模型在处理序列数据时，动态地分配不同的权重给输入序列的各个位置。

### 1.2 发展历程

- **2014年**：Bahdanau等提出基于神经网络的注意力机制，用于机器翻译
- **2017年**：Vaswani等在[[Transformer]]中提出缩放点积注意力和多头注意力，奠定现代LLM基础
- **2019年起**：针对长序列问题，涌现出多种高效注意力变体
- **2022年**：FlashAttention提出，大幅优化注意力计算的内存访问模式
- **2023年**：分组查询注意力（GQA）被广泛应用于大模型，平衡质量和速度

### 1.3 注意力机制的数学本质

注意力机制的核心是计算**查询（Query）**与**键（Key）**的相似度，然后对**值（Value）**进行加权求和：

```
Attention(Q, K, V) = softmax(Score(Q, K))V
```

## 2. 基础注意力机制

### 2.1 加性注意力（Additive Attention）

#### 2.1.1 计算流程

```
Score(Q, K) = v^T tanh(W_q Q + W_k K)
Attention(Q, K, V) = softmax(Score(Q, K))V
```

#### 2.1.2 复杂度分析

- 时间复杂度：O(n d + n m)
- 空间复杂度：O(n m)

#### 2.1.3 特点与适用场景

- 适合Q和K维度不同的场景
- 计算成本较高，但在低维场景下表现良好
- 早期机器翻译模型（如Bahdanau Attention）使用

### 2.2 点积注意力（Dot-Product Attention）

#### 2.2.1 计算流程

```
Score(Q, K) = Q K^T
Attention(Q, K, V) = softmax(Score(Q, K))V
```

#### 2.2.2 复杂度分析

- 时间复杂度：O(n^2 d)
- 空间复杂度：O(n^2)

#### 2.2.3 特点与适用场景

- 计算效率高，可利用GPU并行加速
- 当d较大时，容易出现softmax溢出问题
- 适用于Q和K维度相同的场景

### 2.3 缩放点积注意力（Scaled Dot-Product Attention）

#### 2.3.1 计算流程

```
Score(Q, K) = Q K^T / √d_k
Attention(Q, K, V) = softmax(Score(Q, K))V
```

#### 2.3.2 复杂度分析

- 时间复杂度：O(n^2 d)
- 空间复杂度：O(n^2)

#### 2.3.3 核心改进

- 引入缩放因子1/√d_k，解决点积注意力在高维时的softmax溢出问题
- 当d_k增大时，点积结果的方差会增大，导致softmax函数梯度消失
- 缩放因子保持点积结果的方差为1，使softmax函数处于梯度良好区域

#### 2.3.4 PyTorch实现示例

```python
import torch
import torch.nn.functional as F

def scaled_dot_product_attention(query, key, value, mask=None):
    """
    实现缩放点积注意力
    
    Args:
        query: [batch_size, num_heads, seq_len_q, d_k]
        key: [batch_size, num_heads, seq_len_k, d_k]
        value: [batch_size, num_heads, seq_len_v, d_v]
        mask: [batch_size, 1, seq_len_q, seq_len_k]  # 可选的掩码
    
    Returns:
        output: [batch_size, num_heads, seq_len_q, d_v]
        attention_weights: [batch_size, num_heads, seq_len_q, seq_len_k]
    """
    # 1. 计算点积
    d_k = query.size(-1)
    attention_scores = torch.matmul(query, key.transpose(-2, -1)) / torch.sqrt(torch.tensor(d_k, dtype=torch.float32))
    
    # 2. 应用掩码（如果提供）
    if mask is not None:
        attention_scores = attention_scores.masked_fill(mask == 0, -1e9)
    
    # 3. 计算softmax
    attention_weights = F.softmax(attention_scores, dim=-1)
    
    # 4. 加权求和
    output = torch.matmul(attention_weights, value)
    
    return output, attention_weights
```

## 3. Transformer中的注意力机制

### 3.1 多头注意力（Multi-Head Attention）

#### 3.1.1 核心思想

将Q、K、V通过不同的线性变换投影到多个子空间，在每个子空间中计算注意力，最后将结果拼接并投影得到最终输出。

#### 3.1.2 计算流程

```
# 1. 线性投影
Q_i = Q W_i^Q, K_i = K W_i^K, V_i = V W_i^V

# 2. 缩放点积注意力
head_i = Attention(Q_i, K_i, V_i)

# 3. 拼接并投影
MultiHead(Q, K, V) = Concat(head_1, ..., head_h) W^O
```

#### 3.1.3 复杂度分析

- 时间复杂度：O(n^2 d + h n d)
- 空间复杂度：O(n^2 h + h n d)

#### 3.1.4 优势

- 能够捕获不同子空间的不同注意力模式
- 提高模型的表达能力
- 并行计算多个头，提高效率

### 3.2 自注意力（Self-Attention）

#### 3.2.1 核心思想

使用同一序列作为Q、K、V，让序列中的每个位置关注自身和其他位置。

#### 3.2.2 计算流程

```
SelfAttention(X) = MultiHead(X, X, X)
```

#### 3.2.3 特点与适用场景

- 能够捕获序列内的长距离依赖
- 计算复杂度为O(n^2 d)，随序列长度增长而平方增长
- 是Transformer架构的核心组件

#### 3.2.4 注意力权重热力图

建议插入：注意力权重热力图，展示不同位置之间的注意力强度，直观体现自注意力如何捕获长距离依赖。

### 3.3 因果注意力（Causal/Masked Attention）

#### 3.3.1 核心思想

在自注意力中，限制每个位置只能关注之前的位置，防止信息泄露。

#### 3.3.2 计算流程

```
# 创建下三角掩码
mask = torch.tril(torch.ones(seq_len, seq_len)).bool()

# 应用掩码
attention_scores = attention_scores.masked_fill(mask == 0, -1e9)
```

#### 3.3.3 适用场景

- 仅解码器架构（如GPT系列、[[LLaMA]]、[[Mistral]]）
- 自回归生成任务
- 语言建模（CLM）

#### 3.3.4 实现细节

```python
# 创建因果掩码
seq_len = query.size(-2)
causal_mask = torch.tril(torch.ones(seq_len, seq_len, device=query.device)).view(1, 1, seq_len, seq_len)

# 应用掩码
attention_scores = attention_scores.masked_fill(causal_mask == 0, float('-inf'))
```

## 4. 注意力机制变体

### 4.1 局部注意力（Local Attention）

#### 4.1.1 核心思想

每个位置只关注局部窗口内的位置，减少计算复杂度。

#### 4.1.2 计算流程

```
# 窗口大小为w
for i in range(seq_len):
    window = [i - w//2, i + w//2]
    attention_scores[i, window] = compute_attention(Q[i], K[window], V[window])
```

#### 4.1.3 复杂度分析

- 时间复杂度：O(n w d)
- 空间复杂度：O(n w)

#### 4.1.4 适用场景

- 长序列建模
- 语音处理
- 时间序列预测

### 4.2 稀疏注意力（Sparse Attention）

#### 4.2.1 核心思想

通过某种规则（如随机、块状、固定模式）只计算部分位置对的注意力，减少计算量。

#### 4.2.2 主要变体

- **固定模式注意力**：如Longformer的滑动窗口+全局注意力
- **随机注意力**：随机选择部分Key进行计算
- **块状注意力**：将序列分为块，只计算块内和相邻块的注意力

#### 4.2.3 复杂度分析

- 时间复杂度：O(n m d)，m << n
- 空间复杂度：O(n m)

### 4.3 滑动窗口注意力（Sliding Window Attention）

#### 4.3.1 核心思想

每个位置只关注前k个位置，形成滑动窗口。

#### 4.3.2 计算流程

```
# 窗口大小为k
for i in range(seq_len):
    start = max(0, i - k + 1)
    attention_scores[i, start:i+1] = compute_attention(Q[i], K[start:i+1], V[start:i+1])
```

#### 4.3.3 适用场景

- 自回归生成任务
- GPT-4、[[Mistral]]等模型使用
- 平衡长距离依赖和计算效率

## 5. 高效注意力变体

### 5.1 Linformer

#### 5.1.1 核心创新

使用低秩矩阵近似Key和Value，将复杂度从O(n^2)降低到O(n d)

#### 5.1.2 计算流程

```
# 使用投影矩阵E和F将K和V投影到低维空间
K_proj = E K
V_proj = F V

# 计算注意力
Attention(Q, K, V) = softmax(Q K_proj^T / √d) V_proj
```

#### 5.1.3 复杂度分析

- 时间复杂度：O(n d k + n^2 k)
- 空间复杂度：O(n d + n k)

### 5.2 Performer

#### 5.2.1 核心创新

使用**正交随机特征**（Orthogonal Random Features）近似softmax注意力，将复杂度从O(n^2)降低到O(n d)

#### 5.2.2 计算流程

```
# 使用随机特征映射φ近似exp(Q K^T / √d)
Attention(Q, K, V) ≈ [φ(Q) φ(K)^T V] / [φ(Q) φ(K)^T 1]
```

#### 5.2.3 复杂度分析

- 时间复杂度：O(n d + n m)
- 空间复杂度：O(n d + m d)

### 5.3 FlashAttention

#### 5.3.1 核心创新

通过**分块计算**和**内存访问优化**，减少GPU内存访问次数，实现更快的注意力计算

#### 5.3.2 计算流程

1. 将Q、K、V分块加载到GPU SRAM
2. 在SRAM中计算注意力
3. 只将结果写回HBM，而不是中间结果
4. 利用GPU的并行计算能力

#### 5.3.3 复杂度分析

- 时间复杂度：O(n^2 d)（理论上）
- 实际速度：比标准实现快2-4倍
- 内存占用：减少5-10倍

#### 5.3.4 适用场景

- 所有需要注意力机制的场景
- 长序列建模
- 大规模模型训练

#### 5.3.5 代码示例（概念）

```python
def flash_attention(query, key, value, mask=None):
    """FlashAttention的简化概念实现"""
    batch_size, num_heads, seq_len_q, d_k = query.shape
    seq_len_k = key.shape[2]
    
    # 分块大小
    block_size = 128
    
    output = torch.zeros_like(query)
    
    # 按块计算
    for i in range(0, seq_len_q, block_size):
        # 加载Q块
        q_block = query[:, :, i:i+block_size, :]
        
        for j in range(0, seq_len_k, block_size):
            # 加载K、V块
            k_block = key[:, :, j:j+block_size, :]
            v_block = value[:, :, j:j+block_size, :]
            
            # 计算注意力
            attn = torch.matmul(q_block, k_block.transpose(-2, -1)) / math.sqrt(d_k)
            
            # 应用掩码
            if mask is not None:
                mask_block = mask[:, :, i:i+block_size, j:j+block_size]
                attn = attn.masked_fill(mask_block == 0, -1e9)
            
            attn = F.softmax(attn, dim=-1)
            
            # 加权求和
            out_block = torch.matmul(attn, v_block)
            
            # 累加到输出
            output[:, :, i:i+block_size, :] += out_block
    
    return output
```

### 5.4 MQA（Multi-Query Attention）

#### 5.4.1 核心创新

所有头共享同一组K和V，只保留多头Q

#### 5.4.2 计算流程

```
# 多头Q，单头K、V
Q_1, ..., Q_h = Q W_1^Q, ..., Q W_h^Q
K = K W^K, V = V W^V

# 每个头使用相同的K和V
head_i = Attention(Q_i, K, V)

# 拼接并投影
MQA(Q, K, V) = Concat(head_1, ..., head_h) W^O
```

#### 5.4.3 复杂度分析

- 时间复杂度：O(n^2 d + h n d)
- 空间复杂度：O(n^2 h + n d)（比标准多头注意力低h倍）

### 5.5 GQA（Grouped Query Attention）

#### 5.5.1 核心创新

将头分组，每组共享同一组K和V

#### 5.5.2 计算流程

```
# 分组
num_groups = h // g

# 每组共享K和V
for g in range(num_groups):
    Q_group = Q[:, g*g:(g+1)*g, :, :]
    K_group = K[:, g, :, :].unsqueeze(1).repeat(1, g, 1, 1)
    V_group = V[:, g, :, :].unsqueeze(1).repeat(1, g, 1, 1)
    
    head_group = Attention(Q_group, K_group, V_group)
    groups.append(head_group)

# 拼接并投影
GQA(Q, K, V) = Concat(groups) W^O
```

#### 5.5.3 特点与适用场景

- 平衡内存占用和模型质量
- [[LLaMA 2 70B]]、GPT-4使用
- 比MQA质量更高，比标准多头注意力内存占用更低

## 7. 工程实践与优化

### 7.1 常见陷阱与解决方案

#### 7.1.1 Softmax溢出

**问题**：当注意力分数过大时，softmax计算会溢出

**解决方案**：
- 使用缩放点积注意力
- 减去最大值，进行数值稳定：`softmax(x) = softmax(x - max(x))`
- 使用对数空间计算

#### 7.1.2 KV缓存管理

**问题**：长序列下KV缓存占用大量内存

**解决方案**：
- 使用MQA/GQA减少缓存大小
- 实现动态缓存管理
- 对于超长序列，采用滑动窗口或稀疏注意力

#### 7.1.3 长序列截断

**问题**：超过模型最大序列长度的输入需要截断

**解决方案**：
- 智能截断策略（如保留首尾重要信息）
- 使用长上下文扩展技术（如RoPE扩展）
- 采用结构化注意力机制

### 7.2 优化策略

#### 7.2.1 内存优化

- 使用[[FlashAttention]]减少内存占用
- 采用混合精度训练（FP16/BF16）
- 实现梯度检查点

#### 7.2.2 速度优化

- 利用GPU并行计算
- 使用CUDA内核优化
- 实现批处理
- 采用高效注意力变体

#### 7.2.3 质量优化

- 调整头数和隐藏维度
- 选择合适的注意力机制
- 结合其他架构组件（如[[位置编码]]）

## 8. 注意力机制的未来发展

### 8.1 瓶颈与挑战

- O(n²)复杂度限制了长序列处理
- 内存占用随序列长度平方增长
- 长距离依赖建模困难
- 计算效率待提高

### 8.2 替代方向

#### 8.2.1 状态空间模型（SSM）

- 如Gated Linear Units (GLU)、Hyena、Mamba
- 线性复杂度O(n d)
- 能够捕获长距离依赖
- 适合长序列建模

#### 8.2.2 线性注意力

- 如Performer、Linformer
- 将注意力计算转换为线性操作
- 复杂度O(n d)
- 适合超长序列

#### 8.2.3 混合架构

- 结合注意力机制和状态空间模型
- 如RetNet、H3、Mamba-2
- 兼顾注意力的灵活性和SSM的效率

### 8.3 最新研究进展

#### 8.3.1 FlashAttention-2 (ICML 2024)

- 进一步优化内存访问
- 支持更长的序列长度
- 速度比FlashAttention快1.5-2倍

#### 8.3.2 LongLoRA (NeurIPS 2023)

- 扩展LLM的上下文长度到100K+
- 只训练注意力层的低秩适配器
- 保持模型质量的同时大幅减少训练成本

#### 8.3.3 Grouped-Query Attention for Long Context (ACL 2024)

- 针对长上下文优化GQA
- 提出动态分组策略
- 在128K序列上实现高效推理
