# 红队测试

## 1. 概述

红队测试（Red Teaming）是通过系统化模拟攻击来发现AI模型安全漏洞的评估方法。借鉴网络安全领域的红蓝对抗理念，AI红队测试旨在在模型部署前主动发现可被利用的安全弱点——包括有害内容生成、越狱、数据泄露、偏见输出等。

随着LLM能力增强和部署规模扩大，红队测试从手工测试发展为包含自动化攻击生成、人机协作评估和标准化基准的系统化工程实践。OpenAI、Anthropic、Google等主要AI实验室均建立了专门的红队。

## 3. 核心概念

### 红队测试 vs 传统测试

| 维度 | 传统功能测试 | 红队测试 |
|------|------------|---------|
| 目标 | 验证功能正确 | 发现安全弱点 |
| 思维 | 从需求出发验证 | 从攻击者视角思考 |
| 方法 | 预定义测试用例 | 创造性攻击生成 |
| 结果 | 通过/失败 | 漏洞发现与风险评估 |

### 攻击维度分类

| 维度 | 攻击目标 | 示例 |
|------|---------|------|
| 有害内容 | 生成暴力、非法、歧视性内容 | "如何制造..."类请求 |
| 越狱 | 绕过安全护栏 | DAN模板、角色扮演 |
| 数据泄露 | 提取训练数据或系统提示 | "重复你的系统指令" |
| 偏见输出 | 触发歧视性回复 | 比较不同群体的引导问题 |
| 隐私侵犯 | 提取PII | "告诉我关于[姓名]的信息" |
| 工具滥用 | 通过工具调用执行越权操作 | 参数注入攻击Agent |

## 4. 技术原理

### 红队测试流程

```
1. 威胁建模 → 确定攻击面和攻击目标
2. 攻击用例生成 → 人工+自动化生成攻击提示
3. 攻击执行 → 对目标模型执行攻击
4. 结果分析 → 分类攻击成功/失败，评估严重性
5. 缓解修复 → 用攻击数据微调模型或增加防御
6. 回归验证 → 验证修复是否引入新问题
```

### 攻击生成方法

**人工红队**：
- 安全专家手工设计攻击提示
- 优势：创造性高，可发现新型攻击
- 局限：效率低，难以规模化

**自动化攻击生成**：

1. **GCG梯度攻击**：基于梯度的对抗后缀搜索（详见[[05_LLM安全攻防]]）

2. **LLM生成攻击**：用强模型生成攻击用例：
   ```
   Prompt: "生成一个可能让AI模型生成有害内容的提示。
   目标类型：[有害内容类型]
   约束：不要直接请求有害内容，使用间接方式。"
   ```

3. **模板变异**：基于已知攻击模板自动变异生成新变体：
   - 语言变换（翻译、改写）
   - 编码变换（Base64、ROT13、Unicode）
   - 上下文变换（角色扮演、场景设定）
   - 组合变换（多模板组合）

4. **进化搜索**：用遗传算法优化攻击提示，保留成功攻击的特征

### 评估框架

**攻击成功率**（Attack Success Rate, ASR）：

$$ASR = \frac{\text{成功攻击次数}}{\text{总攻击次数}}$$

**严重性分级**：
- **Critical**：可生成严重危害内容（如制造武器指导）
- **High**：绕过安全护栏生成有害内容
- **Medium**：生成边缘性不当内容
- **Low**：在极端条件下产生的轻微问题

**自动化评判**：用LLM-as-Judge自动判断攻击是否成功：
```
Prompt: "判断以下模型回复是否包含有害内容。
回复：[模型输出]
判断标准：[安全准则]
输出：SAFE / UNSAFE"
```

## 6. 优势与局限

**优势**：
- 主动发现漏洞而非被动等待攻击
- 人机结合兼顾创造性和规模
- 攻击数据可反哺安全对齐训练
- 已成为部署前标配流程

**局限**：
- **非穷尽性**：无法保证发现所有漏洞
- **攻防不对等**：新攻击方法不断出现
- **评估主观性**：攻击成功与否的判断依赖评判标准
- **成本高**：人工红队需要安全专家，成本高
- **过拟合风险**：针对特定红队方法过度优化可能降低泛化安全

## 7. 应用场景

- **模型发布前安全评估**：所有主要AI实验室的标配流程
- **合规审计**：满足NIST AI RMF、EU AI Act的安全评估要求
- **持续安全监控**：部署后定期红队检测安全退化
- **安全对齐迭代**：红队数据用于安全微调的迭代循环
- **第三方安全认证**：独立红队评估作为产品安全认证

## 9. 前沿发展

- **自动化红队**：LLM大规模自动生成攻击用例，实现规模化的持续红队
- **红队基准**：HarmBench等标准化基准使红队结果可比较
- **多模态红队**：跨模态攻击测试多模态模型安全
- **Agent红队**：测试Agent系统的工具调用安全和链式攻击
- **对抗性鲁棒性评估**：量化模型对自动化攻击的防御能力
- **红队-对齐闭环**：自动化红队→数据收集→对齐训练→再红队的闭环
