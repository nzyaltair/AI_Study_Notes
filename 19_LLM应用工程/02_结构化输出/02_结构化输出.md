# 结构化输出

## 背景与发展

大语言模型天然倾向于生成自由文本，而实际应用需要可编程的结构化数据（JSON、XML、函数调用参数等）。结构化输出是连接LLM与外部系统的桥梁，使LLM能与API、数据库、函数调用等编程组件无缝集成。随着[[工具调用]]和Function Calling的普及，结构化输出成为LLM应用工程的关键基础设施。

## 核心思想

通过提示引导、格式约束、解码控制或后处理校验，使LLM输出严格符合预定义的结构化格式（如JSON Schema），实现"自由文本→可编程数据"的可靠转换。

## 技术原理

### 方法分类

| 方法类型 | 实现层面 | 核心机制 | 适用场景 |
|----------|----------|----------|----------|
| 提示工程 | 输入侧 | 通过提示词引导LLM生成结构化输出 | 快速原型、简单结构 |
| 输出格式约束 | 输出侧 | JSON Schema定义严格格式规则 | 复杂结构、高可靠性 |
| 解码层控制 | 解码侧 | 生成过程中实时控制token选择 | 高性能、严格格式 |
| 后处理与校验 | 后处理 | 验证和修正生成的输出 | 多模型兼容场景 |
| 工具集成 | 框架层 | 专用工具链简化实现 | 生产环境 |

### 提示工程方法

在提示中嵌入JSON Schema或格式示例，指导LLM生成符合Schema的输出：

```python
schema = {
    "type": "object",
    "properties": {
        "产品名称": {"type": "string"},
        "价格": {"type": "number"},
        "品牌": {"type": "string"}
    },
    "required": ["产品名称", "价格", "品牌"]
}
prompt = f"请按以下JSON Schema输出：\n{json.dumps(schema)}"
```

### 输出格式约束

利用模型内置的JSON Schema支持或Function Calling，直接传递Schema给模型：

```python
response = client.chat.completions.create(
    model="gpt-4-turbo",
    messages=[{"role": "user", "content": "查询北京天气"}],
    response_format={"type": "json_object"}
)
```

### 解码层控制（Constrained Decoding）

在解码过程中实时过滤不符合格式要求的token，只允许生成符合约束的token序列。核心原理是在每一步生成时，根据当前已生成文本和格式规则，计算合法的token集合，将非法token的logits设为负无穷。

### Grammar-based Decoding

通过定义形式文法（如CFG、BNF）约束模型生成的token序列，确保输出符合特定语法规则。Llama.cpp等推理引擎原生支持grammar约束。

### 后处理与校验

- **正则过滤**：提取JSON部分并尝试解析，失败时尝试修复
- **LLM-as-a-Judge**：使用另一个LLM验证和修正输出

### 主流工具

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| Outlines | 基于JSON Schema的结构化生成 | 生产环境 |
| Guidance | 微软开发，支持条件生成和格式约束 | 复杂条件生成 |
| lm-format-enforcer | 轻量级格式约束 | 高性能场景 |
| vLLM Logits Processors | 高性能推理引擎集成 | 大规模部署 |

## 发展演进

提示词引导 → JSON Mode（2023）→ Function Calling（2023，OpenAI）→ Constrained Decoding（Outlines/Guidance, 2023）→ Grammar-based Decoding（Llama.cpp）→ 原生结构化输出模型

## 关键算法·模型

- **Constrained Decoding**：在token采样时实时过滤非法token
- **Grammar-based Decoding**：基于CFG/BNF文法约束生成
- **Outlines**：基于JSON Schema的结构化生成框架
- **Guidance**：微软的条件生成与格式约束框架
- **Cross-Encoder重排**：用于结构化输出质量校验

## 应用场景

- API调用：LLM生成API调用参数
- [[RAG]]结果解析：从检索结果中提取结构化信息
- [[Agent]]工具调用：Agent生成工具调用参数
- 数据标注：自动提取结构化标注
- 数据库写入：LLM输出直接写入数据库
