# 计算机视觉工程实践与前沿

## 1. 概述

计算机视觉的工程实践是将研究成果转化为可用产品的关键环节。一个 CV 系统的生命周期涵盖数据处理、模型训练、评估调优、部署推理和监控维护。从实验室原型到生产级系统，需要解决性能、延迟、吞吐量、鲁棒性和成本等工程挑战。

本篇梳理 CV 工程实践的主流框架、部署优化技术和前沿发展趋势，作为整个计算机视觉知识体系的实践总结。

## 3. 核心概念

### 3.1 CV 系统管线


### 3.2 推理优化目标

|:---|:---|:---|
| 延迟（Latency） | 单次推理时间 | 实时应用关键 |

| 显存占用 | GPU 内存使用量 | 边缘部署约束 |
| 模型大小 | 存储空间需求 | 移动端部署约束 |
| 精度（Accuracy） | 推理结果准确性 | 核心质量指标 |

## 4. 技术原理

### 4.1 主流框架与工具链

#### OpenCV

传统 CV 的标准库，覆盖图像处理、特征提取、几何变换、视频 I/O 等：

```python
import cv2
# 图像读取
img = cv2.imread('image.jpg')
# 高斯滤波
blurred = cv2.GaussianBlur(img, (5,5), 1.0)
# Canny 边缘检测
edges = cv2.Canny(blurred, 100, 200)
# SIFT 特征
sift = cv2.SIFT_create()
kp, des = sift.detectAndCompute(cv2.cvtColor(img, cv2.COLOR_BGR2GRAY), None)
```

#### PyTorch + TorchVision

深度学习 CV 的主流训练框架：

```python
import torchvision.models as models
# 预训练分类模型
model = models.resnet50(pretrained=True)
# 预训练检测模型
from torchvision.models.detection import fasterrcnn_resnet50_fpn
detector = fasterrcnn_resnet50_fpn(pretrained=True)
```

#### OpenMMLab

涵盖 CV 全任务的模块化工具链：

| 工具 | 任务 |
|:---|:---|
| MMDetection | 目标检测/实例分割 |
| MMSegmentation | 语义/全景分割 |
| MMClassification | 图像分类 |
| MMPose | 姿态估计 |
| MMTracking | 视频跟踪 |

#### Detectron2

Meta 开源的检测/分割研究平台，灵活可扩展。

### 4.2 模型压缩与加速

#### 量化（Quantization）

将浮点权重转换为低精度整数：

- **PTQ（训练后量化）**：直接量化训练好的模型，无需重训练
  - 对称量化：$q = \text{round}(x / s)$, $s = \frac{\max(|x|)}{127}$
  - 非对称量化：$q = \text{round}(x / s) + z$
- **QAT（量化感知训练）**：在训练中模拟量化误差，精度更高

| 精度 | 位宽 | 精度损失 | 加速比 |
|:---|:---|:---|:---|
| FP32 | 32 bit | 基准 | 1× |
| FP16 | 16 bit | <0.5% | ~2× |
| INT8 | 8 bit | 1-3% | ~3-4× |
| INT4 | 4 bit | 3-8% | ~6-8× |

参见 [[12_大模型推理与优化/02_量化技术]]。

#### 知识蒸馏

用大模型（教师）指导小模型（学生）训练：

$$\mathcal{L} = \alpha \cdot \mathcal{L}_{CE}(y, y_s) + (1-\alpha) \cdot \mathcal{L}_{KD}(p_t, p_s)$$

$$\mathcal{L}_{KD} = T^2 \cdot \text{KL}(p_t^T \| p_s^T)$$

其中 $T$ 为温度，$p_t, p_s$ 为教师/学生的 softmax 输出。

参见 [[21_知识蒸馏与模型压缩]]。

#### 剪枝

移除不重要的权重或通道：

- **非结构化剪枝**：逐权重置零，稀疏但需特殊硬件支持
- **结构化剪枝**：移除整个通道/层，无需特殊硬件

参见 [[12_大模型推理与优化/03_剪枝与稀疏化]]。

## 6. 优势与局限

### 工程实践优势

1. **工具链成熟**：从训练到部署有完整开源工具链
2. **部署方案多样**：覆盖云端到边缘全场景
3. **优化技术丰富**：量化/蒸馏/剪枝有成熟方案
4. **社区活跃**：OpenMMLab/Detectron2 等持续更新

### 工程实践挑战

1. **精度-速度权衡**：实时性与精度常需取舍
2. **领域迁移**：研究模型到实际场景的泛化差距
3. **数据分布漂移**：部署后数据分布变化导致性能退化
4. **长尾问题**：罕见类别/场景性能差
5. **多模态融合复杂**：融合多传感器数据工程复杂

## 7. 应用场景

### 7.1 自动驾驶感知

```
多摄像头 + LiDAR → BEV 感知 → 3D 检测 + 车道线 + 分割 → 融合 → 规划
```

关键要求：低延迟（<50ms）、高安全性、多模态融合。

### 7.2 工业质检

```
产线图像 → 缺陷检测/分割 → 分类/定位 → 报警/分拣
```

关键要求：高精度（>99%）、高吞吐、可解释性。

### 7.3 医学影像

```
医学图像（CT/MRI/病理）→ 预处理 → 分割/检测 → 诊断辅助
```

关键要求：合规性（FDA/NMPA）、可解释性、小数据泛化。

### 7.4 安防监控

```
视频流 → 实时检测 + 跟踪 → 行为分析 → 告警
```

关键要求：实时性、多路并发、低误报率。

## 9. 前沿发展

### 视觉基础模型

- **SAM 系列**：Meta SAM/SAM 2 实现通用分割，零样本迁移
- **DINOv2**：自监督视觉基础模型，通用视觉表示
- **Grounding DINO**：开放词汇检测，文本引导检测
- **Depth Anything**：单目深度估计基础模型

### 端到端视觉系统

- **自动驾驶**：从感知到规划的端到端学习（UniAD 等）
- **具身智能**：视觉感知 → 理解 → 行动的闭环
- **视觉-语言-行动模型**：统一视觉理解和机器人控制

### 多模态统一

- **统一架构**：视觉和语言在同一 Transformer 中处理
- **Any-to-Any 生成**：统一图像/视频/文本/音频的生成模型
- **视觉对话**：基于图像的多轮对话交互

### 实时高效视觉

- **Flash Attention**：加速 Transformer 注意力计算
- **神经架构搜索（NAS）**：自动搜索最优网络结构
- **端侧大模型**：在移动设备上运行 ViT/扩散模型
- **稀疏计算**：动态计算图减少冗余计算

### 可信视觉

- **鲁棒性**：对抗攻击防御、分布外泛化
- **公平性**：视觉模型在不同人群上的公平性
- **可解释性**：注意力可视化、概念发现
- **隐私保护**：联邦学习、差分隐私在视觉中的应用

### 新兴方向

- **事件相机视觉**：基于神经形态传感器的视觉处理
- **偏振视觉**：利用光的偏振信息增强场景理解
- **热红外视觉**：夜间/恶劣天气下的视觉感知
- ** underwater 视觉**：水下场景的图像恢复与理解
