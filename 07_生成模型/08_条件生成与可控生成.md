# 条件生成与可控生成

## 1. 概述

条件生成与可控生成是生成模型从"能生成"到"能控制"的关键跨越。条件生成指在给定条件输入（文本、类别、草图等）下生成符合条件的内容；可控生成则进一步强调对生成结果的精细控制——包括结构控制、风格控制、身份保持等。

现代条件生成体系以扩散模型为核心，结合 Classifier-Free Guidance、ControlNet、LoRA 等技术，实现了前所未有的生成可控性。

## 3. 核心概念

### 3.1 条件类型

| 条件类型 | 输入形式 | 代表方法 |
|---------|---------|---------|
| 文本条件 | 自然语言描述 | CLIP/T5 编码 + 交叉注意力 |
| 类别条件 | 类别标签 | 类别嵌入 + AdaLN |
| 图像条件 | 参考图像 | IP-Adapter、Reference |
| 结构条件 | 边缘/深度/姿态图 | ControlNet、T2I-Adapter |
| 风格条件 | 风格图像 | Style Aligned、InstantStyle |
| 身份条件 | 人脸图像 | InstantID、PhotoMaker |
| 布局条件 | 边界框/分割图 | GLIGEN、Layout-Guidance |

### 3.2 条件注入方式

1. **交叉注意力**：文本 token 通过交叉注意力注入 U-Net/DiT
2. **AdaLN**：条件和时间步通过自适应层归一化调制特征
3. **拼接**：条件图与输入直接通道拼接（如 Pix2Pix）
4. **ControlNet 分支**：独立的条件控制网络，输出叠加到主网络
5. **IP-Adapter**：图像条件通过解耦的交叉注意力注入

### 3.3 引导强度与质量-多样性权衡

- **低引导**：高多样性、低保真度、条件一致性弱
- **高引导**：低多样性、高保真度、条件一致性强
- **过引导**：伪影增加、色彩失真

## 4. 技术原理

### 4.1 Classifier-Free Guidance (CFG)

同时训练条件模型 $\epsilon_\theta(x_t, t, c)$ 和无条件模型 $\epsilon_\theta(x_t, t, \varnothing)$（训练时随机丢弃条件），推理时差分引导：

$$\tilde{\epsilon} = \epsilon_\theta(x_t, t, \varnothing) + w \cdot [\epsilon_\theta(x_t, t, c) - \epsilon_\theta(x_t, t, \varnothing)]$$

等价于：

$$\tilde{\epsilon} = (1 + w) \cdot \epsilon_\theta(x_t, t, c) - w \cdot \epsilon_\theta(x_t, t, \varnothing)$$

- $w = 0$：纯条件生成（无引导）
- $w = 1$：标准条件生成
- $w = 3 \sim 7$：常用范围，强条件一致性
- $w > 10$：过引导，可能出现伪影

**数学直觉**：CFG 近似于在 $p(x|c)^w \cdot p(x)^{1-w}$ 分布上采样，$w > 1$ 相当于提高条件分布的权重。

### 4.2 Classifier Guidance

使用外部分类器 $p_\phi(c | x_t)$ 的梯度引导：

$$\tilde{\epsilon} = \epsilon_\theta(x_t, t) - s \cdot \sigma_t \nabla_{x_t} \log p_\phi(c | x_t)$$

**与 CFG 对比**：
- Classifier Guidance 需要额外训练噪声鲁棒分类器
- CFG 更简单（同一模型处理条件和无条件）
- CFG 已成为主流标准

### 4.3 ControlNet

**核心思想**：在预训练扩散模型旁添加一个可训练的副本（ControlNet），接收结构条件输入，输出残差特征注入主网络。

**架构**：
1. 复制 U-Net 的编码器和中间块作为 ControlNet
2. 输入：条件图（如 Canny 边缘图）+ 噪声潜变量
3. ControlNet 各层输出通过零卷积（zero convolution）注入主 U-Net
4. 训练时冻结主网络，只训练 ControlNet

**零卷积**：权重和偏置初始化为 0，确保训练开始时不影响主网络，训练过程中逐步学习调整。

**优势**：
- 不破坏预训练模型的能力
- 可以添加多种结构条件（边缘、深度、姿态、分割图等）
- 可以组合多个 ControlNet

**支持的条件的类型**：
- Canny 边缘、HED 边缘、Scribble 涂鸦
- 深度图（MiDaS、Depth Anything）
- 姿态估计（OpenPose）
- 语义分割图
- 法线图
- 人体姿态/手部姿态

### 4.4 T2I-Adapter

与 ControlNet 类似但更轻量：
- 仅在 U-Net 编码器各尺度添加小型适配器网络
- 参数量远小于 ControlNet
- 适合轻量部署

### 4.5 LoRA (Low-Rank Adaptation)

在预训练模型权重上添加低秩分解的增量：

$$W' = W + \Delta W = W + B A, \quad B \in \mathbb{R}^{d \times r}, A \in \mathbb{R}^{r \times d}, r \ll d$$

**在生成模型中的应用**：
- 微调特定风格/角色
- 训练成本低（仅训练 $A, B$）
- 可组合（多个 LoRA 可叠加）
- 模型小（几 MB ~ 几百 MB）

**训练**：
- 冻结原始权重 $W$
- $A$ 用高斯初始化，$B$ 初始化为 0
- 学习率通常比全量微调高

### 4.6 Textual Inversion

学习新概念的文本嵌入：
1. 选择少量目标概念图像（3-5 张）
2. 为概念分配一个占位符 token（如 `<my-cat>`）
3. 固定模型，仅训练该 token 的嵌入向量
4. 推理时在提示词中使用该 token 引用概念

### 4.7 IP-Adapter

实现图像作为条件（图像 Prompt）：
- 将图像通过 CLIP 图像编码器提取特征
- 通过解耦的交叉注意力注入模型
- 图像和文本条件通过独立的交叉注意力层
- 可同时使用文本和图像条件

**解耦交叉注意力**：

$$\text{Attn}_{img}(Q, K_{img}, V_{img}) = \text{softmax}\left(\frac{Q K_{img}^T}{\sqrt{d}}\right) V_{img}$$

与文本的交叉注意力并行，通过可学习权重组合。

### 4.8 InstantID / PhotoMaker

身份保持生成：
- 提取人脸的身份特征（如 ArcFace 嵌入）
- 将身份嵌入通过投影层注入交叉注意力
- 结合文本提示和参考人脸生成保持身份的图像

### 4.9 DALL-E 3 提示词重写

利用 LLM 增强文本理解：
1. 训练 LLM 将用户简短提示重写为详细描述
2. 用详细描述作为扩散模型的输入
3. 缓解"模型不懂用户想要什么"的问题

## 6. 优势与局限

### 6.1 优势

- **前所未有的可控性**：从文本到结构到身份的全方位控制
- **即插即用**：大多数方法不需要重训基础模型
- **可组合**：多种控制方法可叠加使用
- **个性化定制**：低成本适配特定风格/角色/概念

### 6.2 局限

- **多条件冲突**：不同条件之间可能矛盾
- **引导伪影**：过强引导导致色彩失真和伪影
- **控制精度有限**：复杂场景的精确控制仍有挑战
- **依赖基础模型**：控制方法的能力受限于基础模型
- **训练数据需求**：ControlNet 等需要配对训练数据

## 7. 应用场景

- **创意设计**：精确控制构图、风格、色彩
- **角色设计**：保持角色一致性的多角度生成
- **室内设计**：从草图到效果图
- **电商**：产品图换背景、换风格
- **虚拟试衣**：服装在不同人物上的效果
- **动画制作**：角色姿态和表情控制
- **广告创意**：品牌风格一致的内容生成

## 8. 与其他技术关系

- 条件生成是 [[04_扩散模型]] 的核心应用方向

- LoRA 技术源自大语言模型微调，与参数高效微调章节关联
- ControlNet 的思想（旁路分支）与残差学习相关
- 提示词重写利用 LLM，与大语言模型章节关联

## 9. 前沿发展

- **多条件统一框架**：统一处理文本、图像、结构等多种条件
- **3D 可控生成**：ControlNet 在 3D 生成中的扩展
- **视频可控生成**：时空条件控制视频生成
- **无需训练的控制**：利用注意力操作实现零样本控制（如 Attend-and-Excite）
- **自然语言控制**：通过对话式交互调整生成结果
- **可控性与创造力的平衡**：在精确控制下保持生成多样性
