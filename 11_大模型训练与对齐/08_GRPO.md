# GRPO

## 1. 概述

GRPO（Group Relative Policy Optimization，组内相对策略优化）是DeepSeek提出的强化学习对齐算法，首次在DeepSeekMath（Shao et al., 2024）中提出，后在DeepSeek-R1（2025）中大规模应用于推理模型训练。GRPO简化了传统RLHF的流程，去除了独立的奖励模型和价值网络（Critic），直接使用组内相对优势作为奖励信号。

GRPO的成功证明了无需复杂RLHF流程也能训练出高性能推理模型，特别适合数学推理、代码生成等有明确正确答案的任务。

## 3. 核心概念

### 3.1 组内相对比较

GRPO的核心创新是对同一问题生成一组（Group）候选回答，通过组内相对比较计算每个回答的优势值（advantage）。这种方法不需要训练独立的奖励模型，也不需要Critic网络来估计基线。

### 3.2 去除Critic

传统PPO需要训练一个价值网络（Critic）来估计状态值，作为优势函数的基线。GRPO用组内均值替代Critic，减少了训练开销和显存占用。

### 3.3 规则奖励

GRPO特别适合使用基于规则的奖励函数（如答案正确性检查），这避免了奖励模型的偏差，特别适合有明确正确答案的推理任务。

### 3.4 R1-Zero vs R1

- **DeepSeek-R1-Zero**：直接在基座模型上用GRPO训练，无SFT阶段。证明了纯RL可以训练推理能力，但存在可读性差等问题。
- **DeepSeek-R1**：在R1-Zero基础上增加冷启动SFT数据，结合GRPO训练，性能对标OpenAI o1。

## 4. 技术原理

### 4.1 算法流程

1. 输入问题 $q$，从当前策略 $\pi_\theta$ 采样 $G$ 个回答 $\{o_1, o_2, \ldots, o_G\}$
2. 计算每个回答的奖励 $r_i$（规则奖励或模型奖励）
3. 组内标准化计算优势值：$\tilde{A}_i = (r_i - \text{mean}(r)) / \text{std}(r)$
4. 用PPO变体更新策略 $\pi_\theta$

### 4.2 优势值计算

组内标准化将绝对奖励转化为相对优势：

$$\tilde{A}_i = \frac{r_i - \text{mean}(r_1, \ldots, r_G)}{\text{std}(r_1, \ldots, r_G)}$$

这种方法的关键在于：**不需要绝对奖励值精确，只需要组内相对排序正确**。即使奖励函数有系统性偏差，组内标准化也能消除其影响。

### 4.3 策略优化目标

带KL惩罚和PPO裁剪的优化目标：

$$J_{GRPO} = \mathbb{E}\left[\frac{1}{G}\sum_{i=1}^{G}\left(\min\left(\frac{\pi_\theta(o_i|q)}{\pi_{old}(o_i|q)} \tilde{A}_i, \, \text{clip}\left(\frac{\pi_\theta(o_i|q)}{\pi_{old}(o_i|q)}, 1-\epsilon, 1+\epsilon\right) \tilde{A}_i\right) - \beta \, D_{KL}(\pi_\theta \| \pi_{ref})\right)\right]$$

### 4.4 与RLHF的关键区别

| 特征 | RLHF（PPO） | GRPO |
|------|------------|------|
| 奖励模型 | 需要独立训练 | 不需要（规则奖励或组内比较） |
| 基线计算 | 价值网络（Critic） | 组内均值 |
| 训练成本 | 高（RM + Critic + Actor + Ref） | 低（Actor + Ref） |
| 显存占用 | 大（4个模型） | 小（2个模型） |
| 稳定性 | Critic训练不稳定 | 更稳定（无Critic） |
| 在线性 | 在线（on-policy） | 在线（on-policy） |
| 适用场景 | 通用对齐 | 推理任务（有明确答案） |

### 4.5 奖励函数设计

DeepSeek-R1使用两类基于规则的奖励：

- **正确性奖励**：数学题答案正确性验证、代码通过测试用例
- **格式奖励**：思维链使用 `<think)` 和 `</think)` 标签包裹

这种基于规则的奖励避免了奖励模型的偏差，特别适合有明确正确答案的推理任务。

### 4.6 DeepSeek-R1训练流程

1. **基座模型**：DeepSeek-V3（671B MoE）
2. **冷启动SFT**（仅R1）：用少量高质量CoT数据进行SFT
3. **GRPO推理训练**：以规则奖励为信号，训练推理能力
4. **拒绝采样SFT**：从RL模型生成数据，筛选高质量样本
5. **第二轮GRPO**：结合推理和通用对齐

## 6. 优势与局限

### 优势

- **去除Critic**：减少一个模型的训练，降低显存和计算开销
- **去除奖励模型**：规则奖励避免RM偏差和Hacking
- **训练稳定**：无Critic训练不稳定问题
- **适合推理任务**：规则奖励天然适合有明确答案的任务
- **在线学习**：on-policy采样，探索能力强

### 局限

- **依赖规则奖励**：需要可验证的正确性判断，不适用于开放式对话
- **采样成本高**：每个问题需采样 $G$ 个回答，推理开销大
- **组大小敏感**：$G$ 太小则统计量不稳定，太大则计算开销高
- **仍需参考模型**：需要维护参考模型计算KL约束
- **通用对齐能力有限**：不适合没有明确正确答案的对齐任务

## 7. 应用场景

- **推理模型训练**：数学推理、代码生成等有明确正确答案的任务
- **低成本对齐**：无需训练奖励模型，适合资源受限场景
- **大规模训练**：显存占用小，适合大规模分布式训练
- **RL+CoT训练**：训练模型的思维链推理能力

## 8. 与其他技术关系

- GRPO是[[05_RLHF|RLHF]]的简化变体，去除了奖励模型和Critic
- 与[[06_DPO|DPO]]不同，GRPO仍使用在线强化学习（on-policy）
- GRPO的PPO裁剪机制来源于PPO算法（方向13）
- KL惩罚项参考了RLHF的设计
- GRPO是推理模型训练的关键技术（详见方向10的推理模型与思维链）

## 9. 前沿发展

### 9.1 推理模型训练

- **纯RL训练**：R1-Zero证明可以跳过SFT直接RL训练
- **冷启动数据**：少量高质量CoT数据显著提升RL训练效果
- **多阶段训练**：SFT → RL → 拒绝采样 → SFT → RL的迭代流程

### 9.2 奖励函数改进

- **过程奖励模型（PRM）**：逐步评估推理过程而非只评估最终答案
- **多维度奖励**：结合正确性、可读性、简洁性等多维度
- **自适应奖励**：根据训练阶段动态调整奖励权重

### 9.3 通用化扩展

- **从推理到通用对齐**：将GRPO扩展到没有明确答案的通用对齐任务
- **多模态GRPO**：扩展到视觉/音频等模态
- **GRPO + DPO混合**：结合在线和离线方法的优势
