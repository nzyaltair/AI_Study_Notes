# 模型选择与调参

## 1. 概述

模型选择（Model Selection）与超参数调优（Hyperparameter Tuning）是机器学习工作流中的关键步骤。No Free Lunch 定理表明**不存在普遍最优的模型**——不同模型在不同数据和任务上表现各异。模型选择与调参的目标是：为特定问题找到最优模型（或模型组合）及其最优超参数配置，使泛化性能最大化。

## 2. 发展历史

| 年代 | 里程碑 | 意义 |
|------|--------|------|
| 1974 | Stone 提出交叉验证 | 无偏模型评估方法 |
| 1995 | Vapnik 提出 SRM | 基于理论复杂度的模型选择 |
| 1998 | Bergstra & Bengio 赞成随机搜索 | 高维超参数空间优于网格搜索 |
| 2012 | Bergstra 等提出 TPE（Hyperopt） | 首个实用的贝叶斯优化器 |
| 2018 | Optuna（Preferred Networks） | 现代贝叶斯优化的工业标准 |
| 2018+ | AutoML（Auto-sklearn, H2O） | 端到端自动模型选择与调参 |

## 3. 核心概念

### 3.1 参数 vs 超参数

| 类型 | 定义 | 示例 | 决定方式 |
|------|------|------|---------|
| **参数** | 从数据中学习 | 线性回归的权重 $\mathbf{w}$、神经网络的权重 | 梯度下降/最小二乘 |
| **超参数** | 训练前设定 | 学习率、树深度、正则化系数 $\lambda$ | 交叉验证/贝叶斯优化 |

### 3.2 评估指标

**分类指标**（混淆矩阵为基础）：

| 指标 | 公式 | 关注什么 | 使用场景 |
|------|------|---------|---------|
| **Accuracy** | $\frac{TP+TN}{TP+TN+FP+FN}$ | 整体正确率 | 类别均衡 |
| **Precision** | $\frac{TP}{TP+FP}$ | 预测正类中的真正率 | 减少误报（FP成本高） |
| **Recall** | $\frac{TP}{TP+FN}$ | 真正类中被查出的比例 | 减少漏报（FN成本高） |
| **F1-Score** | $\frac{2 \cdot P \cdot R}{P + R}$ | Precision 和 Recall 调和平均 | 类别不均衡 |
| **AUC-ROC** | ROC 曲线下面积 | 排序能力（与阈值无关） | 二分类排序 |
| **Log Loss** | $-\frac{1}{n}\sum[y\log\hat{y}+(1-y)\log(1-\hat{y})]$ | 概率校准质量 | 概率输出模型 |

**回归指标**：

| 指标 | 公式 | 特点 |
|------|------|------|
| **MSE** | $\frac{1}{n}\sum(y_i - \hat{y}_i)^2$ | 对大误差惩罚重（平方） |
| **RMSE** | $\sqrt{\text{MSE}}$ | 与目标同单位，直观 |
| **MAE** | $\frac{1}{n}\sum|y_i - \hat{y}_i|$ | 对异常值鲁棒 |
| **$R^2$** | $1 - \frac{\sum(y_i - \hat{y}_i)^2}{\sum(y_i - \bar{y})^2}$ | 方差解释比例 |
| **MAPE** | $\frac{100\%}{n}\sum|\frac{y_i-\hat{y}_i}{y_i}|$ | 百分比误差，需 $y>0$ |

### 3.3 模型选择的核心原则

1. **从简单开始**：先训练一个简单基线模型（如逻辑回归/线性回归）
2. **渐进复杂**：逐步尝试更复杂的模型，仅在验证性能提升时才采纳
3. **多模型比较**：不同模型族对数据假设不同，应并行比较
4. **集成优先**：多个模型的组合往往优于单一模型

## 4. 技术原理

### 4.1 交叉验证（Cross-Validation）

交叉验证是评估泛化性能的核心方法，通过在数据的不同子集上反复训练和评估来估计泛化误差。

**K 折交叉验证**：
1. 将数据随机分为 $K$ 等份
2. For $i = 1$ to $K$：
   - 第 $i$ 份作为验证集
   - 其余 $K-1$ 份作为训练集
   - 训练模型并计算验证集误差
3. 取 $K$ 次验证误差的平均作为泛化误差估计

**K 的选择**：
- $K=5$ 或 $K=10$ 是标准做法
- $K$ 大 → 偏差小，但方差可能大（训练集相似度高）
- $K=N$（留一法 LOO）→ 偏差极小，但计算量 $O(N)$ 倍

**分层 K 折**：每折中各类别比例与原始数据一致，特别重要于类别不平衡数据。

**时间序列 CV**：保持时间顺序的前向链验证：
```
[t1 t2 t3] → [t4]       # 折1
[t1 t2 t3 t4] → [t5]    # 折2
[t1 t2 t3 t4 t5] → [t6] # 折3
```

**嵌套交叉验证**：外循环评估泛化性能，内循环调超参数。用于无偏地评估模型选择过程。

### 4.2 超参数搜索方法

#### 网格搜索（Grid Search）

穷举所有参数组合。$n$ 个参数，每个 $k$ 个候选值 → $k^n$ 种组合。

- **优点**：全面、可并行
- **缺点**：维数灾难，大部分计算浪费在无关区域
- **适用**：参数少（≤3）、候选值少的场景

#### 随机搜索（Random Search）

从参数的分布中随机采样 $B$ 次组合。Bergstra & Bengio（2012）证明：当仅有少数超参数真正重要时，随机搜索在相同预算下比网格搜索更高效（因为不会在无关维度上重复）。

- **优点**：不受参数数量限制，可设定搜索预算
- **缺点**：可能错过最优区域
- **技巧**：对于关键参数，用 log-uniform 分布（如学习率、正则化系数）

#### 贝叶斯优化（Bayesian Optimization）

以历史评估结果为指导，智能选择下一组超参数。核心组件：

1. **代理模型**（Surrogate Model）：通常是高斯过程（GP）或 Tree-structured Parzen Estimator（TPE），建模超参数→性能的映射
2. **采集函数**（Acquisition Function）：平衡探索（Exploration）与利用（Exploitation），常用 Expected Improvement（EI）

**贝叶斯优化流程**：
```
初始随机采样 B0 组 → 训练+评估 → 构建代理模型
→ 采集函数选择下一组参数 → 训练+评估 → 更新代理模型
→ 重复直到预算用完
```

**常用框架**：

| 框架 | 算法 | 特点 |
|------|------|------|
| **Optuna** | TPE + 剪枝 | 动态参数空间、试验管理、可视化 |
| **Hyperopt** | TPE | 首个流行框架 |
| **Scikit-optimize** | GP | 高斯过程，小预算下高效 |
| **Ray Tune** | 多种 | 分布式支持，大规模实验 |

**剪枝（Pruning）**：在训练过程中监控中间结果，若某组超参数明显无希望，提前终止该试验（减少计算浪费）。Optuna 内置 Median Pruner、Hyperband 等策略。

### 4.3 学习曲线分析

- **高偏差**：训练/验证误差都高且接近 → 模型欠拟合 → 增加复杂度
- **高方差**：训练误差低、验证误差高 → 模型过拟合 → 需要正则化/更多数据
- **良好拟合**：训练/验证误差接近且都较低

### 4.4 模型选择策略

| 策略 | 方法 | 场景 |
|------|------|------|
| **简单基线** | 用默认超参数的逻辑回归/线性回归做基线 | 所有项目 |
| **HPO** | 对每个候选模型调参后比较 | 模型数量少 |
| **Combined** | 将所有候选模型+超参数放入统一搜索空间 | 模型数量多 |
| **AutoML** | 让 AutoML 系统自动完成模型选择和调参 | 全自动化需求 |

## 6. 优势与局限

### 6.1 优势

- **数据驱动**：用数据而非直觉选择模型和超参数
- **可复现**：标准化的评估流程保证结果可复现
- **防过拟合**：交叉验证防止在验证集上过拟合（但需嵌套CV防超参泄露）
- **自动化**：贝叶斯优化和 AutoML 大幅减少人工调参

### 6.2 局限

- **计算开销**：K 折 × 候选数 × 单次训练时间 可能非常昂贵
- **数据泄露风险**：交叉验证中预处理（如标准化）必须在每折内独立计算
- **嵌套 CV**成本：双层交叉验证使计算量 $O(K_{outer} \times K_{inner})$
- **过拟合验证集**：反复在同一测试集上评估会导致间接过拟合

- **多目标优化**：同时优化精度、推理延迟、模型大小（Pareto 前沿搜索）
- **神经架构搜索（NAS）**：自动搜索最优网络结构
- **AutoML 2.0**：结合元学习（在历史任务上学习调参策略）加速新任务搜索

- **梯度引导的超参数优化**：对可微的超参数（如正则化系数），通过超梯度优化
