# 向量表示与Embedding

## 背景与发展

向量表示是将文本、图像等非结构化数据转换为数值向量的过程。嵌入（Embedding）是神经网络学习得到的低维稠密向量，能捕获语义关系。从One-hot到Word2Vec再到上下文感知嵌入和对比学习嵌入，向量表示技术不断演进。

## 核心思想

将词汇、句子或文档映射到高维空间中的点，使语义相似的内容在嵌入空间中距离更近。现代嵌入通过预训练和对比学习，能够生成上下文感知的语义表示。

## 技术原理

### 嵌入空间性质

- **语义相似度**：语义相似的文本在嵌入空间中距离更近
- **线性关系**：部分语义关系可通过向量运算捕获（如"国王" - "男人" + "女人" ≈ "女王"）
- **各向异性**：嵌入向量倾向于聚集在特定区域，对比学习可缓解此问题

### 传统词嵌入

#### Word2Vec（2013）

通过预测上下文或目标词学习分布式表示：

- **CBOW**：根据上下文预测目标词，适合低频词
- **Skip-gram**：根据目标词预测上下文，适合高频词和生僻词
- 优化：Hierarchical Softmax、Negative Sampling

#### GloVe（2014）

结合全局词共现统计和局部上下文信息：

$$J = \sum_{i,j} f(X_{ij}) (w_i^T w_j + b_i + b_j - \log X_{ij})^2$$

#### FastText（2016）

将词汇分解为字符n-gram，能处理未登录词和形态丰富的语言。

### 上下文感知嵌入

同一词在不同上下文中生成不同向量，捕获多义性：

- **ELMo**（2018）：基于双向LSTM
- **BERT**（2018）：基于[[Transformer]]编码器，输入 = Token Embedding + Segment Embedding + [[位置编码]]
- **GPT系列**：基于Transformer解码器，自回归生成上下文表示

### 对比学习嵌入

通过对比学习优化语义相似度：

- **SimCSE**（2021）：无监督使用Dropout作数据增强，有监督使用NLI数据
- **ConSERT**：多种数据增强策略（随机删除、同义词替换、回译）

### 嵌入粒度

| 粒度 | 生成方式 | 适用场景 |
|------|---------|---------|
| Token-level | 每个token向量 | NER、词性标注、机器翻译 |
| Sentence-level | 平均池化/CLS/专用模型 | 语义相似度、文本分类 |
| Document-level | 整文档单一向量 | 文档检索、推荐系统 |

## 发展演进

One-hot/TF-IDF → **Word2Vec/GloVe**（2013-2014）→ FastText（2016）→ ELMo/BERT（2018）上下文感知 → **SimCSE**（2021）对比学习 → E5/BGE指令微调嵌入

### 嵌入质量评估

- **STS**：Pearson/Spearman相关系数（STS-B数据集）
- **检索**：MRR、NDCG（MSMARCO、BEIR数据集）
- **聚类**：Purity、ARI、NMI

## 应用场景

- **语义检索**：RAG系统的向量召回
- **文本分类与聚类**：嵌入作为特征输入下游模型
- **推荐系统**：用户-物品嵌入匹配
- **多模态对齐**：CLIP将文本和图像嵌入对齐到共享空间
