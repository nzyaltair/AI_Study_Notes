# Agent记忆

## 1. 概述

记忆系统是 `Agent系统` 的核心组件之一，直接影响Agent的决策能力、上下文理解和持续学习能力。受认知科学中人类记忆分类的启发，Agent记忆系统从简单的对话缓存发展为多类型、多层次的复杂架构。

Agent记忆解决的核心问题：LLM的上下文窗口有限，无法在一次对话中保留所有历史信息。记忆系统通过外部存储和智能检索，使Agent能够"记住"过去的交互和知识，实现跨会话的持续学习。

## 3. 核心概念

### 记忆类型分类

**按存储时长**：

| 类型 | 特点 | 实现方式 | 类比 |
|------|------|---------|------|
| 短期记忆 | 容量有限，访问快，易遗忘 | 会话缓存、滑动窗口 | 工作记忆 |
| 工作记忆 | 动态更新，任务导向 | 临时Scratchpad、上下文窗口 | 专注力 |
| 长期记忆 | 容量大，相对稳定 | `向量数据库` 、知识图谱 | 长期记忆 |

**按内容性质**：

| 类型 | 存储内容 | 特点 | 类比 |
|------|---------|------|------|
| 情景记忆 | 特定事件、交互记录 | 有时间戳，事件关联 | 个人经历 |
| 语义记忆 | 事实知识、概念关系 | 结构化，可泛化 | 一般知识 |
| 程序记忆 | 技能、操作流程 | 隐含性，可执行 | 技能习惯 |

### 关键术语

- **Memory Stream**：按时间排序的记忆事件流
- **Reflection**：Agent定期反思记忆，生成高层见解
- **Memory Consolidation**：将短期记忆巩固为长期记忆
- **Forgetting**：基于策略清理低价值记忆
- **Memory Retrieval**：从记忆库中检索相关信息
- **Context Window Management**：管理LLM上下文窗口的有效利用

## 4. 技术原理

### 典型记忆架构

**基于向量数据库的记忆**：将记忆内容向量化后存储，通过相似度检索获取相关记忆。优势是支持语义检索、可扩展性好。

```python
class VectorMemory:
    def __init__(self, embedding_model, vector_db):
        self.embedding_model = embedding_model
        self.vector_db = vector_db

    def store(self, content, metadata=None):
        embedding = self.embedding_model.encode(content)
        self.vector_db.add(embedding, content, metadata)

    def retrieve(self, query, k=5):
        query_embedding = self.embedding_model.encode(query)
        return self.vector_db.similarity_search(query_embedding, k=k)
```

**基于图结构的记忆网络**：将记忆表示为图节点和边，通过图算法进行检索和推理。优势是支持关系推理、可解释性好。

**混合记忆系统**：结合多种架构，短期记忆用会话缓存，长期记忆用向量数据库，语义记忆用知识图谱。

### MemGPT：操作系统式记忆管理

MemGPT将操作系统的内存管理思想引入Agent记忆：

```
┌──────────────────────────────────┐
│       主上下文（Main Context）     │  ← LLM上下文窗口
│  ┌─────────┐  ┌──────────────┐  │
│  │ 系统指令  │  │ 工作记忆区    │  │
│  └─────────┘  └──────────────┘  │
├──────────────────────────────────┤
│     外部记忆（External Memory）    │  ← 向量数据库
│  ┌─────────┐  ┌──────────────┐  │
│  │ 记忆分页  │  │ 档案存储     │  │
│  └─────────┘  └──────────────┘  │
└──────────────────────────────────┘
  ↑ 分页调入/调出（Page In/Out）
```

核心机制：
- **分页（Paging）**：将记忆分页，按需调入主上下文
- **中断（Interrupt）**：上下文将满时触发记忆压缩
- **自编辑（Self-Edit）**：LLM自主决定记忆的更新和删除

### Generative Agents记忆流

斯坦福小镇Agent的记忆架构：

```
记忆流（Memory Stream）：
  [事件1: "早上遇到了John"]          重要性: 3
  [事件2: "John说他要开咖啡店"]       重要性: 8
  [事件3: "在图书馆看了书"]           重要性: 2
  ...
     ↓ 定期反思
反思层（Reflection）：
  [见解: "John可能正在创业"]          重要性: 9
     ↓ 深层反思
高层见解（Higher-level Reflection）：
  [见解: "镇上的年轻人都在创业"]
```

### 记忆生命周期

**编码（写入）**：
1. 内容捕获：从交互中提取值得记忆的信息
2. 预处理：清洗、结构化
3. 特征编码：向量化
4. 元数据添加：时间戳、重要性评分、关联实体

**检索（查询）**：

| 策略 | 机制 | 适用场景 |
|------|------|---------|
| 相似性检索 | 计算向量相似度 | 语义相关记忆 |
| 时间检索 | 按时间戳过滤/衰减 | 最近事件优先 |
| 重要性检索 | 按评分排序 | 关键信息提取 |
| 混合检索 | 加权融合多种策略 | 复杂查询 |

**混合检索评分公式**：

$$\text{Score}(m) = \alpha \cdot \text{Relevance}(m) + \beta \cdot \text{Recency}(m) + \gamma \cdot \text{Importance}(m)$$

其中 $\text{Relevance}$ 为语义相关度，$\text{Recency}$ 为时间近因性（指数衰减），$\text{Importance}$ 为重要性评分。

**更新（遗忘/压缩/摘要）**：

- **遗忘**：基于TTL、访问频率或重要性评分清理低价值记忆
- **压缩**：合并相似记忆、删除冗余、降维
- **摘要**：LLM生成记忆的简洁表示，保留核心信息

### 记忆摘要模式

将长对话逐步摘要，形成层次化结构：

```
原始会话 → 短期摘要 → 长期摘要 → 核心知识
检索时：核心知识 → 长期摘要 → 短期摘要 → 原始会话（逐步扩展）
```

### 基于反思的记忆增强

定期让Agent反思最近的经历，生成新的见解和规则，存入长期记忆供后续决策使用：

```python
def reflect(agent, recent_memories):
    prompt = f"""
    回顾以下最近的交互记录，提取可复用的经验教训：
    {recent_memories}
    
    请生成：
    1. 关键见解（Insights）
    2. 可复用的规则（Rules）
    3. 需要避免的错误（Pitfalls）
    """
    reflections = agent.llm.generate(prompt)
    agent.long_term_memory.store(reflections, type="reflection")
```

## 6. 优势与局限

### 优势

- 实现跨会话的持续记忆，提升用户体验
- 支持个性化适配，记住用户偏好和历史交互
- 通过反思机制从经验中学习，持续改进
- 层次化记忆管理有效利用有限的上下文窗口

### 局限

- **检索质量依赖嵌入模型**：记忆检索精度受嵌入模型质量影响
- **记忆噪声**：大量低价值记忆可能干扰检索
- **一致性挑战**：记忆更新可能导致信息不一致
- **成本开销**：记忆存储和检索增加额外计算成本
- **隐私问题**：记忆中可能包含敏感信息，需要隐私保护
- **遗忘策略设计困难**：难以平衡保留与遗忘

## 7. 应用场景

- **长期对话助手**：跨会话记忆，记住用户偏好和历史
- **个性化Agent**：学习用户习惯，提供个性化服务
- **持续学习Agent**：从经验中积累知识和技能
- **多Agent协作**：共享记忆实现团队知识对齐
- **专业领域助手**：知识图谱记忆支持专业推理
- **游戏NPC**：具有记忆和个性的虚拟角色

## 8. 与其他技术关系

- Agent记忆依赖 `向量数据库` 实现长期存储
- `RAG` 可视为Agent记忆的一种实现（外部知识即记忆）
- `Agent框架` 的记忆模块基于本技术
- 记忆检索质量影响 `提示工程` 的上下文构造
- 多Agent协作需要记忆共享与对齐机制
- `Agent系统` 的反思机制依赖记忆系统

## 9. 前沿发展

- **可学习记忆控制器**：用强化学习训练记忆管理策略，自动学习何时存储、检索和遗忘
- **神经符号记忆**：结合神经网络（语义检索）和符号系统（逻辑推理）的记忆架构
- **记忆压缩技术**：更高效的记忆压缩方法，在有限空间内保留更多信息
- **共享记忆系统**：多Agent间的记忆共享协议和一致性保证
- **隐私保护记忆**：差分隐私、联邦学习等技术保护记忆中的敏感信息
- **记忆可解释性**：记忆检索路径可视化、记忆重要性解释
- **终身学习记忆**：支持Agent在整个生命周期中持续积累和整合知识
- **记忆评估基准**：建立Agent记忆系统的标准化评估方法
