# 目标检测

## 1. 概述

目标检测（Object Detection）是计算机视觉的核心任务之一——同时回答图像中"**有哪些物体**"（分类）和"**在哪里**"（定位）。作为连接图像分类与场景理解的中间环节，目标检测是自动驾驶、安防监控、医学影像分析等应用的基础能力。

目标检测的核心挑战：物体尺度差异大、姿态各异、遮挡严重、背景复杂。

## 2. 发展历史

```
传统方法 (2001-2012)
  │  Haar + AdaBoost (人脸检测), HOG + SVM (行人检测), DPM (可变形部件模型)
  ▼
两阶段时代 (2014-2017)
  │  R-CNN → Fast R-CNN → Faster R-CNN → FPN → Mask R-CNN
  ▼
单阶段崛起 (2016-2020)
  │  YOLO v1 → SSD → RetinaNet (Focal Loss) → YOLO v2/v3/v4/v5 → EfficientDet
  ▼
Transformer时代 (2020-至今)
     DETR → Deformable DETR → DINO → YOLOv8/v9/v10/v11
```

**关键转折**：
- **2014**: R-CNN首次将CNN用于检测，mAP从传统方法的~35%跃升到~60%（PASCAL VOC）
- **2015**: Faster R-CNN提出RPN，实现端到端的区域提议
- **2017**: RetinaNet的Focal Loss解决正负样本严重不均衡，单阶段检测器首次匹敌两阶段
- **2020**: DETR用Transformer消除手工设计的Anchor和NMS

## 3. 核心概念

### 3.1 基本术语

| 术语 | 定义 |
|------|------|
| **Bounding Box (BBox)** | 物体位置的矩形框表示 $(x,y,w,h)$ 或 $(x_1,y_1,x_2,y_2)$ |
| **IoU (Intersection over Union)** | 预测框与真实框的交并比，衡量定位精度 |
| **Anchor Box** | 预设的参考框（不同大小/长宽比），作为检测的先验 |
| **RPN (Region Proposal Network)** | 生成候选区域的小型网络 |
| **NMS (Non-Maximum Suppression)** | 后处理算法，去除重叠的重复检测 |
| **mAP (mean Average Precision)** | 检测任务的核心评估指标 |

### 3.2 mAP 计算

1. 对每个类别，按置信度排序所有预测
2. 计算不同IoU阈值下的Precision-Recall曲线
3. AP = PR曲线下面积
4. mAP = 所有类别AP的平均
5. COCO标准：mAP@[.5:.95]表示10个IoU阈值（0.5~0.95步长0.05）的平均

### 3.3 正负样本定义

- **正样本**：与某ground-truth的IoU > 阈值（如0.5/0.7）
- **负样本**：与所有ground-truth的IoU < 阈值
- **忽略**：介于两者之间，不参与损失计算

## 4. 技术原理

### 4.1 两阶段检测器（Two-Stage）

#### R-CNN (2014)
1. **Selective Search** 生成~2000个候选区域
2. 每个候选区域缩放到固定尺寸 → CNN提取特征
3. SVM分类 + 边界框回归

**问题**：每个候选区域都跑一次CNN，极度缓慢（GPU上一张图~13秒）。

#### Fast R-CNN (2015)
- 整张图只跑一次CNN提取特征图
- 将候选区域映射到特征图上（**RoI Pooling**）
- 统一输出固定尺寸 → 分类 + 回归

**速度提升**：~2秒/张（比R-CNN快150×）。

#### Faster R-CNN (2015)
**核心创新——RPN（Region Proposal Network）**：在特征图上滑动小网络，每个位置预设多种Anchor，直接预测：
- 是否有物体（二分类）
- 边界框的偏移量

将区域提议也变成了可学习的端到端过程。Faster R-CNN = RPN + Fast R-CNN。

#### FPN (Feature Pyramid Network, 2017)
**问题**：高层特征语义强但分辨率低，低层特征分辨率高但语义弱。对小物体检测不利。

**解决方案**：构建特征金字塔 —— 自顶向下的上采样路径+横向连接，融合多尺度特征。每一层金字塔独立预测不同尺度的物体。

### 4.2 单阶段检测器（One-Stage）

#### YOLO v1 (2016)
将检测建模为回归问题：图像分成 $S \times S$ 网格，每个网格预测固定数量的边界框和类别概率。

**特点**：极快（45 FPS），但定位精度较差，对小物体不友好。

#### SSD (2016)
在多尺度特征图上同时预测，引入不同长宽比的默认框（Default Box）。平衡了速度和精度。

#### RetinaNet (2017)
**核心创新——Focal Loss**：

$$\text{FL}(p_t) = -\alpha_t (1-p_t)^\gamma \log(p_t)$$

- $\gamma = 2, \alpha_t = 0.25$（标准设置）
- $(1-p_t)^\gamma$ 项自动降低易分类样本的损失权重
- 解决了单阶段检测器中**正负样本极端不平衡**（1:100000）的核心问题

使单阶段检测器首次在精度上与两阶段匹敌。

#### YOLO后续版本演进

| 版本 | 年份 | 核心改进 |
|------|------|----------|
| YOLOv2 | 2016 | 引入Anchor、BatchNorm、多尺度训练、Darknet-19 |
| YOLOv3 | 2018 | FPN多尺度预测、Darknet-53骨干、多标签分类 |
| YOLOv4 | 2020 | Bag of Freebies/Specials、CSPNet、PANet |
| YOLOv5 | 2020 | PyTorch实现、自动Anchor聚类、易部署 |
| YOLOv8 | 2023 | Anchor-free、解耦头、C2f模块 |
| YOLOv10 | 2024 | NMS-free训练、一致双重分配 |

### 4.3 Transformer检测器

#### DETR (2020)
将目标检测建模为**集合预测问题**——用Transformer的Encoder-Decoder结构直接输出固定数量的预测（含类别或无物体标记），通过**匈牙利算法**做二分匹配计算损失。

**优势**：
- 彻底消除Anchor设计、NMS后处理等手工组件
- 架构简洁统一

**局限**：
- 训练收敛慢（需要500 epochs vs Faster R-CNN的12 epochs）
- 小物体检测效果一般

#### Deformable DETR (2021)
用**可变形注意力**替代全局注意力，只关注参考点周围的少量采样点，大幅加速训练收敛（50 epochs可达Faster R-CNN水平），并提升了小物体检测。

#### DINO (2023)
进一步改进：对比去噪训练、混合查询选择、前向两次预测。COCO达到63.3 AP，刷新DETR系列记录。

## 6. 优势与局限

### 两阶段 vs 单阶段

| 维度 | 两阶段 | 单阶段 |
|------|--------|--------|
| 精度 | 高（候选区域精修） | 逐渐追平 |
| 速度 | 慢 | 快（实时） |
| 小物体 | 好（RPN+高分辨率特征） | 差（需专门设计） |
| 训练复杂度 | 高（多步/多任务） | 低（端到端） |

### 统一挑战
- **遮挡**：部分可见物体难以准确定位和分类
- **小物体**：低分辨率特征图中信息不足
- **长尾分布**：罕见类别样本稀少
- **域迁移**：训练集和部署场景分布差异

## 7. 应用场景

| 场景 | 常用方法 | 关键需求 |
|------|----------|----------|
| 自动驾驶 | YOLO系列、CenterNet | 实时性（>30FPS）、小物体 |
| 安防监控 | YOLOv5/v8 | 边缘部署、实时性 |
| 遥感图像 | Faster R-CNN+FPN | 小物体、旋转框 |
| 医学影像 | Faster R-CNN、YOLO | 高召回率 |
| 工业质检 | YOLOv8-nano、RT-DETR | 极速、轻量化 |
| 文字检测 | EAST、DBNet | 不规则形状 |

## 8. 与其他技术关系

- 目标检测的**骨干网络**（Backbone）直接复用分类预训练的ResNet/EfficientNet等架构 → [[02_经典CNN架构]]
- FPN的多尺度融合设计与[[04_图像分割]]的U-Net跳跃连接思想相通
- DETR的集合预测+匈牙利匹配启发了[[06_CV前沿发展|目标检测的新范式]]
- Anchors → Anchor-free → 端到端预测（DETR），反映"减少手工先验"的发展趋势
- 检测+跟踪的多任务学习正成为视频理解的基础

## 9. 前沿发展

1. **开放词汇检测（Open-Vocabulary Detection）**：不限于训练类别，支持任意文本描述的物体检测（如OWL-ViT、Grounding DINO）
2. **端到端检测**：DETR系列消除NMS的方向持续演进（DINO、Co-DETR、RT-DETR）
3. **实时高精度**：RT-DETR在实时速度下达到50+ AP，满足部署需求
4. **视觉基础模型**：SAM（Segment Anything Model）可用于检测前的位置发现
5. **3D目标检测**：从2D扩展到BEV空间（如自动驾驶中的BEVDet、BEVFormer）
6. **弱监督/半监督检测**：减少标注成本，仅需图像级标签或少量边界框标注
