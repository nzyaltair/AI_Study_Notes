# 多模态RAG

## 1. 概述

多模态RAG（Multimodal RAG，MRAG）是传统文本 `RAG` 的扩展，支持图像、视频、音频等多模态内容的检索与生成。它使用跨模态嵌入模型将不同模态内容映射到统一的向量空间，实现跨模态检索，并使用多模态LLM处理多模态上下文生成回答。

多模态RAG解决了传统文本RAG无法处理视觉、听觉信息的局限，适用于需要理解图像、视频、音频等非文本内容的知识密集型应用。

## 2. 发展历史

| 时间 | 里程碑 | 意义 |
|------|--------|------|
| 2021.01 | CLIP（Radford et al.） | 视觉-文本对比学习，跨模态嵌入基础 |
| 2022 | BLIP-2（Li et al.） | 生成式视觉-语言预训练 |
| 2023.04 | LLaVA（Liu et al.） | 视觉指令微调多模态LLM |
| 2023 | Qwen-VL / CogVLM | 中文优化多模态LLM |
| 2023 | 多模态RAG实践兴起 | 图文混合检索+多模态生成 |
| 2023.10 | Video-LLaVA | 视频理解增强 |
| 2024 | 多模态嵌入模型成熟 | BGE-VL、JINA-CLIP等 |
| 2024 | 端到端多模态RAG | 联合优化多模态检索与生成 |
| 2025 | 多模态RAG标准化 | 主流框架原生支持 |

## 3. 核心概念

### 与传统文本RAG对比

| 维度 | 文本RAG | 多模态RAG |
|------|---------|-----------|
| 数据模态 | 纯文本 | 文本+图像+视频+音频 |
| 嵌入空间 | 单模态文本嵌入 | 跨模态统一嵌入空间 |
| 检索策略 | 文本相似度 | 跨模态相似度 |
| 生成模型 | 文本LLM | 多模态LLM（LLaVA、Qwen-VL、GPT-4V） |
| 文档预处理 | 文本分块、摘要 | 多模态特征提取、标注、对齐 |
| 存储需求 | 文本向量 | 多模态向量+原始媒体文件 |

### 多模态RAG架构

```
┌──────────────────────────────────────────────────┐
│  数据摄入层                                        │
│  文本 → 文本嵌入    图像 → 视觉嵌入 + 文本描述      │
│  视频 → 关键帧 + 音频转文字                         │
├──────────────────────────────────────────────────┤
│  统一向量空间（CLIP / 多模态嵌入模型）                │
├──────────────────────────────────────────────────┤
│  跨模态检索层                                      │
│  文本查询 → 跨模态向量检索 → 多模态文档              │
├──────────────────────────────────────────────────┤
│  多模态生成层                                      │
│  文本+图像+视频 → 多模态LLM → 回答                  │
└──────────────────────────────────────────────────┘
```

## 4. 技术原理

### 多模态数据预处理

| 模态 | 预处理流程 | 关键技术 |
|------|-----------|---------|
| 图像 | 视觉特征提取 + 文本描述生成 | CLIP/ViT编码 + VLM生成caption |
| 视频 | 关键帧提取 + 音频处理 + 时间对齐 | 场景检测 + Whisper语音识别 |
| 音频 | 语音识别 + 音频嵌入 | Whisper + Wav2Vec 2.0 |
| 文档 | OCR + 版面分析 + 图文配对 | PaddleOCR + LayoutLM |

### 跨模态嵌入

**CLIP模型**：通过对比学习将图像和文本映射到统一的512维向量空间，使"猫的图片"和"一只猫"的文本向量接近。

```
CLIP训练目标：
  maximize: sim(image_embed, text_embed) for matching pairs
  minimize: sim(image_embed, text_embed) for non-matching pairs
```

### 跨模态检索策略

| 检索模式 | 原理 | 优点 | 缺点 |
|----------|------|------|------|
| 早期融合 | 多模态数据融合后联合检索 | 保留模态间信息 | 计算复杂 |
| 联合嵌入 | 多模态统一嵌入空间检索 | 跨模态检索自然 | 训练成本高 |
| 晚期融合 | 各模态独立检索后融合 | 效率高 | 丢失模态相关性 |
| 混合检索 | 加权融合多模态相似度 | 灵活可调 | 权重调优困难 |

### 多模态嵌入模型

| 模型 | 类型 | 维度 | 特点 |
|------|------|------|------|
| CLIP | 视觉-文本对比 | 512 | OpenAI，通用基础 |
| Chinese-CLIP | 视觉-文本对比 | 512 | 中文优化 |
| BGE-VL | 视觉-文本 | 1024 | 多语言，高性能 |
| JINA-CLIP | 视觉-文本 | 768 | 开源高性能 |
| DINOv2 | 纯视觉 | 768 | 自监督，视觉特征强 |
| Whisper | 音频识别 | - | 语音转文本 |

### 多模态上下文构建

将检索到的多模态内容组织为结构化提示：

```python
MULTIMODAL_PROMPT = """
## 文本上下文
{text_context}

## 图像参考
[Image 1: {image1_description}]
[Image 2: {image2_description}]

## 视频信息
{video_summary}

## 问题
{user_query}
"""
```

对于原生支持图像输入的多模态LLM，可直接将图像作为输入：

```python
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{
        "role": "user",
        "content": [
            {"type": "text", "text": "基于以下图片回答问题"},
            {"type": "image_url", "image_url": {"url": image_url}},
            {"type": "text", "text": user_query}
        ]
    }]
)
```

## 6. 优势与局限

### 优势

- 支持图文、视频、音频等丰富模态的知识检索
- 跨模态检索打破文本-only的限制
- 多模态LLM可直接理解图像内容
- 适用于视觉密集型领域（医疗、电商、制造）

### 局限

- **存储成本高**：多模态数据（尤其视频）占用大量存储
- **检索精度有限**：跨模态嵌入空间不如纯文本嵌入空间成熟
- **计算开销大**：多模态特征提取和编码成本高
- **模态对齐困难**：不同模态的语义对齐仍不完美
- **多模态LLM限制**：图像分辨率限制、视频长度限制、成本高

## 7. 应用场景

- **图文问答系统**：用户上传图片提问
- **视频内容检索与摘要**：从视频库中检索相关片段
- **医疗影像报告分析**：结合影像和病历生成诊断辅助
- **电商商品图文检索**：以图搜商品 + 商品知识问答
- **多模态知识库管理**：混合文档、图表、视频的知识库
- **教育辅助**：教材图文检索与解释
- **制造业质检**：产品图像检索 + 缺陷分析

## 8. 与其他技术关系

- 多模态RAG以 `基础RAG` 和 `进阶RAG` 为基础
- `向量数据库` 提供多模态向量存储
- `多模态模型` 提供感知与生成能力
- CLIP等跨模态模型是多模态RAG的嵌入基础
- `Agent系统` 可集成多模态RAG扩展感知范围

## 9. 前沿发展

- **端到端多模态RAG**：联合优化多模态检索与生成，不再分阶段处理
- **视频RAG**：从关键帧检索走向时序理解，支持长视频问答
- **3D多模态RAG**：结合3D场景理解，支持空间推理
- **具身多模态记忆**：机器人Agent的多模态记忆系统
- **多模态嵌入模型升级**：更高维度、更强跨模态对齐能力的嵌入模型
- **实时多模态RAG**：流式视频/音频的实时检索增强
- **多模态RAG评估**：建立多模态RAG评估基准和指标
