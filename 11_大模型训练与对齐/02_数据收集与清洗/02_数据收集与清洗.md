# 数据收集与清洗

## 背景与发展

数据是大语言模型训练的基础，其质量直接决定模型最终性能。随着预训练数据规模从 GPT-2 的 WebText（40GB）增长到 LLaMA 3 的 15T tokens，数据处理的工程化和自动化程度不断提升。业界形成了"数据质量优先于数量"的共识，高质量数据的缩放效果显著优于低质量数据。

## 核心思想

构建高质量、多样化、合规的训练数据集，涵盖数据收集、格式标准化、去重、质量过滤、脱敏和领域配比六个核心环节。数据处理 Pipeline 的质量是预训练成功的关键前提。

## 技术原理

### 数据来源

| 数据集 | 规模 | 特点 |
|-------|------|------|
| CommonCrawl | 100T+ tokens | 涵盖网页内容，需重度清洗 |
| C4 | 10T tokens | 清洗后的 CommonCrawl 子集 |
| The Pile | 800B tokens | 22 个高质量数据集混合 |
| Wikipedia | 70B tokens | 结构化知识 |
| GitHub | 100B tokens | 多语言代码 |
| arXiv | 10B tokens | 学术论文 |

### 数据去重

**精确去重**：使用 SHA-256 等哈希算法识别完全相同的文本，速度快但无法处理近似重复。

**近似去重**：

- **[[MinHash]]**：通过最小哈希签名快速估算 Jaccard 相似度，适合大规模数据
- **[[SimHash]]**：生成 64/128 位指纹哈希，通过 Hamming 距离判断相似性
- **LSH（Locality Sensitive Hashing）**：基于局部敏感哈希的近似近邻搜索

工业实践：LLaMA 3 结合 4-gram 精确去重与 MinHash 近似去重，去重率达 40%-50%；Qwen 2 采用多阶段去重（文档级→段落级→语义级），去重率达 60%。

### 质量过滤

**基础过滤**：长度过滤、字符比例过滤、重复模式过滤。

**语言识别**：使用 fasttext 等工具识别并保留目标语言文本。

**困惑度过滤**：用预训练语言模型计算文本困惑度，过滤低质量文本。高质量文本通常困惑度适中。

**模型辅助过滤**：训练专门的数据质量评估模型，对文本进行多维度评分（语法正确性、信息密度、多样性）。

### 毒性与有害内容过滤

- **毒性分类**：识别仇恨言论、歧视性内容
- **有害内容过滤**：过滤暴力、色情、虚假信息
- 工具：Perspective API、Detoxify

### 敏感信息脱敏（PII）

- **PII 类型**：姓名、身份证号、手机号、邮箱、地址
- **脱敏策略**：替换、模糊化、删除
- 工具：Microsoft Presidio、spaCy NER

### 领域数据配比与采样

- **均匀采样**：各领域按固定比例混合
- **加权采样**：根据领域重要性调整采样权重
- **基于困惑度的加权**：根据不同来源语料的困惑度动态调整采样权重

### 合成数据

- **Self-Instruct**：让模型生成指令-响应对
- **Evol-Instruct**：迭代优化生成的指令，逐步增加复杂度
- **UltraChat**：生成多轮对话数据

## 发展演进

The Pile (2020) → 去重+规则过滤 → 困惑度过滤 (GPT-3 时代) → 模型辅助质量评估 (LLaMA 2/3) → 合成数据增强 → 自进化数据飞轮

## 关键算法·模型

- **MinHash + LSH**：大规模近似去重的标准方案
- **fasttext**：快速语言识别
- **Self-Instruct**：合成指令数据生成
- **DQI（Data Quality Index）**：多维度数据质量评估框架

## 应用场景

- **预训练数据准备**：LLaMA/Qwen/DeepSeek 的数据处理 Pipeline
- **SFT 数据构建**：指令多样性、难度分布、拒绝采样
- **偏好数据收集**：人类标注、AI 生成、Pair-wise 比较
- **领域适配**：医疗、法律、金融领域数据清洗与配比

## 与其他技术关系

- 数据准备是 [[预训练流程]] 的前置环节
- 分词与 Tokenization 是数据处理的关键步骤
- 去重算法基于信息论中的相似度度量
- 合成数据生成依赖已有大模型的能力
