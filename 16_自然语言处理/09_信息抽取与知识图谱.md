# 信息抽取与知识图谱

## 1. 概述

信息抽取（Information Extraction, IE）研究如何从非结构化文本中提取结构化信息。它是将"文本"转化为"知识"的核心技术，包含命名实体识别（NER）、关系抽取（RE）、事件抽取（EE）三个递进层次。知识图谱（Knowledge Graph, KG）则以图结构组织抽取的知识——节点是实体，边是实体间的关系。

信息抽取是 NLP 从"理解语言"走向"构建知识"的桥梁，也是搜索引擎、问答系统、推荐系统和智能分析的基础设施。在大模型时代，LLM 正在改变信息抽取的范式——从专门训练的抽取模型转向基于提示的生成式抽取。

## 3. 核心概念

### 3.1 信息抽取的层次

```
文本 → NER（实体） → RE（关系） → EE（事件） → 知识图谱
```

| 任务 | 输入 | 输出 | 示例 |
|:---|:---|:---|:---|
| NER | 文本 | 实体列表 | "张三在北京工作" → PER:张三, LOC:北京 |
| 关系抽取（RE） | 文本 + 实体 | 关系三元组 | (张三, 工作地点, 北京) |
| 事件抽取（EE） | 文本 | 事件结构 | 事件类型:就业, 触发词:工作, 论元:张三(雇员),北京(地点) |
| 实体链接（EL） | 实体 + 知识库 | 知识库 URI | "苹果" → dbpedia:Apple_Inc. |

### 3.2 知识图谱的表示

知识图谱以三元组形式存储知识：

$$(h, r, t) \quad \text{head entity, relation, tail entity}$$

示例：
- (姚明, 职业, 篮球运动员)
- (姚明, 出生地, 上海)
- (姚明, 国籍, 中国)

### 3.3 远程监督（Distant Supervision）

利用已有知识库自动标注训练数据：

1. 从知识库获取三元组 $(h, r, t)$
2. 在文本中找到同时包含 $h$ 和 $t$ 的句子
3. 假设该句子表达了关系 $r$

**噪声问题**：同一句子中的两个实体可能有多种关系，远程监督引入噪声标签。

## 4. 技术原理

### 4.1 命名实体识别

参见 [[02_词法分析]] 中的 NER 部分。核心方法：BiLSTM-CRF、BERT-CRF。

### 4.2 关系抽取

#### 基于句法的方法

利用依存路径提取关系特征：

```
句子：张三 就职于 微软
依存路径：张三 ←nsubj← 就职于 →dobj→ 微软
关系：(张三, 就职于, 微软)
```

#### 神经网络方法

将实体对及其上下文编码为向量，分类关系类型：

- **PCNN（Piecewise CNN）**：以实体位置为界将句子分为三段，分别卷积
- **BERT-based RE**：在 `[CLS]` 表示上分类，或将实体标记注入输入

#### 联合抽取

同时进行 NER 和 RE，避免级联误差：

- **参数共享**：NER 和 RE 共享编码器，分别解码
- **统一框架**：将 NER 和 RE 统一为表格填充问题

### 4.3 事件抽取

事件抽取识别文本中发生的事件及其参与者：

```
句子：张三昨日在北京发布了新产品

事件类型：产品发布
触发词：发布
论元：
  - 发布者：张三（Agent）
  - 时间：昨日（Time）
  - 地点：北京（Location）
  - 对象：新产品（Product）
```

事件抽取比关系抽取更复杂——一个事件涉及多个论元和角色。

### 4.4 实体链接（Entity Linking）

将文本中的实体提及映射到知识库中的实体：

1. **候选生成**：根据表面名称检索候选实体
2. **实体消歧**：根据上下文选择正确实体

如 "苹果" 在科技语境下链接到 Apple Inc.，在饮食语境下链接到 Apple（fruit）。

### 4.5 知识图谱嵌入

将实体和关系映射到低维向量空间：

| 模型 | 打分函数 | 特点 |
|:---|:---|:---|
| TransE | $\|h + r - t\|$ | 简单高效，不适合一对多关系 |
| TransH | $\|h_\perp + r - t_\perp\|$ | 处理一对多/多对多关系 |
| TransR | $\|M_r h + r - M_r t\|$ | 关系特定空间 |
| RotatE | $\|h \circ r - t\|$ | 复数空间旋转 |
| ComplEx | $\text{Re}(h \cdot r \cdot \bar{t})$ | 复数嵌入 |

用于知识图谱补全（预测缺失的三元组）和知识问答。

## 5. 关键方法/模型

| 方法 | 任务 | 核心思想 | 优势 |
|:---|:---|:---|:---|
| BiLSTM-CRF | NER | 序列标注 + CRF | 经典、效果好 |
| BERT-CRF | NER | 预训练 + CRF | 当前主流 |
| PCNN + Attn | RE | 分段卷积 + 注意力 | 远程监督去噪 |
| SpanBERT | IE | Span 预训练 | 适合 Span 任务 |
| UIE | 统一 IE | 统一抽取框架 | 一个模型多任务 |
| LLM-based IE | IE | 提示工程 + 生成 | 零样本、灵活 |

## 6. 优势与局限

### 优势

1. **结构化知识**：将非结构化文本转化为可计算的结构化知识
2. **知识推理**：知识图谱支持多跳推理和关联分析
3. **可积累**：知识图谱可持续更新和扩展

### 局限

1. **标注成本高**：IE 需要大量标注数据，尤其事件抽取
2. **远程监督噪声**：自动标注引入错误标签
3. **跨领域迁移难**：训练于新闻的模型在医疗/法律领域效果差
4. **知识更新**：实体和关系随时间变化，知识图谱需持续维护
5. **开放世界**：无法穷举所有实体和关系类型

## 7. 应用场景

- **搜索引擎**：Google Knowledge Graph 提供结构化搜索结果
- **智能问答**：基于知识图谱的 KBQA
- **推荐系统**：基于知识图谱的推荐
- **金融风控**：公司关系网络分析、风险传导预测
- **医疗**：药物-疾病-基因关系抽取
- **情报分析**：人物关系网络、事件追踪
- **RAG**：GraphRAG 增强知识检索

## 9. 前沿发展

- **LLM 生成式抽取**：用 LLM 直接生成结构化输出（JSON/三元组）
- **统一信息抽取（UIE）**：一个模型处理 NER/RE/EE 等所有 IE 任务
- **开放世界 IE**：不预定义实体/关系类型，开放抽取
- **多模态知识图谱**：从图像、视频中抽取实体和关系
- **GraphRAG**：知识图谱增强的检索增强生成
- **LLM + 知识图谱**：LLM 利用知识图谱进行事实约束，减少幻觉
- **自动知识图谱构建**：LLM 自主从文本构建和维护知识图谱
