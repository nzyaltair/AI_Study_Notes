# 数据收集与清洗

## 1. 概述

数据是大语言模型训练的基础，其质量直接决定模型最终性能。随着预训练数据规模从GPT-2的WebText（40GB）增长到LLaMA 3的15T tokens，数据处理的工程化和自动化程度不断提升。业界形成了"数据质量优先于数量"的共识——高质量数据的缩放效果显著优于低质量数据。

数据工程涵盖数据收集、格式标准化、去重、质量过滤、脱敏和领域配比六个核心环节，是预训练成功的关键前提。

## 3. 核心概念

### 3.1 数据来源

| 数据集 | 规模 | 特点 |
|-------|------|------|
| CommonCrawl | 100T+ tokens | 涵盖网页内容，需重度清洗 |
| C4 | 10T tokens | 清洗后的CommonCrawl子集 |
| The Pile | 800B tokens | 22个高质量数据集混合 |

| StackExchange | 20B tokens | 问答对 |

### 3.2 数据质量维度

- **准确性**：信息正确，无事实错误
- **多样性**：覆盖多种主题、风格、难度
- **信息密度**：单位文本包含的有效信息量
- **语言规范性**：语法正确，表达清晰
- **合规性**：无毒无害，无敏感信息

## 4. 技术原理

### 4.1 数据去重

**精确去重**：使用SHA-256等哈希算法识别完全相同的文本，速度快但无法处理近似重复。

**近似去重**：

- **MinHash**：通过最小哈希签名快速估算Jaccard相似度，适合大规模数据
- **SimHash**：生成64/128位指纹哈希，通过Hamming距离判断相似性
- **LSH（Locality Sensitive Hashing）**：基于局部敏感哈希的近似近邻搜索

工业实践：LLaMA 3结合4-gram精确去重与MinHash近似去重，去重率达40%-50%；Qwen 2采用多阶段去重（文档级→段落级→语义级），去重率达60%。

### 4.2 质量过滤

**基础过滤**：长度过滤、字符比例过滤、重复模式过滤（如n-gram重复率）。

**语言识别**：使用fasttext等工具识别并保留目标语言文本。

**困惑度过滤**：用预训练语言模型计算文本困惑度，过滤低质量文本。高质量文本通常困惑度适中。

**模型辅助过滤**：训练专门的数据质量评估模型，对文本进行多维度评分（语法正确性、信息密度、多样性）。LLaMA系列使用fasttext分类器判断文本是否"类似维基百科"。

### 4.3 毒性与有害内容过滤

- **毒性分类**：识别仇恨言论、歧视性内容
- **有害内容过滤**：过滤暴力、色情、虚假信息
- 工具：Perspective API、Detoxify

### 4.4 敏感信息脱敏（PII）

- **PII类型**：姓名、身份证号、手机号、邮箱、地址
- **脱敏策略**：替换、模糊化、删除
- 工具：Microsoft Presidio、spaCy NER

### 4.5 领域数据配比与采样

- **均匀采样**：各领域按固定比例混合
- **加权采样**：根据领域重要性调整采样权重
- **基于困惑度的加权**：根据不同来源语料的困惑度动态调整采样权重
- **数据配比定律**（Data Mixing Laws）：近期研究表明数据混合比例对模型性能有显著影响，可通过小规模实验预测最优配比

### 4.6 合成数据生成

- **Self-Instruct**：让模型生成指令-响应对
- **Evol-Instruct**：迭代优化生成的指令，逐步增加复杂度
- **UltraChat**：生成多轮对话数据
- **数据飞轮**：模型生成数据 → 训练更强模型 → 生成更高质量数据

## 5. 关键方法/模型

### 5.1 关键算法

| 算法 | 用途 | 特点 |
|------|------|------|
| MinHash + LSH | 大规模近似去重 | 标准方案，O(n)复杂度 |
| fasttext | 语言识别 | 快速轻量 |
| SimHash | 近似去重 | 适合短文本 |
| Self-Instruct | 合成指令数据 | 模型自举生成 |
| DQI | 数据质量评估 | 多维度评分 |

### 5.2 工业级数据管线

- **LLaMA 3管线**：CC数据 → 去重 → 质量过滤 → 毒性过滤 → 配比混合
- **Qwen 2管线**：多语言采集 → 多阶段去重 → 模型质量评分 → 领域配比
- **DeepSeek管线**：代码数据增强 + 数学数据增强 + 通用文本

## 6. 优势与局限

### 优势

- **系统化**：从采集到配比的完整流程，可复现
- **可扩展**：Pipeline可处理万亿级token
- **质量可控**：多维度过滤确保数据质量

### 局限

- **高质量数据枯竭**：互联网高质量文本增长速度低于模型训练需求
- **过滤偏差**：质量过滤可能无意中排除某些领域的有效文本
- **去重损失**：过度去重可能删除有价值的重复信息（如法律条文引用）
- **合成数据风险**：模型生成的数据可能包含错误信息或模式坍缩

## 7. 应用场景

- **预训练数据准备**：LLaMA/Qwen/DeepSeek的数据处理Pipeline
- **SFT数据构建**：指令多样性、难度分布、拒绝采样
- **偏好数据收集**：人类标注、AI生成、Pair-wise比较
- **领域适配**：医疗、法律、金融领域数据清洗与配比

## 9. 前沿发展

### 9.1 数据质量Scaling

研究表明，数据质量比数据数量更重要。高质量数据的缩放曲线显著优于低质量数据，"数据质量Scaling Laws"正在被系统研究。

### 9.2 合成数据

- **模型自生成数据**：用强模型生成高质量训练数据
- **数据飞轮**：模型生成 → 训练 → 生成更高质量数据的循环
- **验证机制**：通过规则或模型验证合成数据的正确性

### 9.3 数据配比优化

- **Data Mixing Laws**：系统研究不同领域数据混合比例对性能的影响
- **动态配比**：训练过程中动态调整数据配比
- **课程学习**：按难度递增的顺序组织训练数据

### 9.4 多模态数据

- 统一处理文本、图像、视频、音频数据
- 跨模态对齐数据构建
- 多模态合成数据生成
