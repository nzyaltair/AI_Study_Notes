# 推理加速与并行

## 背景与发展

大语言模型推理面临三大瓶颈：计算瓶颈（自注意力 $O(n^2)$ 复杂度）、内存瓶颈（模型参数和 [[KV-Cache]] 占用大量显存）、带宽瓶颈（显存带宽限制参数访问速度）。随着模型规模和上下文长度增长，这些瓶颈日益突出。

推理加速技术从早期的算子融合和模型并行，发展到 PagedAttention、连续批处理、Speculative Decoding 等系统级创新。vLLM、TensorRT-LLM 等框架将推理吞吐量提升了数倍至数十倍。

## 核心思想

通过计算优化（算子融合、FlashAttention）、内存优化（KV-Cache 管理、PagedAttention）、并行策略（张量并行、流水线并行）和调度优化（连续批处理、Speculative Decoding），在保持模型精度的前提下最大化吞吐量、降低延迟和成本。

## 技术原理

### 算子融合

将多个独立算子合并为复合算子，减少内核启动和中间结果的内存访问开销。典型应用包括 FlashAttention 和 TensorRT-LLM 的内核融合。

### FlashAttention

重新设计注意力计算的内存访问模式，将注意力计算分块在 GPU SRAM 中执行，减少对 HBM 的读写：

- 速度提升 2-4x，内存使用降低 50-70%
- 支持更长序列，计算复杂度仍为 $O(n^2)$ 但 IO 大幅优化

### PagedAttention

借鉴 OS 虚拟内存管理，将 [[KV-Cache]] 划分为固定大小的块，通过页表管理不连续内存：

- 内存利用率提升 60-70%
- 支持变长序列和连续批处理
- 减少 OOM 风险

### 连续批处理（Continuous Batching）

传统批处理需等待同 batch 所有请求完成。连续批处理在每步动态加入新请求、移除已完成请求：

- GPU 利用率从 30-40% 提升到 80-90%
- 吞吐量提升 3-4x
- 延迟降低 30-50%

### 张量并行

将模型权重矩阵在多个 GPU 上分割，并行计算后通过 All-Reduce 聚合结果：

- 适合超大模型（>30B），单 GPU 显存不足的场景
- 计算速度与 GPU 数量近似线性增长
- 通信开销随 GPU 数量增加

### 流水线并行

将模型层在多个 GPU 上分割，形成流水线处理不同层的计算：

- 适合层数较多的模型
- 存在流水线气泡，GPU 利用率有损失

### Speculative Decoding

使用小模型提前生成多个候选 token，大模型一次性验证：

- 端到端延迟降低 30-50%
- 吞吐量提升 2-3x
- 生成质量与大模型完全一致

### Offloading

将部分模型参数或 KV-Cache 从 GPU 转移到 CPU 内存或 NVMe 存储：

- 支持比 GPU 显存更大的模型
- 代价是增加延迟、降低吞吐量
- 典型实现：llama.cpp GPU-CPU offloading、DeepSpeed ZeRO-Inference

## 发展演进

算子融合 -> FlashAttention（2022）-> PagedAttention/vLLM（2023）-> 连续批处理 -> Speculative Decoding -> FP8 推理

## 关键算法·模型

### 主流推理引擎对比

| 引擎 | 核心优化 | 并行支持 | 量化支持 | 适用场景 |
|------|---------|---------|---------|---------|
| vLLM | PagedAttention、连续批处理 | 张量并行 | INT4/INT8/AWQ | 高并发在线服务 |
| TensorRT-LLM | 算子融合、Tensor Core | 张量/流水线并行 | INT4/INT8/FP8 | 低延迟高性能 |
| llama.cpp | CPU/GPU 混合 | 单机 | GGUF INT4/INT8 | 边缘设备 |
| SGLang | RadixAttention | 张量并行 | INT4/INT8 | Agent 应用 |
| DeepSpeed-FastGen | Dynamic Splitfuse | 张量/流水线并行 | INT4/INT8 | 超大模型 |

### LLaMA-2-70B 性能对比（A100 80GB, INT4）

| 引擎 | 延迟（ms/token） | 吞吐量（tokens/s） |
|------|-----------------|-------------------|
| vLLM | 120 | 267 |
| TensorRT-LLM | 90 | 356 |
| TGI | 180 | 178 |
| DeepSpeed | 200 | 160 |

## 应用场景

- **高并发在线服务**：vLLM + 连续批处理 + PagedAttention
- **低延迟实时推理**：TensorRT-LLM + FP8 + Speculative Decoding
- **超大模型部署**：张量并行 + 流水线并行 + Offloading
- **边缘/本地部署**：llama.cpp + CPU/GPU 混合推理
- **Agent 应用**：SGLang + 结构化生成

## 与其他技术关系

- [[KV-Cache]] 优化是推理加速的基础
- [[量化]] 减少模型参数和 KV-Cache 的显存占用
- [[剪枝与稀疏化]] 减少计算量，N:M 稀疏获得硬件加速
- PagedAttention 是 [[vLLM]] 的核心创新
- FlashAttention 优化 [[注意力机制]] 的内存访问
- 连续批处理与 Speculative Decoding 提升系统吞吐量
