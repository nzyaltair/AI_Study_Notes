# 视觉语言模型

## 1. 概述

视觉语言模型（Vision-Language Model, VLM）是连接视觉与语言两个模态的 AI 系统，使模型能够"看图说话"（图像描述、视觉问答）和"按文理解"（图文匹配、视觉推理）。VLM 是多模态 AI 的核心方向，也是 [[00_大语言模型核心架构_综述|大语言模型]] 向多模态扩展的关键路径。

VLM 解决的核心问题：
- **跨模态语义鸿沟**：图像像素与文本 token 处于完全不同的表示空间，需要桥梁对齐
- **零样本视觉理解**：无需为每个视觉任务标注数据，通过自然语言描述即可完成任务
- **视觉推理与对话**：不仅识别图像内容，还能基于图像进行复杂推理和多轮对话
- **文档与图表理解**：理解结构化视觉信息（表格、公式、图表、文档）

## 3. 核心概念

### 3.1 架构范式

VLM 的发展经历了四种主要架构范式：

| 范式 | 架构 | 代表模型 | 特点 |
|:---|:---|:---|:---|
| 对比学习 | 双塔编码器 + 对比损失 | CLIP, ALIGN | 简单高效，零样本能力强，但不擅长生成 |
| 桥接式 | 视觉编码器 + 桥接模块 + LLM | BLIP-2, Flamingo | 冻结大模型，只训练桥接模块，高效 |
| 投影式 | 视觉编码器 + 投影层 + LLM | LLaVA, Qwen-VL | 简单投影连接，端到端微调 |
| 原生多模态 | 从头训练统一模型 | GPT-4V, Gemini | 所有模态共享参数，能力最强但训练成本最高 |

### 3.2 视觉编码器

将图像转换为视觉特征序列的模块：
- **ViT（Vision Transformer）**：将图像分割为 $16 \times 16$ patch，线性投影为 token 序列，加位置编码后输入 Transformer
- **CLIP ViT**：CLIP 预训练的视觉编码器，提供与语言对齐的视觉特征，是大多数 VLM 的默认选择
- **InternViT**：6B 参数大规模视觉编码器，InternVL 使用
- **SigLIP**： sigmoid 损失替代 softmax 的 CLIP 变体，更适合大批量训练

### 3.3 视觉-语言对齐

将对齐方式分类：

| 方法 | 原理 | 代表 |
|:---|:---|:---|
| 线性投影 | MLP 将视觉特征映射到 LLM 嵌入空间 | LLaVA |
| Q-Former | 可学习 Query token 通过交叉注意力提取视觉特征 | BLIP-2 |
| 交叉注意力 | LLM 层间插入交叉注意力层 | Flamingo, CogVLM |
| 原生融合 | 视觉 token 与文本 token 直接拼接为统一序列 | GPT-4V, Gemini |

### 3.4 视觉 Token 压缩

视觉 token 数量直接影响计算效率，压缩方法包括：
- **Q-Former 查询压缩**：BLIP-2 用 32 个 Query 提取固定数量特征
- **Perceiver Resampler**：Flamingo 用可学习 latents 压缩变长视觉特征
- **Token 合并/剪枝**：相似 token 合并，冗余 token 剪枝
- **动态分辨率**：Qwen2-VL 根据图像内容动态调整 patch 数量

## 4. 技术原理

### 4.1 CLIP：对比学习对齐

CLIP 是跨模态对齐的奠基性工作。核心思想：在大规模图文对上，通过对比学习将匹配的图文对在共享语义空间中拉近，不匹配的推远。

**架构**：
```
图像 → ViT 图像编码器 → 图像特征 I ∈ R^D
文本 → Transformer 文本编码器 → 文本特征 T ∈ R^D
                              ↓
              对称 InfoNCE 对比损失
```

**对称 InfoNCE 损失**：

$$\mathcal{L}_{CLIP} = \frac{1}{2N}\sum_{i=1}^N\left[-\log \frac{\exp(I_i \cdot T_i / \tau)}{\sum_{j=1}^N \exp(I_i \cdot T_j / \tau)} - \log \frac{\exp(I_i \cdot T_i / \tau)}{\sum_{j=1}^N \exp(I_j \cdot T_i / \tau)}\right]$$

前半部分是图像→文本方向，后半部分是文本→图像方向。$\tau$ 为可学习温度参数。

**零样本分类**：构造文本提示 "a photo of a {label}"，计算图像与各类别文本的相似度，选择最高的。

### 4.2 BLIP-2：Q-Former 桥接

BLIP-2 的核心创新是 Q-Former——一个轻量级 Transformer，用可学习 Query 从冻结的视觉编码器中提取与语言相关的特征。

**Q-Former 结构**：
- $N_q$ 个可学习 Query token（通常 32-64 个）
- 共享自注意力层（Query 之间交互）
- 交叉注意力层（Query 关注视觉特征）
- FFN 层

**训练阶段**：
1. **阶段一**：冻结视觉编码器和 LLM，训练 Q-Former 学习从图像中提取文本相关特征（图文对比 + 图文匹配 + 图像描述生成三任务）
2. **阶段二**：将 Q-Former 输出通过线性层投影到 LLM 输入空间，训练投影层使 LLM 能理解视觉特征

### 4.3 LLaVA：投影式架构

LLaVA 证明了简单的线性投影即可有效连接视觉编码器和 LLM：

```
图像 → CLIP ViT → 视觉特征 H_v → MLP 投影 → 视觉 Token H_p
                                                   ↓
文本 → Tokenization → 文本 Token → [H_p; 文本 Token] → LLM → 输出
```

**投影公式**：

$$H_p = \text{CLIP\_ViT}(x_{image}) \cdot W_{project} \in \mathbb{R}^{N_v \times D_{LLM}}$$

**两阶段训练**：
1. **预训练对齐**：用 595K 图文对，冻结 ViT 和 LLM，只训练投影层 $W_{project}$
2. **指令微调**：用 158K 多模态指令数据（GPT-4 生成），冻结 ViT，微调投影层和 LLM

### 4.4 Flamingo：交叉注意力融合

Flamingo 在冻结的 LLM 层间插入交叉注意力层，使 LLM 能"查看"视觉信息：

```
文本 LLM 层 → 自注意力 → 新增交叉注意力(关注视觉特征) → FFN → 下一层
```

- **Perceiver Resampler**：将变长视觉特征压缩为固定数量的 latents
- **门控交叉注意力**：初始化为零的门控参数，训练时逐步打开，保护 LLM 原有能力
- **少样本能力**：通过交错图文示例实现 in-context learning

### 4.5 训练策略详解

| 阶段 | 目标 | 数据规模 | 冻结策略 | 学习内容 |
|:---|:---|:---|:---|:---|
| 视觉-语言预训练 | 跨模态对齐 | 数亿图文对 | 冻结 LLM + 视觉编码器 | 投影/桥接模块 |
| 指令微调 | 多模态对话 | 数十万指令 | 冻结视觉编码器 | 投影层 + LLM |
| 偏好对齐 | 人类偏好 | 数万偏好对 | 全参或 LoRA | 对齐人类偏好 |

**指令数据构建**（LLaVA 方法）：
1. 给 GPT-4 提供图像描述和目标指令
2. 让 GPT-4 生成多模态指令-回答对
3. 覆盖对话、详细描述、复杂推理三类任务

## 6. 优势与局限

### 6.1 优势

- **零样本能力**：CLIP 无需特定任务训练即可完成零样本分类
- **通用理解**：一个模型处理多种视觉-语言任务
- **自然交互**：用户可用自然语言描述需求，无需专业接口
- **推理能力**：现代 VLM 不仅能识别，还能基于图像进行复杂推理

### 6.2 局限

- **幻觉问题**：VLM 可能"看到"图像中不存在的内容（视觉幻觉）
- **细粒度理解不足**：对微小物体、精确计数、空间关系理解仍有困难
- **分辨率限制**：固定分辨率编码丢失高分辨率细节
- **视觉 token 冗余**：大量视觉 token 增加计算开销但未必都有用
- **模态偏置**：语言先验可能覆盖视觉证据，导致"看图说话"而非"看图说真话"
- **幻觉评估困难**：缺乏标准化的多模态幻觉评估基准

## 7. 应用场景

- **多模态对话**：带图片的智能助手（GPT-4V, Gemini）
- **视觉问答（VQA）**：基于图像回答自然语言问题
- **OCR 与文档理解**：表格、公式、图表、发票识别（Qwen-VL）
- **图像检索**：文本到图像搜索（CLIP）
- **多模态 Agent**：UI 自动化操作、视觉导航
- **教育辅助**：拍照解题、可视化教学
- **医疗影像分析**：结合病历的医学影像理解
- **自动驾驶**：场景理解与决策辅助
- **内容审核**：图文联合安全审核

## 8. 与其他技术关系

- VLM 以 [[00_大语言模型核心架构_综述|大语言模型]] 为语言推理基座，LLM 提供语义理解和生成能力
- 视觉编码器基于 [[05_卷积神经网络|ViT]] 或 CNN 架构，ViT 将图像分块序列化
- [[08_Transformer与注意力机制|Transformer]] 是跨模态融合的统一架构，交叉注意力连接模态
- CLIP 对齐技术也用于 [[02_图像生成|图像生成]] 的文本条件编码
- VLM 扩展了 [[19_LLM应用工程/04_Agent系统|Agent]] 的感知能力，是 GUI Agent 的核心
- VLM 的指令微调方法继承自 [[11_大模型训练与对齐|大模型训练]] 的 SFT 范式
- 参见综述：[[00_多模态AI]]

## 9. 前沿发展

- **原生多模态**：从桥接式走向从头训练的统一模型（GPT-4o），消除模态间信息瓶颈
- **动态分辨率**：Qwen2-VL 的 Naive Dynamic Resolution，根据图像内容自适应调整 patch 数
- **视觉 Token 压缩**：TokenPacker、LLaVA-PruMerge 等方法减少视觉 token 数量
- **多模态推理链**：Visual Chain-of-Thought，将 CoT 推理扩展到视觉领域
- **高分辨率理解**：支持 4K+ 图像输入，用于遥感、医学等高精度场景
- **视频理解**：从图像理解扩展到视频时序理解（LLaVA-NeXT-Video, Video-LLaVA）
- **多图理解**：支持多图输入进行对比、排序、推理
- **幻觉缓解**：通过 DPO/VHD 等方法减少视觉幻觉
- **高效训练**：LoRA、QLoRA 等参数高效微调方法在 VLM 中的应用
