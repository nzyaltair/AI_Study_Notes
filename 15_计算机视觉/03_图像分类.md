# 图像分类

## 1. 概述

图像分类（Image Classification）是计算机视觉最基础的任务：给定一张图像，输出其所属的语义类别。它要求模型回答"这张图片里是什么"的问题，是目标检测、图像分割等更复杂视觉任务的基础。

图像分类的发展史几乎就是计算机视觉的发展史——从手工特征 + SVM，到 CNN 的端到端学习，再到 Vision Transformer 的注意力范式，每一次突破都推动了整个领域的进步。

本篇从 CV 任务的视角梳理图像分类的发展脉络、核心方法、评估体系和工程实践。关于 CNN 和 ViT 的架构细节，参见 [[05_卷积神经网络]] 和 [[06_视觉Transformer]]。

## 3. 核心概念

### 3.1 任务定义

给定输入图像 $I \in \mathbb{R}^{H \times W \times C}$，学习映射 $f_\theta: I \to y$，其中 $y \in \{1, 2, \ldots, K\}$ 为类别标签。

- **单标签分类**：每张图像只有一个类别
- **多标签分类**：一张图像可属于多个类别
- **细粒度分类**：区分同一大类下的子类（如鸟类品种、汽车型号）
- **零样本分类**：对未见类别进行分类（如 CLIP）

### 3.2 评估指标

| 指标 | 公式 | 说明 |
|:---|:---|:---|
| Top-1 Accuracy | $\frac{\text{正确预测数}}{\text{总样本数}}$ | 最常用的分类指标 |
| Top-5 Accuracy | 预测的前 5 类包含真实标签的比例 | ImageNet 标准指标 |
| Precision / Recall | $P = \frac{TP}{TP+FP}$, $R = \frac{TP}{TP+FN}$ | 每类指标 |
| Macro F1 | 各类 F1 的算术平均 | 对类别不平衡敏感 |
| Confusion Matrix | $K \times K$ 矩阵 | 可视化分类错误分布 |

### 3.3 数据集

| 数据集 | 类别数 | 图像数 | 特点 |
|:---|:---|:---|:---|
| MNIST | 10 | 70,000 | 手写数字，入门基准 |
| CIFAR-10/100 | 10/100 | 60,000 | 小尺寸（32×32），快速实验 |
| ImageNet-1K | 1,000 | 1.28M | 大规模分类标准基准 |
| ImageNet-21K | 21,841 | 14M | 大规模预训练数据 |
| Places-365 | 365 | 1.8M | 场景分类 |
| CUB-200 | 200 | 11,788 | 细粒度鸟类分类 |
| Stanford Dogs | 120 | 20,580 | 细粒度犬种分类 |
| JFT-300M | 18,291 | 300M | Google 内部大规模数据 |

## 4. 技术原理

### 4.1 传统方法时代（2012 之前）

经典流程：**手工特征提取 + 编码 + 分类器**

1. **特征提取**：SIFT、HOG、LBP 等局部描述子
2. **特征编码**：
   - **BoVW（Bag of Visual Words）**：K-Means 聚类构建视觉词典，统计直方图
   - **Fisher Vector**：用 GMM 建模特征分布，编码一阶/二阶统计量
   - **VLAD（Vector of Locally Aggregated Descriptors）**：编码特征与聚类中心的残差
3. **分类器**：线性 SVM、核 SVM、随机森林

### 4.2 深度学习时代（2012 至今）

端到端学习取代了手工特征 + 分类器的分步流程：

$$\text{图像} \xrightarrow{\text{CNN/ViT backbone}} \text{特征向量} \xrightarrow{\text{全连接层}} \text{logits} \xrightarrow{\text{Softmax}} \text{类别概率}$$

**交叉熵损失**：

$$\mathcal{L}_{CE} = -\sum_{i=1}^{K} y_i \log \hat{p}_i, \quad \hat{p}_i = \frac{e^{z_i}}{\sum_j e^{z_j}}$$

**标签平滑（Label Smoothing）**：

$$\tilde{y}_i = (1-\epsilon) y_i + \frac{\epsilon}{K}$$

防止模型过度自信，提升泛化。

### 4.3 数据增强

数据增强是提升分类性能的关键技术：

| 类别 | 方法 | 说明 |
|:---|:---|:---|
| 几何变换 | 随机裁剪、翻转、旋转、缩放 | 增加位置/方向不变性 |
| 颜色变换 | 色彩抖动、亮度/对比度调整 | 增加光照不变性 |
| Cutout | 随机遮挡图像区域 | 提升对遮挡的鲁棒性 |
| MixUp | $\tilde{x} = \lambda x_i + (1-\lambda) x_j$, $\tilde{y} = \lambda y_i + (1-\lambda) y_j$ | 线性混合样本和标签 |
| CutMix | 将区域替换为另一张图的区域 | 比 MixUp 保留更多空间信息 |
| RandAugment | 从增强列表中随机采样并应用 | 自动化增强策略 |
| AutoAugment | 强化学习搜索最优增强策略 | 数据驱动增强 |

### 4.4 迁移学习

利用预训练模型加速新任务学习：

- **特征提取**：冻结 backbone，只训练分类头
- **微调**：以较小学习率训练整个网络
- **领域自适应微调**：在不同学习率下分层微调

**预训练数据规模效应**：在 ImageNet-21K 或 JFT-300M 上预训练的模型在下游任务上泛化显著优于 ImageNet-1K 预训练。

### 4.5 训练技巧

现代分类模型训练的关键技术：

- **学习率调度**：Cosine Annealing、Warmup + Cosine Decay
- **正则化**：Dropout、Stochastic Depth、Weight Decay
- **归一化**：BatchNorm（CNN）/ LayerNorm（ViT）
- **EMA（指数移动平均）**：维护模型权重的 EMA 副本，推理时使用 EMA 权重
- **混合精度训练**：FP16/BF16 加速训练，减少显存

### 5.3 零样本分类

**CLIP** 通过对比学习对齐图像与文本嵌入，实现零样本分类：

$$\hat{y} = \arg\max_i \frac{\exp(\text{sim}(I, T_i) / \tau)}{\sum_j \exp(\text{sim}(I, T_j) / \tau)}$$

其中 $T_i = \text{TextEncoder}(\text{"a photo of a } [class_i]\text{"})$。

参见 [[14_多模态AI/01_视觉语言模型]]。

## 6. 优势与局限

### 优势

1. **任务定义清晰**：输入输出明确，易于评估和比较
2. **基准完善**：ImageNet 等数据集提供标准化评测
3. **成熟度高**：从研究到产品部署的完整工具链
4. **迁移性强**：预训练分类模型是众多视觉任务的 backbone

### 局限

1. **单标签假设局限**：现实图像通常包含多个物体
2. **长尾分布问题**：类别频率不平衡导致少数类性能差
3. **分布外泛化困难**：对未见领域/风格泛化能力有限
4. **对抗脆弱性**：微小人眼不可见的扰动可导致错误分类

## 8. 与其他技术关系

- **与卷积神经网络的关系**：CNN 是图像分类的核心架构，ResNet/EfficientNet 等是分类 SOTA 的代表。参见 [[05_卷积神经网络]]。
- **与视觉 Transformer 的关系**：ViT 开创了 Transformer 在图像分类上的应用。参见 [[06_视觉Transformer]]。
- **与目标检测的关系**：分类是检测的基础——检测器需要分类 backbone 提取特征并判断区域类别。参见 [[04_目标检测]]。
- **与自监督学习的关系**：自监督预训练（MAE/DINO）为分类提供了强大的初始化表示。参见 [[09_自监督视觉学习]]。
- **与多模态 AI 的关系**：CLIP 实现了零样本分类，将分类从闭集扩展到开放词汇。参见 [[14_多模态AI]]。

## 9. 前沿发展

- **基础模型分类**：SAM/CLIP 等大规模预训练模型实现通用视觉理解，分类作为零样本能力
- **多模态分类**：结合图像与文本/音频的多模态分类
- **持续学习**：在不遗忘旧类的前提下学习新类别
- **可解释分类**：注意力可视化、概念瓶颈模型提升分类可解释性
- **鲁棒性研究**：分布外泛化、对抗鲁棒性、公平性
- **高效分类**：在边缘设备上实现实时高精度分类（知识蒸馏、量化）
