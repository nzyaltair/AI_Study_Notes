# 位置编码（LLM 视角）

## 一句话理解

位置编码为排列不变的自注意力注入序列顺序信息；现代 LLM 几乎统一采用旋转位置编码（RoPE），并通过 YaRN / NTK-aware 等外推技术从训练长度扩展到数十倍长的推理上下文。

## 概述

本笔记从**大语言模型架构实践**的视角看位置编码，只关注 LLM 选型与长上下文外推；位置编码的完整机制、推导与所有方法对比见权威笔记 [[04_位置编码]]（位于 08_Transformer与注意力机制 方向）。

## LLM 主流位置编码选型

| 模型 | 位置编码 | 训练长度 | 最大长度 | 外推方法 |
|:---|:---|:---|:---|:---|
| GPT-2 | 可学习 | 1024 | 1024 | 无 |
| BLOOM | ALiBi | 2K | 8K+ | 原生外推 |
| LLaMA 2 | RoPE | 4K | 4K | - |
| LLaMA 3 | RoPE | 8K | 128K | YaRN |
| Mistral | RoPE | 8K | 32K | - |
| Qwen 2 | RoPE | 32K | 128K | NTK-aware / YaRN |
| DeepSeek-V3 | RoPE | 4K-32K | 128K | YaRN |

要点：

- **RoPE 已是事实标准**：LLaMA / Qwen / Mistral / DeepSeek 等主流开源 LLM 全部采用 RoPE。
- **ALiBi 走向边缘**：原生外推优势被 RoPE + 外推技巧（YaRN）取代，仅在 BLOOM / MPT 等少数模型中使用。
- **可学习嵌入不再适用于 LLM**：最大长度固定、无法外推，仅在 BERT / GPT-2 等短上下文时代模型中使用。

## 长上下文外推（LLM 关键实践）

长上下文 LLM（128K → 1M token）的核心挑战是把短上下文训练的 RoPE 模型外推到远超训练长度的推理场景：

- **位置插值（PI）**：线性压缩位置索引，4-8x，用于 LLaMA-2-32K。
- **NTK-aware**：调整 RoPE 基频，8-32x，社区方案、Qwen 1.5 采用。
- **YaRN**：分频段处理（高频外推、低频插值、中频过渡）+ 注意力缩放补偿，16-64x，当前最主流方案，LLaMA 3 / DeepSeek-V3 采用。
- **LongRoPE**：进化搜索非均匀频率插值，128x+，微软方案。
- **Dynamic NTK**：推理时根据当前缓存长度动态调整 $\theta$，vLLM 默认。

外推方法的完整对比与公式推导见 [[04_位置编码#4.6 长度外推方法]]。

## 与 LLM 架构其他组件的耦合

- **与注意力变体**：RoPE 需与 [[02_注意力机制|MQA/GQA/MLA]] 协同设计——MLA（DeepSeek）将 RoPE 应用于解耦的注意力部分。
- **与长上下文机制**：外推是 [[10_大语言模型核心架构/07_长上下文与记忆机制|长上下文与记忆机制]] 的基础，配合滑动窗口 / 注意力 sink（StreamingLLM）实现超长上下文。
- **与 KV-Cache**：Dynamic NTK 在推理时根据 KV-Cache 长度动态调整位置编码，详见 [[12_大模型推理与优化/01_KV-Cache机制|KV-Cache]]。

## 前沿发展

- **NoPE**：Decoder-only 模型中因果掩码已隐含位置信息，短序列可完全不用位置编码（Kazemnejad et al., 2023），长序列外推能力仍待验证。
- **Self-Extend**：推理时无需微调的位置扩展方法。
- **SSM 路线**：Mamba 等架构通过递归状态天然编码位置，绕开显式位置编码。
