# 注意力机制原理

## 1. 概述

注意力机制（Attention Mechanism）是一种让模型在处理序列数据时，能够动态地将计算资源分配到输入中最相关部分的技术。其核心思想是**软寻址（Soft Addressing）**：给定一个查询（Query），模型计算它与所有键（Key）的相似度，得到注意力权重，再对值（Value）进行加权求和，从而获得与当前查询最相关的信息。

- **解决的问题**：传统 Seq2Seq 将整个输入序列压缩为固定长度的上下文向量，长序列信息丢失严重。注意力机制允许解码器在每一步动态地关注输入序列的不同部分，打破了信息瓶颈。
- **历史意义**：注意力机制从 Seq2Seq 的辅助模块发展为 Transformer 的核心组件，是深度学习架构设计中最具影响力的思想之一。

## 3. 核心概念

### 3.1 Query-Key-Value 框架

注意力机制的三个核心要素，借鉴自信息检索系统：

| 概念 | 角色 | 类比 |
|:---|:---|:---|
| **Query（查询）** | 当前需要查找的信息 | 搜索关键词 |
| **Key（键）** | 每个位置信息的"标签" | 索引项 |
| **Value（值）** | 每个位置的实际内容 | 文档内容 |

计算流程：Query 与所有 Key 计算相似度 → softmax 得到注意力权重 → 对 Value 加权求和。

### 3.2 注意力分数（Attention Score）

衡量 Query 与 Key 之间相关性的标量值。不同的分数函数定义了不同的注意力变体：

| 类型 | 公式 | 特点 | 提出者 |
|:---|:---|:---|:---|
| **加性注意力** | $e_{t,i} = v_a^T \tanh(W_a s_{t-1} + U_a h_i)$ | MLP 计算，参数多 | Bahdanau 2014 |
| **点积注意力** | $e_{t,i} = s_t^T h_i$ | 零额外参数，可矩阵化 | Luong 2015 |
| **通用点积** | $e_{t,i} = s_t^T W_a h_i$ | 带可学习线性变换 | Luong 2015 |
| **缩放点积** | $e_{t,i} = \frac{s_t^T h_i}{\sqrt{d_k}}$ | 缩放防梯度消失 | Vaswani 2017 |

### 3.3 注意力权重的 softmax 归一化

$$\alpha_{t,i} = \frac{\exp(e_{t,i})}{\sum_{j=1}^{T} \exp(e_{t,j})}$$

注意力权重 $\alpha_{t,i} \geq 0$ 且 $\sum_i \alpha_{t,i} = 1$，构成概率分布。这使注意力具有**可微性**——梯度可以通过权重反向传播到所有位置。

### 3.4 上下文向量（Context Vector）

$$c_t = \sum_{i=1}^{T} \alpha_{t,i} \cdot v_i$$

上下文向量是所有 Value 的注意力加权求和，代表"当前查询最需要的信息"。

## 4. 技术原理

### 4.1 注意力的一般形式

注意力机制可统一表示为：

$$\text{Attention}(Q, K, V) = \text{softmax}(\text{score}(Q, K)) V$$

其中 $\text{score}(\cdot)$ 是相似度函数。不同 score 函数的选择影响计算效率和表达能力。

### 4.2 缩放点积注意力的数学推导

给定 $Q \in \mathbb{R}^{n \times d_k}$, $K \in \mathbb{R}^{n \times d_k}$, $V \in \mathbb{R}^{n \times d_v}$：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

**缩放因子的必要性**：假设 $Q, K$ 的元素为均值 0、方差 1 的独立随机变量，则 $QK^T$ 的每个元素是 $d_k$ 个独立项之和，方差为 $d_k$。当 $d_k$ 较大时（如 64），点积值远大于 0，softmax 输出趋近于 one-hot 分布，梯度趋近于零（饱和区）。除以 $\sqrt{d_k}$ 使方差归一化为 1，保持梯度稳定。

**计算复杂度**：
- 时间复杂度：$O(n^2 \cdot d_k)$（$QK^T$ 为 $n \times n \times d_k$，$AV$ 为 $n \times n \times d_v$）
- 空间复杂度：$O(n^2)$（注意力矩阵）

### 4.3 加性注意力 vs 乘性注意力

**加性注意力（Bahdanau）**：

$$e_{t,i} = v_a^T \tanh(W_a s_{t-1} + U_a h_i)$$

- 通过 MLP 计算相似度，引入可学习参数 $W_a, U_a, v_a$
- 在 $d_k$ 较小时性能与点积相当
- 无法利用矩阵乘法优化，计算效率低

**乘性注意力（Luong / Transformer）**：

$$e_{t,i} = \frac{q^T k}{\sqrt{d_k}}$$

- 直接点积，无额外参数（或仅一个投影矩阵）
- 可高效利用矩阵乘法（GEMM），GPU 友好
- 在 $d_k$ 较大时优于加性注意力（Vaswani et al. 2017 实验验证）

### 4.4 注意力的三种范式

| 范式 | Q 来源 | K/V 来源 | 应用场景 |
|:---|:---|:---|:---|
| **自注意力（Self-Attention）** | 当前序列 | 当前序列 | 序列内全局依赖建模 |
| **交叉注意力（Cross-Attention）** | 序列 A | 序列 B | 编码器-解码器交互、多模态融合 |
| **因果注意力（Causal Attention）** | 当前序列（带掩码） | 当前序列（带掩码） | 自回归生成 |

### 4.5 注意力权重的信息论视角

从信息论角度，注意力权重 $\alpha_{t,i}$ 构成位置 $t$ 对输入各位置的信息分布。该分布的**熵**衡量了注意力的"集中度"：

$$H(t) = -\sum_i \alpha_{t,i} \log \alpha_{t,i}$$

- 低熵：注意力高度集中（如对齐翻译中的对应词）
- 高熵：注意力分散（如全局语义聚合）

研究表明，Transformer 的注意力头自发地形成了不同的注意力模式：一些头关注局部窗口，一些头关注固定位置（如句首 `[CLS]`），一些头关注语义相关的远程 token。

## 5. 关键方法/模型

### 5.1 Bahdanau 注意力（加性注意力）

**论文**：Bahdanau, Cho & Bengio, *Neural Machine Translation by Jointly Learning to Align and Translate* (2014)

在 Seq2Seq 编码器-解码器架构中，解码器每一步计算与编码器所有隐藏状态的加性注意力分数，得到动态上下文向量。翻译质量在长句上大幅超越无注意力基线。

### 5.2 Luong 注意力（乘性注意力）

**论文**：Luong, Pham & Manning, *Effective Approaches to Attention-based Neural Machine Translation* (2015)

系统化了注意力计算方式，提出 dot、general、concat 三种分数函数，并引入了 **Input Feeding** 机制（将上一步的注意力上下文作为解码器输入）。

### 5.3 自注意力（Self-Attention）

**论文**：Vaswani et al., *Attention Is All You Need* (2017)

将注意力的 Query、Key、Value 统一为同一序列的线性投影，使序列内任意两个位置直接交互。路径长度为 $O(1)$，支持完全并行化。详见 [[02_Self-Attention与多头注意力]]。

### 5.4 交叉注意力（Cross-Attention）

Query 来自一个序列，Key/Value 来自另一个序列。应用场景包括：
- 编码器-解码器交互（T5、原始 Transformer）
- 多模态融合（CLIP、Flamingo：视觉特征作为 K/V，文本作为 Q）
- 检索增强生成（RAG：检索文档作为 K/V）

## 6. 优势与局限

### 优势

1. **动态信息聚合**：放弃固定瓶颈，按需查询，信息无损
2. **全局感受野**：任意两个位置直接交互，路径长度 $O(1)$
3. **可解释性**：注意力权重提供直观的输入-输出对齐可视化
4. **并行友好**：点积注意力可高效矩阵化，GPU 并行计算
5. **统一框架**：自注意力、交叉注意力、因果注意力统一于 QKV 范式

### 局限

1. **二次复杂度**：标准注意力 $O(n^2)$ 的时间和空间复杂度限制长序列应用
2. **位置无关**：注意力本身是排列不变的，需要额外的位置编码
3. **注意力冗余**：研究发现大量注意力头是冗余的，可被剪枝
4. **中间层注意力难以解释**：浅层注意力有可辨识的模式，深层注意力往往难以直观理解

## 7. 应用场景

| 场景 | 注意力类型 | 说明 |
|:---|:---|:---|
| 机器翻译 | 交叉注意力 + 自注意力 | 编码器自注意力编码源语言，解码器交叉注意力对齐源-目标 |
| 文本生成 | 因果自注意力 | GPT 系列自回归生成 |
| 文本理解 | 双向自注意力 | BERT 编码器双向建模 |
| 图像描述 | 交叉注意力 | CNN 编码图像特征，RNN/Transformer 解码器交叉注意力生成描述 |
| 语音识别 | 自注意力 + 交叉注意力 | Whisper、Conformer |
| 多模态融合 | 交叉注意力 | CLIP（图文对齐）、Flamingo（视觉注入语言模型） |
| 蛋白质结构 | 自注意力 + 交叉注意力 | AlphaFold 2 的 Evoformer |

## 8. 与其他技术关系

- **与 Seq2Seq 的关系**：注意力机制诞生于 Seq2Seq 框架，作为解码器对编码器的动态查询机制。参见 [[06_序列模型/04_Seq2Seq与注意力机制]]。
- **与 Self-Attention 的关系**：自注意力是注意力的特例（Q=K=V 来自同一序列），是 [[02_Self-Attention与多头注意力|Transformer 的核心]]。
- **与位置编码的关系**：注意力本身无法感知位置，需要 [[04_位置编码|位置编码]] 补充顺序信息。
- **与高效注意力的关系**：标准注意力的 $O(n^2)$ 复杂度催生了 [[05_高效注意力机制|大量高效注意力变体]]。
- **与 softmax 的关系**：注意力的权重归一化依赖 softmax，理解 softmax 的梯度行为是理解注意力训练稳定性的基础。

## 9. 前沿发展

- **注意力可解释性**：通过机械可解释性（Mechanistic Interpretability）方法分析注意力头的功能（如 Indirect Object Identification、copy head 等）
- **注意力替代**：Mamba 等 SSM 架构在不使用注意力的情况下实现线性时间的序列建模，挑战注意力的必要性
- **注意力稀疏化**：研究发现注意力矩阵天然稀疏，硬稀疏注意力（如 Sparse Attention、BigBird）在保持性能的同时降低复杂度
- **注意力与检索的融合**：RAG 场景中将注意力池从固定序列扩展到动态检索的外部知识库
- **多模态注意力**：统一注意力架构处理文本、图像、音频、视频等多模态输入
