# 进阶RAG

## 背景与发展

基础RAG面临四大挑战：查询-文档语义鸿沟、检索噪声与冗余、上下文窗口限制、静态检索范式缺乏反馈。进阶RAG通过引入查询优化、多向量检索、融合检索、上下文压缩和索引结构演进，构建更准确、高效、鲁棒的检索增强生成系统。

## 核心思想

在基础RAG的检索-生成流程中，引入多阶段优化：查询前优化（重写/分解）、检索中增强（多向量/融合/递归）、检索后处理（压缩/重排）、索引结构演进（层次化/图检索），以及训练感知的联合优化。

## 技术原理

### 查询优化

**查询重写**：使用LLM将原始查询重写为更有利于检索的形式，提升召回率15-30%。

**HyDE（Hypothetical Document Embeddings）**：先生成假设性答案文档，用该文档的嵌入进行检索，解决查询与文档在嵌入空间中的语义不匹配问题。在事实性任务上Recall@5提升20-40%。

```python
class HyDERetriever:
    def retrieve(self, query):
        # 1. 生成假设性答案
        hypothetical_doc = self.llm.generate(f"回答：{query}")
        # 2. 用假设文档的嵌入检索
        embedding = self.embedding_model.encode(hypothetical_doc)
        # 3. 检索相关文档
        return self.vector_db.similarity_search(embedding, k=5)
```

**子查询分解**：将复杂多跳查询分解为多个子查询，分别检索后合并结果。

### 检索增强

**多向量检索**：为每个文档生成多个向量表示（摘要、关键词、问题），分别检索后融合。

**递归检索**：先检索父文档，再在父文档内部进行细粒度检索。配合父子分块策略：父分块提供上下文，子分块用于精确匹配。

**融合检索**：结合稠密检索（语义）、稀疏检索（BM25）和词汇检索，使用倒数排名融合（RRF）合并结果：

$$\text{RRF}(d) = \sum_{r \in R} \frac{1}{k + \text{rank}_r(d)}$$

其中 $R$ 为检索结果集列表，$k$ 通常取60。

### 上下文压缩与重排

**压缩策略**：

| 策略 | 原理 | 压缩率 | 信息保留 |
|------|------|--------|----------|
| 提取式摘要 | 选择最重要的句子 | 30-50% | 高 |
| 生成式摘要 | LLM生成摘要 | 70-90% | 中 |
| 选择性压缩 | 仅保留相关段落 | 40-60% | 高 |

**多阶段重排**：粗排（基于原始分数）→ 精排（Cross-Encoder）→ 多样性重排（去重）。

### 索引结构演进

**层次化摘要索引**：为文档创建多级摘要索引（文档级→章节级→段落级→原始内容），实现从粗到细的层次化检索。

**图检索**：将文档和实体构建为图结构，支持基于关系的检索和推理。结合知识图谱进行实体扩展和关系遍历。

### 训练感知RAG

**RAG微调**：使用领域数据通过对比学习微调嵌入模型，通过二分类微调重排模型。

**端到端可训练RAG**：构建可微分的检索-生成管道，使用Gumbel-Softmax实现可微采样，联合优化检索损失和生成损失：

$$\mathcal{L} = \alpha \cdot \mathcal{L}_{\text{answer}} + \beta \cdot \mathcal{L}_{\text{retrieval}}$$

## 发展演进

基础RAG → 查询重写/HyDE（2023）→ 多向量/融合检索（2023）→ Self-RAG（2024，自我评估检索需求）→ CRAG（2024，纠正机制）→ Agentic RAG（2024，Agent式动态检索）

## 关键算法·模型

- **HyDE**：假设文档嵌入检索（Gao et al., 2022）
- **RRF**：倒数排名融合（Cormack et al., 2009）
- **Self-RAG**：LLM自评估检索需求（Asai et al., 2023）
- **CRAG**：纠正RAG机制（Yan et al., 2024）
- **ColBERT**：延迟交互检索模型（Khattab & Zaharia, 2020）

## 应用场景

- 复杂多跳问答系统
- 企业级知识库（需要高精度检索）
- 专业领域问答（法律、医疗）
- 多源异构数据检索
