# 参数高效微调

## 1. 简介

参数高效微调（Parameter-Efficient Fine-Tuning, PEFT）是一种微调[[../../5.大语言模型核心/大语言模型|大型语言模型]]的技术，通过仅调整模型的一小部分参数，在保持模型性能的同时，显著降低计算和内存需求。

## 2. 核心概念

### 2.1 为什么需要参数高效微调？

- **计算资源限制**：全参数微调需要大量GPU内存和计算资源
- **存储成本**：每个微调模型需要单独存储，参数高效微调可显著减少存储需求
- **部署效率**：多个微调模型可共享基础模型，简化部署
- **灾难性遗忘**：全参数微调可能导致模型忘记预训练知识

### 2.2 PEFT在LLM训练流水线中的位置

```
预训练 → 监督微调（可选） → 参数高效微调 → 部署
```

## 3. 主流方法

### 3.1 LoRA（Low-Rank Adaptation）

- **核心思想**：在模型层之间插入低秩矩阵，仅训练低秩矩阵参数
- **工作原理**：
  1. 将权重更新分解为两个低秩矩阵的乘积
  2. 仅训练这两个低秩矩阵
  3. 推理时将低秩矩阵与基础模型权重合并
- **优点**：
  - 内存效率高，训练速度快
  - 推理开销小
  - 支持多任务模型共享
- **适用场景**：大型语言模型微调，多任务学习

详细内容请参考：[[LoRA/LoRA|LoRA]]

### 3.2 QLoRA（Quantized LoRA）

- **核心思想**：结合LoRA和量化技术，进一步降低内存需求
- **工作原理**：
  1. 将基础模型量化为4位精度
  2. 在量化模型上应用LoRA
  3. 使用梯度检查点和分页优化内存使用
- **优点**：
  - 可在单个GPU上微调大型模型
  - 保持较高的模型性能
  - 训练速度快
- **适用场景**：超大型语言模型微调，资源受限环境

详细内容请参考：[[QLoRA/QLoRA|QLoRA]]

### 3.3 Adapter

- **核心思想**：在模型层之间插入小型适配器模块，仅训练适配器参数
- **工作原理**：
  1. 在模型的关键层之间插入适配器
  2. 适配器包含两个线性层和一个激活函数
  3. 仅训练适配器参数
- **优点**：
  - 结构灵活，可自定义适配器设计
  - 支持不同的适配器架构
  - 可组合多个适配器
- **适用场景**：多任务学习，领域适应

详细内容请参考：[[Adapter/Adapter|Adapter]]

### 3.4 Prefix Tuning

- **核心思想**：优化输入前缀，不修改模型权重
- **工作原理**：
  1. 在输入前添加可训练的前缀向量
  2. 仅训练前缀向量
  3. 模型权重保持不变
- **优点**：
  - 内存效率高
  - 适合生成任务
  - 可快速切换不同任务
- **适用场景**：生成任务，多任务学习

### 3.5 Prompt Tuning

- **核心思想**：优化软提示，作为模型的输入条件
- **工作原理**：
  1. 学习一组软提示嵌入
  2. 将软提示与硬提示结合使用
  3. 仅训练软提示参数
- **优点**：
  - 参数效率极高
  - 适合特定任务微调
  - 可快速适应新任务
- **适用场景**：特定任务微调，少样本学习

## 4. 工程实践

### 4.1 主流框架

- **PEFT库**：Hugging Face推出的参数高效微调库，支持多种PEFT方法
- **LoRA官方实现**：Microsoft推出的LoRA官方实现
- **QLoRA实现**：支持4位量化的LoRA实现
- **Adapter-Hub**：适配器模块的集合库

### 4.2 典型工作流程

1. **选择基础模型**：根据任务选择合适的预训练模型
2. **选择PEFT方法**：根据资源和任务需求选择合适的PEFT方法
3. **数据准备**：准备任务相关的数据集
4. **配置PEFT参数**：设置秩、学习率、批次大小等参数
5. **训练模型**：使用PEFT方法训练模型
6. **评估模型**：评估模型在验证集上的表现
7. **部署模型**：将PEFT适配器与基础模型合并部署

### 4.3 关键参数选择

| 参数 | 推荐范围 | 说明 |
|------|----------|------|
| LoRA秩（r） | 8-64 | 秩越高，性能越好，但计算和内存需求增加 |
| LoRA alpha | 2*r | 缩放因子，通常设置为秩的2倍 |
| 学习率 | 1e-5-5e-4 | 高于全参数微调的学习率 |
| 批次大小 | 4-32 | 根据GPU内存调整 |
| 训练轮次 | 3-10 | 通常比全参数微调少 |

## 5. 工具与框架

### 5.1 PEFT库使用示例

```python
from peft import LoraConfig, get_peft_model
from transformers import AutoModelForCausalLM, AutoTokenizer

# 加载基础模型
model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-2-7b-hf")
tokenizer = AutoTokenizer.from_pretrained("meta-llama/Llama-2-7b-hf")

# 配置LoRA
lora_config = LoraConfig(
    r=16,  # LoRA秩
    lora_alpha=32,  # 缩放因子
    target_modules=["q_proj", "v_proj"],  # 目标模块
    lora_dropout=0.05,  # Dropout率
    bias="none",  # 不训练偏置
    task_type="CAUSAL_LM"  # 任务类型
)

# 应用LoRA
model = get_peft_model(model, lora_config)

# 查看可训练参数
model.print_trainable_parameters()
```

### 5.2 QLoRA使用示例

```python
from peft import LoraConfig, get_peft_model
from transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig

# 配置4位量化
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_use_double_quant=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.bfloat16
)

# 加载量化模型
model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-70b-hf",
    quantization_config=bnb_config,
    device_map="auto"
)

# 配置LoRA
lora_config = LoraConfig(
    r=64,
    lora_alpha=128,
    target_modules=["q_proj", "v_proj", "k_proj", "o_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)

# 应用LoRA
model = get_peft_model(model, lora_config)
```

## 6. 最佳实践

### 6.1 方法选择

| 场景 | 推荐方法 | 原因 |
|------|----------|------|
| 资源受限环境 | QLoRA | 内存需求最低，可在单GPU上微调大型模型 |
| 多任务学习 | LoRA/Adapter | 支持多任务模型共享，部署效率高 |
| 生成任务 | Prefix Tuning | 专为生成任务优化 |
| 特定任务微调 | Prompt Tuning | 参数效率极高，适合特定任务 |

### 6.2 训练技巧

- **使用合适的学习率**：PEFT通常需要比全参数微调更高的学习率
- **合理设置LoRA秩**：根据模型大小和任务复杂度调整秩
- **选择合适的目标模块**：通常选择注意力层的查询和值投影矩阵
- **使用梯度检查点**：进一步降低内存需求
- **监控训练过程**：关注训练损失和验证性能，避免过拟合

### 6.3 常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 性能不如全参数微调 | LoRA秩过低、训练数据不足 | 增加LoRA秩、增加训练数据、延长训练时间 |
| 训练不稳定 | 学习率过高、批次大小过小 | 降低学习率、增加批次大小、使用梯度裁剪 |
| 推理速度慢 | LoRA模块过多、合并权重开销大 | 减少LoRA模块数量、推理前合并权重 |
