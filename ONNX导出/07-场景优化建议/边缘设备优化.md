# 模块7：应用场景优化建议

## 边缘设备优化

边缘设备（移动设备、IoT设备）通常具有有限的计算资源、内存和电池容量。优化ONNX模型以适应这些约束是边缘AI部署的关键挑战。

---

### 模型量化（Model Quantization）

量化是将模型参数从高精度浮点数（如FP32）转换为低精度整数（如INT8、INT16）的技术，可以显著减少模型大小和推理时间。

#### 1. 动态量化（Dynamic Quantization）

- **原理**：权重在模型转换时静态量化，激活值在推理时动态量化
- **适用场景**：CPU推理，特别是INT8支持的硬件
- **优点**：无需校准数据，实现简单
- **缺点**：推理时有量化开销，速度提升不如静态量化

```python
import torch
import onnx
from onnxruntime.quantization import quantize_dynamic, QuantType

# 动态量化ONNX模型
quantized_model = quantize_dynamic(
    'model_fp32.onnx',
    'model_int8_dynamic.onnx',
    weight_type=QuantType.QUInt8,  # 或 QInt8
    per_channel=True,  # 每通道量化更精确
    reduce_range=True   # 减少激活范围
)
```

#### 2. 静态量化（Static Quantization）

- **原理**：权重和激活都使用量化参数，需要校准数据来确定激活的量化范围
- **适用场景**：要求最高推理性能的场景
- **优点**：推理速度最快，内存占用最小
- **缺点**：需要代表性校准数据，可能出现精度损失

```python
from onnxruntime.quantization import quantize_static, CalibrationDataReader
import onnx

# 定义校准数据读取器
class MyCalibrationDataReader(CalibrationDataReader):
    def __init__(self, data_path):
        self.data = self.load_data(data_path)

    def get_next(self):
        if len(self.data) > 0:
            return {"input": self.data.pop(0)}
        return None

    def rewind(self):
        self.data = self.load_data(data_path)

# 静态量化
quantized_model = quantize_static(
    'model_fp32.onnx',
    'model_int8_static.onnx',
    calibration_data_reader=MyCalibrationDataReader('calibration_data.npy'),
    quant_format=QuantFormat.QOperator,  # 或 QDQ
    activation_type=QuantType.QUInt8,
    weight_type=QuantType.QUInt8,
    per_channel=True
)
```

#### 3. 量化感知训练（QAT - Quantization-Aware Training）

- **原理**：在训练过程中模拟量化误差，让模型适应量化
- **适用场景**：对精度要求高，且能重新训练的场景
- **优点**：精度损失最小，通常比训练后量化效果更好
- **缺点**：需要重新训练，计算成本高

```python
# PyTorch中的QAT示例
import torch
import torch.nn as nn
from torch.quantization import quantize_qat

class QuantizableModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.quant = nn.quantized.QuantStub()
        self.dequant = nn.quantized.DeQuantStub()
        self.conv = nn.Conv2d(3, 64, 3, padding=1)
        self.relu = nn.ReLU()

    def forward(self, x):
        x = self.quant(x)
        x = self.conv(x)
        x = self.relu(x)
        x = self.dequant(x)
        return x

# 准备QAT模型
model_fp32 = QuantizableModel()
model_fp32.qconfig = torch.quantization.get_default_qat_qconfig('fbgemm')
torch.quantization.prepare_qat(model_fp32, inplace=True)

# 正常训练过程...
# 训练完成后转换为量化模型
model_quantized = torch.quantization.convert(model_quantized.eval(), inplace=False)
torch.onnx.export(model_quantized, ...)
```

### 模型剪枝（Pruning）

剪枝通过移除模型中不重要的权重或神经元来减小模型大小和计算量。

#### 剪枝技术类型

1. **非结构化剪枝**
   - 移除单个权重（稀疏矩阵）
   - 压缩率较高但需要特殊硬件/库支持

2. **结构化剪枝**
   - 移除整个滤波器、通道或层
   - 保持密集矩阵格式，通用硬件友好

```python
import torch.nn.utils.prune as prune

# 对卷积层进行剪枝
for module in model.modules():
    if isinstance(module, torch.nn.Conv2d):
        prune.l1_unstructured(
            module,
            name='weight',
            amount=0.3  # 剪枝30%的权重
        )
        prune.remove(module, 'weight')

# 剪枝后重新训练微调
# 导出为ONNX
torch.onnx.export(model, ...)
```

#### 剪枝工作流程

```
原始模型 → 剪枝 → 微调 → 验证 → (循环) → 最终剪枝模型 → ONNX导出
```

**2. 模型加载与推理**

```java
import ai.onnxruntime.OnnxTensor;
import ai.onnxruntime.OnnxTensorInfo;
import ai.onnxruntime.OrtEnvironment;
import ai.onnxruntime.OrtSession;

public class ONNXModelInference {
    private OrtSession session;
    private OrtEnvironment env;

    public ONNXModelInference(Context context, String modelPath) throws OrtException {
        env = OrtEnvironment.getEnvironment();
        OrtSession.SessionOptions options = new OrtSession.SessionOptions();

        // 设置线程数（根据设备CPU核心数调整）
        options.setIntraOpNumThreads(4);
        options.setInterOpNumThreads(2);

        // 启用NNAPI（如果设备支持）
        if (isNNAPIEnabled()) {
            options.addNnapi();
        }

        session = env.createSession(modelPath, options);
    }

    public float[] runInference(float[] inputData) throws OrtException {
        // 创建输入张量
        OnnxTensor inputTensor = OnnxTensor.createTensor(env, inputData,
            new long[]{1, 3, 224, 224}); // batch, channels, height, width

        // 运行推理
        OrtSession.Result result = session.run(Collections.singletonMap("input", inputTensor));

        // 获取输出
        float[] output = (float[]) result.get(0).getValue();

        inputTensor.close();
        result.close();

        return output;
    }
}
```

**3. 性能优化建议**

```java
// 启用图优化
OrtSession.SessionOptions options = new OrtSession.SessionOptions();
```

**2. Swift示例**

```swift
import ONNXRuntime

class ONNXInference {
    let session: OrtSession

    init(modelPath: String) throws {
        let options = OrtSession.SessionOptions()

        // 启用CoreML（苹果神经网络加速）
        options.addCoreML()

        // 设置线程数
        options.setIntraOpNumThreads(4)

        session = try OrtSession(modelPath: modelPath, sessionOptions: options)
    }

    func run(input: [Float]) throws -> [Float] {
        let inputTensor = try OrtTensor(data: input,
                                       shape: [1, 3, 224, 224])

        let outputs = try session.run(inputs: ["input": inputTensor])

        guard let outputTensor = outputs["output"] as? OrtTensor else {
            throw InferenceError.invalidOutput
        }

        return try outputTensor.data()
    }
}
```

### 硬件特定优化

#### ARM NEON优化

ARM NEON是ARM处理器的SIMD（单指令多数据）扩展，ONNX Runtime自动为ARM平台启用NEON优化。

**验证NEON支持**

```bash
# 检查CPU特性
cat /proc/cpuinfo | grep neon

# 运行ONNX Runtime时启用详细日志
onnxruntime_test --log_verbosity 0 2>&1 | grep NEON
```

**通过SessionOptions控制**

```cpp
OrtSession::SessionOptions options;
options.SetIntraOpNumThreads(std::thread::hardware_concurrency());
options.SetInterOpNumThreads(2);
// NEON自动启用，无需额外配置
```

#### Apple Neural Engine（ANE）优化

Apple Silicon（M1/M2/M3）设备上的ANE可以通过CoreML执行提供卓越的性能。

**最佳实践**

1. **模型兼容性**
   - 使用CoreML转换器验证ONNX模型是否支持ANE

```python
import onnx
import onnx_coreml

# 检查模型兼容性
model = onnx.load('model.onnx')
target = onnx_coreml.convert(model,
    image_input_names=['input'],
    preferred_net_architecture='neural_engine')
```

2. **ONNX Runtime配置**

```swift
let options = OrtSession.SessionOptions()
options.addCoreML()  // 启用CoreML后端
options.disableCoreMLCustomOp() // 确保使用ANE而非CPU fallback

// 对于M1/M2设备，NEON和CoreML可能同时使用
```

3. **性能调优**

| 操作 | M1/M2 CPU | M1/M2 GPU | M1/M2 Neural Engine |
|-----|-----------|-----------|---------------------|
| 矩阵乘法 | ~50 GFLOPS | ~2.6 TFLOPS | 专加速器 |
| 内存带宽 | ~68 GB/s | ~68 GB/s | 专用内存 |
| 功耗 | 中等 | 较高 | 最优 |
| 适用模型 | 小型模型 | 中大型模型 | 推荐的网络层 |

### 量化示例：移动端BERT模型

#### 1. 准备模型

```python
from transformers import AutoModelForSequenceClassification
import torch

# 加载预训练BERT
model = AutoModelForSequenceClassification.from_pretrained(
    'bert-base-uncased',
    num_labels=2
)

# 准备示例输入
dummy_input = torch.zeros(1, 128, dtype=torch.long)  # batch=1, seq_len=128

# 导出为ONNX（FP32）
torch.onnx.export(
    model,
    (dummy_input,),
    'bert_base_fp32.onnx',
    input_names=['input_ids'],
    output_names=['logits'],
    dynamic_axes={
        'input_ids': {0: 'batch_size', 1: 'seq_len'}
    },
    opset_version=13
)
```

### 精度与大小权衡表

| 优化方法 | 模型大小减少 | 精度影响 | 推理加速 | 适用场景 |
|---------|-------------|---------|---------|---------|
| **FP32 → FP16** | 50% | 极小（<0.5%） | 1.5-2x | GPU设备、高端移动设备 |
| **INT8动态量化** | 50-75% | 小（1-3%） | 2-3x | CPU推理、通用场景 |
| **INT8静态量化** | 50-75% | 中等（2-5%） | 3-4x | 高性能要求场景 |
| **INT4量化** | 75% | 较大（5-10%） | 4-6x | 超低功耗设备 |
| **稀疏剪枝（50%）** | 50% | 小（依赖模型） | 1.2-1.5x | 支持稀疏推理的硬件 |
| **通道剪枝（30%）** | 30% | 小（1-2%） | 1.3-1.8x | 通用场景 |

**选择建议**

1. **首次部署**：建议从INT8动态量化开始，实现简单且精度损失小
2. **性能瓶颈**：如果推理速度仍不满足，尝试INT8静态量化或INT4
3. **超低功耗**：考虑模型剪枝+BERT→DistilBERT知识蒸馏
4. **精度敏感**：使用QAT或者更大模型（如DistilBERT而非量化BERT）

**常见问题排查**

| 问题 | 症状 | 解决方案 |
|-----|------|---------|
| OOM崩溃 | 内存不足 | 使用int8量化、减少batch size、启用内存优化 |
| 推理慢 | 延迟>预期 | 检查EP选择、启用所有图优化、调整线程数 |
| 精度偏差 | 结果与预期不符 | 检查输入预处理、量化校准、OP兼容性 |
| 电池消耗大 | 高温、耗电快 | 使用ANE/GPU、降低推理频率、合并推理请求 |

---

### 关键要点

1. **量化是首选**：对于大多数边缘应用，INT8量化提供最佳性价比
2. **硬件适配很重要**：充分利用设备原生加速器（ANE、NNAPI、GPU）
3. **验证不可少**：始终对比量化前后的精度指标
4. **平衡是关键**：根据实际需求平衡精度、速度和内存占用
5. **测试真实设备**：模拟器结果可能与真机差异较大
