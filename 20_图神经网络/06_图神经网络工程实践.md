# 图神经网络工程实践

## 1. 概述

将 GNN 从论文实验推向生产系统面临诸多工程挑战：大规模图的内存瓶颈、训练效率、部署延迟、分布式训练等。本笔记覆盖 GNN 的主流框架（PyTorch Geometric、DGL）、大规模图训练技术（采样、分区、分布式）、典型工程案例和部署最佳实践。

GNN 工程化的核心矛盾：**全图训练精度高但不可扩展，采样训练可扩展但引入噪声**。工业级 GNN 系统必须在精度、速度、内存之间取得平衡。

## 3. 核心概念

### 3.1 GNN 框架对比

| 框架 | 开发者 | 特点 | 适用场景 |
|:---|:---|:---|:---|
| **PyG** | TU Dortmund | PyTorch 原生，API 简洁，模型丰富 | 研究、中小规模图 |
| **DGL** | NYU/AWS | 支持多后端，消息传递优化 | 研究、工业部署 |

### 3.2 大规模图训练的挑战

| 挑战 | 原因 | 解决方案 |
|:---|:---|:---|
| 内存不足 | 全图邻接矩阵 + 特征 + 中间激活 | 邻居采样、子图采样 |
| 计算瓶颈 | 邻居指数膨胀（$L$ 层 → $d^L$ 邻居） | 限制采样数、层次化训练 |
| 通信开销 | 分布式训练中图分区跨机通信 | 图分区优化、缓存 |
| 特征存储 | 大图节点特征 TB 级 | 分布式 KV 存储 |

### 3.3 采样策略

| 策略 | 原理 | 优势 | 代表 |
|:---|:---|:---|:---|
| 邻居采样 | 每层采样固定数量邻居 | 简单可控 | GraphSAGE |
| 层采样 | 按层控制总采样数 | 避免指数膨胀 | FastGCN, LADIES |
| 子图采样 | 采样子图整图训练 | 保留局部结构 | GraphSAINT, Cluster-GCN |
| 重要度采样 | 按重要度概率采样 | 减少方差 | PinSage, GAS |

## 4. 技术原理

### 4.1 PyG 核心设计

PyG 基于稀疏 COO 格式表示图：

```python
import torch
from torch_geometric.data import Data

# 图表示: edge_index 为 [2, num_edges] 的边索引
edge_index = torch.tensor([[0, 1, 1, 2], [1, 0, 2, 1]], dtype=torch.long)
x = torch.randn(3, 16)  # 3个节点，16维特征
data = Data(x=x, edge_index=edge_index)

# GCN 层
from torch_geometric.nn import GCNConv
conv = GCNConv(16, 32)
h = conv(x, edge_index)  # [3, 32]
```

**核心抽象**：
- `Data`：单图容器（x, edge_index, edge_attr, y）
- `Batch`：多图批处理（通过虚拟边连接多图）
- `MessagePassing`：消息传递基类，自动处理聚合
- `NeighborLoader`：邻居采样数据加载器

### 4.2 DGL 核心设计

DGL 基于稀疏矩阵运算：

```python
import dgl
import torch.nn as nn

# 创建图
g = dgl.graph(([0, 1, 1, 2], [1, 0, 2, 1]))
g.ndata['h'] = torch.randn(3, 16)

# GCN 层
from dgl.nn import GraphConv
conv = GraphConv(16, 32)
h = conv(g, g.ndata['h'])
```

**核心优势**：
- 统一的消息传递 API（`g.update_all`）
- C++ 后端优化的稀疏操作
- 内置多种采样器（`NeighborSampler`, `MultiLayerNeighborSampler`）
- 分布式训练支持

### 4.3 Cluster-GCN

Cluster-GCN 将大图分区为多个子图簇，每个 batch 在一个簇上训练：

**算法**：
1. 用图分区算法（如 METIS）将图分为 $k$ 个簇
2. 每个 batch 随机选一个簇（或几个簇合并）
3. 在子图上做全图 GCN 前向/反向传播
4. 批间随机打乱

**优势**：
- 每个 batch 内是全图卷积（无采样噪声）
- 内存与簇大小成正比，可控
- 适合深层 GNN

**改进**：聚类边界信息丢失 → 选取相邻簇合并 + 边界节点重复采样。

### 4.4 GraphSAINT

GraphSAINT 从**图级别**采样——先采样子图，再在子图上训练：

**采样器**：
- **Random Node**：随机选节点，导出子图
- **Random Edge**：随机选边，导出关联子图
- **Random Walk**：随机游走采样子图

**归一化**：子图上的聚合需要归一化校正以补偿采样偏差：

$$\hat{A}_{\text{subgraph}} = \tilde{D}^{-1/2} (\tilde{P} \odot \tilde{A}) \tilde{D}^{-1/2}$$

其中 $\tilde{P}$ 是采样概率矩阵，用于校正采样偏差。

### 4.5 分布式 GNN 训练

**图分区策略**：

| 策略 | 原理 | 优势 | 劣势 |
|:---|:---|:---|:---|
| METIS 分区 | 最小化跨分区边 | 通信少 | 分区成本高 |
| 随机分区 | 随机分配节点 | 简单 | 通信量大 |
| 哈希分区 | 按节点 ID 哈希 | 均衡 | 通信量大 |

**训练范式**：
- **全复制**：每台机器存完整图，只分 batch（小图适用）
- **图分区**：每台机器存一部分图，跨机通信邻居特征
- **参数服务器**：图数据存参数服务器，worker 拉取邻居特征

### 4.6 GNN 推理优化

| 技术 | 原理 | 适用场景 |
|:---|:---|:---|
| 预计算传播 | 离线计算 $\tilde{D}^{-1/2}\tilde{A}\tilde{D}^{-1/2}X$，在线只做 MLP | 直推式 GCN |
| 嵌入缓存 | 缓存节点嵌入，增量更新 | 动态图 |
| 量化 | FP32 → FP16/INT8 | 部署 |
| 图压缩 | 合并相似节点，剪枝冗余边 | 大图加速 |

### 5.3 训练流程模板

```
1. 数据准备
   - 加载图数据 (邻接矩阵/边列表 + 节点特征)
   - 数据预处理 (特征归一化, 标签划分)

2. 选择训练策略
   - 小图 (<100K 节点): 全图训练
   - 中图 (100K-10M): 邻居采样 (NeighborLoader)
   - 大图 (>10M): Cluster-GCN / GraphSAINT / 分布式

3. 模型训练
   - GCN/GAT/GIN 前向传播
   - 交叉熵损失 (节点分类) 或 对比损失 (链路预测)
   - AdamW 优化器 + 学习率调度

4. 评估
   - 在 OGB 基准上对比
   - 分析过平滑现象 (层数 vs 性能)

5. 部署
   - 预计算嵌入 + 向量检索 (在线推理)
   - 或实时 GNN 推理 (需要邻居采样)
```

## 6. 优势与局限

### 6.1 优势

- **框架成熟**：PyG 和 DGL 提供了从研究到生产的完整工具链
- **可扩展性**：采样和分布式技术使 GNN 能处理十亿级图
- **标准化**：OGB 基准提供了公平的评估平台
- **工业验证**：Pinterest、Alibaba 等大规模应用证明了 GNN 的生产价值

### 6.2 局限

- **采样-精度权衡**：采样减少计算但可能损失精度
- **分布式复杂度**：图分区和跨机通信的工程难度高
- **实时推理延迟**：动态图的实时 GNN 推理延迟可能不满足线上要求
- **特征存储瓶颈**：大图节点特征存储和访问是系统瓶颈
