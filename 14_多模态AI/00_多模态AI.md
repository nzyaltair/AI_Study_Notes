# 多模态 AI 综述

## 1. 概述

多模态 AI（Multimodal AI）是能够**同时感知、理解和生成多种模态数据**（文本、图像、音频、视频等）的人工智能系统。与单模态模型不同，多模态 AI 通过跨模态对齐与融合，将不同感官通道的信息映射到统一的语义空间，实现超越单一模态的感知、推理与生成能力。

多模态 AI 解决的核心问题：
- **跨模态理解**：让模型能"看图说话""听音辨意"，将一种模态的信息转换为另一种模态的语义表示
- **多模态推理**：综合多种模态信息进行联合推理和决策（如视觉问答、图表分析）
- **多模态生成**：以一种或多种模态为条件，生成另一种模态的内容（如文生图、文生视频）
- **统一感知**：模拟人类多感官协同感知的能力，构建更接近通用智能的系统

[[大语言模型]] 的快速发展为多模态模型提供了统一的语言推理基座，Transformer 架构成为跨模态融合的统一框架。当前多模态 AI 正从"桥接式"架构（视觉编码器 + LLM）走向"原生多模态"统一模型，最终目标是实现 Any-to-Any 的统一多模态生成与理解。

## 3. 核心概念

### 3.1 模态（Modality）

模态指信息的表示形式或感知通道。常见模态包括：
- **文本**：自然语言，离散 token 序列
- **图像**：像素矩阵，空间分布信息
- **音频**：时序波形或频谱
- **视频**：时空序列，图像 + 时间维度

### 3.2 跨模态对齐（Cross-modal Alignment）

将不同模态的信息映射到**共享语义空间**，使语义相似的跨模态内容在该空间中距离相近。对齐是多模态 AI 的基础——只有对齐后，不同模态的信息才能被统一处理和推理。

主要对齐方法：
- **对比学习对齐**（CLIP）：拉近匹配图文对，推远不匹配对
- **桥接模块对齐**（BLIP-2 Q-Former）：学习可查询 token 提取视觉特征
- **投影层对齐**（LLaVA）：线性/MLP 映射视觉特征到 LLM 嵌入空间
- **原生融合**（GPT-4o）：训练阶段即统一处理多模态 token

### 3.3 多模态融合（Multimodal Fusion）

在对齐之后，将不同模态的特征进行整合以支持下游任务：

| 融合策略 | 原理 | 代表 |
|:---|:---|:---|
| 早期融合 | 特征层直接拼接或相加 | ViLBERT |
| 晚期融合 | 各模态独立处理后加权组合 | 集成方法 |
| 交叉注意力融合 | 一个模态作为 Q，另一个作为 K/V | Flamingo |
| Token 序列融合 | 多模态 token 拼接为统一序列 | LLaVA, GPT-4V |

### 3.4 视觉 Token 化

将连续的图像/视频转换为离散 token 序列，使其能被 Transformer 处理：
- **Patch 化**：ViT 将图像分割为 $16 \times 16$ patch，线性投影为 token
- **时空 Patch 化**：视频在空间和时间维度同时分块（Sora）
- **量化离散化**：VQ-VAE 将图像编码为离散 token（DALL·E 1）

### 3.5 条件生成

以一种或多种模态为条件，生成另一种模态的内容：
- **文本条件图像生成**：CLIP 文本编码 + 扩散模型（Stable Diffusion）
- **图像条件视频生成**：首帧 + 文本 → 视频（Sora, Stable Video Diffusion）
- **多模态条件生成**：文本 + 参考图像 → 风格化生成（IP-Adapter）

## 4. 技术原理

### 4.1 CLIP 对比学习对齐

CLIP 是跨模态对齐的奠基性工作。图像编码器（ViT）和文本编码器（Transformer）分别编码两个模态，通过对称 InfoNCE 损失对齐：

$$\mathcal{L}_{CLIP} = \frac{1}{2N}\sum_{i=1}^N\left[-\log \frac{\exp(S_{ii}/\tau)}{\sum_j \exp(S_{ij}/\tau)} - \log \frac{\exp(S_{ii}/\tau)}{\sum_j \exp(S_{ji}/\tau)}\right]$$

其中 $S_{ij} = I_i \cdot T_j$ 为图像-文本余弦相似度，$\tau$ 为可学习温度参数。

InfoNCE 下界互信息：$\mathcal{L}_{NCE} \geq -\log N + I(I; T)$

零样本分类：用文本描述类别（"a photo of a {label}"），选择与图像相似度最高的类别。

### 4.2 LLaVA 投影对齐

LLaVA 以最简方式将视觉信息接入 LLM：

$$H_p = \text{CLIP\_ViT}(x_{image}) \cdot W_{project} \in \mathbb{R}^{N_v \times D_{LLM}}$$

$$H_{input} = [\text{Embed}(x_{text}), H_p]$$

两阶段训练：(1) 图像-文本对齐预训练（冻结视觉编码器和 LLM，只训练投影层）；(2) 多模态指令微调（冻结视觉编码器，微调投影层和 LLM）。

### 4.3 BLIP-2 Q-Former

Q-Former 包含 $N_q$ 个可学习 Query token，通过交叉注意力从冻结的视觉编码器提取与语言相关的特征：

$$\text{Attn}(Q, K_v, V_v) = \text{softmax}\left(\frac{Q K_v^T}{\sqrt{D}}\right) V_v$$

Query token 在自注意力和交叉注意力中迭代更新，最终输出固定数量的视觉特征 token。

### 4.4 Classifier-Free Guidance（CFG）

条件生成中平衡质量与多样性的核心技术：

$$\tilde{\epsilon} = \epsilon_\theta(z_t, t, \varnothing) + w \cdot \left(\epsilon_\theta(z_t, t, c) - \epsilon_\theta(z_t, t, \varnothing)\right)$$

引导强度 $w > 1$ 增强条件一致性，$w = 1$ 为标准条件生成，$w = 0$ 为无条件生成。

### 4.5 多模态训练策略

| 阶段 | 目标 | 数据 | 冻结策略 |
|:---|:---|:---|:---|
| 预训练对齐 | 学习跨模态对齐 | 大规模图文对 | 冻结 LLM 和视觉编码器 |
| 指令微调 | 多模态对话能力 | 多模态指令数据 | 冻结视觉编码器 |
| 偏好对齐 | 对齐人类偏好 | 多模态偏好数据 | 全参数或 LoRA 微调 |

## 6. 优势与局限

### 6.1 优势

- **更全面的感知**：融合多种模态信息，捕捉单模态无法表达的语义
- **更强的推理能力**：多模态信息互补，提升推理准确性
- **更自然的人机交互**：支持语音、图像、视频等多种交互方式
- **更广泛的应用范围**：覆盖创意设计、内容创作、自动驾驶、医疗诊断等
- **通向通用智能的路径**：多模态感知是 AGI 的必要条件

### 6.2 局限

- **计算资源需求高**：多模态模型参数量大，训练和推理成本高
- **数据对齐困难**：高质量多模态配对数据稀缺
- **模态不平衡**：语言模态通常主导，视觉/音频模态容易被忽略
- **评估困难**：多模态任务评估标准不统一，缺乏全面基准
- **幻觉问题**：多模态模型同样存在"看图说谎"的幻觉现象
- **安全风险**：Deepfake、多模态越狱攻击等新型安全威胁

## 8. 与其他技术关系

- **大语言模型**：LLM 是多模态模型的语言推理基座，提供强大的语义理解和生成能力。参见 [[10_大语言模型核心架构]]
- **Transformer**：Transformer 是跨模态融合的统一架构，自注意力机制天然支持多模态 token 混合。参见 [[08_Transformer与注意力机制]]
- **生成模型**：扩散模型是图像/视频生成的核心范式。参见 [[07_生成模型]]
- **计算机视觉**：ViT、目标检测等 CV 技术为多模态模型提供视觉感知基础。参见 [[15_计算机视觉]]
- **自然语言处理**：NLP 的预训练和指令微调方法直接迁移到多模态领域。参见 [[16_自然语言处理]]

- **模型部署**：多模态模型的部署涉及多模态数据流处理、流式推理等工程挑战。参见 [[18_模型部署与工程化]]

## 9. 前沿发展

- **原生多模态统一模型**：从"视觉编码器 + LLM"的桥接式架构走向从头训练的统一模型（GPT-4o, Chameleon），所有模态共享同一 Transformer 参数
- **Any-to-Any 生成**：一个模型支持任意模态到任意模态的转换（文本↔图像↔音频↔视频）
- **实时多模态交互**：GPT-4o 实现了端到端实时语音+视觉交互，延迟低至 200ms
- **长视频理解与生成**：从短视频片段到分钟级长视频的时序建模
- **世界模型**：视频生成被视为通向物理世界模拟器的路径（Sora 作为"世界模拟器"）
- **多模态推理增强**：将 Chain-of-Thought 扩展到多模态推理（视觉 CoT）
- **高效多模态模型**：通过 token 压缩、动态分辨率、MoE 等技术降低计算成本
- **具身智能**：多模态 Agent 从数字世界走向物理世界，实现机器人操作
- **多模态安全**：多模态越狱防御、Deepfake 检测、水印技术
