# 卷积运算与基础组件

## 1. 概述

卷积神经网络的核心在于其**基本运算组件**的设计。理解卷积运算的数学本质、各类变体的适用场景、以及池化/归一化/激活函数等配套机制，是深入掌握CNN架构设计的前提。本章系统梳理CNN的基础组件及其工作原理。

## 2. 发展历史

| 时期 | 进展 |
|------|------|
| 1980s | 福岛邦彦 Neocognitron 引入简单/复杂细胞交替结构（卷积+池化雏形） |
| 1989 | LeCun 使用可训练的卷积核+反向传播训练CNN |
| 2012 | AlexNet 引入ReLU激活函数、Dropout正则化、重叠池化 |
| 2014 | VGG 证明3×3小卷积核堆叠优于大卷积核 |
| 2015 | Batch Normalization（Ioffe & Szegedy）使深层网络稳定训练成为可能 |
| 2016 | 空洞卷积（Dilated Convolution）在语义分割中广泛使用 |
| 2017 | MobileNet推广深度可分离卷积，大幅降低计算量 |
| 2018 | Group Normalization提供batch-size无关的归一化方案 |

## 3. 核心概念

### 3.1 基础术语

| 术语 | 含义 |
|------|------|
| **卷积核（Kernel/Filter）** | 形状为 $k \times k \times C_{in}$ 的可学习权重张量 |
| **特征图（Feature Map）** | 卷积层输出的 $H \times W \times C_{out}$ 张量 |
| **步长（Stride, S）** | 卷积核每次滑动的像素间距 |
| **填充（Padding, P）** | 输入边界补零的行/列数 |
| **膨胀率（Dilation, D）** | 卷积核元素之间的间距 |
| **通道数（Channels）** | 特征图的深度维度，对应卷积核数量 |

### 3.2 卷积 vs 互相关

深度学习中使用的"卷积"在数学上严格定义为**互相关**（Cross-correlation）：卷积核不进行翻转。数学卷积需先将核翻转180°，但在可学习的背景下两者等价，因此约定俗成统称"卷积"。

## 4. 技术原理

### 4.1 标准卷积（Standard Convolution）

对于输入 $I \in \mathbb{R}^{H \times W \times C_{in}}$ 和卷积核 $K \in \mathbb{R}^{k \times k \times C_{in} \times C_{out}}$，输出 $O \in \mathbb{R}^{H' \times W' \times C_{out}}$：

$$O_{c}(i,j) = \sum_{c'=1}^{C_{in}} \sum_{m=0}^{k-1} \sum_{n=0}^{k-1} I_{c'}(i+m, j+n) \cdot K_{c}(c', m, n) + b_c$$

**输出尺寸公式**（无空洞）：

$$H' = \left\lfloor \frac{H - k + 2P}{S} \right\rfloor + 1$$

**参数量**：$k^2 \times C_{in} \times C_{out}$

**计算量（FLOPs）**：$2 \times k^2 \times C_{in} \times C_{out} \times H' \times W'$

### 4.2 1×1 卷积（Pointwise Convolution）

$k=1$ 的特殊卷积，在各位置独立地对通道进行线性组合。本质上是跨通道的**线性投影**，不改变空间维度的感受野。

**核心用途**：
- 升降维：调整特征图的通道数
- 跨通道信息融合
- 构建Bottleneck结构（ResNet中先降维再升维）

**参数量**：$1 \times C_{in} \times C_{out}$，远少于标准卷积。

### 4.3 分组卷积（Grouped Convolution）

将输入通道分为 $G$ 组，每组独立进行卷积，最后拼接输出。AlexNet最早因双GPU内存限制而使用该设计。

**效果**：参数量与计算量降至标准卷积的 $1/G$。

**参数量**：$\frac{k^2 \times C_{in} \times C_{out}}{G}$

**后续影响**：分组卷积启发了深度可分离卷积的设计，ResNeXt将其提升为架构设计维度。

### 4.4 深度可分离卷积（Depthwise Separable Convolution）

将标准卷积分解为两步：

1. **Depthwise Convolution**：每个输入通道用单一 $k \times k$ 卷积核独立处理，不跨通道交互
2. **Pointwise Convolution**：用 $1\times1$ 卷积融合跨通道信息

**参数量对比**：
- 标准卷积：$k^2 \times C_{in} \times C_{out}$
- 深度可分离：$\underbrace{k^2 \times C_{in}}_{\text{Depthwise}} + \underbrace{C_{in} \times C_{out}}_{\text{Pointwise}}$

**压缩比**：$\frac{1}{C_{out}} + \frac{1}{k^2}$。当 $C_{out} \gg k$ 时，如3×3卷积输出通道大时，计算量约为标准卷积的 $1/9$。

> **代表模型**：MobileNet v1/v2/v3、Xception

### 4.5 空洞卷积（Dilated / Atrous Convolution）

在卷积核元素之间插入零值，以**不增加参数**的代价扩大感受野。膨胀率 $d$ 表示核元素间隔为 $d-1$ 个零。

**等效核大小**：$k' = k + (k-1)(d-1)$

**核心用途**：
- 语义分割中保持分辨率的同时获得大感受野（DeepLab系列的核心组件）
- 避免池化带来的空间信息损失

**潜在问题**：Gridding Effect——大膨胀率下部分像素从未被利用，导致信息不连续。DeepLabV3+通过不同膨胀率的混合设计缓解。

**感受野公式**（考虑膨胀）：

$$RF_l = RF_{l-1} + (k_l - 1) \times d_l \times \prod_{i=1}^{l-1} S_i$$

### 4.6 转置卷积（Transposed Convolution）

也称为**分数步长卷积**或反卷积（Deconvolution），用于将低分辨率特征图上采样到高分辨率。

**工作原理**：不是卷积的逆运算，而是通过"输入展开"的方式，将每个输入像素乘以整个卷积核后添加到对应输出位置（重叠区域累加）。

**输出尺寸**：$H' = (H-1) \times S + k - 2P$

**应用**：语义分割（FCN、U-Net）、图像生成（DCGAN）、超分辨率。

**棋盘效应（Checkerboard Artifact）**：当卷积核大小不能被步长整除时，转置卷积会在输出中产生不均匀的重叠，形成棋盘状伪影。可通过核大小=步长的倍数来避免，现代生成模型多用上采样+标准卷积替代。

### 4.7 池化（Pooling）

在空间维度上进行降采样，增强**平移不变性**并减少计算量。

| 类型 | 操作 | 特点 |
|------|------|------|
| **最大池化（Max Pooling）** | 取窗口内最大值 | 保留显著特征，对纹理/边缘敏感 |
| **平均池化（Average Pooling）** | 取窗口内均值 | 平滑特征，对背景信息保留更好 |
| **全局平均池化（GAP）** | 对整个特征图取均值 | 取代Flatten+全连接层，参数量极小 |
| **自适应池化** | 指定输出尺寸自动计算参数 | 支持任意尺寸输入 |

**GAP的重要性**：ResNet和GoogLeNet用GAP替代全连接分类头，显著减少参数，缓解过拟合，并自然支持可变尺寸输入。CAM（类激活映射）可视化也依赖GAP。

**池化 vs 步长卷积**：现代架构（ResNet、ConvNeXt）倾向于用步长为2的卷积替代池化层进行降采样，因为卷积的下采样是可学习的。

### 4.8 归一化层（Normalization）

| 方法 | 归一化维度 | 特点 |
|------|------------|------|
| **BatchNorm** | 沿 batch × H × W | 最常用，依赖batch size；训练/推理行为不一致 |
| **LayerNorm** | 沿 C × H × W | 不依赖batch，适合小batch和RNN/Transformer |
| **InstanceNorm** | 沿 H × W | 用于风格迁移等逐样本任务 |
| **GroupNorm** | 通道分组后沿 H×W | 折中方案，batch无关，适合小batch训练和检测/分割 |

**BatchNorm 核心公式**：

$$\hat{x} = \frac{x - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}, \quad y = \gamma \hat{x} + \beta$$

其中 $\mu_B$、$\sigma_B^2$ 为当前batch的统计量，$\gamma$、$\beta$ 为可学习的缩放和偏移参数。

**BatchNorm的关键作用**：
- 缓解内部协变量偏移（Internal Covariate Shift）
- 允许更大学习率，加速收敛
- 提供正则化效果，减轻对Dropout的依赖

### 4.9 激活函数（Activation Functions）

| 函数 | 公式 | 特点 |
|------|------|------|
| **ReLU** | $f(x) = \max(0, x)$ | CNN标配，简单高效，梯度要么0要么1 |
| **Leaky ReLU** | $f(x) = \max(\alpha x, x)$ | 缓解"死亡ReLU"问题（负区小梯度） |
| **GELU** | $x \cdot \Phi(x)$ | Transformer和ConvNeXt使用，平滑近似 |
| **SiLU/Swish** | $x \cdot \sigma(x)$ | EfficientNet使用，自门控机制 |
| **Mish** | $x \cdot \tanh(\ln(1+e^x))$ | 平滑非单调，YOLOv4使用 |

**ReLU死亡问题**：当大量输入落入负区时，梯度为零，神经元永久失活。可通过合理初始化（He初始化）和适当学习率缓解。

### 4.10 Dropout

训练时以概率 $p$ 随机丢弃神经元输出，等效于训练大量子网络的集成。CNN中通常在**全连接层**后使用（$p \appr 0.5$），卷积层较少使用Dropout。

现代CNN中BatchNorm的普及使Dropout的使用减少，但Transformer中仍有重要地位。

## 5. 关键方法对比

### 5.1 卷积变体选择指南

| 场景 | 推荐卷积类型 | 理由 |
|------|-------------|------|
| 通用分类 | 标准3×3卷积 | 良好的特征提取能力与参数效率平衡 |
| 移动端部署 | 深度可分离卷积 | 大幅降低计算量和模型大小 |
| 语义分割 | 空洞卷积 | 不降采样扩大感受野 |
| 通道数调整 | 1×1卷积 | 参数量极小 |
| 多分支架构 | 分组卷积 | 降低计算量，提供多样性 |
| 图像生成上采样 | 上采样+标准卷积 | 避免转置卷积的棋盘效应 |

### 5.2 归一化层选择指南

| 场景 | 推荐 |
|------|------|
| 分类任务（大batch） | BatchNorm |
| 检测/分割（小batch） | GroupNorm / SyncBN |
| Transformer / 序列模型 | LayerNorm |
| 风格迁移 / 生成 | InstanceNorm |

## 6. 优势与局限

### 卷积运算的核心优势
- **平移等变性**：$f(T(x)) = T(f(x))$，保证同一物体在不同位置的检测一致性
- **参数效率**：权值共享使卷积层参数量与输入尺寸无关
- **局部归纳偏置**：天然适合图像等具有局部相关性的数据

### 局限
- 缺乏**长程依赖建模**能力，感受野的扩大需要线性堆叠层
- 卷积核是**静态的**，无法根据输入内容动态调整
- 对**旋转/大尺度变换**不敏感，需要数据增强或专门设计

## 7. 应用场景

- **图像分类/检测/分割**：所有现代CNN的基础运算
- **特征提取（Backbone）**：输出多尺度特征图供下游任务使用
- **视频处理**：3D卷积扩展时间维度
- **图像生成**：转置卷积用于上采样
- **点云/图数据**：图卷积将卷积概念推广到非欧几里得数据

## 8. 与其他技术关系

- 卷积运算本质是**线性代数**中矩阵乘法的特殊形式——通过Toeplitz矩阵将卷积转换为矩阵乘（im2col操作）
- 深度可分离卷积的思想在**[[Transformer]]**的Multi-Head Attention中也有体现：Depthwise（每头独立） vs Pointwise（头间融合）
- **[[04_深度学习基础]]**中的优化方法、正则化、初始化策略直接适用于CNN训练
- 空洞卷积是 **[[04_图像分割]]** Deeplab系列的关键技术

## 9. 前沿发展

1. **大卷积核回归**：早期CNN（AlexNet 11×11）→ 小卷积核堆叠（VGG 3×3）→ ConvNeXt重新使用7×7卷积，证明更大卷积核在现代化设计下仍有竞争力
2. **动态卷积（Dynamic Convolution）**：根据输入动态调整卷积核权重（如CondConv、DY-Conv）
3. **可变形卷积（Deformable Convolution v2）**：让卷积核的采样位置根据输入内容自适应偏移，DETR的替代方案
4. **非卷积的视觉骨干**：MLP-Mixer、gMLP用纯MLP替代卷积的尝试
5. **训练技巧**：SyncBN（跨GPU同步批归一化）、EMA（指数移动平均模型用于最终推理）
