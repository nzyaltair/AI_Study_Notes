# 模型选择与调参

## 背景与发展

在[[机器学习]]中，选择合适的模型和优化超参数是提高性能的关键步骤。不同模型适用于不同数据分布和任务类型，超参数调优直接影响模型泛化能力。No Free Lunch 定理表明，没有普遍最优的模型，需根据数据和任务选择。

## 核心思想

模型选择是在候选模型中选取最适合数据和任务的模型；调参是寻找最优超参数组合使模型泛化性能最大化。两者都依赖交叉验证来评估泛化能力，避免过拟合验证集。

## 技术原理

### 交叉验证

- **K 折交叉验证**：数据分 K 份，每次用 K-1 份训练 1 份验证，循环 K 次取平均
- **分层 K 折**：保持每折中类别比例一致
- **留一法（LOO）**：K=N 的极端情况，计算量大但数据利用率高
- **时间序列 CV**：前向链验证，保证时间顺序

### 超参数调优方法

- **网格搜索（Grid Search）**：穷举所有参数组合，全面但计算量大
- **随机搜索（Random Search）**：随机采样参数组合，在高维参数空间中比网格搜索更高效
- **贝叶斯优化**：利用历史评估结果构建代理模型，智能选择下一组参数
  - TPE（Tree-structured Parzen Estimator）
  - 高斯过程
  - 工具：Optuna、Hyperopt

### 评估指标

| 任务类型 | 指标 | 说明 |
|---------|------|------|
| 分类 | Accuracy | 正确预测比例 |
| 分类 | Precision / Recall | 精确率 / 召回率 |
| 分类 | F1-Score | Precision 和 Recall 的调和平均 |
| 分类 | AUC-ROC | ROC 曲线下面积 |
| 回归 | MSE / RMSE | 均方误差 / 均方根误差 |
| 回归 | MAE | 平均绝对误差 |
| 回归 | $R^2$ | 决定系数 |

### 模型选择策略

1. 先用简单模型（线性回归/逻辑回归）建立基线
2. 逐步尝试更复杂的模型
3. 多个模型组合（[[集成学习]]）往往优于单一模型

## 发展演进

网格搜索 $\to$ 随机搜索 $\to$ 贝叶斯优化 $\to$ AutoML 自动化模型选择与调参。

## 关键算法·模型

- **交叉验证**：K 折、分层 K 折、时间序列 CV
- **贝叶斯优化**：TPE、高斯过程代理模型
- **AutoML**：自动化特征工程 + 模型选择 + 超参数优化

## 应用场景

- **机器学习**：所有 ML 项目的标准流程
- **深度学习**：学习率、batch size、网络结构等超参数调优
- **LLM 微调**：LoRA rank、学习率等超参数

## 与其他技术关系

- 模型评估依赖统计理论
- 交叉验证防止过拟合，与偏差-方差权衡相关
- 贝叶斯优化基于高斯过程理论
- 模型选择与[[集成学习]]的策略密切相关
