# 解码与采样策略

## 1. 概述

解码与采样策略是从模型预测的概率分布中选择下一个 token 的方法，直接决定生成文本的质量、多样性、可控性和效率。大语言模型的文本生成能力不仅取决于模型本身，也深受解码策略的影响。

早期的解码策略以贪心搜索和波束搜索为主，适合机器翻译等确定性任务。随着开放式生成任务（对话、创意写作）的兴起，Top-K 采样、Top-P（核采样）等随机性策略成为主流。Speculative Decoding（2023）将解码效率提升到新水平，Medusa/EAGLE/Lookahead（2024）进一步突破逐 token 串行限制。

## 2. 发展历史

| 年代 | 里程碑 | 意义 |
|------|--------|------|
| 2014 | Beam Search 普及 | 序列到序列任务的标准解码 |
| 2018 | Nucleus Sampling（Holtzman et al.）| Top-P 采样，开放式生成突破 |
| 2019 | Top-K Sampling | 控制采样多样性 |
| 2020 | Contrastive Search | 对比搜索平衡质量和多样性 |
| 2021 | Classifier-Free Guidance | 条件引导生成 |
| 2023 | Speculative Decoding（Leviathan et al.）| 投机解码，无损加速 |
| 2023 | 约束解码普及 | 结构化输出（JSON/语法约束）|
| 2024 | Medusa / EAGLE | 多 token 预测投机解码 |
| 2024 | Lookahead Decoding（Fu et al.）| 无草稿模型并行解码 |

## 3. 核心概念

解码策略在以下维度间权衡：

- **质量 vs 多样性**：确定性策略质量高但单调，随机性策略多样但可能失控
- **速度 vs 开销**：简单策略计算快，复杂策略增加计算成本
- **可控性 vs 创造性**：约束解码确保格式，自由采样激发创意
- **精度 vs 速度**：投机解码在保持精度的前提下加速生成

## 4. 技术原理

### 4.1 确定性解码

**贪心搜索**（Greedy Search）：每步选概率最高的 token。

- 确定性输出，计算开销低
- 易产生重复和退化（"loop" 现象）
- 适合代码生成、数学计算等需要确定性的任务

**波束搜索**（Beam Search）：维护 $k$ 个候选序列，每步扩展所有候选并保留概率最高的 $k$ 个：

$$\text{score}(y_{1:t}) = \sum_{i=1}^{t} \log P(y_i | y_{1:i-1}, x)$$

- 适合翻译、摘要等对质量要求高的任务
- 缺乏多样性，不适合开放对话
- 可配合长度惩罚（length penalty）控制输出长度

### 4.2 随机性采样

**温度采样**（Temperature Scaling）：通过温度参数 $T$ 调整概率分布平滑度：

$$P'(w_i) = \frac{\exp(\text{logit}_i / T)}{\sum_j \exp(\text{logit}_j / T)}$$

- $T \to 0$：接近贪心搜索，生成确定
- $T = 1$：原始概率分布
- $T \to \infty$：趋于均匀分布，生成随机

**Top-K 采样**：只从概率最高的 $K$ 个 token 中采样。
- $K$ 较小时偏向质量
- $K$ 较大时偏向多样性
- 固定 $K$ 不能自适应不同分布形状

**Top-P 采样**（核采样，Nucleus Sampling）：从累积概率达到 $P$ 的最小 token 集合中采样。
- 自适应调整候选集大小
- 分布尖锐时候选集小（偏向质量），分布平缓时候选集大（偏向多样性）
- 比 Top-K 更灵活，是开放式生成的首选

**Top-K + Top-P 组合**：先 Top-K 过滤再 Top-P 过滤，结合两者优点。实际系统中常同时使用。

### 4.3 高级采样策略

**对比搜索**（Contrastive Search）：
- 在候选 token 中选择既高概率又与上下文区分度大的 token
- 兼顾连贯性和多样性
- 有效缓解重复问题

**Classifier-Free Guidance（CFG）**：
通过对比条件和无条件生成的概率引导生成方向：

$$\text{logits} = \text{logits}_{\text{cond}} + \beta \cdot (\text{logits}_{\text{cond}} - \text{logits}_{\text{uncond}})$$

- $\beta > 0$ 增强条件信号的影响
- 常用于图像生成，也在文本生成中应用

**Min-P 采样**：动态设定最小概率阈值（基于最大概率 token 的比例）：
$$\text{threshold} = \text{min\_p} \times \max(P(w_i))$$
保留所有概率超过阈值的 token。比 Top-P 更自适应。

### 4.4 约束解码

限制生成的 token 必须遵循特定规则，通过 LogitsProcessor 实现：

- **前缀约束**：生成必须以特定前缀开头
- **词汇约束**：生成必须包含特定词
- **语法约束**：遵循正则表达式或上下文无关文法
- **JSON 结构约束**：确保输出是合法 JSON
- **工具调用约束**：确保输出符合函数调用格式

**实现方式**：
- LogitsProcessor：在每个解码步骤修改 logits
- 状态机引导：基于 DFA/CFG 的结构化解码
- Outlines / Guidance / JSON Mode：主流框架的结构化生成方案

### 4.5 Speculative Decoding

使用小模型生成候选 token 序列，大模型并行验证：

1. 小模型用采样策略生成 $k$ 个候选 token
2. 大模型并行计算这 $k$ 个 token 的概率
3. 接受匹配的 token，在首个拒绝处重新生成

**接受规则**（拒绝采样保证质量无损）：
- 若 $\pi_{\text{draft}}(y_i) \leq \pi_{\text{target}}(y_i)$：接受
- 若 $\pi_{\text{draft}}(y_i) > \pi_{\text{target}}(y_i)$：以 $\pi_{\text{target}}/\pi_{\text{draft}}$ 概率接受

**效果**：延迟降低 30-50%，生成质量与大模型完全一致。

### 4.6 多 token 预测变体

| 方法 | 原理 | 特点 |
|------|------|------|
| **Speculative Decoding** | 独立小模型草拟 | 需要额外草稿模型，接受率约 50-70% |
| **Medusa** | 大模型添加多个预测头 | 每头预测未来第 $i$ 个 token，无需独立模型 |
| **EAGLE** | 基于大模型隐藏状态自回归草拟 | 接受率更高（70-85%），加速比更大 |
| **Lookahead Decoding** | Jacobi 迭代并行解码 | 无需草稿模型，无需训练 |
| **Self-Speculative** | 跳过部分层作为草稿模型 | 无需额外模型，通过层跳过加速 |

### 4.7 重复惩罚与控制

- **Repetition Penalty**：降低已出现 token 的概率，$P'(w) = P(w) / \text{penalty}$（若 $w$ 已出现）
- **Frequency Penalty**：按出现次数递增惩罚
- **Presence Penalty**：出现即惩罚（不按次数）
- **Length Penalty**：控制生成长度

## 5. 关键方法与模型

### 5.1 场景策略选择

| 场景 | 推荐策略 | 参数建议 |
|------|---------|---------|
| 代码生成 | 贪心 + 约束 | temperature=0.2, top_k=10 |
| 数学计算 | 贪心 | temperature=0.1, top_k=5 |
| 创意写作 | 温度 + Top-P | temperature=0.9, top_p=0.95 |
| 对话生成 | 温度 + Top-K + Top-P | temperature=0.7, top_k=50, top_p=0.9 |
| 摘要/翻译 | 波束搜索 | num_beams=4, length_penalty=1.2 |
| 实时服务 | Speculative + 温度 | temperature=0.7, top_p=0.9 |
| JSON 输出 | 约束解码 | temperature=0.1, grammar=JSON |

### 5.2 代表论文

## 6. 优势与局限

**优势**：
- 采样策略灵活控制生成风格和多样性
- 约束解码确保结构化输出，支撑 Agent 和工具调用
- 投机解码在不损失质量的前提下显著加速
- 多 token 预测进一步突破串行瓶颈

**局限**：
- 贪心和波束搜索易产生重复退化
- 采样参数需针对不同模型和任务调优
- 约束解码可能限制生成质量
- 投机解码加速比依赖草稿模型质量和任务类型
- 高温度下可能出现幻觉和不连贯

## 7. 应用场景

- **代码生成**：贪心搜索 + 约束解码确保语法正确
- **对话系统**：温度采样 + Top-P 平衡质量和多样性
- **翻译/摘要**：波束搜索保证流畅性
- **实时服务**：Speculative Decoding 降低延迟
- **结构化输出**：约束解码确保 JSON/XML/函数调用格式
- **创意写作**：高温度 + Top-P 最大化多样性
- **Agent 工具调用**：约束解码 + 低温度确保格式正确

- **多模态解码**：统一文本、图像、音频的解码策略
