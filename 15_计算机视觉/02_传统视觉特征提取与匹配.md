# 传统视觉特征提取与匹配

## 1. 概述

传统视觉特征提取是深度学习之前计算机视觉的核心技术。它通过手工设计的算法从图像中提取具有**不变性**（对尺度、旋转、光照、视角变化的不变性）的局部特征，用于图像匹配、物体识别、三维重建等任务。

尽管深度学习已主导大多数视觉任务，传统特征方法在以下场景仍不可替代：无训练数据的几何计算（如 SLAM 中的特征跟踪）、实时嵌入式系统（ORB 特征用于移动端 AR）、图像拼接与全景生成等。

本篇涵盖边缘检测、角点检测、局部特征描述子（SIFT/SURF/ORB）、区域特征（HOG）和特征匹配与几何验证（RANSAC）。

## 3. 核心概念

### 3.1 特征点（Keypoint）

特征点是图像中具有显著局部变化的点，应满足：
- **可重复性**：在不同图像（不同视角/光照）中能被重复检测到
- **可区分性**：不同特征点应有不同的描述子
- **局部性**：只依赖局部邻域，对遮挡鲁棒
- **数量足够**：足以进行匹配和识别

### 3.2 不变性

特征方法的核心设计目标是实现各种变换下的不变性：

| 不变性 | 含义 | 方法 |
|:---|:---|:---|
| 平移不变 | 特征不受位置影响 | 滑动窗口扫描 |
| 尺度不变 | 特征不受大小影响 | 尺度空间（DoG/图像金字塔） |
| 旋转不变 | 特征不受方向影响 | 主方向估计 |
| 光照不变 | 特征不受亮度影响 | 梯度归一化 |
| 仿射不变 | 特征不受视角变化影响 | 仿射归一化（较难实现） |

### 3.3 特征管道

完整的传统特征流程：

```
检测 → 定位 → 方向估计 → 描述子提取 → 匹配 → 几何验证
```

## 4. 技术原理

### 4.1 Harris 角点检测

角点是两个边缘交汇的点，在两个方向上都有大的梯度变化。

**自相关矩阵（结构张量）**：

$$M = \sum_{x,y} w(x,y) \begin{pmatrix} I_x^2 & I_x I_y \\ I_x I_y & I_y^2 \end{pmatrix}$$

其中 $I_x, I_y$ 为图像梯度，$w(x,y)$ 为高斯窗口函数。

**角点响应函数（Harris）**：

$$R = \det(M) - k \cdot \text{trace}(M)^2 = \lambda_1 \lambda_2 - k(\lambda_1 + \lambda_2)^2$$

- $R$ 大正值 → 角点（两个特征值都大）
- $R$ 大负值 → 边缘（一个特征值大，一个小）
- $|R|$ 小 → 平坦区域

**Shi-Tomasi 改进**：$R = \min(\lambda_1, \lambda_2)$，直接取较小特征值，在某些场景更稳定。

### 4.2 SIFT（尺度不变特征变换）

SIFT 是最经典的手工特征方法，完整解决了尺度、旋转和部分光照不变性问题。

#### 步骤 1：尺度空间极值检测

构建高斯金字塔和 DoG（Difference of Gaussian）金字塔：

$$L(x,y,\sigma) = G(x,y,\sigma) * I(x,y)$$

$$D(x,y,\sigma) = L(x,y,k\sigma) - L(x,y,\sigma)$$

在 DoG 金字塔的 3D 空间（$x, y, \sigma$）中检测局部极值点——与 26 个相邻点（同层 8 个 + 上下层各 9 个）比较。

#### 步骤 2：关键点定位

对 DoG 函数进行泰勒展开，亚像素级精确定位，并去除低对比度点和边缘响应点（Hessian 矩阵阈值）。

#### 步骤 3：方向估计

在关键点邻域内计算梯度方向直方图（36 bins，每 10°），取峰值作为主方向：

$$m(x,y) = \sqrt{(L(x+1,y) - L(x-1,y))^2 + (L(x,y+1) - L(x,y-1))^2}$$

$$\theta(x,y) = \arctan\left(\frac{L(x,y+1) - L(x,y-1)}{L(x+1,y) - L(x-1,y)}\right)$$

#### 步骤 4：描述子构建

在关键点 $16 \times 16$ 邻域内，划分为 $4 \times 4$ 子区域，每个子区域计算 8 方向梯度直方图：

$$\text{SIFT 描述子} = 4 \times 4 \times 8 = 128 \text{ 维向量}$$

最终归一化并阈值截断以增强光照不变性。

### 4.3 SURF（加速鲁棒特征）

SIFT 的加速版本，核心改进：

- **积分图**（Integral Image）：$II(x,y) = \sum_{x' \leq x, y' \leq y} I(x',y')$，任意矩形区域求和只需 4 次运算
- **Hessian 矩阵近似**：用方框滤波（Box Filter）近似高斯二阶导数，配合积分图实现快速计算
- **描述子**：64 维（较 SIFT 的 128 维短），基于 Haar 小波响应

### 4.4 ORB（Oriented FAST and Rotated BRIEF）

专为实时应用设计的特征方法，是 OpenCV 的默认特征：

- **FAST 角点检测**：比较中心像素与圆周 16 个像素的亮度，连续 12 个以上亮/暗则为角点
- **方向估计**：计算角点邻域的图像矩质心方向 $\theta = \arctan(m_{01}/m_{10})$
- **BRIEF 描述子**：在关键点邻域随机选取像素对，比较亮度大小生成二进制串（256 位）
- **旋转 BRIEF**：根据关键点方向旋转采样模式

**优势**：速度极快（比 SIFT 快两个数量级），二进制描述子匹配用汉明距离（XOR + popcount），适合移动端和实时 SLAM。

### 4.5 HOG（方向梯度直方图）

HOG 是区域级特征，用于目标检测（特别是行人检测）：

1. **梯度计算**：$G_x, G_y$ 用简单差分 $[-1, 0, 1]$
2. **Cell 划分**：将图像划分为 $8 \times 8$ 像素的 Cell
3. **方向直方图**：每个 Cell 内计算 9 方向（0°-180°，每 20°）的梯度直方图
4. **Block 归一化**：将 $2 \times 2$ Cell 组成 Block，L2 归一化
5. **拼接**：所有 Block 的直方图拼接为最终特征向量

HOG + 线性 SVM 在行人检测上取得了当时的 SOTA 性能。

### 4.6 特征匹配

#### 暴力匹配

计算所有描述子对的距离，对每个查询特征找最近邻：
- **L2 距离**（SIFT/SURF 浮点描述子）
- **汉明距离**（ORB/BRIEF 二进制描述子）

#### 比率测试（Lowe's Ratio Test）

$$\text{ratio} = \frac{d(\text{最近邻})}{d(\text{次近邻})} < \text{阈值（通常 0.7-0.8）}$$

过滤模糊匹配——如果最近邻和次近邻距离相近，说明匹配不可靠。

#### FLANN（快速近似最近邻）

基于 KD-Tree（浮点描述子）或层次聚类（二进制描述子）的近似最近邻搜索，在大量特征时比暴力匹配快得多。

### 4.7 RANSAC（随机抽样一致）

RANSAC 是从含噪声匹配中估计几何模型的经典方法：

1. **随机采样**：从匹配集中随机选取最小样本（如计算单应矩阵需 4 对点）
2. **模型估计**：用样本计算变换参数
3. **内点统计**：计算所有匹配点对该模型的拟合误差，统计内点数
4. **迭代**：重复 N 次，选择内点数最多的模型
5. **精炼**：用所有内点重新拟合模型

**迭代次数估计**：$N = \frac{\log(1-p)}{\log(1-w^s)}$，其中 $p$ 为置信概率（如 0.99），$w$ 为内点比例，$s$ 为最小样本数。

## 5. 关键方法/模型

### 5.1 方法对比

| 特征 | 检测器 | 描述子 | 维度 | 尺度不变 | 旋转不变 | 速度 | 专利 |
|:---|:---|:---|:---|:---|:---|:---|:---|
| Harris | Harris 角点 | — | — | ✗ | ✗ | 快 | 无 |
| SIFT | DoG | SIFT | 128 | ✓ | ✓ | 慢 | 曾有（已过期） |
| SURF | Hessian | SURF | 64 | ✓ | ✓ | 中 | 有 |
| ORB | FAST | rBRIEF | 256bit | ✓ | ✓ | 极快 | 无 |
| AKAZE | 非线性尺度空间 | AKAZE | 486 | ✓ | ✓ | 中 | 无 |

### 5.2 经典应用：图像拼接

全景图拼接的标准流程：
1. SIFT/ORB 特征提取
2. 特征匹配 + Lowe 比率测试
3. RANSAC 估计单应矩阵 $\mathbf{H}$
4. 图像变换与拼接
5. 融合（多频段融合消除接缝）

## 6. 优势与局限

### 优势

1. **几何不变性好**：SIFT/ORB 对尺度、旋转、光照变化具有良好不变性
2. **无需训练**：纯算法方法，直接适用于任意新场景
3. **高效匹配**：二进制描述子（ORB）可在毫秒级完成匹配
4. **可解释性强**：每个步骤的数学原理清晰
5. **适合几何任务**：SLAM、运动恢复结构、图像配准等任务的首选

### 局限

1. **语义理解能力弱**：只能匹配局部纹理模式，无法理解物体语义
2. **纹理稀疏区域失效**：在弱纹理、重复纹理、遮挡区域检测困难
3. **光照剧变鲁棒性不足**：极端光照变化仍可能导致匹配失败
4. **无法端到端优化**：特征提取与下游任务分离，非联合最优

## 8. 与其他技术关系

- **与图像处理的关系**：特征提取依赖图像处理基础——梯度计算、高斯平滑、形态学操作。参见 [[01_计算机视觉基础与图像处理]]。
- **与 CNN 的关系**：深度学习的卷积层可视为可学习的特征提取器，在语义任务上远超手工特征，但在几何不变性上传统方法仍有优势。参见 [[05_卷积神经网络]]。
- **与三维视觉的关系**：SIFT/ORB 是 SfM 和 SLAM 系统的基础组件。参见 [[08_三维计算机视觉]]。
- **与目标检测的关系**：HOG + SVM 是经典检测方法，后被 CNN 检测器取代。参见 [[04_目标检测]]。

## 9. 前沿发展

- **学习型局部特征**：SuperPoint、R2D2 等用深度网络学习关键点检测和描述子，在匹配精度上超越 SIFT
- **图神经网络匹配**：SuperGlue 用 GNN + 注意力机制进行特征匹配，显著提升宽基线匹配性能
- **可微 SLAM**：将特征提取和位姿估计端到端联合优化
- **无监督特征学习**：利用多视图一致性自监督学习局部描述子
