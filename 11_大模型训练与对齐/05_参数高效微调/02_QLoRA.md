# QLoRA

## 背景与发展

QLoRA（Quantized LoRA）由 Dettmers et al. (2023, University of Washington) 提出，是 [[LoRA]] 与 4-bit 量化技术的结合。QLoRA 允许在单张 GPU（如 24GB 显存的 RTX 4090）上微调 70B 参数的大语言模型，将大模型微调的硬件门槛降低了约 10 倍，同时保持接近全参数微调的性能。

## 核心思想

将基础模型权重量化到 4-bit 精度存储，在前向和反向传播时按需解量化为 16-bit 进行计算，同时在量化模型上应用 LoRA 微调。这样既大幅减少了模型内存占用，又通过 LoRA 的低秩矩阵保持了梯度更新的精度。

## 技术原理

### NF4 量化

QLoRA 使用 NF4（Normalized Float 4-bit）量化方案，针对正态分布的权重进行优化：

1. **权重分组**：将权重矩阵分为大小为 64 的组
2. **组内归一化**：每组权重归一化到均值为 0、方差为 1
3. **量化映射**：将归一化后的权重映射到 4-bit 的 16 个值上，这些值通过正态分布的分位数确定
4. **存储**：存储 4-bit 量化权重和每组缩放因子

### 双重量化

对量化后的缩放因子再次进行量化（8-bit），进一步减少约 0.4 bits/param 的内存开销。

### 前向传播

1. 从内存加载 4-bit 量化权重
2. 按需解量化为 BF16/FP16
3. 执行 LoRA 前向传播：$y = Wx + BAx$

### 反向传播

1. 计算损失对输出的梯度 $\partial L / \partial y$
2. 计算 LoRA 矩阵梯度：$\partial L / \partial A = B^T (\partial L / \partial y) x^T$
3. 仅更新 LoRA 矩阵 $A$、$B$，量化权重保持不变

### 关键优化

- **零初始化 LoRA**：$B$ 初始化为零，确保初始模型行为与原始模型一致
- **分页优化器**：使用 NVIDIA 统一内存，在 GPU 内存不足时自动将优化器状态转移到 CPU
- **梯度检查点**：仅保存部分层激活值，反向传播时重新计算

### 显存对比

| 模型规模 | 全参数微调 | QLoRA 微调 |
|---------|-----------|-----------|
| 7B | ~30 GB | ~4 GB |
| 13B | ~60 GB | ~6 GB |
| 70B | ~300 GB | ~24 GB |

### 关键参数

| 参数 | 推荐值 | 说明 |
|------|---------|------|
| 量化类型 | NF4 | 专为正态分布权重优化 |
| 双重量化 | 开启 | 进一步减少内存 |
| 计算 dtype | bfloat16 | 平衡性能和精度 |
| LoRA 秩 $r$ | 16-64 | 大模型可用较小秩 |

## 发展演进

INT8 量化 (2022) → QLoRA (2023, 4-bit NF4) → GPTQ-QLoRA (2023) → AQLoRA (2024，自适应精度)

## 关键算法·模型

- **NF4 量化**：正态分布优化的 4-bit 量化方案
- **双重量化**：对缩放因子再量化
- **分页优化器**：GPU-CPU 内存统一管理
- **bitsandbytes**：QLoRA 的核心量化库

## 应用场景

- **单卡微调大模型**：在消费级 GPU 上微调 70B 模型
- **资源受限环境**：大幅降低微调硬件门槛
- **快速实验**：低成本尝试不同微调配置
- **多任务部署**：量化基模 + 多组 LoRA 适配器

## 与其他技术关系

- QLoRA 是 [[LoRA]] 与模型量化的结合
- 量化技术是推理优化的重要内容
- QLoRA 可与 [[监督微调]]、[[DPO]] 等训练阶段结合
- 分页优化器依赖 CUDA 统一内存机制
