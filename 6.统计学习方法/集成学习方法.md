# 集成学习方法

## 背景

集成学习（Ensemble Learning）通过组合多个基学习器来提升泛化性能，是[[机器学习|机器学习]]竞赛和工业实践中的利器。"三个臭皮匠顶个诸葛亮"是其核心哲学——多个弱学习器组合可以构成强学习器。

## 核心思想

通过一定的策略将多个基学习器（base learner）的预测结果进行组合，使集成模型的性能优于任何单一模型。关键在于基学习器既要有一定的准确性，又要有多样性（diversity）。

## 技术原理

### Bagging（Bootstrap Aggregating）

并行训练多个独立基学习器，通过投票/平均组合结果。

- **Bootstrap采样**：从训练集中有放回抽取n个样本（约36.8%样本未被采样，称为OOB样本）
- **随机森林**（Random Forest）是Bagging的典型代表：
  - 在Bagging基础上，每次分裂时随机选择部分特征
  - 特征随机性增加了树之间的多样性
  - 优点：抗过拟合、能评估特征重要性、并行训练
  - 缺点：模型体积大、可解释性不如单棵树

### Boosting

串行训练基学习器，每个新模型重点关注前一个模型的错误。

#### AdaBoost

- 每轮增加错分样本的权重，减少正确分类样本的权重
- 最终模型为弱学习器的加权组合
- 损失函数：指数损失

#### 梯度提升树（GBDT）

- 每轮拟合当前模型的负梯度（残差）
- 本质：在函数空间中进行梯度下降
- 损失函数可替换（平方损失、Huber损失、LogLoss等）

#### 现代梯度提升框架

| 框架 | 核心创新 | 优势 |
|------|---------|------|
| **XGBoost** | 二阶导数、正则化、列采样 | 精度高、防过拟合 |
| **LightGBM** | Leaf-wise生长、GOSS、EFB | 训练快、内存少 |
| **CatBoost** | 类别特征处理、Ordered Boosting | 类别数据友好、防泄露 |

**XGBoost目标函数**：

$$\mathcal{L} = \sum_{i=1}^n l(y_i, \hat{y}_i) + \sum_{k=1}^K \Omega(f_k), \quad \Omega(f) = \gamma T + \frac{1}{2}\lambda \|\mathbf{w}\|^2$$

**LightGBM关键优化**：
- **Leaf-wise生长**：选择增益最大的叶子分裂（vs Level-wise的层序生长）
- **GOSS**（Gradient-based One-Side Sampling）：保留梯度大的样本，随机采样梯度小的样本
- **EFB**（Exclusive Feature Bundling）：将互斥特征捆绑，降维加速

### Stacking与Voting

#### Voting

- **硬投票**：多数表决
- **软投票**：对预测概率取平均（更推荐）

#### Stacking

多层模型组合策略：
1. 第一层：训练K个不同的基学习器
2. 用K折交叉验证生成元特征（避免数据泄露）
3. 第二层：用元模型（如逻辑回归）基于元特征训练

```python
# Stacking 示例伪代码
from sklearn.model_selection import KFold

# 第一层基模型
base_models = [RandomForest(), XGBoost(), LightGBM()]
# 第二层元模型
meta_model = LogisticRegression()

# K折生成元特征
for train_idx, val_idx in KFold(n_splits=5).split(X):
    for model in base_models:
        model.fit(X[train_idx], y[train_idx])
        meta_features[val_idx] = model.predict(X[val_idx])

# 训练元模型
meta_model.fit(meta_features, y)
```

## 发展演进

Bagging（1996）→ AdaBoost（1997）→ 随机森林（2001）→ GBDT → XGBoost（2014）→ LightGBM（2017）→ CatBoost（2017）

集成方法在表格数据竞赛（Kaggle）中长期占据统治地位，但在图像、文本等领域已被[[深度学习]]超越。

## 应用领域

- **表格数据竞赛**：Kaggle竞赛中几乎所有冠军方案都使用集成
- **金融风控**：信用评分、欺诈检测（XGBoost/LightGBM为主）
- **点击率预估**：广告系统中的CTR预估
- **特征重要性分析**：随机森林/XGBoost的特征重要性用于[[特征工程|特征选择]]

## 与其他技术关系

- 集成方法以[[监督学习方法|决策树]]为最常用的基学习器
- 模型融合策略依赖[[模型选择与调参|交叉验证]]防止数据泄露
- Boosting的梯度下降思想与[[深度学习|神经网络优化]]相通
- XGBoost/LightGBM等已成为[[Python用于机器学习|数据科学工具栈]]标配
- Stacking中的元模型思想在[[大语言模型|LLM路由]]中也有应用
