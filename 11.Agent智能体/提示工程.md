# 提示工程

## 背景

提示工程（Prompt Engineering）是与LLM交互的核心技术。通过精心设计提示词，可以引导LLM生成更准确、更有用的输出。提示工程是LLM应用开发的基本功，也是[[Agent智能体|Agent]]系统设计的基础。

## 核心思想

LLM的行为由输入提示决定。通过结构化的提示设计、示例引导和推理引导，可以显著提升LLM的输出质量和可控性。

## 技术原理

### 基础提示技术

| 技术 | 原理 | 适用场景 |
|------|------|---------|
| **零样本** | 直接提问，不提供示例 | 简单任务 |
| **少样本** | 提供3-5个示例引导 | 特定格式/领域任务 |
| **角色提示** | "你是一个XX专家..." | 设定专业角色 |
| **指令提示** | 明确任务要求与约束 | 格式化输出 |

### 思维链（CoT）

引导模型展示推理过程：

```
让我们逐步思考：
1. 首先...
2. 然后...
3. 因此...
答案：...
```

详见[[推理模型与思维链|推理模型与思维链]]。

### ReAct框架

Thought → Action → Observation循环：

```
Thought: 用户想知道天气，我需要查询
Action: get_weather("北京")
Observation: 晴天，25°C
Thought: 获取到了天气信息，可以回答
Final Answer: 北京今天晴天，25°C。
```

### 高级提示技术

| 技术 | 原理 | 应用 |
|------|------|------|
| **自洽性** | 多次采样取多数投票 | 提高准确性 |
| **思维树** | 树形搜索多个推理路径 | 规划类任务 |
| **自我修正** | 生成后审查和修改 | 代码生成 |
| **Decomposition** | 将复杂任务分解为子任务 | 多步推理 |
| **Meta-prompting** | 用LLM生成提示 | 自动优化 |

### 提示工程化

- **模板化**：使用变量和模板引擎管理提示
- **版本控制**：对提示进行版本管理
- **A/B测试**：对比不同提示效果
- **评估体系**：建立提示质量评估指标
- **工具**：LangChain PromptTemplate、PromptHub、DSPy

### 结构化输出

通过提示引导LLM输出结构化格式：

```python
# Function Calling 方式
response = llm.chat(
    messages=[{"role": "user", "content": "..."}],
    tools=[{
        "type": "function",
        "function": {
            "name": "get_weather",
            "parameters": {"type": "object", "properties": {...}}
        }
    }]
)

# JSON Mode 方式
response = llm.chat(
    messages=[...],
    response_format={"type": "json_object"}
)
```

## 发展演进

手工提示 → Few-Shot → CoT（2022）→ ReAct → 自动化提示优化（DSPy, 2023）→ 提示成为"软编程"

趋势：从手工调提示走向自动化提示优化，提示工程正在成为一门系统化的工程学科。
